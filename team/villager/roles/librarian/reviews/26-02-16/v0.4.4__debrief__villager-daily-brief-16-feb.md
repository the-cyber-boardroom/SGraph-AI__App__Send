## Questions for the Human

### Priority Questions (Blocking Work)

1. **Subdomain DNS ownership:** Does the human have direct access to Route 53 for `sgraph.com` and `sgraph.ai`? Or does another party manage DNS? This determines whether DevOps can act directly.

2. **SA pipeline Lambda placement:** Should the SA REST API live on the existing Admin Lambda, or should it be a separate Lambda function? The brief mentions "two Lambda functions" (Public and Admin) but the SA pipeline is a significant new surface area.

3. **Translator ROLE.md creation:** Should the Conductor create the Translator ROLE.md based on the role definition brief, or does the human want to review/approve the ROLE.md before it goes live?

4. **CloudFront log data -- actual sample:** The DPO/AppSec need to examine actual log data to confirm IP addresses are absent. Does the human have a sample log file, or should DevOps pull one from the S3 bucket?

### Clarification Questions (Non-Blocking)

5. **"Klingon" language scope:** The brief says "no full content needed." Should the Translator produce even a few key strings (error messages, button labels) in Klingon for the Easter egg effect, or is the dropdown entry alone sufficient?

6. **Physical card printing:** Does the human have a preferred print vendor, or should the Designer produce print-ready files for any standard vendor?

7. **Evidence pack timeline:** The DPO/AppSec/GRC evidence collection is described as running in parallel. Is there a target date for the articles, or is this open-ended?

8. **Beta invite token format:** The brief mentions `community-beta-001` as an example. Is this the desired token naming convention, or should the Ambassador propose a naming scheme?

## Process Updates Required

### 1. Daily Memo Workflow

The brief defines a new mandatory workflow for processing human input. The Conductor must add this to `.claude/villager/CLAUDE.md`:

1. Architect + Librarian read first (always)
2. Librarian catalogues (always)
3. Produce a debrief document (always)
4. Impacted roles update themselves (only impacted roles)
5. Human reviews the debrief
6. Clarifications flow back

### 2. Briefing Packs

A new process for packaging work for agents. Key principles:
- Every pack lives in its own dedicated folder
- Minimum viable pack: folder + BRIEF.md + .issues/tasks.issues
- Code context uses method streams (actual code with file paths and line numbers)
- Librarian and Architect lead pack creation

The first pack to create: **CloudFront log pipeline** (workstream-level).
