## Key Findings

### What These Briefs Are

**Doc 334 (Ephemeral Infra Next Phase):** Seven next-phase features beyond the Day 48 MVP: UI split, AMI management, vault server as instance type, Docker management, remote shell, Prometheus endpoints, Stacks. This brief is addressed to Explorer Dev + DevOps.

**Doc 335 (Firefox Browser Plugin):** Firefox + MITM proxy as the first real validation of the Day 48 plugin architecture. Interactive browsing (human) vs. Playwright's programmatic model. Addressed to Explorer Dev.

**Doc 336 (SGit Four-Layer Refactoring):** Villager brief for the SGit-AI/SGit-AI__CLI repo. Four-layer code separation, transaction logging, step decomposition, PKI readiness. Not work for this repo — work for the SGit Villager team.

### Architectural Highlights (from Architect Review)

- **Vault server as instance type** is the most significant proposal — it completes the ephemeral platform triangle (Compute + Browser + Storage) and enables Stacks
- **Stacks** is the highest-complexity item (launcher, dependency order, partial failure handling)
- **SGit four-layer** is a prerequisite for PKI — Villager team should complete it before any PKI Explorer work begins
- **Firefox MITM script upload** is arbitrary code execution — AppSec review needed before shipping
