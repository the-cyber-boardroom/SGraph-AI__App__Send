## What's in the 04/29 Late Batch (Quick Reference)

### Ephemeral Infra Next Phase (PROPOSED)
Seven features extending the ephemeral infrastructure platform. Priority order:
1. Split creation/live UI (P1 — UI refactoring)
2. AMI management (P1 — enables fast AMI-based launch)
3. SG/Send vault server instance type (P2 — storage primitive)
4. Docker container management (P2 — manage containers inside instances)
5. Remote shell Option A (P2 — API-based, no SSH/SSM)
6. Prometheus endpoints (P3 — metrics on instances)
7. Stacks Phase 1 (P3 — multi-instance launch shortcut)

### Firefox Browser Plugin (PROPOSED)
- Plugin under `plugins/firefox/`
- Interactive browsing vs Playwright's programmatic model
- MITM proxy with uploadable intercept scripts
- Vault storage per session (credentials, scripts, logs, screenshots)
- AMI bake captures configured Firefox profiles

### SGit Four-Layer Refactoring (PROPOSED)
- Four layers: Crypto (pure) → Core (state changes) → Network (API) → Plugins (read-only)
- No upward dependencies — enforced by folder structure
- Transaction logging on every Core action (replay, monitoring, audit)
- Step-based decomposition with per-step timing (performance visibility)
- Feature flags for plugins (`config.json`)
- Estimated: 8–10 dev-days phased; PKI can start after Phase 1
