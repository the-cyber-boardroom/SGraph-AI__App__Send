### Non-Blocking — Ephemeral Compute Architecture

The architect review flags two medium-priority concerns for the Ephemeral Compute platform:

1. **VPC NAT Gateway default:** Recommend default to NO internet access from private nodes (opt-in). This reduces cost significantly for ephemeral stacks.
2. **External spec security:** `setup.sh` execution from third-party repos is a security risk. Safe for MVP (only own repos). Must be addressed before the spec ecosystem is opened to third parties.

These are not blocking for any current work stream — they are design decisions for future Ephemeral Compute development.
