# Debrief: Preview Thumbnails for PDF & Markdown in Gallery

**Version:** v0.16.26
**Date:** 20 March 2026
**Team:** Explorer
**Session:** `claude/start-explorer-team-gGRC0`

---

## Executive Summary

Gallery view now shows **real rendered thumbnails** for PDFs and Markdown files, not just generic icons. When a folder is uploaded, the browser generates visual previews of every file type — PDF first pages, markdown-rendered text, video frames, and image thumbnails — all stored in the `_preview/` folder inside the zip. Recipients see a rich visual gallery immediately on decryption.

This was a Phase 2 milestone: the `_preview/` infrastructure built in v0.2.12 (image thumbnails only) now supports **all document types**.

## What Was Delivered

### 1. Upload Side — PDF, Markdown & Video Thumbnail Generation (v0.2.13)

**File:** `sgraph_ai_app_send__ui__user/v0/v0.2/v0.2.13/_common/js/components/send-upload/send-upload-v0213.js`

**Three bugs fixed that prevented v0.2.13 from working:**

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| v0.2.13 code never executed | Overrode `_startFolderZip` which is a no-op since v0.2.3. Actual zip path is `_v023_compressFolder` | Override `_v023_compressFolder` instead |
| PDF.js failed to load | CDN URLs used ES module `.mjs` which doesn't set `window.pdfjsLib` | Switched to non-module `pdf.min.js` (v3.11.174) with multiple CDN fallbacks |
| Markdown thumbnails failed | SVG foreignObject approach is blocked by browser security (tainted canvas) | Rewrote to use direct Canvas API text rendering |

**What's generated per file type:**

| Type | Thumbnail Method | Output |
|------|-----------------|--------|
| **Image** | Canvas resize (v0.2.12) | JPEG, max 200px wide |
| **PDF** | pdf.js renders page 1 to canvas | JPEG, scaled to 200px wide |
| **Markdown** | Direct canvas text drawing with heading/list/code parsing | JPEG, 200x260px |
| **Video** | First frame capture at 1-second mark | JPEG, scaled to 200px wide |
| **Audio** | Duration metadata extraction | No visual thumbnail (metadata only) |

**Markdown canvas renderer** parses and renders: H1-H6 headings with underlines, code blocks with grey background, blockquotes with left bar, bullet/numbered lists, horizontal rules, and regular text with word wrapping. Inline formatting (`**bold**`, `_italic_`, `[links]()`) is stripped cleanly. Content truncated with a fade-out gradient.

## Architecture: How It Works End-to-End

```
SENDER (upload)                              RECIPIENT (gallery)

1. User selects folder                       5. Decrypt zip in browser
2. v0.2.13 generates thumbnails:             6. v0.2.8 finds _preview/ folder
   - Images → canvas resize → JPEG          7. Reads _manifest.json
   - PDFs → pdf.js page 1 → JPEG            8. Loads JPEG thumbnails
   - MDs → canvas text render → JPEG        9. Displays in gallery grid
   - Videos → frame capture → JPEG
3. Thumbnails stored in _preview/
   _preview/
     _manifest.json
     metadata/file-001.meta.json
     thumbnails/file-001.thumb.jpg
4. Zip encrypted + uploaded
```

**Key property:** The server never sees any of this. Thumbnails are generated client-side, encrypted inside the zip, and only visible after decryption. Zero-knowledge architecture fully preserved.

---

## Key Decision

**Canvas text rendering over SVG foreignObject for markdown thumbnails.** The SVG foreignObject approach produces prettier output (full CSS styling) but fails silently across browsers due to canvas tainting security restrictions. The canvas text approach is 100% reliable, produces readable thumbnails, and has zero external dependencies.
