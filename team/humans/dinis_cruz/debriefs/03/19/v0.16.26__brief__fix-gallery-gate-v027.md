# Brief: Fix Gallery Gate in v0.2.7 Download Overlay

**Version:** v0.16.26
**Date:** 2026-03-19
**Team:** Explorer
**Target file:** `sgraph_ai_app_send__ui__user/v0/v0.2/v0.2.3/_common/js/components/send-download/send-download-v027.js`

---

## Problem

The gallery page (`/gallery`) only works when **every file** in the zip is a "viewable" type (image, PDF, or markdown). If even one file is a `.txt`, `.json`, `.html`, or unknown type, the entire gallery falls back to the plain file-browser view.

### Root cause

The gate is in **v0.2.6** (`send-download-v026.js` line 105):

```js
if (!isGalleryPage() || !this._zipTree || !allFilesAreViewable(this._zipTree)) {
    return _v025_renderZipLayout.call(this, timingHtml, sendAnotherHtml);
}
```

`allFilesAreViewable()` calls `isGalleryCompatible()` which calls `getFileType()` — returns `'other'` for `.txt`, `.json`, `.html`, `.csv`, `.xml`, `.yaml`, `.log`, etc. If **any** file returns `'other'`, the gate fails and the whole gallery is skipped.

**v0.2.7** currently delegates to v0.2.6's `_renderZipLayout`, so it inherits this gate.

### What v0.2.5 did differently

v0.2.5 had a different gate: `allFilesAreImages()` — only images. v0.2.6 expanded it to images + PDF + markdown, but still requires ALL files to match.

---

## Fix: v0.2.7 should bypass v0.2.6's gate on gallery pages

### Strategy

In v0.2.7's `_renderZipLayout` override, **do NOT delegate to v0.2.6** when on a gallery page. Instead, render the gallery directly, handling all file types:

1. **Images** → thumbnail (existing behaviour)
2. **PDF** → PDF icon card (existing v0.2.6 behaviour)
3. **Markdown** → markdown icon card (existing v0.2.6 behaviour)
4. **Everything else** → generic document icon card with file extension badge

### Specific changes to `send-download-v027.js`

#### Change 1: Replace the `_renderZipLayout` override (around line 105)

Current code:
```js
SendDownload.prototype._renderZipLayout = function(timingHtml, sendAnotherHtml) {
    var origTree = this._zipTree;
    if (isGalleryPage() && this._zipTree) {
        this._zipTree = filterGalleryFiles(this._zipTree);
    }
    var html = _v026_renderZipLayout.call(this, timingHtml, sendAnotherHtml);
    // ...rest of function
```

New approach:
```js
SendDownload.prototype._renderZipLayout = function(timingHtml, sendAnotherHtml) {
    // On gallery pages, render directly — bypass v0.2.6's allFilesAreViewable gate
    if (isGalleryPage() && this._zipTree) {
        var files = filterGalleryFiles(this._zipTree);
        if (files.length > 0) {
            return this._v027_renderGallery(files, timingHtml, sendAnotherHtml);
        }
    }
    // Non-gallery pages: delegate to v0.2.6 as before
    var html = _v026_renderZipLayout.call(this, timingHtml, sendAnotherHtml);
    html = html.replace('>Save All</button>', '>Save locally</button>');
    // ...mode switcher injection (existing code)...
    return html;
};
```

#### Change 2: Add `_v027_renderGallery` method

This is essentially v0.2.6's gallery render code, but:
- Already receives filtered files (no `allFilesAreViewable` gate)
- Adds a generic "document" card type for unknown file types
- Includes the mode-switcher and "Save locally" text from existing v0.2.7

```js
SendDownload.prototype._v027_renderGallery = function(files, timingHtml, sendAnotherHtml) {
    var zipName  = this._zipOrigName || 'gallery';
    var sizeStr  = this.transferInfo ? this.formatBytes(this.transferInfo.file_size_bytes) : '';
    var self     = this;

    // Count by type — add 'other' category
    var counts = { image: 0, pdf: 0, markdown: 0, other: 0 };
    files.forEach(function(e) {
        var t = getFileType(e.name);     // from v0.2.6's scope
        if (counts[t] !== undefined) counts[t]++;
        else counts.other++;
    });

    var autoMode = files.length <= 14 ? 'large' : files.length <= 30 ? 'grid' : 'compact';

    // Generate cards — reuse v0.2.6's thumb HTML for known types,
    // add generic card for 'other' type
    var thumbsHtml = files.map(function(entry, idx) {
        var type = getFileType(entry.name);
        var ext  = (entry.name || '').split('.').pop().toLowerCase();

        if (type === 'image') {
            // Same as v0.2.6 image thumb
            return '<div class="v026-thumb" data-index="' + idx + '" data-type="image">' +
                '<div class="v026-thumb__img" id="v026-thumb-' + idx + '"></div>' +
                '<div class="v026-thumb__label">' + self.escapeHtml(entry.name) + '</div></div>';
        } else if (type === 'pdf') {
            // Same as v0.2.6 PDF thumb
            return '<div class="v026-thumb v026-thumb--doc" data-index="' + idx + '" data-type="pdf">' +
                '<div class="v026-thumb__icon v026-thumb__icon--pdf">' + ICON.pdf + '</div>' +
                '<div class="v026-thumb__label">' + self.escapeHtml(entry.name) + '</div></div>';
        } else if (type === 'markdown') {
            // Same as v0.2.6 markdown thumb
            return '<div class="v026-thumb v026-thumb--doc" data-index="' + idx + '" data-type="markdown">' +
                '<div class="v026-thumb__icon v026-thumb__icon--md">' + ICON.markdown + '</div>' +
                '<div class="v026-thumb__label">' + self.escapeHtml(entry.name) + '</div></div>';
        } else {
            // NEW: generic document card for unknown types
            return '<div class="v026-thumb v026-thumb--doc" data-index="' + idx + '" data-type="other">' +
                '<div class="v026-thumb__icon v026-thumb__icon--other">' +
                    '<span class="v027-ext-badge">' + self.escapeHtml(ext.toUpperCase()) + '</span>' +
                '</div>' +
                '<div class="v026-thumb__label">' + self.escapeHtml(entry.name) + '</div></div>';
        }
    }).join('');

    // Build meta label including 'other' count
    var metaParts = [];
    if (counts.image)    metaParts.push(counts.image + (counts.image === 1 ? ' image' : ' images'));
    if (counts.pdf)      metaParts.push(counts.pdf + (counts.pdf === 1 ? ' PDF' : ' PDFs'));
    if (counts.markdown) metaParts.push(counts.markdown + (counts.markdown === 1 ? ' doc' : ' docs'));
    if (counts.other)    metaParts.push(counts.other + (counts.other === 1 ? ' file' : ' files'));

    var hash = window.location.hash || '';
    var folderUrl = window.location.pathname.replace('/gallery', '/download') + hash;
    var switchBtn = '<a class="btn btn-sm btn-secondary v027-mode-switch" href="' + folderUrl + '" title="View as folder">' +
        ICON.folder + ' Folder view</a>';

    // Return the gallery HTML (same structure as v0.2.6 but with mode switcher)
    return '<div class="v026-gallery">' +
        '<div class="v026-gallery__header">' +
            // ... header left (icon, name, badge, meta, status) ...
            // ... header right (Save locally button, mode switch, info button) ...
        '</div>' +
        // ... size mode buttons ...
        '<div class="v026-grid" id="v026-grid" data-mode="' + autoMode + '">' + thumbsHtml + '</div>' +
        '</div>' +
        '<div id="zip-info-panel" ...>...</div>';
};
```

**Note:** The actual HTML structure should mirror v0.2.6's gallery output exactly (copy from v0.2.6 lines 119-175), just with:
- "Save locally" instead of "Save All"
- Mode switcher button added
- The `metaParts` including the "other" count

#### Change 5: Access to v0.2.6's `getFileType` function

The `getFileType` function is scoped inside v0.2.6's IIFE. v0.2.7 can't access it. Two options:

**Option A (preferred):** Duplicate `getFileType` in v0.2.7 — it's only ~10 lines:
```js
var IMAGE_EXTS = ['png','jpg','jpeg','gif','webp','svg','bmp','ico'];
var PDF_EXTS   = ['pdf'];
var MD_EXTS    = ['md','markdown'];
function getFileType(name) {
    var ext = (name || '').split('.').pop().toLowerCase();
    if (IMAGE_EXTS.indexOf(ext) !== -1) return 'image';
    if (PDF_EXTS.indexOf(ext) !== -1)   return 'pdf';
    if (MD_EXTS.indexOf(ext) !== -1)    return 'markdown';
    return 'other';
}
```

**Option B:** Use the global `FileTypeDetect` if available (loaded from `file-type-detect.js`).
