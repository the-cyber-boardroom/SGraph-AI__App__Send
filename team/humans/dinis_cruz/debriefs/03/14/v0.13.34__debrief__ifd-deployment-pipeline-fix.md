## Executive Summary

The download page on dev.send.sgraph.ai was broken because a stale `send-download.js` file in S3 `latest/` was the v0.2.1 override (not the v0.2.0 base class). This session diagnosed the root cause, fixed the deployment pipeline to deploy both v0.2.0 and v0.2.1 in order, added recovery tools (`--clean-latest`, `--rebuild-latest`), and documented the entire IFD deployment model.

## Key Decisions

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | Workflow deploys all versions in chain (v0.2.0 → v0.2.1), not just latest patch | Makes deployment idempotent — doesn't depend on prior S3 state |
| 2 | Override files must use version-suffixed names (`-v021.js`) | Prevents filename collisions that cause stale file ghosts in the overlay |
| 3 | `--rebuild-latest` replays from S3 release archives | Nuclear recovery option — but normal deploys use local git source (safer) |
| 4 | Guard clauses required in all IFD override files | Turns hard crashes into diagnosable warnings when base class is missing |
