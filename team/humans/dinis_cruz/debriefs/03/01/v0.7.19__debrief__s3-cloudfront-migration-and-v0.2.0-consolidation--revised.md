## Human Decisions Incorporated

All answers from the human's review have been incorporated:

1. **No backward compatibility needed** — no live customers. Only `send.sgraph.ai` domain is the stable anchor.
2. **Consistent flow sgraph.ai → send.sgraph.ai** — separate identity but natural transition. Wider page layout. Shared visual language.
3. **send.sgraph.ai = 100% file transfer** — clean, focused, fast. No docs, no how-tos, no team pages.
4. **Link back to sgraph.ai** — for documentation, how-tos, team explanations, pricing, etc.
5. **Wider page layout** — match sgraph.ai layout width and structure.
6. **API paths move to `/api/*`** — clean break, simplifies CloudFront to 2 behaviors.
7. **v0.2.0 IFD with progression design** — v0.2.0 code frozen after completion. Next changes go to v0.2.1, v0.2.2, etc. Components designed with overridable method boundaries.
8. **No IFD file size concern** — S3 hosting loads everything quickly.
9. **`latest/` + version path** — same as sgraph.ai and other sites.
10. **Version manifest file** — `manifest.json` lists all files for `latest/` copy. Helps CI pipeline.
11. **3 deployment environments** — dev.send.sgraph.ai (CD on dev), main.send.sgraph.ai (CD on main merge), send.sgraph.ai (manual trigger after QA).
12. **100% symmetric flow** — confirmed.
13. **Pre-render i18n** — en-gb base, all locale files in side-by-side folders, exact same model as sgraph.ai.
14. **Same S3 bucket, different prefix** — CF mapping per environment.
15. **CloudFront with two origins** — S3 (default) + Lambda (`/api/*`).

## Architecture Simplification

The revised architecture is significantly simpler than v1:

**CloudFront behaviors: from 10 to 2**

```
v1 (10 behaviors):
  /transfers/* → Lambda
  /presigned/* → Lambda
  /vault/*     → Lambda
  /join/*      → Lambda
  /info/*      → Lambda
  /mcp         → Lambda
  /mcp/*       → Lambda
  /set-cookie  → Lambda
  /docs*       → Lambda
  /openapi.json → Lambda
  *            → S3

v2 (2 behaviors):
  /api/*       → Lambda
  *            → S3
```
