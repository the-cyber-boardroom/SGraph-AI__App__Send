## Executive Summary

Three-role analysis (Architect, Dev, DevOps) of how to migrate ALL static pages from send.sgraph.ai to S3+CloudFront, keeping only API calls on Lambda. Combined with i18n support (15 locales) and v2.0 IFD consolidation focused on symmetric key encryption features only (vault/PKI/data rooms excluded).

**Key finding:** The separation is clean. The current codebase already has symmetric key, vault, and data room features architecturally distinct. The API client already uses relative URLs that work unchanged with CloudFront same-origin routing.

## Key Decisions for Review

### 1. Architecture: CloudFront with Two Origins
- **S3 (default):** All static content (HTML, CSS, JS, fonts, locale pages)
- **Lambda (explicit behaviors):** API endpoints only (/transfers/*, /presigned/*, /vault/*, /join/*, /info/*, /mcp*, /docs*)
- **Same origin advantage:** UI and API on same domain = no CORS needed

### 2. No `/api/` Prefix Change (Yet)
Current Lambda routes don't use `/api/` prefix. Changing would break existing clients. Instead, explicit CloudFront behaviors per API route prefix. Migrate to `/api/` prefix in a future version.

### 3. v2.0 = Symmetric Key Only
- **Include:** Upload, download, preview, transfer history, access gate, transparency, 15-locale i18n
- **Exclude:** Vault (vault.html, vault-api, vault-crypto), Data Rooms (join, room), SSH keygen, card assets
- **API surface:** 18 endpoints (down from ~44)

### 4. IFD Version: v2/v2.0/v2.0.0/
Major version because: collapses 9 IFD layers, changes serving architecture, removes features, adds i18n. Not an incremental v0.1.x.

### 5. Pre-rendered i18n (Same as sgraph.ai)
Build-time pre-rendering for page-level text + embedded `<script>` block for Web Component translations. No runtime JSON fetching. Same 15 locales as sgraph.ai.

### 6. Same S3 Bucket, New Prefix
Reuse existing `745506449035--static-sgraph-ai--eu-west-2` bucket. New prefix: `websites/send-sgraph-ai/`.

## What's Hard

1. **IFD consolidation of send-download** — 740 lines of monkey-patch overrides on a 400-line base. 5 layers to merge into one coherent class.
2. **IFD consolidation of send-upload** — 5 layers of overrides, multipart upload logic, token validation flow.
3. **13 new locale translation files** — each with ~160 keys to translate.
