## What's New vs v0.20.17

### Phase 2: Theme Control (Full Object)

The `"theme"` field now accepts an object (v2) or a string (v1 backward compat):

```json
"theme": {
  "mode":    "dark",
  "accent":  "#00d4aa",
  "font":    "mono",
  "density": "comfortable"
}
```

| Field | What it does |
|-------|-------------|
| `mode` | `"dark"`, `"light"`, `"auto"` (auto = light) |
| `accent` | Sets `--plr-accent` CSS var → nav active indicator, image border |
| `font` | `"sans"`, `"serif"`, `"mono"`, `"system"` |
| `density` | `"compact"`, `"comfortable"`, `"spacious"` — controls section padding |

### Phase 2: Component-Level Styling Props

**Hero:**
- `height`: `"small"` (150px) / `"medium"` (250px) / `"large"` (400px) / `"full"` (100vh)
- `align`: `"left"` / `"center"` / `"right"` — text alignment in overlay
- `overlay`: `"gradient"` / `"dark"` / `"light"` / `"solid"` / `"none"`

**Section:**
- `layout`: `"full-bleed"` / `"narrow"` (640px) / `"wide"` (1200px)
- `background`: any CSS colour, or `"alt"` for subtle alternating shade
- `divider`: `"line"` (default) / `"space"` (no line, extra padding) / `"none"`

**Image:**
- `align`: `"left"` / `"center"` / `"right"`
- `border`: `true` — 1px accent-coloured border
- `shadow`: `true` — subtle drop shadow
- `rounded`: `true` — 8px border radius

**Gallery:**
- `aspect`: `"16:9"` / `"4:3"` / `"1:1"` / `"auto"`
- `gap`: `"none"` / `"small"` / `"medium"` / `"large"`
- `show_captions`: explicit boolean (default: show when captions array is non-empty)

**Slides:**
- `controls`: `"bottom"` (new default — nav below image) / `"top"` (old behaviour)
- `transition`: `"fade"` (CSS opacity) / `"none"`
- `autoplay`: `true` — auto-advances every 3 s; stops on first user click

### Phase 3: New Components

**`cards`** — navigable card grid for hub pages:
```json
{
  "type": "cards",
  "props": {
    "columns": 2,
    "items": [
      { "title": "...", "description": "...", "image": "path.jpg", "link": "subfolder/" }
    ]
  }
}
```
Whole card is clickable; navigates to `link` using the same folder/file navigation as the rest of the Browse UI.

**`columns`** — side-by-side layout:
```json
{
  "type": "columns",
  "props": { "ratio": "1:2", "gap": "medium", "vertical_align": "top" },
  "children": [
    { "type": "image", "props": { "file": "crop.jpg" } },
    { "type": "text",  "props": { "content": "..." } }
  ]
}
```
Ratios: `1:1`, `1:2`, `2:1`, `1:3`, `3:1`. Stacks to single column below 768px.

### Source View Improvements

- **Copy button** (`⎋ Copy JSON`) in source bar — copies raw JSON to clipboard with `✓ Copied` / `✗ Failed` feedback. Falls back to `execCommand` for older browsers.
- **`_page.json` direct click** now opens the tab in **source view** by default (the intent of clicking the file is to read the JSON). Folder click and auto-open still open rendered view.

---

## Backward Compatibility

All new props are optional with sensible defaults. Existing `_page.json` files (v1 — `"theme": "dark"` string) render identically. No migration needed.

---

## Quick Reference: The v2 Example from the Proposal

The root `_page.json` from the proposal (`"theme"` object + `"cards"` + `"layout": "wide"`) will now render correctly. The agent authoring `_page.json` can write:

```json
{
  "title": "SG/Send Strategic Pack",
  "theme": { "mode": "dark", "accent": "#00d4aa", "font": "sans", "density": "comfortable" },
  "components": [
    { "type": "hero", "props": { "title": "...", "height": "large", "align": "center" } },
    { "type": "section", "props": { "title": "Articles", "layout": "wide" },
      "children": [
        { "type": "cards", "props": { "columns": 2, "items": [ ... ] } }
      ]
    }
  ]
}
```

## What Remains from the Proposal (Not Implemented)

| Item | Reason deferred |
|------|----------------|
| Source view line numbers | Nice to have; requires wrapping each line in a `<span>` — adds complexity, lower priority |
| `hero_style` theme prop (top-level gradient behaviour) | Redundant with component-level `overlay` prop on `hero` |
| `"auto"` mode inheriting Browse shell dark/light | Requires reading shell CSS variables at runtime; deferred to future session |
| `accordion` component | Not in v2 proposal scope |
