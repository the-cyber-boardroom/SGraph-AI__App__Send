## What Was Delivered

### 1. Content-Hashed Gallery Folders (`_gallery.{hash}/`)

The gallery folder now includes a hash of all file content hashes in its name. The hash was already being computed but wasn't used — now it is. This enables deduplication and cache-friendliness.

**Critical fix:** Gallery folder is now **only generated for gallery delivery**. When users choose "Browse" or "Download", the zip is clean — zero modifications to uploaded files (apart from existing dot-file/empty-dir cleanup).

### 2. Print-Friendly Markdown (`SgPrint` Module)

New `SgPrint` utility module (194 lines) opens a professional A4 print preview window with:
- SG/Send branded toolbar with Print and Close buttons
- Clean A4 styling with proper margins, headings, code blocks, tables
- Cross-browser paper size options (removed Chrome-limiting `@page size: A4`)

### 3. Unified Delivery UX — Three Cards Always Visible

Upload Step 2 now always shows Browse → Gallery → Download. Smart defaults (gallery for folders, download for single files), consistent ordering, proper card hover states.

### 4. Folder View Shows Everything

Removed ALL filtering from folder tree view. The folder view now shows exactly what's in the zip — including `_gallery.{hash}/` if present. This is correct: what you see is what you download.

## Key Decisions

1. **`_gallery.{hash}/` not `_gallery/`** — content-addressable, already had the hash computed
2. **Gallery-only generation** — folder delivery = zero modifications to user files
3. **No folder filtering** — folder view is a faithful representation of the zip contents
4. **PDF generation deferred** — belongs in a future gallery version with user-controlled options (briefing document created)
