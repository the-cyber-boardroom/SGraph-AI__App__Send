## Executive Summary

Four Explorer roles (Librarian, Architect, Dev, Designer) analysed the v0.10.14 dev brief for building vault.sgraph.ai as a pure HTML/JS application on top of send.sgraph.ai. The unanimous finding across all roles: **the existing infrastructure is sufficient for MVP with zero server-side changes**.

### The Big Insight

The existing transfer API (`/transfers/create` + `/transfers/upload` + `/transfers/complete` + `/transfers/download`) already provides everything the vault needs. A vault is just a collection of encrypted transfers — the server has no idea it's hosting a vault. This aligns perfectly with the brief's "server knows nothing" principle.

## Key Decisions (Consensus Across Roles)

| # | Decision | Rationale |
|---|----------|-----------|
| D-1 | **Zero server-side changes for MVP** | Existing transfer API suffices. Path-based storage deferred to Phase 2 to avoid leaking structure to server. |
| D-2 | **User Lambda only (not Admin Lambda)** | User Lambda is public-facing, has transfer endpoints, uses token auth (not cookies). Avoids CORS complexity. |
| D-3 | **PBKDF2-SHA256, 600k iterations** | Native Web Crypto API — no WASM dependency. Argon2 upgrade path for Phase 3. |
| D-4 | **Symmetric encryption only (no RSA)** | Single vault key → KDF → AES-256-GCM. Eliminates PKI dependency. Sharing = sharing the key. |
| D-5 | **Reuse v0.2.0 SendComponent patterns** | Shadow DOM, separated CSS/HTML/JS, design tokens. More modern than admin's light DOM approach. |
| D-6 | **17 locales from day one** | Reuse existing i18n infrastructure (locale JSONs, generate_i18n_pages.py, URL-based routing). |
| D-7 | **Flag-free language selector** | Use language names, not flags (one flag ≠ one language). Culturally inclusive. |

## Multi-Cultural Design Highlights

- **Text expansion budgets** — layout tested with German (30-40% longer than English)
- **RTL ready** — CSS Logical Properties throughout (margin-inline-start, not margin-left)
- **Locale-aware formatting** — `Intl.NumberFormat` for file sizes, `Intl.DateTimeFormat` for dates
- **~90 translatable strings** catalogued with pluralisation support
- **Accessibility** — WAI-ARIA TreeView, keyboard navigation, WCAG AA contrast
- **Responsive** — mobile-first, 5 breakpoints, tree as bottom-sheet drawer on mobile
