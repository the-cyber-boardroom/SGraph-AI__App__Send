# v0.19.5 Debrief: Presigned API Rename — `abort` → `cancel`

**Version:** v0.19.5 | **Date:** 28 March 2026 | **Session:** Explorer — API terminology cleanup

---

## Summary

Renamed the presigned multipart upload "abort" operation to "cancel" across the entire API surface — backend routes, service layer, event names, status values, all UI versions, and tests. This is a **breaking change to the HTTP API path** (`/api/presigned/abort/` → `/api/presigned/cancel/`). The new vault presigned endpoints were built with `cancel` from the start.

---

## What Changed

### API Endpoint

```
BEFORE:  POST /api/presigned/abort/{transfer_id}/{upload_id}
AFTER:   POST /api/presigned/cancel/{transfer_id}/{upload_id}
```

No change to request/response format — only the URL path.

### Backend (Python)

| Location | Before | After |
|---|---|---|
| `Routes__Presigned.py` — route path | `/api/presigned/abort/{transfer_id}/{upload_id}` | `/api/presigned/cancel/{transfer_id}/{upload_id}` |
| `Routes__Presigned.py` — method name | `abort__transfer_id__upload_id()` | `cancel__transfer_id__upload_id()` |
| `Service__Presigned_Urls.py` — method name | `abort_multipart_upload()` | `cancel_multipart_upload()` |
| `Service__Presigned_Urls.py` — event action | `multipart_aborted` | `multipart_cancelled` |
| `Service__Presigned_Urls.py` — return status | `{ "status": "aborted" }` | `{ "status": "cancelled" }` |

### What Was NOT Changed

The **AWS S3 SDK call** `self.s3.client().abort_multipart_upload()` was left as-is — that's the AWS API method name, not our terminology. This is an internal implementation detail that the CLI team never interacts with.

## Impact on CLI Team

### If you call the presigned abort endpoint today

Update the URL path:

```python
# Before
POST /api/presigned/abort/{transfer_id}/{upload_id}

# After
POST /api/presigned/cancel/{transfer_id}/{upload_id}
```

### If you use the `Vault__API` client

The new vault presigned endpoints use `cancel` from the start:

```
POST /api/vault/presigned/cancel/{vault_id}
```

### Response format change

```json
// Before
{ "transfer_id": "abc123", "status": "aborted" }

// After
{ "transfer_id": "abc123", "status": "cancelled" }
```

If your code checks `status == "aborted"`, update it to `status == "cancelled"`.

### Event log entries

If you inspect transfer metadata events, the action value changed:

```json
// Before
{ "action": "multipart_aborted", ... }

// After
{ "action": "multipart_cancelled", ... }
```
