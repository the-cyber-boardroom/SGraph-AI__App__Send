**File:** `send-download.js`

## Workflow Reflection — Claude Web + SG/Send Vaults

This session is a milestone for how the agentic team operates. The entire v0.3.0 IFD consolidation — a complex multi-file refactoring across 4 components, a P0 bug fix, CI pipeline rewrite, and full audit of 17 IFD overlays — was executed by Claude Web (claude.ai) using sg-send-cli vaults as the collaboration channel.

### What worked

**Vaults as the patch handoff layer.** Claude Web wrote code, generated `git format-patch` patches, and pushed them to an encrypted vault. The human pulled patches from the vault, reviewed the diffs in GitHub, and applied with `git am`. This gives the human full control over what lands on dev — Claude never pushes directly to the repo.

**Claude Web doing work that Claude Code couldn't finish.** The previous Claude Code session timed out after 232 tool calls trying to write the upload orchestrator. Claude Web picked up the session handoff docs, read the full context (CLAUDE.md, reality document, role files, gap analysis, architect review), and completed the remaining work across multiple patches — including finding and fixing the P0 bug that Claude Code had introduced.

**The session handoff pattern works.** The previous session left detailed handoff docs (`IFD__Send_UI__SESSION-HANDOFF__v030-non-upload-parity.md` and `v0.16.36__review__v0-3-0-non-upload-gap-analysis.md`) with exact file paths, line numbers, and reference overlays. This meant zero ramp-up time — Claude Web could start producing patches immediately.

**Feature audit grounded in reality.** By reading every v0.2.x overlay header and cross-referencing against v0.3.0 code, the audit caught issues (M1–M4) that the previous gap analysis had either missed or marked as "already present." The reality document discipline forced honest assessment.

### What could improve

**Vault key hygiene.** Vault keys were initially included in the debrief (now sanitised). Keys should always be shared out-of-band, never committed to the repo — even in docs. This is the same principle as the zero-knowledge guarantee: sensitive material stays off the server.

**Patch testing before merge.** Claude Web can syntax-check JS files but can't run the full app in a browser. The human has to do the functional testing. A future improvement would be a CI step that runs the patches through Playwright smoke tests before merge.

**The v0.2.x → v0.3.0 transition should have been caught earlier.** The `FriendlyCrypto.resolve()` bug (0016) was introduced when the v0.3.0 rewrite was first written — possibly by an LLM that hallucinated the API. The lesson: when rewriting code that calls external modules, verify the actual method signatures exist, don't assume from memory.

### The bigger picture

This workflow — human briefs → agent reads context → agent produces patches → vault transfer → human reviews and merges — is becoming a reliable pattern. The vault layer means Claude Web (which has no git access) can participate in the same development workflow as Claude Code sessions. The patches are standard `git format-patch` output, reviewable in any diff tool, and applied with standard `git am`. Nothing proprietary in the pipeline.

The 18-role team structure means each session operates with clear boundaries. This session was pure Dev work — no architecture decisions, no product direction, no security reviews. The Conductor's session handoff routed the right work to the right role with the right context.
