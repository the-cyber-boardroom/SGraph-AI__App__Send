## Architecture Summary

**Before:** 1 monolithic file (7,221 lines) with 15 IFD overlays chain-mutating prototypes

**After:** 22 focused files:
```
send-upload/
  send-upload.js          (~800 lines — orchestrator + upload engine)
  send-upload.css          (184 lines — orchestrator styles)
  upload-crypto.js         (120 lines — friendly key utilities)
  upload-file-utils.js     (160 lines — file type detection)
  upload-step-select/      (3 files — Step 1: file selection)
  upload-step-delivery/    (3 files — Step 2: delivery mode)
  upload-step-share/       (3 files — Step 3: share mode)
  upload-step-confirm/     (3 files — Step 4: confirmation)
  upload-step-progress/    (3 files — Step 5: progress)
  upload-step-done/        (3 files — Step 6: results)
```

**Communication pattern:** Properties down, CustomEvents up (bubbles + composed for Shadow DOM crossing)

**Key design decisions:**
- Each sub-component uses Shadow DOM for style encapsulation
- CSS loaded via `<link>` in shadow root (custom properties cascade through)
- HTML templates fetched once, cached on class static property
- No build step required — vanilla JS, no bundler
- Upload engine stays in orchestrator (not split into sub-components)
- Carousel timer managed by orchestrator, passes current message as property
