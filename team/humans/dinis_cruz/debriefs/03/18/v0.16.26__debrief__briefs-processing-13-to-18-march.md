## What Was Processed — High-Level Themes

### Theme 1: Vault Architecture Overhaul (03/17 — dominant theme)

The vault-redesign session (11 documents) produced a complete architectural overhaul:
- **Self-describing file IDs** — `{type}-{derivation}-{mutability}-{hex_id}` eliminates dual-write
- **Sub-tree model** — one tree per directory, on-demand loading, vault open in 3 requests
- **Four-layer security** — Modes A (full zero-knowledge), B (structural visibility), C (signed plaintext), D (pure VCS)
- **11-phase CLI rewrite** — ~300 LOC added, ~370 removed (net smaller, more capable)
- **Three-client convergence** — CLI, browser, UI all on one crypto model (PBKDF2 passphrase)

**Architect verdict:** Sound. Self-describing IDs are the right abstraction. Sub-tree model proven by simulation with real vault data.

### Theme 2: PKI & Vault Positioning (03/14)

Five positioning documents define SG Vault as "Git with integrated trust":
- Four PKI operating modes (anonymous → countersigned)
- Three-layer collaboration model (remote → local → clone)
- SG Vault Hub vision (GitHub for encrypted vaults, change packs)
- User keys external to vault (never stored in `bare/`)

### Theme 3: Upload UX & Distribution (03/16)

Messaging pivot: "share large files and folders" (not "encrypted transfer"):
- Three-step flow (upload → distribution → credentials)
- Three sharing modes (simple token, full link, separate key)
- 10GB upload limit, pricing at 5 GBP = 10GB upload + 20GB bandwidth
- ASCII mockups, state machines, component inventory in team response

### Theme 4: Revenue & Use Cases (03/13 + 03/16)

### Theme 5: New Capabilities (03/17–03/18)
