## Summary

The gallery view for image-only zip transfers is fully working end-to-end. This session added view mode controls (compact/grid/large) with smart auto-selection based on image count.

## What Was Delivered

### 1. Gallery View — Working End-to-End
- **Thumbnail grid** renders when visiting `/gallery/` with an image-only zip
- **Lightbox** opens on click with full-size image, prev/next navigation (arrows + keyboard), counter (e.g. "2 / 4")
- **Save All** downloads the complete zip; **Save Image** (lightbox) downloads individual images
- **Keyboard shortcuts:** Arrow keys / j/k navigate, Escape closes, 's' saves

### 2. View Mode Toggle (NEW — this session)
- Three view modes accessible via toggle buttons in the gallery header:
  - **Compact** — `minmax(120px)`, 1:1 aspect ratio, smaller labels — best for 9+ images
  - **Grid** (default) — `minmax(180px)`, 4:3 aspect ratio — best for 3-8 images
  - **Large** — `minmax(300px)`, 16:10 aspect ratio — best for 1-2 images
- **Smart auto-selection:** view mode automatically chosen based on image count (<=2: large, 3-8: grid, 9+: compact)
- User can override at any time via the toggle buttons
- `align-content: start` fixes the empty-space issue — thumbnails pack to the top

- SVG icons for each mode: 3x3 grid (compact), 2x2 grid (grid), stacked rows (large)
- Follows existing v0.2.5 surgical overlay pattern — no new files needed
