# Debrief — Vault Key Compatibility: Workspace ↔ Vault Alignment

**Version:** v0.10.47
**Date:** 4 March 2026
**Session type:** Explorer
**Branch:** `claude/vault-project-review-89StF`

---

## Executive Summary

This session fixed three bugs that prevented the workspace (workspace.sgraph.ai) from correctly sharing vaults with the vault UI (vault.sgraph.ai). The root cause: the vault key format changed from 3-part (`passphrase:vault_id:settings_transfer_id`) to 2-part (`passphrase:vault_id`) in v0.1.1, but several components were not updated.

**After this fix:** the same vault key works identically on both vault.sgraph.ai and workspace.sgraph.ai. A vault created on either site can be opened on the other.

---

## What Was Fixed

### Bug 1: vault-app.js `_updateVaultKey` splice corruption (CRITICAL)

**Files:** `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.0/_common/js/components/vault-app/vault-app.js` and `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.1/_common/js/components/vault-app/vault-app.js`

**Problem:** `_updateVaultKey()` used `parts.splice(-2, 2, vaultId, settingsId)` to reconstruct the vault key. For a 2-part key like `mypass:abc123`:
- `parts = ['mypass', 'abc123']`
- `splice(-2, 2, ...)` replaces ALL parts → `['newVaultId', 'settingsId']`
- **Passphrase is lost.** The key becomes `vaultId:settingsId` — unusable.

**Fix:** Replaced with `this._vault.getVaultKey()` — the same clean pattern already used in vault-shell.js (which was fixed earlier).

### Bug 2: Workspace missing sg-vault-crypto.js (BLOCKING)

**File:** `sgraph_ai_app_send__ui__workspace/v0/v0.1/v0.1.0/index.html`

**Problem:** The workspace loaded 3 vault libraries (sg-send-crypto, sg-send, sg-vault) but was missing `sg-vault-crypto.js`. The vault UI loads 4 libraries — sg-vault-crypto must come before sg-vault because `SGVault.open()` calls `SGVaultCrypto.parseVaultKey()` and `SGVaultCrypto.deriveKeys()` internally. Without it, the workspace would throw `SGVaultCrypto is not defined` at runtime when opening any vault.

**Fix:** Added the missing script tag in the correct load order position.

### Bug 3: Workspace stale 3-part key references (UX)

**File:** `sgraph_ai_app_send__ui__workspace/v0/v0.1/v0.1.0/components/vault-panel/vault-panel.js`

**Problem:** Comment and input placeholder said `passphrase:vault_id:settings_transfer_id`, which would confuse users into thinking they need a 3-part key. The v0.1.1 format is 2-part: `passphrase:vault_id`.

**Fix:** Updated comment (line 6) and placeholder (line 303) to `passphrase:vault_id`.

## Key Technical Context (for other sessions)

### Vault Key Format (v0.1.1)

The vault key is **2 parts**: `passphrase:vault_id`

- **passphrase** — user-chosen, used for PBKDF2 key derivation (600k iterations → AES-256-GCM key)
- **vault_id** — hex ID of the vault's root transfer on send.sgraph.ai

The old 3-part format (`passphrase:vault_id:settings_transfer_id`) is obsolete. `SGVaultCrypto.parseVaultKey()` in v0.1.1 expects exactly 2 parts.

### How Vault ↔ Workspace Sharing Works

Both sites:
1. Load the **same** sg-vault.js v0.1.1 from `static.sgraph.ai` CDN
2. Talk to the **same** `send.sgraph.ai` backend for encrypted storage
3. Use `SGVault.open(sgSend, vaultKey)` which:
   - Parses vault key → extracts passphrase + vault_id
   - Fetches encrypted vault settings from send.sgraph.ai (by vault_id)
   - Derives AES-256-GCM key from passphrase via PBKDF2
   - Decrypts vault tree → folder structure + file references

Same key = same vault = same files. The server only stores encrypted blobs.

### The `_updateVaultKey` Pattern

Three components had this method:
- **vault-shell.js** — already fixed (uses `vault.getVaultKey()`) ✅
- **vault-app.js** — fixed in this session ✅
- **vault-browser.js** — was always fine (just emits an event, doesn't reconstruct) ✅
- **workspace vault-panel.js** — never had this method (passes key through cleanly) ✅

The correct pattern is always: `this._vault.getVaultKey()`. Never reconstruct manually.

### Workspace Architecture (for reference)

The workspace (`sgraph_ai_app_send__ui__workspace/v0/v0.1/v0.1.0/`) is a complete document transformation tool:
- **vault-panel.js** — encrypted file browser (uses SGVault)
- **document-viewer.js** — multi-format renderer (markdown, code, HTML, images, PDF)
- **llm-chat.js** — prompt input + streaming LLM responses
- **llm-connection.js** — OpenRouter + Ollama provider management
- **prompt-library.js** — 5 built-in prompt templates
- **workspace-shell.js** — 5-zone layout orchestrator
- **Debug panels** — messages, events, API logger

All communication via EventBus (`window.sgraphWorkspace.events`). All vault operations via SGVault library loaded from CDN. All LLM calls are client-side (zero-knowledge).
