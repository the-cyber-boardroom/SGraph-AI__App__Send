### New Pattern: Role Activity Logs

Each role now has `team/roles/{role}/activity-log.md` — a version-pinned, commit-pinned table tracking all reviews. This solves the "where did we leave off?" problem:
- Architect, Dev, Historian, Journalist, Librarian all have activity logs
- Each entry records version, commit hash, link to review, and description
- New sessions read the activity log to know exactly where to resume
