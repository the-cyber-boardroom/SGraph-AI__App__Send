## Summary

The QA team completed a full review of v0.3.0, capturing **66 issues** across 7 documents. The Architect triaged all 66 and approved **17 low-risk fixes** for the release. All 17 have been implemented and pushed. **47 issues deferred** (architecture changes, features, high-risk modifications).

## Fixes Implemented

### Security (P1)

| # | Issue | Fix |
|---|---|---|
| **U-030** | "Link + key separate" email leaked the decryption key — defeated the entire purpose of separate mode | `_openEmailLink()` now checks `_shareMode`: separate mode uses `linkOnlyUrl` only, token mode uses friendly token URL, combined uses combined URL |

### Stability (P1)

| # | Issue | Fix |
|---|---|---|
| **FT-019** | Gallery crashes on large uploads (~160 MB with videos) and was the default delivery mode | Changed default from `'gallery'` to `'browse'` in `upload-file-utils.js`. Gallery still available as option. |

### Regressions Restored (G-001, G-002, U-001)

| # | Issue | Fix |
|---|---|---|
| **G-001** | Markdown lightbox had no styled typography — just browser defaults in a narrow white box | Added `width: 70vw`, `box-shadow: 0 8px 32px rgba(0,0,0,0.5)`, `padding: 2.5rem 3rem`, `max-width: 900px`. Full descendant typography (tables, code, blockquotes, headings, links, hr) was added in the earlier session commit. |
| **G-002** | Lightbox missing Print and Share buttons from v0.2.x | Print button added (earlier commit) with SgPrint for markdown. Share/Copy Link button added with clipboard feedback ("Copied!" for 2s). |
| **U-001** | File/Text toggle was chunky bordered buttons instead of v0.2.x compact pills | Removed border, reduced padding to `0.375rem 0.75rem`, reduced font to `0.8rem`. |

### CSS Quality Fixes

| # | Issue | Fix |
|---|---|---|
| **U-003** | Footer version nearly invisible (0.7 * 0.6 = 0.42 opacity), took two lines | Version now `display: inline` on same line, opacity raised to 0.8, padding reduced to `0.75rem`. |
| **U-010** | "Encrypt & Send" button text unreadable on hover — lost contrast | `.btn-primary:hover` now explicitly sets `color: var(--color-bg, #0A1628)` to maintain dark text on teal. |
| **D-001** | Single text/code files shown on dark background (#0d1117) | Changed `.sb-file__code` to `background: #fafafa; color: #1a1a1a` — light background matching browse view markdown. |
| **FT-013** | Gallery print rendered with dark background | Added `.sg-gallery { background: white !important; }` to existing `@media print` block. |

### Upload Wizard Logic

| # | Issue | Fix |
|---|---|---|
| **U-017** | Access Token bar and Test Files visible during all wizard steps — dangerous during encrypt (clicking "Change Token" loses progress) | Hidden once wizard advances past Step 1 idle. Re-shown on back navigation to Step 1. |
| **U-021** | Two "Email Link" buttons on Done page (header + body) | Removed the in-body email button from `upload-step-done.js`. Header button remains. |
| **U-028** | Two Encrypt buttons on Confirm page (header + body) with different labels | Removed the in-body "Encrypt & Send" button from `upload-step-confirm.js`. Header "Encrypt & Upload →" button remains. |
| **U-019** | Email text awkward — "Secure file for you" repeated, no context | Rewritten: "I've sent you an encrypted file via SG/Send." with appropriate link per share mode. |
| **U-020** | Email used raw hash URL instead of the share mode user selected | Token mode now uses friendly token URL. Separate mode uses link-only URL. |
| **D-006b** | Email `mailto:` opened in same tab, navigating away | Added `'_blank'` target to `window.open()`. |

### Browse View

| # | Issue | Fix |
|---|---|---|
| **FT-016** | Files unsorted in folder tree — numbered files (1-5) appeared in reverse/random order | Added `localeCompare` with `{ numeric: true }` for natural alphanumeric sort. |
| **U-006** | Dropping files outside the drop zone opened them in browser tab | Added document-level `dragover` and `drop` `preventDefault()` to `index.html`. |
