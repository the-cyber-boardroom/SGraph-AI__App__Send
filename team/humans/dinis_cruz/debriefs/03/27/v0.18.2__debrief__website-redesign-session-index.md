## 6. Tools Repo Integration Strategy

**File:** `../../../../../roles/dev/reviews/03/27/v0.18.2__dev-review__tools-repo-integration-strategy.md`

How sgraph.ai relates to `SGraph-AI__Tools` (`https://github.com/the-cyber-boardroom/SGraph-AI__Tools`).

**Core principle:** One place for shared code. sgraph.ai is master for the top/bottom banner. __Tools is master for tools and `<sg-public-viewer>`. All sites consume via `static.sgraph.ai` (pinned versions).

**Key decisions:**
- Tools (SSH Keygen, File Encryptor, Key Generator, Vault Browser etc.) are a significant credibility asset — link to them from the main site nav
- `<sg-site-header>` and `<sg-site-footer>` to be built here and published to __Tools
- All new Web Components follow IFD versioning (`v0/v0.1/v0.1.0/component.js`)
- `static.sgraph.ai` is the production pin channel — CI verifies version availability before deploy
- tools.sgraph.ai nav needs updating to match new sgraph.ai nav

## Key Decisions Made This Session

| Decision | Rationale |
|----------|-----------|
| North star: "encrypted + browsable + server never sees" | No competitor offers all three |
| Hero: "Share files. Browse them. Nobody can read them. Not even us." | Adds the differentiator ("browse them") to the proven line |
| Nav adds Tools, removes Architecture and Team | Tools is credibility; Architecture/Team are wrong audience |
| "Already have a token? →" in header | Highest-volume visitor who wants to USE the product, not learn |
| Six-sentence privacy policy visible on homepage | Marketing weapon, not just legal document |
| Test counts removed from homepage | Span multiple repos, number misleading without context |
| Proton Drive = client-side E2EE too | Encryption is NOT our differentiator vs Proton — browsing UX is |
| /agents/ → qa.send.sgraph.ai | Wrong audience on marketing site |
| /architecture/ → /security/ (301) | Right content, wrong name and framing |
| Deploy /security/ BEFORE removing /architecture/ | SEO continuity |
| Web Components with IFD versioning throughout | One place for shared code |
| sgraph.ai owns master banner; publishes to __Tools | Header/footer consistency across all SGraph properties |
| qa.send.sgraph.ai is the proof layer | Every sgraph.ai claim links to QA site for verification |
| Screenshots: Dinis will share via SG/Send vault | Human task — not in code |
