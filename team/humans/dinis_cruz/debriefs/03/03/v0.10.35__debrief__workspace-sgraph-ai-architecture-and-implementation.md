## Key Findings

### 1. No Backend Changes Needed (Phase 1)

All 10 vault API endpoints the workspace needs already exist. The workspace is purely a new frontend that calls existing APIs.

### 2. Link, Don't Copy

Shared JS/CSS files are **imported at runtime** from `vault.sgraph.ai`, not copied into the workspace repo. This ensures all `.sgraph.ai` sites run the same version of shared components. Can pin to specific release URLs if needed.

| Category | Count | Notes |
|----------|-------|-------|
| **Linked from vault.sgraph.ai** | 10 files (~2,500 lines) | event-bus, messages-service, vault-crypto, vault-api, file-type-detect, markdown-parser, debug panels, CSS, fonts |
| **Adapted** (new files, inspired by existing) | 3 components | workspace-shell, vault-panel, document-viewer |
| **Built new** | 5 components (~2,050 lines) | llm-connection, llm-chat, llm-provider, prompt-library, manifest-manager |

### 3. LLM Strategy: OpenRouter + Ollama (APPROVED)

OpenRouter gives browser-friendly access to all major models (Claude, GPT-4, Gemini, plus free models for testing) through one API. Ollama provides free local inference for maximum privacy.

### 4. Source/Transform Folder Pairs with Manifest

Each workspace project uses a folder pair (source/ + transform/). Both documents are independently editable and versionable. A manifest file per pair tracks which source version produced which transformation. Version-by-filename.

### 5. Phase 1 File Types: Text, HTML, Markdown

Over-indexing on three types that cover 90%+ of use cases, each with a different preview mode. JSON, CSV, images, PDF come later.

### 6. Main Shell Left-Nav

The workspace shell has a main navigation (like the admin shell) that loads different panels/views. The document transformation view is one panel — more views will be added over time.
