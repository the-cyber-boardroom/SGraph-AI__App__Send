## Executive Summary

Vault v0.1.1 is implemented and committed. This is a **major milestone** — the first time the admin shell architecture pattern has been extracted and reused in another product (vault). The session also fixed a critical S3 deployment bug where all CI environments were deploying to the same folder.

---

## What's New (Firsts)

### 1. First Admin Shell Pattern Reuse

The admin console's 3-column layout (v0.1.3) was successfully extracted and adapted for the vault. This proves the pattern is reusable across products — the same Light DOM shell, EventBus, debug sidebar architecture now powers both admin and vault.

**Components adapted:** vault-shell, EventBus, ConfigManager, MessagesService, messages-panel, events-viewer, api-logger.

### 2. First Multi-Format File Preview

Vault can now preview files inline without downloading:
- Markdown (rendered with security-first parser in sandboxed iframe)
- Images (blob URL rendering)
- PDF (embedded iframe)
- Code (line-numbered with language detection)
- Audio and Video (native HTML5 controls)
- Text (plain pre-formatted)
- Binary (fallback with file info)

### 3. First Cross-Product Module Sharing

Three modules were copied from send.sgraph.ai into vault:
- **FileTypeDetect** — file type detection by extension
- **MarkdownParser** — security-first markdown-to-HTML
- **JSZip** — vendor library

This validates the Architect's Tier 1/2/3 module classification and the phased sharing strategy (D3: copy now, shared folder later, npm package future).

### 4. First Hierarchical Tree View

Vault-tree-view replaces the flat table browser with a collapsible folder tree — the first tree navigation component in the SGraph UI family.

### 5. First Full Drag-and-Drop Upload Overlay

Vault-upload-dropzone provides a full-area overlay for drag-and-drop file uploads, with path-aware targeting (files go to the currently selected folder).

### 6. First Localhost Development Demo

The vault v0.1.1 was built and tested entirely on localhost — no cloud infrastructure required. This demonstrates the offline-first development capability that is a core architectural advantage of the zero-knowledge design. **Important for investor narrative:** the entire product works offline.

### 7. First Branch-Isolated S3 Deployments

Fixed a critical CI/CD bug: all three deployment workflows (vault, send, website) were deploying to the same S3 base folder regardless of branch. Now each environment gets its own isolated path:

```
websites/sgraph-vault/dev/     ← dev branch
websites/sgraph-vault/main/    ← main branch
websites/sgraph-vault/prod/    ← production
```

This applies across all three sites (vault, send, website).

## Key Decisions Made

1. **Light DOM shell + Shadow DOM children** — vault-shell is Light DOM (like admin-shell) while vault-entry, vault-upload, vault-share stay Shadow DOM for style isolation
2. **Copy patterns, not code inheritance** — admin shell is not designed for inheritance; extracting patterns is safer and faster
3. **Branch-based S3 isolation** — dev/main/prod each get their own S3 prefix, preventing accidental overwrites
