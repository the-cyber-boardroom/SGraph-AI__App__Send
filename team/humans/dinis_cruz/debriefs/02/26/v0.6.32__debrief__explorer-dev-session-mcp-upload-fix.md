## Deliverables

### 1. MCP Upload Fix (Code)

**Problem:** Claude.ai MCP uploads were intermittently corrupted (`size: 0`) because the upload tool had no explicit parameter for the encrypted payload. File names were also lost because the SGMETA envelope wasn't being applied.

**Fix:** Added `data: str = ''` parameter to `transfers_upload` — MCP clients pass base64-encoded encrypted payload as an explicit tool argument. Browser uploads completely unaffected (parameter defaults to empty).

## Key Decisions

1. **Added `data` parameter instead of changing the MCP transport** — keeps backward compatibility, explicit tool argument is more reliable than request body interception
2. **SGMETA wrapping stays client-side** — the filename must be encrypted with the content (server never sees it), so wrapping happens before encryption in the MCP client, same as the browser
3. **Kept `unwrap_mcp_payload` as fallback** — older MCP clients that send `{"data": "<base64>"}` in the request body still work
