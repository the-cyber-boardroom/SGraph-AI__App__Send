## What Happened Today

The timeline on 26 February 2026:

1. **Morning:** `data` parameter fix deployed (v0.6.32) -- unblocked MCP uploads
2. **Morning:** SGMETA documentation added to Sherpa guide
3. **Afternoon:** Claude.ai Sherpa Agent generates PDF, encrypts it, uploads via MCP, emails download link via n8n to AWS WorkMail
4. **Afternoon:** You receive the email, click the link, browser decrypts in 251ms, PDF renders inline with filename preserved

## Key Decisions

5 decisions recorded by the Historian (D-M008-001 to D-M008-005):

1. **`data` parameter instead of MCP transport change** -- simple, backwards-compatible, immediately proven
2. **SGMETA wrapping stays client-side** -- server never sees filenames, zero-knowledge preserved
3. **Composable MCP tools** -- Sherpa Agent uses send.sgraph.ai MCP + n8n MCP in a single workflow
4. **SGMETA documented for AI agents** -- Claude.ai understood the binary format and implemented it correctly
5. **n8n as email bridge** -- email delivery via MCP tool call, no custom code
