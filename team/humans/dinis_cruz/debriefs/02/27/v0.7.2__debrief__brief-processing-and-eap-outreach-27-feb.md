## Key Decisions Needed

### 1. MCP Security Talk — Discrepancy Flag
The MCP security speaker notes describe several patterns as "in production" that **do not exist in code**: two-stage Security Analyst, state machine execution control, dynamic permissions, containerized isolation, signed Issues-FS. The Explorer team recommends either:
- (a) Building the Security Analyst gate first to make the core claim accurate, or
- (b) Reframing the talk language from "deployed" to "designed and in progress"

### 2. One-Time Secret Links
The outreach brief (Brief 2) depends on `/secret` endpoints for one-time access code delivery. These do not exist. Decision needed: build `/secret` first, or use existing room invite links for the initial campaign?

### 3. FastAPI WorkMail — Separate Service or Part of SG/Send?
The spec recommends Pydantic (conflicts with Type_Safe rule). Is this a separate project with its own rules, or part of the SG/Send codebase?

### 4. Automation Approval Thresholds
The Sherpa inbox brief asks: at what point can the Sherpa act without human approval? GRC sign-off required. What is the initial bar?
