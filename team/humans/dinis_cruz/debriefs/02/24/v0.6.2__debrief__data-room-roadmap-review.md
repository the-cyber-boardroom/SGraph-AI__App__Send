## Key Decisions for Dinis

The Architect identified 4 decisions that should be resolved before Phase 3 implementation:

| # | Decision | Recommended | Alternative |
|---|----------|-------------|-------------|
| 1 | Data Room = vault with metadata overlay? | **Yes** (reuse vault infrastructure) | New first-class entity (more code, cleaner separation) |
| 2 | Key hierarchy (room_key → folder_key → file_key)? | **Yes** (HKDF derivation, adding user = 1 wrap operation) | Keep per-file wrapping (simpler but O(n) on user add) |
| 3 | Audit trail in Memory-FS or dedicated store? | **Memory-FS for MVP** (NS_AUDIT namespace) | S3 append-only log (better for production) |
| 4 | Session management for room recipients? | **Room-scoped session token** (lightweight, no OAuth) | Full JWT/OAuth (overkill for MVP) |
