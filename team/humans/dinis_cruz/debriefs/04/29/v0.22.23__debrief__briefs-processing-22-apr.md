The observability pipeline spans this repo, a CLI repo, an AMI bake pipeline, and AWS infrastructure. No document currently maps which repo owns which component. The Architect review recommends producing this map before any implementation begins.

**Action:** Architect/Conductor to produce component ownership map before observability implementation sprint starts.

### 2. Security Cleaning Is an Attack Surface (High Risk)

Stage 1 (security cleaning) processes full MITM proxy logs containing bearer tokens, API keys, and PII. A missed regex pattern means plaintext secrets in Elastic. The Architect review recommends AppSec sign-off before production use.

**Action:** AppSec review of Stage 1 design before any real production logs are processed.

### 3. Open Decision #25 Still Open: WhatsApp Backup — Build or Document?

The WhatsApp brief explicitly says "I do not want to build this, but somebody should build it." The decision on whether to build a reference implementation vs. write a developer guide for integrators is still open.

**Action:** Conductor/Alchemist decision on scope.

### 4. Open Decision #26: S3 vs Vault for Raw Log Storage

High-volume raw logs → S3 (pragmatic). Sensitive logs with request/response bodies → vault (privacy). The arch brief defers the per-source decision.

**Action:** Architect + DevOps per-source decision before Stage 0 implementation.

### 5. Open Decision #27: Lens Visualisation Experiment

Lens is the modern Kibana visualisation type. Current programmatic dashboard creation uses legacy types. The dev brief requests an experiment before committing to production dashboard templates.

**Action:** Dev to run experiment; confirm feasibility before dashboard template design.
