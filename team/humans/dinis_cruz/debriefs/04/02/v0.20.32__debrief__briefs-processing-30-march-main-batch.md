## Executive Summary

March 30 was the LLM infrastructure design day. Six substantive documents plus a daily brief define how the LLM tooling should evolve — not incrementally, but architecturally.

The central insight is the **one-shot feedback loop**: instead of appending every LLM response to a growing chat history, each cycle extracts the valuable findings, prunes the stale content, and produces a refined bundle of roughly constant size. The context does not explode. A standard 10-message chat reaches ~15,000 tokens; a one-shot loop cycle stays at ~500–600 tokens throughout.

The breakthrough connection: **the bundle lives in a vault**. Each cycle is a commit. Each exploration is a branch. This means SGit — which we built for encrypted file storage — is simultaneously the correct infrastructure for managing LLM conversation state. The `sg-git-graph` component (also proposed in this batch) makes both SGit vault histories and LLM conversation threads visible as interactive branch graphs, using the same component and the same data model.

Two other significant items:
- **Chat component stabilisation** (five bug fixes + Model Chooser) — Villager-appropriate work that should be done soon
- **OWASP presentation** — a submission opportunity for the community, with Ambassador leading
