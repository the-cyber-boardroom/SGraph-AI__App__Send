## Key Decisions to Review

**Decision 1 — PLR context banner still missing (P1)**
The designer review identifies that new users don't understand they're looking at an AI-generated rich-media page. A `type: "banner"` block showing page metadata was identified as P1 but not yet implemented. This is the highest-priority UX debt in the PLR.

**Decision 2 — Two theme selectors, not synced**
There is now a theme selector in edit mode AND a theme selector on the toolbar. If a user changes the theme from the toolbar (no edit mode) and then opens edit mode, the edit-mode selector shows "— keep current —" rather than the actual current theme. Dev review flags this as "cosmetically confusing but not a bug." Is this acceptable, or should the two controls be synced?

**Decision 3 — Profile page still unbuilt (P0)**
Doc 246 (08 Apr) identified the profile page + credit activation as the immediate blocker for the £5 credit outreach to ~50 users. This has been carried forward through two master indexes. No session has picked it up. Should this be the next explicit session target?

**Decision 4 — Vault large file: E2E test gap**
The `GET /api/vault/presigned/read-url/{vault_id}/{file_id}` endpoint has unit tests. The full round-trip (client calls presigned endpoint → fetches from S3 → decrypts client-side) has not been E2E-tested. This is a gap if large file support is being relied on in production.
