## Vault Branching: Now Complete

The vault now has a proper version-control-like UX for web editing:

| Action | UX |
|--------|----|
| **Edit files in the browser** | Commits land on your clone branch (not named branch) |
| **Push** (teal button, ↑N) | Appears when you have ahead commits. Fast-forwards the named branch. |
| **Pull** (blue-teal button, ↓N) | Appears when you're behind the named branch. Fast-forwards your clone. |
| **Refresh** (↺ button) | Remounts the file tree from server HEAD (all vault types) |

This is the same push/pull model as git, implemented transparently in the browser. The clone branch ref is HMAC-derived from the passphrase (deterministic, no key management overhead).

## PLR Context Banner: What It Is

The April 10 session documented the banner *placement* fix (outside the card frame, `908d9dd`). This session adds the banner *feature origin* (`40ea89d`):

- New `type: "banner"` block in `_page.json` — explicit banner content
- **Auto-banner:** automatically inserts at the top of any page whose first block is not a hero or banner — new users see a page title strip even with zero authoring effort
- `page.meta`: `{ author, date, tags, badge }` — optional metadata shown in the auto-banner
- `"banner": false` at page root: suppresses completely
