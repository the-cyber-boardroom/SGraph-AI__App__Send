## Key Findings

### The session manager gap
The VNC brief (doc 314) identifies a "new FastAPI service with Docker socket access" for session management. This service is more complex than the SG/Send Lambda and has no prior blueprint. The Architect flags it as the most complex new service in the batch. Do not start implementation without a dedicated architecture document.

### Evidence packs as immediate demo — no new code needed
The evidence pack vault use cases (doc 316) can be created TODAY using existing infrastructure. An Ambassador agent session with a vault key can create a news story vault and a risk acceptance vault using sgit. These would be the "show, don't tell" demos the website needs. High value, zero engineering cost.

### Villager video hardening is unblocked
Doc 322 (video capture hardening) is a Villager brief for an already-existing feature. It does not depend on any of the other 10 briefs. The Villager team can start immediately. Web Worker viability (Open Decision #21) is the only research dependency.

### Google app:data proof of concept exists but is not shipped
Doc 317 notes "the proof of concept is working" for client-side Google OAuth + app:data read/write. This is a working prototype, not a shipped product. Open Decision #22 (cross-domain OAuth) must resolve before the prototype becomes a product feature.
