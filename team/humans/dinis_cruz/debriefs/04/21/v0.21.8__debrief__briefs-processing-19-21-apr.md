## Key Decisions Needing Your Review

### 1. Auth MVP: Cognito Dropped — Vault Key as Credential (Doc 297)

The Architect has traced through four design positions and landed on:
- **No Cognito, no Firebase, no master password**
- Vault key stored in the browser's native password manager (Credential Management API)
- One component: `<sg-vault-picker>` (~150 lines vanilla JS)
- Google sign-in demoted to optional vault personalisation

**Verdict:** Architecturally sound. Zero-knowledge guarantee is stronger than any Cognito-based design. Zero new server code required.

**Six questions need your input before Stage 2 ships** (documented in doc 297 §13):
1. Should shared-vault auto-save to password manager be suppressed for one-time shares?
2. `sg1.` prefix on stored credential — yes or no?
3. Recovery UX at vault creation (Sherpa to design)
4. Multiple accounts in shared browser profile — just document in UX?
5. AppSec threat-model review of form fallback pattern
6. Vault name leakage to Google/Apple password manager — acceptable?
