## Key Code Delivered (All Code-Verified)

### 1. `sg-site-header` Web Component — v1.0.4 through v1.0.6

The sgraph.ai site header is now a Shadow DOM Web Component. Three rapid versions shipped in one session:
- **v1.0.4:** Replaces all 5 inline `<header>` elements. Slot-based: `slot='cta'` (token bar) and `slot='locale'` (locale picker).
- **v1.0.5:** Self-configuring profiles — `site="Send"` is all a page needs. Nav, token bar, home URL, active-link detection all automatic.
- **v1.0.6:** Environment-aware cross-site links — links from `dev.sgraph.ai` to tools stay on `dev.tools.sgraph.ai`, not `tools.sgraph.ai`.

Path: `sgraph_ai__website/v0/v0.2/v0.2.0/_common/js/components/sg-site-header/v1/v1.0/{v1.0.4,v1.0.5,v1.0.6}/`

### 2. 10 Homepage Section Web Components

All homepage sections are now isolated Shadow DOM components with separate `.js`, `.css`, `.html` files:

| Component | Section |
|-----------|---------|
| `sg-send-hero` | Hero with inline token input |
| `sg-vault-patterns` | Human/Agent/Agent-Agent pattern cards |

| `sg-vault-primitives` | Three agentic primitives (browser automation, encrypted VFS, code execution sandbox) |
| `sg-use-cases` | Five use-case rows with real screenshots |
| `sg-tools-section` | Browser AI tools cards |

**Critical fix:** All components now declare `static jsUrl = import.meta.url` — required for `SgComponent` to resolve `.html`/`.css` templates from the component's location, not the base class's CDN host.

## Architect Flag for Your Attention

The Architect review flags a non-obvious requirement: any `SgComponent` subclass **must** declare `static jsUrl = import.meta.url`. Without it, template fetches fail silently with 403 errors. This should be added to the `SgComponent` extension guide or enforced with a runtime check. See [Architect Review](../../../../../roles/architect/reviews/04/24/v0.22.6__architect-review__code-delivery-23-24-apr.md) for full detail.
