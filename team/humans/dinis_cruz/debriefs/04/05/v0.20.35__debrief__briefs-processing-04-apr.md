## Executive Summary

Today's Librarian session catalogued the 04/04 batch — **backend infrastructure and agentic capabilities day**. Three design documents, one architecture pattern formalisation, and three technical reference documents. No code changes. All deliverables are briefs-processing outputs (catalogue, role reviews, reality document update).

**The three design threads:**

1. **Backend storage restructured** with a typed, flat `by-id-from__*` grouping hierarchy. Transfers move from a flat sea of IDs to a logically organised structure that enables per-user cleanup, cost attribution, and vault alignment — without any client-side changes.

2. **Agentic LLM component suite designed** — six components forming a complete browser-native agentic loop. Gemma 4 (released 2 April 2026, Apache 2.0) is the primary target, with native trained-in tool calling. The `sg-sandbox` component is the most security-sensitive addition to the platform to date.

3. **Deterministic-safe-paths pattern formalised** as a reusable architecture primitive. This pattern was already in use informally in SGit and storage code. Formalising it with a shared function, salt registry, and implementation checklist means future features can be built against it with predictable security properties.
