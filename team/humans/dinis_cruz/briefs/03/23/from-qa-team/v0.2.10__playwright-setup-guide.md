# Running Playwright + Chromium in Claude Code Sessions

**Version:** v0.2.10

Guide for setting up headless browser automation in a Claude Code web environment.
Validated on 2026-03-23 in a Claude Code session on this repo.

---

## Why This Is Non-Obvious

Claude Code sessions run on Linux containers with Python 3.11 as the default.
Playwright needs Chromium system libraries that aren't pre-installed, and some
packages (like `sgraph-ai-app-send`) require Python 3.12+. This guide covers
both problems.

---

## Step 1: Create a Python 3.12 Virtual Environment

The default Python is 3.11, but 3.12 is available at `/usr/bin/python3.12`.

```bash
python3.12 -m venv /tmp/venv312
source /tmp/venv312/bin/activate
python --version  # Should show Python 3.12.x
```

**Important:** Every subsequent command must be prefixed with
`source /tmp/venv312/bin/activate &&` since shell state doesn't persist
between tool calls in Claude Code.

---

## Step 2: Install Python Dependencies

```bash
source /tmp/venv312/bin/activate && pip install -r requirements.txt
```

This installs Playwright (Python package), pytest, FastAPI, Pillow, etc.

---

## Step 3: Install Chromium + System Libraries

This is the key step — `--with-deps` installs the OS-level libraries
(libasound, libfreetype, etc.) that Chromium needs:

```bash
source /tmp/venv312/bin/activate && python -m playwright install --with-deps chromium
```

This takes ~30-60 seconds. It downloads the Chromium binary and installs
system packages via apt.

**Only install `chromium`** — no need for firefox or webkit.

---

## Step 4: Install SG/Send (for v030 Local-Stack Tests)

The v030 tests run against a local in-memory SG/Send server. The app isn't
on PyPI, so clone it from GitHub:

```bash
git clone --depth 1 -b dev https://github.com/the-cyber-boardroom/SGraph-AI__App__Send /tmp/sgraph-send-ref
```

Install it without full dependency resolution (some deps aren't on PyPI),
then install the required deps individually:

```bash
source /tmp/venv312/bin/activate && pip install -e /tmp/sgraph-send-ref --no-deps
source /tmp/venv312/bin/activate && pip install osbot-fast-api osbot-fast-api-serverless fastapi-mcp mgraph-ai-service-cache mgraph-ai-service-cache-client issues-fs-cli
```

Verify the import works:

```bash
source /tmp/venv312/bin/activate && python -c "from sgraph_ai_app_send.lambda__user.testing.Send__User_Lambda__Test_Server import setup__send_user_lambda__test_server; print('OK')"
```

---

## Step 5: Run the Tests

```bash
# Single test file
source /tmp/venv312/bin/activate && python -m pytest tests/qa/v030/test__access_gate.py -v --tb=short

# All v030 tests
source /tmp/venv312/bin/activate && python -m pytest tests/qa/v030/ -v --tb=short
```

Expected result for access_gate: 3 passed, 1 skipped (the "no gate" test
skips because the test server has the gate enabled).

---

## What Happens Under the Hood

1. **`send_server` fixture** — starts a real SG/Send FastAPI server on a random port with in-memory storage
2. **`ui_server` fixture** — builds a static UI directory from the `sgraph_ai_app_send__ui__user` package, serves it with `python -m http.server` on localhost:10062
3. **`browser` fixture** — launches headless Chromium via Playwright
4. **`page` fixture** — creates a fresh browser page per test
5. **`screenshots` fixture** — captures PNGs via CDP (`Page.captureScreenshot`) and saves them to `sg_send_qa__site/pages/use-cases/{use_case}/screenshots/`

The tests navigate the real UI, interact with the real API, and capture
real screenshots — no mocks, no patches.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `Package requires Python >=3.12` | Use `python3.12 -m venv`, not the default python |
| `No module named 'osbot_fast_api'` | Run the `pip install` line from Step 4 |
| `No module named 'fastapi_mcp'` | Same — install deps individually |
| `playwright install` hangs | Add `--with-deps chromium` (not bare `install`) |
| Tests skip with "not installed" | SG/Send import failed — check Step 4 |
| Screenshots not captured | Tests skipped — check `ui_server` fixture logs |
| `UI directory not found` | The UI package version changed — check `UI_VERSION` in conftest.py |
