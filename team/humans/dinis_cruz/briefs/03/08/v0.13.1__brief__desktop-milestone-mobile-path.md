## Milestone: SGraph Desktop is Live

The desktop app went from "let's explore options" (this morning's brief) to **signed, running, and downloadable** in the same day. Built with Tauri, deployed via GitHub Actions, Apple code-signed.

**What shipped:**
- macOS desktop app ("SGraph Desktop 3")
- Apple Gatekeeper approved — "Chrome downloaded it for malicious software and none was detected"
- Full SG/Send UI running inside the app (v0.13.0, UI v0.2.0)
- Left sidebar with navigation: Send, Vault, Grid/Tools, Settings
- 224 credits remaining visible, "Change Token" button, language selector (EN-GB)
- Test files section with drag-to-upload capability
- Status bar: "https://send.sgraph.ai — Active — Ready"

**Why Tauri, not Electron:**
- App size: ~10-15MB (Tauri) vs ~150MB+ (Electron)
- Uses the OS native WebView (WebKit on macOS, WebView2 on Windows) instead of bundling Chromium
- Rust backend — faster, more secure, lower memory footprint
- Same codebase targets desktop AND mobile (iOS + Android)

## Mobile App: How Hard Is It?

### Answer: Not Very — Tauri 2.0 Has Native Mobile Support

Tauri 2.0 (stable since late 2024) supports iOS and Android as first-class targets from the same codebase. Since SGraph Desktop is already a Tauri app, adding mobile is an extension, not a rewrite.

### What's Needed

**Step 1: Initialise mobile targets (< 1 hour)**

```bash
cd sgraph-desktop
npx tauri android init    # Creates Android project structure
npx tauri ios init         # Creates iOS project structure (macOS only)
```

This scaffolds the native project files (Xcode project for iOS, Gradle project for Android) and configures the Tauri build pipeline to target mobile.

**Step 2: Run on simulator/device**

```bash
npx tauri ios dev          # Launch in iOS simulator
npx tauri android dev      # Launch in Android emulator
```

Hot-module replacement works on mobile — frontend changes live-reload on the device/emulator.

**Step 3: Adapt the UI for mobile**

This is where the real work is. The current SG/Send UI is designed for desktop-width screens. On mobile:

| Element | Desktop | Mobile Adaptation Needed |
|---|---|---|
| Left sidebar navigation | Vertical icon bar | Bottom tab bar or hamburger menu |
| Drop zone | Large drag-and-drop area | "Browse File" button (no drag-and-drop on mobile) |
| File list | Horizontal cards | Vertical list |
| Language selector | Top-right dropdown | Settings screen |
| Upload progress | Inline bars | Full-screen progress view |

The Web Components architecture helps here — each component can query its container size and adapt its layout. An `<sg-file-manager>` component that renders as a horizontal card grid on desktop and a vertical list on mobile is one component, not two.

**Step 4: Mobile-specific features**

| Feature | What It Enables |
|---|---|
| **Camera upload** | Take a photo → encrypt → upload. Tauri has a barcode scanner plugin; camera access is available. |
| **Share sheet integration** | "Share via SG/Send" appears in the iOS/Android share menu. Select a file from any app → encrypted upload. |
| **Biometric auth** | Use Face ID / fingerprint to unlock the vault or confirm a send. Tauri has a biometric plugin. |
| **Push notifications** | "Your file was downloaded" notifications. |
| **Offline queue** | Encrypt files while offline, upload when connectivity returns. |

**Step 5: Distribution**

| Platform | Distribution |
|---|---|
| **iOS** | TestFlight for beta → App Store for production. Apple Developer Program required ($99/year). |
| **Android** | APK direct download for beta → Google Play Store for production. Google Developer account ($25 one-time). |
| **Both** | GitHub Actions CI/CD (same pipeline as desktop, extended with mobile build targets) |

### Estimated Effort

| Phase | Effort | What You Get |
|---|---|---|
| **Basic mobile build** | 1-2 days | SG/Send running on iOS simulator + Android emulator. Full functionality but desktop layout. |
| **Responsive UI** | 3-5 days | Mobile-adapted layout, touch-optimised interactions, bottom navigation. |
| **Mobile-specific features** | 1-2 weeks | Share sheet, biometric, camera upload, push notifications. |
| **App Store submission** | 2-3 days | Screenshots, descriptions, review process (Apple review can take 1-3 days). |

The basic mobile build could be running by end of day if someone starts now. The responsive UI adaptation is the bulk of the work.

### What's Free from the Desktop Investment

Because the Tauri desktop app already exists:

- **Build pipeline:** GitHub Actions CI/CD is set up. Adding mobile targets extends the existing pipeline.
- **Code signing:** Apple signing is working. The same Apple Developer account handles iOS.
- **Rust backend:** Any Rust code written for desktop (local storage, crypto helpers, etc.) works on mobile.
- **Web UI:** The entire SG/Send frontend (HTML, JS, CSS, Web Components) runs unchanged in the mobile WebView. Only layout adaptations needed.
- **Tauri plugins:** Same plugin API on desktop and mobile. A plugin written for desktop (e.g., filesystem access) works on mobile with platform-specific implementations in Swift (iOS) or Kotlin (Android).

## Connection to Today's Briefs

| Brief | How Desktop/Mobile Connects |
|---|---|
| **Web Components architecture** | Components that adapt to container size work on desktop AND mobile without code changes |
| **sg-layout** | The fractal layout framework works in any WebView — desktop or mobile. Panel sizes adapt to screen. |
| **Chrome extension** | The extension handles desktop browsers. The Tauri app handles desktop native. The mobile app handles phones. Three form factors, one UI codebase. |
| **Translation engineering** | The `<sg-translations>` service component works identically in desktop and mobile WebViews. |
| **Identity/PKI** | Biometric auth on mobile (Face ID, fingerprint) could be the strongest identity verification — "this device is mine, and my face unlocks it." |
