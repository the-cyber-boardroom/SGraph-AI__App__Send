## The Strategic Case for a Mobile App

The mobile app isn't just about being on another platform. It's three things:

1. **A new monetisation channel.** App Store and Google Play enable direct purchase (£5 for the app, with £5 of credits included). This is simpler than Stripe for casual users.
2. **A credibility signal.** "We have apps on iOS, Android, desktop, and web" is a fundraising and enterprise sales story. It signals maturity.
3. **Unique mobile use cases.** Camera capture, share sheet integration, biometric auth, and on-the-go secrets management are capabilities that only exist on a phone.

The implementation cost is low because Tauri 2.0 builds iOS and Android from the same codebase we already have. The web loads fast (CDN-based, all static). The question isn't "can we build it?" — it's "what use cases justify it?"

## What the App Does on Day One (Current Features Only)

Even with zero new features, the current SG/Send web UI on a phone provides:

| Feature | Mobile Experience |
|---|---|
| **Upload a file** | "Browse File" button → pick from Photos, Files, or camera roll → encrypted → uploaded |
| **Upload a photo/video** | Take a photo → encrypt → share via link. One-tap secure photo sharing. |
| **Receive a file** | Open a link → view/download in the app. Preview PDFs, images, videos. |
| **Friendly keys** | Type `apple-mango-56` on a phone keyboard — designed for mobile input |
| **One-time secrets** | Open a one-time link → see the secret → it's consumed. Perfect for mobile. |
| **Multiple languages** | pt-PT, en-GB, etc. — the same i18n system works in the app |

The killer mobile moment: **someone sends you a confidential file via WhatsApp link. You tap it. SG/Send opens in the app. You see the file. No download to your camera roll. No copy sitting in your email. It was in the app, encrypted, and now it's gone.** That's a use case that doesn't work as cleanly in a browser.

## Monetisation Models

### Option A: Paid App with Included Credits (Recommended for Launch)

- **Price:** £4.99 on App Store / Google Play
- **Includes:** £5 worth of credits (5GB at £1/GB)
- **Effect:** Zero-friction purchase. Buy the app = ready to use. No separate Stripe checkout needed.
- **Apple/Google take:** 30% on first year, 15% thereafter. So £4.99 → ~£3.50 net. But the user is worth more in retention.

### Option B: Free App with In-App Credit Purchase

- **Price:** Free download
- **Credits:** Buy via in-app purchase (£4.99 for 5GB, £9.99 for 12GB, etc.)
- **Effect:** Lower barrier to download. But adds friction at purchase time.

### Option C: Free App, Credits via Web Only

- **Price:** Free download
- **Credits:** Purchase only via send.sgraph.ai (Stripe). No in-app purchase.
- **Effect:** Avoids Apple/Google 30% cut entirely. But disconnects purchase from app experience.

**Recommendation:** Start with **Option A** (paid app, included credits) for the "just works" experience. Add in-app credit purchases later. Keep web/Stripe as the primary channel for power users who want more credits.

**The provisioning problem is solved for mobile:** Unlike web (where we had the post-purchase gap), the App Store handles everything. User downloads → app opens with credits → ready to go. No tokens to enter, no welcome pages, no confusion.

## Unique Mobile Capabilities

Features that only make sense or work best on a phone:

| Feature | What It Does | Why Mobile |
|---|---|---|
| **Camera → Encrypt → Share** | Take a photo, one tap to encrypt and share | Camera is native to mobile |
| **Share sheet integration** | "Share via SG/Send" appears in iOS/Android share menu | OS-level integration |
| **Biometric vault access** | Face ID / fingerprint to unlock vault and confirm sends | Phone has biometrics |
| **QR code scanning** | Scan a QR code to receive a file (instead of typing a link) | Phone has camera |
| **Push notifications** | "Your file was downloaded" / "New file received" | Phone is always with you |
| **Mobile-only viewing** | Sender restricts: "can only be opened on a mobile device" | Device attestation |
| **Offline queue** | Encrypt files while offline, auto-upload when connected | Mobile goes offline frequently |
| **NFC sharing (future)** | Tap phones to exchange encrypted files | Tauri has NFC plugin |

### Share Sheet Is the Killer Feature

If "Share via SG/Send" appears in the iOS share menu:

1. User is in any app (Photos, Files, WhatsApp, etc.)
2. Taps Share → selects "SG/Send"
3. File is encrypted and uploaded
4. User gets a link to share
5. Done — never left the original app

This makes SG/Send as accessible as AirDrop but works over the internet, with encryption, with anyone (not just Apple users).

## Competitive Landscape: Mobile File Sharing

### Direct Competitors on Mobile

| App | Free Tier | Paid | E2E Encrypted | Mobile App | Key Differentiator |
|---|---|---|---|---|---|
| **WeTransfer** | 2GB, 7-day expiry | $6.99-$25/mo (up to 200GB) | Partial (TLS + AES, but server can access) | iOS + Android | Simplicity, brand recognition |
| **Smash** | Unlimited size, free | Paid for priority | No E2E | iOS + Android + Mac | No file size limits on free tier |
| **Proton Drive** | 5GB storage | From €3.99/mo | Yes (true E2E) | iOS + Android | Privacy brand (Proton Mail ecosystem) |
| **Tresorit** | 5GB | From $10/mo | Yes (true E2E) | iOS + Android | Enterprise compliance (HIPAA, GDPR) |
| **Internxt** | 10GB | From $1.49/mo | Yes (true E2E, open source) | iOS + Android | Open source, no file size limits |
| **FileMail** | 5GB | From $10/mo | No E2E | iOS + Android | Large file support |
| **pCloud** | 10GB | From $49.99/year | Optional (Crypto add-on) | iOS + Android | Lifetime plans, crypto folder |
| **SG/Send** | 5 free downloads | Pay-per-use (£1/GB) | **Yes (true zero-knowledge, browser-side)** | **Desktop (shipped) + Mobile (building)** | **Zero-knowledge, pay-per-use, open source** |

### Where SG/Send Wins

1. **True zero-knowledge.** WeTransfer, Smash, and FileMail encrypt in transit (TLS) but the server CAN see your files. SG/Send encrypts in the browser — the server literally never sees plaintext. Only Proton Drive, Tresorit, and Internxt match this, and they're all subscription-based.

2. **Pay-per-use.** No subscription. No monthly fee. Buy credits, use them. This is unique in the market. Every competitor is either free-with-limits or subscription.

3. **Open source.** The code is auditable. Proton and Internxt are also open source — this is the trust tier. WeTransfer, Tresorit, pCloud are not.

4. **No account required for basic use.** Like WeTransfer, you can use SG/Send without creating an account. But unlike WeTransfer, your files are truly encrypted.

### Where SG/Send Needs to Catch Up

1. **Brand recognition.** WeTransfer has massive awareness. We have zero.
2. **File size limits.** Smash has no size limit on free. We're at 5GB.
3. **Mobile apps.** Every competitor has them. We're building ours now.
4. **Storage/sync.** Proton Drive, pCloud, Tresorit offer cloud storage + sync. We offer transfers + vaults (different model but overlapping use cases).

### The WeTransfer AI Controversy (July 2025)

WeTransfer updated their ToS to allow using user files for AI training, triggering a boycott. They reversed the policy, but the trust damage is done. This is a positioning opportunity: **"Your files are encrypted before they leave your device. We can't train AI on files we can't see."**

## App Store Presence: What to Prepare

### iOS App Store

| Item | What's Needed |
|---|---|
| **Apple Developer Account** | $99/year (may already have for desktop signing) |
| **App name** | "SG/Send" or "SG Send — Encrypted File Sharing" |
| **Category** | Productivity or Utilities |
| **Screenshots** | 6.7" (iPhone 15 Pro Max), 6.1" (iPhone 15), 5.5" (iPhone 8 Plus) |
| **Description** | 4000 chars max. Lead with: "Send files encrypted in your browser. The server never sees your data." |
| **Privacy label** | "Data Not Collected" (our strongest signal — we literally don't collect user data) |
| **App Review** | 1-3 days. Encryption requires an export compliance declaration (straightforward for AES-256). |

### Google Play Store

| Item | What's Needed |
|---|---|
| **Google Developer Account** | $25 one-time |
| **App name** | Same as iOS |
| **Category** | Tools or Productivity |
| **Screenshots** | Phone + tablet sizes |
| **Data safety** | "Data not shared with third parties", "Data encrypted in transit", "Data not collected" |

### The Privacy Label Is Our Weapon

Both App Store and Google Play require privacy disclosures. Most file sharing apps disclose: device identifiers, usage data, email address, etc. SG/Send can truthfully declare: **"Data Not Collected."** Zero-knowledge means zero data collection. This is visible to every user who views the app listing — and it's a massive differentiator.
