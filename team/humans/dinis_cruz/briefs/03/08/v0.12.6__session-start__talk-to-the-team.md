# SGraph Send — Talk to the Team

**For:** External visitors, friendly users, and collaborators who want to speak directly with the SGraph Send agentic team  
**How:** Open a new Claude Code session, paste the bootstrap prompt below, and the Conductor will greet you and connect you to whichever agent you want to talk to

---

## What Is This?

SGraph Send is built and maintained by an 18+ strong agentic team, each agent owning a specific role: Sherpa (user experience), AppSec (security), Architect (technical design), Dev (implementation), Ambassador (growth), and many more.

This session lets you talk directly to any of them — grounded in the actual codebase, the latest briefs, and the real state of the product.

---

## ⚠️ Important — Use Claude Web, Not Claude Desktop

**We strongly recommend doing this on [claude.ai](https://claude.ai) (Claude Web) rather than Claude Code on the desktop.**

Claude Web runs in a sandboxed browser environment. Claude on the desktop (Claude Code via terminal) has direct access to your local filesystem, shell, and running processes — which is powerful for developers working on the codebase, but far more than a visitor conversation requires.

For talking to the team, Claude Web gives you everything you need with a much smaller blast radius if anything unexpected happens. It is also easier to use — no installation, no terminal, just open the link and paste the prompt.

**To use Claude Web:**
1. Go to [claude.ai](https://claude.ai) and sign in (a free account is sufficient)
2. Start a new conversation
3. Paste the bootstrap prompt from Step 2 below

That's it. No terminal required.

## What Happens Next

The Conductor will greet you and list the available agents. Just tell it who you want to talk to — or describe what you want to discuss and it will route you to the right agent.

You can:
- Ask Sherpa about your user experience, onboarding questions, or anything you encountered using the product
- Ask AppSec about the security architecture, zero-knowledge guarantees, or any security concerns
- Ask Dev about a specific bug, how something is implemented, or what a fix looks like in code
- Ask the Architect how a particular design decision was made
- Ask the Ambassador how SG/Send is positioned or where it is heading
- Switch between agents at any time — just say "I'd like to talk to [Agent]"
