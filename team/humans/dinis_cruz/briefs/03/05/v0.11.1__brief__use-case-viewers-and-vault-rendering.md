## Use-Case Driven Workflows on the Website

The website should drive users to specific use cases with specific viewers. It is not about encryption. It is not about the technology. It is about the flow. Each use case has its own URL, its own UI, its own workflow.

| Use Case | URL | What It Does |
|---|---|---|
| **Share PDF** | `send.sgraph.ai/pdf/` | Upload PDF, get a link, recipient views in browser |
| **Share Gallery** | `send.sgraph.ai/gallery/` | Upload photos/videos, recipient sees a gallery |
| **Share Video** | `send.sgraph.ai/video/` | Upload video, recipient plays in browser |
| **Share Sensitive Documents** | `send.sgraph.ai/room/` | Upload folder, get a data room link |
| **Share a Secret** | `send.sgraph.ai/secret/` | Type text, encrypt, one-time self-destructing link |

Each has a different UI, sequence of events, and workflow. Different URLs means different SEO, different landing pages, different funnels. Eventually these can be branded for specific industries.

**For Ambassador and Sherpa:** These need to be on both sgraph.ai and send.sgraph.ai. Each is a landing page answering a specific user need. The user arrives because they want to share a video. The encryption is why it is better, but the use case is why they came.

## Vault Content Rendering: How Fast Can It Be?

If someone has an HTML page stored in a vault, how fast can we render it so it feels like a normal website?

### The Loader Pattern

A small JavaScript loader: fetches the encrypted blob, decrypts client-side, renders the HTML. The user sees a link click, a brief loader, then the page appears.

### Two Key Delivery Modes

| Mode | How | URL Pattern | Use Case |
|---|---|---|---|
| **Symmetric (hash fragment)** | Key in URL hash (never sent to server) | `send.sgraph.ai/v/abc123#key=xyz` | Quick sharing via WhatsApp/email |
| **PKI (URL path)** | Public key identifier in URL, only matching private key holder decrypts | `send.sgraph.ai/v/pubkey_prefix/abc123` | Targeted delivery, specific recipient only |

The loader should detect whether the client already has keys installed. If PKI keys are present from a previous session, decrypt automatically with no key prompt: zero-click experience.

### Performance Targets

| Step | Target |
|---|---|
| Load the loader JS | < 100ms |
| Fetch encrypted blob | < 500ms |
| Decrypt (AES-256-GCM, hardware-accelerated) | < 100ms |
| Render HTML | < 100ms |
| **Total for small HTML** | **< 1 second** |

For larger content (media, big documents), show a progress bar. For small HTML pages, explore embedding the encrypted data directly in the loader HTML (no second request, single HTTP roundtrip).
