# Folder Sharing Launch: "Share Large Files and Folders"

**version** v0.16.10
**date** 16 Mar 2026
**from** Human (project lead)
**to** Ambassador (lead), Sherpa, Designer, Developer
**type** Brief

---

## Context

The folder view is looking really good. The latest version has powerful features that are ready for users. The time is right to launch.

## The Message

The lead message should be: **Share large files and share large folders.**

Folders are where SG/Send has a genuine differentiator. This is not just "send a file." It is "send a structured folder with files that can be browsed, previewed, and selectively downloaded." Before we even add features like streaming downloads and simple download paths, the current folder view is strong enough to launch.

This needs the Ambassador and Sherpa involved to refine the messaging. The conversation to have:

- Does "share large files and folders" resonate more than "encrypted file transfer"?
- Should "large" be in the message at all, or is it implied by the 10GB limit?
- How do we position folder sharing against competitors who only offer flat file transfer?

## What to Launch With

- Folder view as the primary sharing experience
- Demo folders on the website (public demo packs with leaked decryption keys so people can try it)
- Videos and screenshots showing the folder browsing experience
- Updated website messaging: shift from "encrypted transfer" to "share large files and folders"

## What's Coming Next (But Not Needed for Launch)

- Gallery view for image-heavy folders
- Streaming/progressive downloads (start working with files before the full download completes)
- Simple download IDs (short, predictable tokens for easy sharing)

These are in parallel development but the folder launch does not depend on them.

| # | Criterion |
