## 1. Content Review — Urgent

**Assigned to:** Sherpa + Ambassador

The website content needs an immediate review pass. Issues identified:

### Inaccurate or Speculative Claims

There are references to pricing that don't match reality — for example, claims about file hosting costing "£5,000" or similar figures. These need to be removed or corrected.

**Principle:** The website must be 100% focused on reality. No speculative pricing for competitors, no made-up statistics, no claims we can't back up. Every statement should be either factually verifiable or clearly our own positioning.

**Review checklist:**
- Remove any competitor pricing claims that aren't sourced
- Verify all technical claims match the actual product (AES-256-GCM, zero-knowledge, client-side encryption — all correct)
- Verify pricing matches the agreed model (£0.01/file, £5 starter pack, 500 files)
- Check all links work (send.sgraph.ai, GitHub repo, etc.)
- Verify the "Free Early Access available. Open source. From £0.01 per file." tagline is accurate
- Review each language variant for AI translation artefacts (awkward phrasing, wrong terminology)

### Pricing Page Table Layout

The pricing page tables break in some non-English languages (text overflow, column width issues). Developer needs to fix the responsive layout to handle longer text strings in German, Croatian, Portuguese, etc.

**Action for Developer:** Fix the pricing page table CSS to handle variable-length text across all 17 language variants. Test in at least DE-DE, HR-HR, and PT-PT (languages with longer words).

The current site has two distinct user paths that should be made more prominent:

**Action for Designer:** Make these two paths more visually distinct. The current "Get Started Free" and "See Pricing" buttons partially do this, but the flow from pricing page → actual purchase → token delivery needs to be clearer.

## 4. News and Content Section

**Assigned to:** Journalist (lead), Ambassador

The website needs a news/blog section to create a sense of momentum and activity. When someone visits sgraph.ai, they should see that this is a living, active project.

### Content Types

| Type | Frequency | Example |
|---|---|---|
| **Launch announcements** | Per release | "v0.10.1: sgraph.ai goes multilingual in 17 countries" |
| **Feature highlights** | Weekly | "How zero-knowledge encryption protects your files" |
| **Use case stories** | Bi-weekly | "Sending credentials securely with SG/Send + Claude" |
| **Technical deep-dives** | Monthly | "Inside our client-side encryption: AES-256-GCM explained" |
| **Community updates** | As needed | "Help us improve translations — here's how" |

### Multilingual Content Strategy

The news section creates a translation cost problem: every article × 17 language variants = significant AI translation effort. The human hit Claude Code usage limits yesterday just translating the site.

**Strategy: Tiered content translation**

| Tier | What | Languages | Rationale |
|---|---|---|---|
| **Always translate** | Core website pages (home, pricing, product, about) | All 17 variants | These are the permanent pages that drive conversion |
| **Translate on demand** | Major announcements, launch posts | Top 5 languages (EN-GB, DE-DE, ES-ES, FR-FR, PT-PT) + EN-US | Balance reach vs cost |
| **English only** | Technical deep-dives, docs, detailed guides | EN-GB only | Flag with "🇬🇧 EN-GB only" indicator |
| **Community translated** | Anything the community volunteers to translate | Per contribution | Builds engagement and reduces cost |

### Docs Site: English Only (For Now)

The documentation site (docs.sgraph.ai or sgraph.ai/docs/) will be English only for now. This is a deliberate decision to avoid translation cost on content that changes frequently.

Display a clear indicator: "🇬🇧 Documentation is currently available in English (UK) only. Want to help translate? [Contribute →]"

This turns the limitation into a community engagement opportunity.

## 5. Multilingual Outreach Strategy

**Assigned to:** Ambassador

Now that the website is multilingual, outreach can be language-targeted. This is a significant advantage — most competitor products only have English websites.

### LinkedIn Posts in Local Languages

**The idea:** Post on LinkedIn in different languages, targeting specific communities:

- "Para mis amigos en España — SG/Send ahora está disponible en español..." with a screenshot of the Spanish site
- "Für meine Freunde in Deutschland — SG/Send ist jetzt auf Deutsch verfügbar..." with a German screenshot
- "Para os meus amigos em Portugal — SG/Send agora disponível em português..."

**Why this works:**
- LinkedIn posts in the reader's native language stand out in a feed dominated by English
- The screenshot of the website in their language demonstrates the product immediately
- It signals respect for their language and culture — not just an English product with a Google Translate button
- Each post targets a different audience segment with minimal content overlap
