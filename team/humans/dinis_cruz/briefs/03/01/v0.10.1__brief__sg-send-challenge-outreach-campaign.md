## The Concept

Instead of "please try my product," frame the outreach as a challenge:

**"Can you find the buy button, buy £5 of credits, send a file, and get it viewed?"**

This reframes the ask from a favour into a game. It's lighter, more engaging, and — critically — it's also a usability test. If they can't complete the challenge, we learn where the UX breaks.

---

## The Challenge Flow

```
1. Recipient receives the challenge (WhatsApp, LinkedIn, email)
   "Here's the SG/Send Challenge..."

2. Go to sgraph.ai
   → Find the pricing page
   → Find the "Buy Credits" button
   → Buy £5 of credits via Stripe

3. Use your new credits on send.sgraph.ai
   → Send an encrypted file to someone (or back to themselves)
   → Get the file viewed/downloaded by the recipient

4. Send a message THROUGH SG/Send confirming you completed it
   → This proves they used the product end-to-end

5. Receive back (via SG/Send) a £5 Amazon voucher link
   → Click to activate
   → Net cost to them: £0
```

**Why this works:**
- It's a challenge, not a sales pitch — people are more likely to engage
- It tests the entire funnel (discovery → pricing → payment → usage → sharing)
- Step 4 (confirm via SG/Send) forces a second use of the product — they're now both sender and recipient
- The Amazon voucher arrives via SG/Send, demonstrating the product delivers value
- Every step generates data about where people get stuck

---

## What We Learn From Each Step

| Step | If They Succeed | If They Fail |
|---|---|---|
| Find the pricing page | Navigation is clear | Navigation needs work |
| Find the buy button | CTA is visible | CTA is buried or confusing |
| Complete Stripe checkout | Payment flow works | Stripe integration issues, friction points |
| Send a file | Core product works, token delivery was smooth | Token delivery failed, UX confusion, onboarding gaps |
| Get file viewed | Sharing flow works, recipient experience is good | Download link confusing, decryption issues |
| Send confirmation via SG/Send | They understand the product well enough to use it independently | They need more guidance |

## Campaign Materials

### Material 1: The Challenge Card (SVG/Image)

A shareable image for WhatsApp, LinkedIn DM, and email. Should be visually clean, not salesy.

**Design direction:**

```
┌─────────────────────────────────────────────┐
│                                             │
│     🔐 The SG/Send Challenge                │
│                                             │
│     Can you send an encrypted file          │
│     that nobody can read — not even us?     │
│                                             │
│     1. Go to sgraph.ai                      │
│     2. Find the buy button (£5)             │
│     3. Send an encrypted file               │
│     4. Get someone to open it               │
│     5. Tell me you did it — via SG/Send     │
│                                             │
│     Complete the challenge and I'll          │
│     send you a £5 Amazon voucher            │
│     — through SG/Send, of course.           │
│                                             │
│     sgraph.ai                               │
│                                             │
└─────────────────────────────────────────────┘
```

**Formats needed:**
- SVG (for WhatsApp — verify WhatsApp supports SVG, otherwise PNG)
- PNG (for LinkedIn, email, universal fallback)
- Multiple language variants (at minimum: EN, ES, PT, DE, FR)

### Material 2: Detailed Description (Long Form)

A longer version for email or LinkedIn post that explains the context:

**Elements:**
- What SG/Send is (one paragraph)
- Why you're asking (honest — "I built this and need real users to test the full flow")
- The challenge (step by step)
- The reward (£5 Amazon voucher)
- The time commitment ("about 10 minutes")
- The link (sgraph.ai)

**Tone:** Personal, honest, not corporate. "I've been building something and I'd love your help testing it."

### Material 3: LinkedIn Post Variants

**English version (public — without voucher code):**
> I've been building SG/Send — encrypted file transfer where nobody can read your files, not even us. The website just went live in 17 countries and 12 languages.
>
> Here's a challenge: can you go to sgraph.ai, buy £5 of credits, and send an encrypted file? DM me when you've done it.
>
> [Screenshot of the site in their language]

**Multilingual variants:**
- Spanish: "Para mis amigos en España..." with ES-ES screenshot
- Portuguese: "Para os meus amigos em Portugal..." with PT-PT screenshot  
- German: "Für meine Freunde in Deutschland..." with DE-DE screenshot

The multilingual angle is the hook — a LinkedIn post in Spanish with a screenshot of a Spanish website stands out in a feed full of English content.

### Material 4: WhatsApp Message (Personal)

Short, direct, personal:

> Hey [name]! I've been building something and could use your help testing it. It's called SG/Send — encrypted file sharing.
>
> The challenge: go to sgraph.ai, buy £5 of credits, send a file, and tell me you did it — through SG/Send itself.
>
> Complete it and I'll send you a £5 Amazon voucher back through the app. Takes about 10 minutes.
>
> Interested?

Wait for their "yes" before sending the detailed steps. Don't front-load everything.

## Voucher Distribution Workflow

```
1. Human sends challenge message (WhatsApp/LinkedIn/email)
2. Recipient completes the challenge
3. Recipient sends confirmation message via SG/Send
4. Human receives the confirmation
5. Human creates a SG/Send transfer containing the Amazon voucher link
6. Sends back via SG/Send
7. Recipient activates the voucher

Tracking: log each outreach (who, when, channel, language)
         log each completion (who, when, which steps completed)
         log each voucher sent (who, when, voucher code)
```

**For now this is manual and that's fine.** The human tracks it in a spreadsheet or the data room. The important thing is that every interaction happens through or because of SG/Send.

---

## Sherpa's Role

The challenge is also an implicit Sherpa test:

- When users hit the pricing page, can they find their way without help?
- If they email Sherpa with questions, what do they ask? (This reveals UX gaps)
- The paths users take tell us what's intuitive and what's not

**Action for Sherpa:** Be ready for incoming questions from challenge participants. Log every question — these are free usability insights.
