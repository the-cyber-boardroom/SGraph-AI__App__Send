# sg-layout Implementation Guide

**Version:** v0.1.3
**Date:** 18 March 2026
**Audience:** LLM sessions and developers integrating sg-layout into SGraph projects

---

## What sg-layout Is

`sg-layout` is a vanilla JS Web Component that provides a VS Code-style dockable panel system. It supports:

- Rows, columns, and tabbed stacks
- Drag-to-dock (rearrange panels by dragging tabs)
- Resizable splits with drag handles
- Panel locking (pin panels in place)
- Nested/fractal layouts (sg-layout inside sg-layout)
- JSON serialization for save/restore
- CSS custom property theming

**No frameworks. No build step. One import and you're running.**

## Quick Start

### 1. Import

```javascript
import { SgLayout }   from '/core/sg-layout/v0.1.0/sg-layout.js';
import { SGL_EVENTS } from '/core/sg-layout/v0.1.0/sg-layout-events.js';
```

The `SgLayout` import registers the `<sg-layout>` custom element automatically.

### 2. Create and Mount

```javascript
const layout = document.createElement('sg-layout');
layout.style.width  = '100%';
layout.style.height = '100%';
document.getElementById('container').appendChild(layout);
```

**Important:** The parent container must have explicit dimensions. `sg-layout` fills 100% of its parent — if the parent has no height, nothing renders.

### 3. Set a Layout

```javascript
layout.setLayout({
    type: 'row', id: 'root', sizes: [0.25, 0.75],
    children: [
        { type: 'stack', id: 's-sidebar', activeTab: 0, tabs: [
            { type: 'tab', id: 't-nav', title: 'Navigation', tag: 'div', state: {} }
        ]},
        { type: 'stack', id: 's-main', activeTab: 0, tabs: [
            { type: 'tab', id: 't-content', title: 'Content', tag: 'div', state: {} }
        ]}
    ]
});
```

Or use the HTML attribute:

```javascript
layout.setAttribute('layout', JSON.stringify(tree));
```

### 4. Populate Panels

```javascript
const navEl = layout.getPanelElement('t-nav');
navEl.innerHTML = '<ul><li>Item 1</li><li>Item 2</li></ul>';

const contentEl = layout.getPanelElement('t-content');
contentEl.innerHTML = '<h1>Hello World</h1>';
```

That's it. You have a two-panel layout with a draggable split handle.

## Layout Tree Reference

The layout is a JSON tree with four node types:

### Tab (leaf node — a single panel)

```javascript
{
    type: 'tab',
    id: 'unique-id',          // Must be unique across the layout
    title: 'Panel Title',     // Displayed in the tab bar
    tag: 'div',               // HTML element tag to create
    state: {},                // Arbitrary data — survives serialization
    locked: false             // Optional. true = no drag, no drop, no close
}
```

### Stack (contains one or more tabs)

```javascript
{
    type: 'stack',
    id: 'unique-id',
    activeTab: 0,             // Index of the visible tab
    tabs: [ /* tab nodes */ ]
}
```

### Row (horizontal split — children side by side)

```javascript
{
    type: 'row',
    id: 'unique-id',
    sizes: [0.3, 0.7],       // Proportions. Must sum to ~1.0
    children: [ /* stacks, rows, or columns */ ]
}
```

### Column (vertical split — children stacked top to bottom)

```javascript
{
    type: 'column',
    id: 'unique-id',
    sizes: [0.6, 0.4],
    children: [ /* stacks, rows, or columns */ ]
}
```

### Rules

- `sizes` array length must match `children` array length
- `sizes` values are proportions (0.0 to 1.0), should sum to 1.0
- Rows and columns can nest arbitrarily
- Every tab `id` must be unique within the layout
- `tag` can be any valid HTML element tag, including `'sg-layout'` for nesting

## Common Layout Patterns

### IDE Layout (sidebar + editor + console)

```javascript
{
    type: 'row', id: 'root', sizes: [0.2, 0.8],
    children: [
        { type: 'stack', id: 's-sidebar', activeTab: 0, tabs: [
            { type: 'tab', id: 't-files', title: 'Files', tag: 'div', state: {}, locked: true }
        ]},
        { type: 'column', id: 'c-main', sizes: [0.7, 0.3],
            children: [
                { type: 'stack', id: 's-editor', activeTab: 0, tabs: [
                    { type: 'tab', id: 't-main',  title: 'main.js',    tag: 'div', state: {} },
                    { type: 'tab', id: 't-style', title: 'styles.css', tag: 'div', state: {} }
                ]},
                { type: 'stack', id: 's-console', activeTab: 0, tabs: [

                    { type: 'tab', id: 't-terminal', title: 'Terminal', tag: 'div', state: {} }
                ]}
            ]
        }
    ]
}
```

### Three Equal Columns

```javascript

    type: 'row', id: 'root', sizes: [0.333, 0.334, 0.333],
    children: [
        { type: 'stack', id: 's1', activeTab: 0, tabs: [

        ]},
        { type: 'stack', id: 's3', activeTab: 0, tabs: [
            { type: 'tab', id: 't3', title: 'Right', tag: 'div', state: {} }
        ]}
    ]

```

### Single Panel (no splits)

```javascript
{
    type: 'stack', id: 's-only', activeTab: 0, tabs: [
        { type: 'tab', id: 't-only', title: 'Main', tag: 'div', state: {} }
    ]
}
```

## Public API

All methods are on the `<sg-layout>` element instance.

### Layout

| Method | Returns | Description |
|--------|---------|-------------|
| `setLayout(tree)` | void | Render a layout from JSON |
| `getLayout()` | object \| null | Get the current layout as JSON |

### Panels

| Method | Returns | Description |
|--------|---------|-------------|
| `addPanel({ tag, title, state, locked })` | string (id) | Add a panel (creates a new stack) |
| `addTabToStack(stackId, { tag, title, state, locked }, activate?)` | string \| null | Add a tab to an existing stack |
| `removePanel(id)` | void | Remove a panel/tab by id |
| `getPanelElement(id)` | HTMLElement \| null | Get the DOM element for a panel |
| `getPanelBounds(id)` | `{ width, height }` \| null | Get a panel's current size |

### Locking

| Method | Returns | Description |
|--------|---------|-------------|
| `lockPanel(id)` | void | Lock a panel (no drag, drop, or close) |
| `unlockPanel(id)` | void | Unlock a panel |
| `isPanelLocked(id)` | boolean | Check lock state |

### Fractal (Nested Layouts)

| Method | Returns | Description |
|--------|---------|-------------|
| `getChildLayouts()` | Map\<string, SgLayout\> | All nested sg-layout instances |
| `getParentLayout()` | SgLayout \| null | Parent layout (if nested) |
| `getDepth()` | number | Nesting depth (0 = root) |
| `walkLayouts(visitor)` | void | Walk entire fractal tree. `visitor(layout, depth)` |

## Events

Import event constants — never hardcode strings:

```javascript
import { SGL_EVENTS } from '/core/sg-layout/v0.1.0/sg-layout-events.js';
```

### Listening

```javascript
layout.events.on(SGL_EVENTS.LAYOUT_CHANGED, (detail) => {
    console.log('Tree changed:', detail.tree);
});
```

### Key Events

| Constant | Payload | When |
|----------|---------|------|
| `LAYOUT_CHANGED` | `{ tree }` | Any structural change (drag, close, add) |
| `LAYOUT_READY` | `{}` | All panels registered after setLayout |
| `TAB_CHANGED` | `{ stackId, tabId }` | User switches active tab |
| `PANEL_RESIZED` | `{ id, width, height }` | Panel size changed (split drag, window resize) |
| `PANEL_FOCUSED` | `{ id }` | Panel gains focus |
| `PANEL_SHOWN` | `{ id }` | Panel becomes visible (tab activated) |
| `PANEL_HIDDEN` | `{ id }` | Panel hidden (different tab activated) |
| `PANEL_CLOSED` | `{ id }` | Panel removed from layout |
| `DRAG_START` | `{ panelId, sourceLayoutId }` | Tab drag begins |
| `DRAG_END` | `{ panelId, accepted }` | Tab drag ends (accepted = successfully docked) |

### Command Events (dispatch to trigger actions)

| Constant | Payload | Effect |
|----------|---------|--------|
| `CMD_ADD_PANEL` | `{ tag, title, state, target }` | Add a panel |
| `CMD_REMOVE_PANEL` | `{ id }` | Remove a panel |
| `CMD_FOCUS_PANEL` | `{ id }` | Focus/activate a panel |
| `CMD_SET_LAYOUT` | `{ tree }` | Replace the layout |

### Lifecycle Events (from hosted components)

| Constant | Payload | Effect |
|----------|---------|--------|
| `SET_TITLE` | `{ panelId, title }` | Update a tab's title |
| `REQUEST_FOCUS` | `{ panelId }` | Request focus from within a panel |
| `REQUEST_CLOSE` | `{ panelId }` | Request close from within a panel |

## Saving and Restoring Layouts

### Save

```javascript
const tree = layout.getLayout();
localStorage.setItem('my-layout', JSON.stringify(tree));
```

### Restore

```javascript
const saved = localStorage.getItem('my-layout');
if (saved) {
    layout.setLayout(JSON.parse(saved));
} else {
    layout.setLayout(defaultLayout);
}
```

### Auto-save on change

```javascript
layout.events.on(SGL_EVENTS.LAYOUT_CHANGED, ({ tree }) => {
    localStorage.setItem('my-layout', JSON.stringify(tree));
});
```

### Component State

If a hosted element implements `getLayoutState()`, sg-layout calls it during serialization and merges the result into `tab.state`:

```javascript
class MyEditor extends HTMLElement {
    getLayoutState() {
        return { scrollTop: this.scrollTop, file: this.currentFile };
    }
}
```

## Theming

Override CSS custom properties on `<sg-layout>` or any ancestor:

```css
sg-layout {
    /* Geometry */
    --sgl-handle-size: 4px;          /* Resize handle width */
    --sgl-tab-height: 28px;          /* Tab bar height */
    --sgl-panel-header-height: 32px; /* Single-panel title bar height */
    --sgl-min-panel-size: 80px;      /* Minimum panel dimension */

    /* Colours */
    --sgl-bg: #1A1A2E;              /* Root background */
    --sgl-surface: #162040;          /* Panel/tab background */
    --sgl-surface-hover: #253254;    /* Hover state */
    --sgl-border: #222d4d;           /* Border colour */
    --sgl-accent: #4ECDC4;          /* Active tab, resize handle highlight */
    --sgl-text: #e2e8f0;            /* Primary text */
    --sgl-text-muted: #5a6478;      /* Secondary text */

    /* Motion */
    --sgl-transition-speed: 150ms;
}
```

### Light Theme Example

```css
sg-layout.light {
    --sgl-bg: #f5f5f5;
    --sgl-surface: #ffffff;
```

Nested layouts automatically get `[data-depth="N"]` attributes (0 = root). Predefined depth colours go up to depth 4.

## Panel Locking

Lock panels to prevent users from dragging, dropping onto, or closing them.

```javascript
// Via layout tree (at creation)
{ type: 'tab', id: 't-nav', title: 'Navigation', tag: 'div', state: {}, locked: true }

// Via API (after creation)
layout.lockPanel('t-nav');
layout.unlockPanel('t-nav');
layout.isPanelLocked('t-nav'); // → true/false
```

**Behaviour when locked:**

| Action | Result |
|--------|--------|
| Drag tab | Blocked — cursor stays `default` |
| Drop onto stack | Blocked if ALL tabs in stack are locked |
| Close button | Hidden |
| Resize handles | Still work (locking is per-panel, not per-split) |

A stack with mixed locked/unlocked tabs still accepts drops — only the unlocked tabs can be dragged away.

## Drag-to-Dock

Works automatically. No configuration needed.

**How users interact:**

1. Grab a tab (or single-panel title bar) and drag
2. Hover over a target panel — a cross-shaped overlay appears:
   ```
          [▲]
       [◀][●][▶]
          [▼]
   ```
3. Drop on a zone:
   - **▲ Top** — split target vertically, new panel above
   - **▼ Bottom** — split target vertically, new panel below
   - **◀ Left** — split target horizontally, new panel left
   - **▶ Right** — split target horizontally, new panel right
   - **● Center** — merge as a new tab in the target stack
4. A dashed preview rectangle shows where the panel will land

**Cross-layout drag is intentionally scoped to a single sg-layout instance.** Panels cannot be dragged between separate sg-layout elements. This is by design — nested sg-layouts are treated as independent workspaces.

## Nested / Fractal Layouts

Use `tag: 'sg-layout'` to embed a layout inside a layout:

```javascript
{
    type: 'tab',
    id: 't-nested',
    title: 'Sub-Layout',
    tag: 'sg-layout',        // ← Creates a child sg-layout
    state: {
        layout: {             // ← The child's layout tree
            type: 'column', id: 'nested', sizes: [0.5, 0.5],
            children: [
                { type: 'stack', id: 'ns1', activeTab: 0, tabs: [
                    { type: 'tab', id: 'nt1', title: 'Top', tag: 'div', state: {} }
                ]},
                { type: 'stack', id: 'ns2', activeTab: 0, tabs: [
                    { type: 'tab', id: 'nt2', title: 'Bottom', tag: 'div', state: {} }
                ]}
            ]
        }
    }
}
```

Navigate the tree programmatically:

```javascript
layout.walkLayouts((childLayout, depth) => {
    console.log(`Depth ${depth}:`, childLayout.getLayout());
});
```

## Gotchas & Tips

1. **Parent must have height.** `sg-layout` is `width: 100%; height: 100%`. If the parent is `height: auto`, nothing renders.

2. **Wait a frame after mounting.** `getPanelElement()` returns null until the component renders. Use `await new Promise(r => requestAnimationFrame(r))` or listen for `LAYOUT_READY`.

3. **IDs must be unique.** Across ALL tabs in the layout (including nested layouts). Duplicate IDs cause undefined behaviour.

4. **Sizes must match children.** `sizes: [0.3, 0.7]` with 3 children will break. Always keep them in sync.

5. **No build step required.** Import paths must be valid browser-resolvable URLs. Use absolute paths from the CDN or relative paths from your HTML file.

6. **sg-layout is in Shadow DOM.** You cannot style internal elements (tabs, handles) with global CSS. Use CSS custom properties for theming.

7. **`state` is your escape hatch.** Store any data you need per-panel in `state: {}`. It serializes with `getLayout()` and restores with `setLayout()`.

8. **Single-panel stacks show a title bar, not a tab bar.** The title bar is draggable just like tabs.

9. **`removePanel` cleans up the tree.** When a stack empties, the parent row/column adjusts. When only one child remains, the container is promoted (flattened).

10. **Import order doesn't matter.** `SgLayout` self-registers as `<sg-layout>` on import. The `SGL_EVENTS` import is just constants.
