## 1. Batch Read (New — High Priority)

### 1.1 Add `read` Operation to Batch Endpoint

The existing `POST /api/vault/batch/{vault_id}` supports `write`,
`write-if-match`, and `delete`. Add `read`:

```python
elif op_type == 'read':
    file_id      = op.get('file_id', '')
    payload      = self.vault_service.read(vault_id=vault_id, file_id=file_id)
    if payload:
        results.append(dict(op='read', file_id=file_id, status='ok',
                            data=base64.b64encode(payload).decode('ascii')))
    else:
        results.append(dict(op='read', file_id=file_id, status='not_found'))
```

**No authentication on batch read** — same as individual reads. Data is
encrypted; knowing the file_id doesn't help without the vault key.

### 1.2 Server-Side Caching by Mutability

The self-describing file IDs enable automatic caching. Parse the mutability
segment from the file_id:

```python
def _cache_policy(self, file_id):
    parts = file_id.replace('bare/', '').split('/')[-1].split('-')
    if len(parts) >= 3:
        mutability = parts[2]
        if mutability == 'imm':
            return dict(cache=True, ttl=None)       # forever
        elif mutability == 'snw':
            return dict(cache=True, ttl=5)           # 5 seconds
    return dict(cache=False)                         # muw or unknown
```

For batch reads: `obj-cas-imm-*` → serve from cache. `ref-pid-muw-*` → always
read fresh from storage.

## 2. Health Check Endpoint (New)

```python
@route_path('/vault/health/{vault_id}')
def health__vault_id(self, vault_id):
    manifest_path = self.vault_service.vault_manifest_path(vault_id)
    exists = self.vault_service.storage_fs.file__exists(manifest_path)
    if exists:
        return dict(status='ok', vault_id=vault_id)
    raise HTTPException(status_code=404, detail='Vault not found')
```

Unauthenticated. Serves as Lambda warm-up and vault existence check.
CLI calls this before push/pull/clone.

## 3. Route Changes

### 3.1 No Breaking Changes

The `{file_id:path}` capture in routes handles both old (`bare/data/obj-xxx`)
and new (`bare/data/obj-cas-imm-xxx`) formats. Self-describing IDs are longer
but contain no characters that break URL parsing.

### 3.2 Consider: File ID Validation (Defence in Depth)

```python
import re
VALID_FILE_ID = re.compile(r'^[a-zA-Z0-9/_-]{1,128}$')
```

Rejects path traversal attempts. Not strictly necessary (Storage_FS sandboxes
paths) but good practice.

## 4. Storage — No Changes

`Service__Vault__Pointer` is a flat key-value store. It doesn't interpret
file_ids. The self-describing IDs are longer strings but work identically:

```python
def vault_payload_path(self, vault_id, file_id):
    return f'transfers/vault/{vault_id}/{file_id}/payload'
```

`bare/data/obj-cas-imm-aeaa531d8450` → `transfers/vault/aaaabbp1/bare/data/obj-cas-imm-aeaa531d8450/payload`.

### 4.1 List Endpoint — Consider Auth

With deterministic addressing, `list` is only needed for:
- Enumerating pending change packs
- Admin/debug (vault browser's raw file listing)

**Recommendation:** Require auth token for `list`. Normal vault operations don't need it.

## 5. Compare-and-Swap Enforcement

The server currently allows `write-if-match` on any file. With self-describing
IDs, the server could enforce that compare-and-swap is only used on `*-muw-*`
files and simple writes on `*-imm-*` files. This is defence in depth —
clients should already follow these rules, but the server can enforce them.

```python
if op_type == 'write-if-match':
    parts = file_id.split('-')
    if len(parts) >= 3 and parts[2] == 'imm':
        # Immutable files should not use compare-and-swap
        results.append(dict(op=op_type, file_id=file_id, status='error',
                            detail='immutable files cannot use write-if-match'))
        break
```
