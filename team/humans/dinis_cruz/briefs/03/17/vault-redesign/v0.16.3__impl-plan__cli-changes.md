## 1. ID Naming Convention

All file IDs change to the four-segment `{type}-{derivation}-{mutability}-{hex_id}` format.

### 1.1 Crypto Layer (`Vault__Crypto.py`)

**Add** `derive_branch_ref_file_id(read_key, vault_id, branch_name)` method.

**Update** `derive_keys()` — return dict uses new ID format:

```python
def derive_keys(self, passphrase, vault_id):
    ...
    ref_file_id          = 'ref-pid-muw-' + self.derive_ref_file_id(read_key_bytes, vault_id)
    branch_index_file_id = 'idx-pid-muw-' + self.derive_branch_index_file_id(read_key_bytes, vault_id)
    return dict(
        read_key_bytes        = read_key_bytes,
        read_key              = read_key_bytes.hex(),
        write_key_bytes       = write_key_bytes,
        write_key             = write_key_bytes.hex(),
        ref_file_id           = ref_file_id,
        branch_index_file_id  = branch_index_file_id,
        passphrase            = passphrase,
        vault_id              = vault_id,
    )
```

**Delete:** `derive_tree_file_id()`, `derive_settings_file_id()`, `TREE_DOMAIN`, `SETTINGS_DOMAIN`.

### 1.2 Object Store (`Vault__Object_Store.py`)

Object IDs change from `obj-{hash}` to `obj-cas-imm-{hash}`:

```python
def _compute_id(self, ciphertext):
    raw_hash = self.crypto.compute_object_id(ciphertext)
    return f'obj-cas-imm-{raw_hash}'
```

Local path: `bare/data/obj-cas-imm-{hash}`.  
Server file_id: `bare/data/obj-cas-imm-{hash}`.  
Identical.

**Delete:** `use_v2` flag, `_fallback_path()`, all v1 compat logic.

### 1.3 Ref Manager (`Vault__Ref_Manager.py`)

Refs use `ref-pid-muw-{hmac}` or `ref-pid-snw-{hmac}` IDs.
Local path: `bare/refs/ref-pid-muw-{hmac}`.

**Delete:** `use_v2` flag, `read_head()`/`write_head()` (v1 API), `_head_path()`, `list_refs()`.

### 1.4 Branch Manager (`Vault__Branch_Manager.py`)

**Delete:** `find_branch_index_id()` — replaced by deterministic derivation.

Branch index uses `idx-pid-muw-{hmac}` as ID. Named branch ref uses `ref-pid-muw-{hmac}`.
Clone branch ref uses `ref-pid-snw-{hmac}` (derived from vault key + branch name).

### 1.5 Key IDs

Keys use `key-rnd-imm-{random}`. Local path: `bare/keys/key-rnd-imm-{random}`.

### 1.6 Change Packs

Change packs use `pkg-rnd-imm-{ts}_{random}`. Local path: `bare/pending/pkg-rnd-imm-{ts}_{random}`.

## 2. Sub-Tree Model in Commit

Replace flat tree building in `Vault__Sync.commit()`.

### Current (flat)

```python
new_tree = Schema__Object_Tree(schema='tree_v1')
for path in sorted(new_file_map.keys()):
    new_tree.entries.append(Schema__Object_Tree_Entry(path=path, blob_id=...))
```

### New (sub-tree, bottom-up)

```python
def _build_sub_trees(self, directory, file_map, old_tree_map, read_key, obj_store):
    """Build sub-tree objects bottom-up. Returns root tree object ID."""
    # 1. Group files by parent directory
    # 2. For each directory (deepest first):
    #    a. Build tree entries: files → blob_id, subdirs → tree_id
    #    b. Encrypt name into name_enc, size into size_enc
    #    c. Encrypt and store tree object → obj-cas-imm-{hash}
    #    d. If hash matches old tree → reuse (no upload needed)
    # 3. Return root tree object ID
```

Tree entries use encrypted-only fields: `name_enc`, `size_enc`, `content_hash_enc`.
No plaintext `path`, `name`, `size`, `content_hash`.

## 3. Sync Engine Changes (`Vault__Sync.py`)

### 3.1 Init — Deterministic IDs

Use HMAC-derived IDs for named branch ref and branch index.
Clone branch ref derived from vault key + branch name.

### 3.2 Push — No Dual-Write, Named Branch Merge Commit

**Remove:** all dual-write logic, `browser_ref_id`, `browser_index_id`.

**Add:** merge commit on named branch (spec 6.3 step 6), signed by named branch key.

**Change:** batch operations use self-describing IDs:
- `bare/data/obj-cas-imm-{hash}` (not `bare/data/obj-{hash}`)
- `bare/refs/ref-pid-muw-{hmac}` (not `bare/refs/ref-{random}`)
- Compare-and-swap only on `*-muw-*` files

### 3.3 Clone — Graph-Walking

Replace `api.list_files(vault_id, 'bare/')` + download everything.

Primary flow: ref → commit → root tree → sub-trees (depth levels) → blobs → branch index → keys → register.

Lite clone mode: same `bare/` structure but objects fetched on demand via transparent interception in `Vault__Object_Store.load()`.

### 3.4 Pull — Tree Comparison with Sub-Tree Shortcut

Compare root tree entries. If a sub-tree ID is unchanged, skip the entire folder.
Only recurse into changed sub-trees. This is the main performance win of the sub-tree model.

### 3.5 `_upload_bare_to_server()` — Rewrite

Replace `os.walk(bare_dir)` with explicit upload of known files using self-describing IDs.
No more mirroring the bare directory — upload specific objects, refs, indexes, and keys.

### 3.6 Add Tracking File

Write `local/tracking.json` after every pull/push for fast LCA.

## 4. Schema Changes

### 4.1 `Schema__Object_Tree_Entry`

**Remove:** `path`, `name`, `size`, `content_hash`  
**Keep:** `name_enc`, `size_enc`, `content_hash_enc`, `blob_id`, `tree_id`

### 4.2 `Schema__Object_Commit`

**Remove:** `parent` (single), `timestamp` (ISO), `message` (plaintext), `version`  
**Add:** `message_enc`  
**Keep:** `parents[]`, `timestamp_ms`, `tree_id`, `branch_id`, `signature`, `author_key_id`, `author_signature`, `attestations`

### 4.3 `Schema__Branch_Index`

**Remove:** `index_id` (now deterministic, computed not stored)

### 4.4 `Schema__Branch_Meta`

`head_ref_id` now stores `ref-pid-muw-{hmac}` or `ref-pid-snw-{hmac}`.
`public_key_id` now stores `key-rnd-imm-{random}`.
