## Implementation Order

The changes are sequenced so each step is independently testable. Each phase
produces a working (but incomplete) system. Tests are updated alongside code.

```
Phase 1: Schemas & ID format        (break the build, fix forward)
Phase 2: Crypto layer                (new derivations, remove legacy)
Phase 3: Object store & ref manager  (self-describing IDs)
Phase 4: Branch manager              (deterministic index IDs)
Phase 5: Sub-tree model              (new commit algorithm)
Phase 6: Sync engine — init          (deterministic IDs in init)
Phase 7: Sync engine — push          (remove dual-write, add merge commit)
Phase 8: Sync engine — clone         (graph-walking, lite clone)
Phase 9: Sync engine — pull          (sub-tree comparison)
Phase 10: API client                 (batch read)
Phase 11: Cleanup                    (delete all legacy code)
```

## Phase 1: Schema Changes

**Goal:** Update all Type_Safe schemas to the new field set. This will
intentionally break existing code — we fix forward in subsequent phases.

**Current:**
```python

    content_hash     : Safe_Str__Content_Hash    = None   # plaintext hash
    name_enc         : Safe_Str__Encrypted_Value = None   # encrypted name
    size_enc         : Safe_Str__Encrypted_Value = None   # encrypted size
    content_hash_enc : Safe_Str__Encrypted_Value = None   # encrypted hash
```

**New:**
```python
class Schema__Object_Tree_Entry(Type_Safe):
    blob_id          : Safe_Str__Object_Id       = None   # file → obj-cas-imm-{hash}
    tree_id          : Safe_Str__Object_Id       = None   # folder → obj-cas-imm-{hash}
    name_enc         : Safe_Str__Encrypted_Value = None   # AES-GCM encrypted name (base64)
    size_enc         : Safe_Str__Encrypted_Value = None   # AES-GCM encrypted size (base64)
```

**Removed:** `path`, `name`, `size`, `content_hash`.

Every reference to `entry.path` or `self._entry_path(entry)` must be replaced

**New:**

### 1.3 `Schema__Branch_Index` — Remove `index_id`

```python

    schema     : Safe_Str__Schema_Version = None
    branches   : list[Schema__Branch_Meta]
```

**Removed:** `index_id` — the ID is now deterministic (HMAC-derived),
computed by the caller, not stored inside the index.

### 1.4 `Schema__Branch_Meta` — No Structural Changes

- `private_key_id` → `key-rnd-imm-{random}`

```python
def encrypt_metadata(self, read_key: bytes, plaintext: str) -> str:
    """Encrypt a metadata string, return base64-encoded ciphertext."""
    data = plaintext.encode('utf-8')
    ciphertext = self.encrypt(read_key, data)
    return base64.b64encode(ciphertext).decode('ascii')

def decrypt_metadata(self, read_key: bytes, b64_ciphertext: str) -> str:
    """Decrypt a base64-encoded metadata ciphertext, return plaintext string."""
    ciphertext = base64.b64decode(b64_ciphertext)
    data = self.decrypt(read_key, ciphertext)
    return data.decode('utf-8')
```

## Phase 5: Sub-Tree Model

This is the largest single change. The flat tree algorithm in `commit()` is
replaced with a bottom-up sub-tree builder.

### 5.1 New Method: `_build_sub_trees()`

```python
def _build_sub_trees(self, directory: str, file_map: dict,
                     old_root_tree_id: str, read_key: bytes,
                     obj_store: Vault__Object_Store) -> str:
    """Build sub-tree objects bottom-up from the working directory.

    Args:
        directory: working directory path
        file_map: {relative_path: {size, content_hash}} from _scan_local_directory
        old_root_tree_id: previous commit's root tree ID (for reuse detection)
        read_key: vault read key
        obj_store: object store for storing tree/blob objects

    Returns:
        Root tree object ID (obj-cas-imm-{hash})
    """
    # Step 1: Group files by parent directory
    dir_contents = {}  # { dir_path: [(name, relative_path)] }
    for rel_path in sorted(file_map.keys()):
        parts = rel_path.split('/')
        if len(parts) == 1:
            # Root-level file
            dir_contents.setdefault('', []).append((parts[0], rel_path))
        else:
            # File in a sub-directory
            dir_path = '/'.join(parts[:-1])
            filename = parts[-1]
            dir_contents.setdefault(dir_path, []).append((filename, rel_path))
            # Ensure all parent directories are registered
            for i in range(1, len(parts) - 1):
                parent = '/'.join(parts[:i])
                dir_contents.setdefault(parent, [])

    # Step 2: Build trees bottom-up (deepest directories first)
    tree_ids = {}  # { dir_path: obj-cas-imm-{hash} }
    for dir_path in sorted(dir_contents.keys(), key=lambda p: p.count('/'), reverse=True):
        entries = []

        # Add file entries
        for name, rel_path in dir_contents[dir_path]:
            if rel_path in file_map:
                file_info  = file_map[rel_path]
                local_file = os.path.join(directory, rel_path)
                with open(local_file, 'rb') as f:
                    content = f.read()

                # Check if blob already exists (content-addressed dedup)
                file_hash    = self.crypto.content_hash(content)
                encrypted    = self.crypto.encrypt(read_key, content)
                blob_id      = obj_store.store(encrypted)

                entry = Schema__Object_Tree_Entry(
                    blob_id          = blob_id,
                    name_enc         = self.crypto.encrypt_metadata(read_key, name),
                    size_enc         = self.crypto.encrypt_metadata(read_key, str(len(content))),
                    content_hash_enc = self.crypto.encrypt_metadata(read_key, file_hash),
                )
                entries.append(entry)

        # Add sub-directory entries (already built in previous iterations)
        for child_dir in sorted(dir_contents.keys()):
            if child_dir == dir_path:
                continue
            # Is child_dir a direct child of dir_path?
            if dir_path == '':
                if '/' not in child_dir:  # direct child of root
                    folder_name = child_dir
                else:
                    continue
            elif child_dir.startswith(dir_path + '/') and '/' not in child_dir[len(dir_path)+1:]:
                folder_name = child_dir.split('/')[-1]
            else:
                continue

            if child_dir in tree_ids:
                entry = Schema__Object_Tree_Entry(
                    tree_id  = tree_ids[child_dir],
                    name_enc = self.crypto.encrypt_metadata(read_key, folder_name),
                )
                entries.append(entry)

        # Build and store tree object
        tree_obj   = Schema__Object_Tree(schema='tree_v1', entries=entries)
        tree_json  = json.dumps(tree_obj.json()).encode()
        tree_enc   = self.crypto.encrypt(read_key, tree_json)
        tree_id    = obj_store.store(tree_enc)
        tree_ids[dir_path] = tree_id

    return tree_ids.get('', tree_ids.get(list(tree_ids.keys())[0] if tree_ids else '', ''))
```

### 5.3 Update `_checkout_tree()` — Recursive Tree Extraction

Currently iterates flat entries. New version walks sub-trees recursively:

```python
def _checkout_tree(self, directory, tree_id, obj_store, read_key, base_path=''):
    """Recursively extract files from a tree into the working directory."""
    ciphertext = obj_store.load(tree_id)
    tree_json  = json.loads(self.crypto.decrypt(read_key, ciphertext))
    tree       = Schema__Object_Tree.from_json(tree_json)

    for entry in tree.entries:
        name = self.crypto.decrypt_metadata(read_key, str(entry.name_enc))
        full_path = os.path.join(base_path, name) if base_path else name

        if entry.blob_id:
            # File — decrypt and write
            blob_ct   = obj_store.load(str(entry.blob_id))
            plaintext = self.crypto.decrypt(read_key, blob_ct)
            file_path = os.path.join(directory, full_path)
            os.makedirs(os.path.dirname(file_path), exist_ok=True)
            with open(file_path, 'wb') as f:
                f.write(plaintext)
        elif entry.tree_id:
            # Folder — recurse
            self._checkout_tree(directory, str(entry.tree_id),
                               obj_store, read_key, full_path)
```

### 5.4 Update `status()` — Walk Sub-Trees for Comparison

`status()` currently compares flat path entries. With sub-trees, it needs to
walk both the committed tree and the working directory, building flat path
maps from both for comparison. The simplest approach:

```python
def _flatten_tree(self, tree_id, obj_store, read_key, prefix=''):
    """Walk sub-trees recursively, return {path: {blob_id, content_hash}}."""
    result = {}
    ciphertext = obj_store.load(tree_id)
    tree_json  = json.loads(self.crypto.decrypt(read_key, ciphertext))
    tree       = Schema__Object_Tree.from_json(tree_json)

    for entry in tree.entries:
        name = self.crypto.decrypt_metadata(read_key, str(entry.name_enc))
        full_path = f'{prefix}/{name}' if prefix else name

        if entry.blob_id:
            content_hash = self.crypto.decrypt_metadata(read_key, str(entry.content_hash_enc))
            result[full_path] = dict(blob_id=str(entry.blob_id), content_hash=content_hash)
        elif entry.tree_id:
            result.update(self._flatten_tree(str(entry.tree_id), obj_store, read_key, full_path))

    return result
```

Then compare against `_scan_local_directory()` as before.

### 8.1 Graph-Walking Clone (Replace List-All)

```python
def clone(self, vault_key, directory, on_progress=None):

    # Step 2: Walk commit → tree → blobs (graph-following)
    self._fetch_commit_and_trees(vault_id, commit_id, obj_store, read_key, sg_dir, _p)

    # Step 3: Read branch index (deterministic, 1 request)
    index_data    = self.api.read(vault_id, f'bare/indexes/{branch_index_file_id}')
    branch_index  = self._decrypt_branch_index(index_data, read_key)

    # Step 4: Fetch keys referenced by branch index
    for branch in branch_index.branches:
        self._fetch_key(vault_id, str(branch.public_key_id), sg_dir)
        if branch.private_key_id:
            self._fetch_key(vault_id, str(branch.private_key_id), sg_dir)

    # Step 5: Create new clone branch + register
    ...
```

### 8.2 Lite Clone Mode (Future — Design Now)

Add a `lazy` parameter to `Vault__Object_Store`:

```python
class Vault__Object_Store(Type_Safe):
    vault_path : Safe_Str__Vault_Path = None
    crypto     : Vault__Crypto
    api        : Vault__API           = None   # NEW: for on-demand fetch
    vault_id   : str                  = ''     # NEW: for on-demand fetch

    def load(self, object_id: str) -> bytes:
        path = self.object_path(object_id)
        if os.path.isfile(path):
            with open(path, 'rb') as f:
                return f.read()

        # Not local — fetch on demand if API is configured
        if self.api and self.vault_id:
            data = self.api.read(self.vault_id, f'bare/data/{object_id}')
            if data:
                os.makedirs(os.path.dirname(path), exist_ok=True)
                with open(path, 'wb') as f:
                    f.write(data)
                return data

        raise FileNotFoundError(f'Object not found: {object_id}')
```

This is the transparent interception layer. Same code paths, lazy fetching.
Objects accumulate locally over time. The `api` and `vault_id` fields are
only set for lite clones — full clones leave them as None.

### 9.2 Sub-Tree Merge with Shortcut

The three-way merge (`Vault__Merge`) needs updating for sub-trees.
The key optimisation: if a sub-tree ID is the same in both branches,
skip it entirely.

```python
def _compare_tree_ids(self, base_tree_id, ours_tree_id, theirs_tree_id):
    """Sub-tree shortcut: if IDs match, no need to recurse."""
    if base_tree_id == ours_tree_id == theirs_tree_id:
        return 'unchanged'
    if base_tree_id == ours_tree_id:
        return 'theirs_changed'
    if base_tree_id == theirs_tree_id:
        return 'ours_changed'
    if ours_tree_id == theirs_tree_id:
        return 'both_same'
    return 'both_changed'   # must recurse
```

Only when both sides changed the same sub-tree does the merge recurse
into that folder. This is O(changed folders) not O(all files).

### 10.1 New Method: `batch_read()`

```python
def batch_read(self, vault_id: str, file_ids: list) -> dict:
    """Batch read multiple files in one request.

    Returns dict mapping file_id → bytes (payload) or None (not found).
    """
    operations = [{'op': 'read', 'file_id': fid} for fid in file_ids]
    result = self.batch(vault_id, '', operations)  # no write_key needed for reads
    payloads = {}
    for r in result.get('results', []):
        if r.get('status') == 'ok' and r.get('data'):
            payloads[r['file_id']] = base64.b64decode(r['data'])
        else:
            payloads[r['file_id']] = None
    return payloads
```

Note: the `batch()` method currently requires `write_key` in the header.
For read-only batches, the server should accept requests without `write_key`.
This may need a backend change (see backend impl plan).
