# Vault-Based CRM and Email Outreach Workflow

**version** v0.16.17
**date** 17 Mar 2026
**from** Human (project lead)
**to** Developer (lead), Sherpa, Ambassador
**type** Dev brief

## The Problem

There are early access users and paying customers who have credits but have not used them. We need to communicate with them: feature announcements, encouragement to try new capabilities, personalised outreach. But the current approach (using an LLM to generate and send full HTML emails via MCP) is too slow. A 20KB HTML email takes too long to generate and assemble in a single LLM session.

## The Solution: EML Files in Vaults

Split the workflow into two phases:

**Phase 1: Create the emails.** In one session (or a dedicated UI), create a set of EML files. An EML file is a standard email format: headers (to, from, subject, CC), body (HTML or plain text), attachments. Create them from an HTML template with templating variables, with optional LLM personalisation per user.

**Phase 2: Send the emails.** Share the vault (or folder) containing the EML files with a Claude session that has MCP access to WorkMail. That session reads each EML file and sends it. Or use a programmatic tool that talks directly to the WorkMail API.

The vault becomes the outbox. EML files move from an `outbox/` folder to a `sent/` folder after sending. Delivery receipts and metadata are added alongside the sent files. The vault is the audit trail.

## Vault as Mini-CRM

Take this further: build a vault-based CRM for all users and contacts.

Per contact, store:
- Basic information (name, email, company)
- LinkedIn data (scraped or manually added)
- Semantic knowledge graph (key facts, interests, relationship history)
- Message history (every EML sent and received)
- Interaction log (when they signed up, what they purchased, when they last used the product)

Each contact could be a folder in the vault, or eventually each contact could have their own vault (vaults inside vaults).

### LLM Personalisation

With a per-contact knowledge graph, LLM calls become targeted and cheap:
- "Write a one-paragraph personalised message for this user based on their profile" (small LLM call, not generating the whole email)
- "This user is a CISO. Emphasise the zero-knowledge architecture."
- "This user bought credits 2 weeks ago and has not used them. Encourage them to try the folder sharing feature."

The LLM does not generate the entire email. It generates the personalised paragraph. The template handles the rest. This is fast and cost-effective.

### Multi-Language Support

Combine with the translation pipeline: generate the personalised message, then translate for users in different locales. Portuguese users get Portuguese emails. The LLM personalisation and translation are separate, composable steps.

## Backup and Durability

The vault-as-database pattern introduces a risk: losing the vault key means losing the CRM data. Mitigations:

- Vault key stored securely (OS keychain, separate vault, hardware key)
- Regular vault export as encrypted zip (backup to S3, local storage, or another vault)
- Eventually: SG/Send Hub (the GitHub-equivalent) provides hosted backup for vaults
