## Abstract (For Submission)

The security industry is about to undergo a Cambrian explosion. The same agentic AI capabilities that are transforming software development will enable thousands of new security companies to emerge, replacing the expensive, poorly integrated incumbent solutions that have left organisations unprepared for what comes next.

This talk argues that the OWASP community is perfectly positioned to lead this explosion. The people in this room have the domain expertise, the open source ethos, and the technical depth to build the next generation of security companies. What they have lacked until now is the ability to scale: to go from a boutique consultancy serving a handful of clients to a product company serving thousands.

Agentic AI changes this equation. Using a case study of building SG/Send (a zero-knowledge encrypted platform) with a team of 20+ AI agents, this talk demonstrates how a single practitioner can build, test, deploy, and maintain enterprise-grade software at a fraction of the traditional cost. The talk provides concrete examples of AppSec companies that attendees could create tomorrow, built on open source foundations, across every domain in the OWASP Top 10 and beyond.

## The Argument

### The Problem: Incumbents Have Failed

The current security landscape:

- Expensive software that was never well integrated
- Solutions that were never well connected across the stack
- Tools that did a "decent job" but created the environment we have now
- Identity systems with crazy vulnerabilities
- Authorisation and verification systems that are fundamentally broken
- Organisations utterly unprepared for what is coming next

Cyber security has never been about security. It is about changing the business to protect itself. Building resilience. Understanding the side effects of what you create. The incumbents sold tools. What organisations needed was transformation.

### The Catalyst: Software Explosion

We are not going to have fewer developers. We are going to have more. Every team, every part of the business, will create mini apps. The volume of software being created is about to explode across every level of maturity:

- Vibe-coded prototypes (Explorer phase, minimal security)
- Team-built internal tools (Villager phase, some security)
- Enterprise production systems (Town Planner phase, full security)
- Autonomous agents (the ultimate insider threat: powerful, dangerous, and operating at machine speed)

Each level needs security. The incumbents cannot serve all of them. The market needs thousands of focused companies, not a dozen generalist platforms.

### The Opportunity: Cambrian Explosion

The OWASP community has been creating boutique security companies for years. Consultancies. Niche tools. Specialised services. They just never scaled. The typical pattern: a few practitioners, a handful of clients, custom work, limited growth.

Agentic AI changes the scaling equation:

- One practitioner with AI agents can do the work of a team
- Open source foundations mean the code is not the moat (services and expertise are)
- Domain expertise (which the OWASP community has in abundance) is what the agents amplify
- The same agentic workflows that build software can build security tools

### The Model: Open Source, Sovereignty, Services

The business model for these new companies:

- **Open source code** (because code becomes disposable, value is in services)
- **Sovereignty** (customers deploy in their own environment, no vendor lock-in)
- **Services on top** (customisation, integration, managed security, expertise)
- **Market pressure** (the organic food model: companies that use well-maintained open source are more trustworthy)

### The Case Study: SG/Send

A live demonstration of the model in practice:

- 20+ AI agents, organised into Explorer/Villager/Town Planner teams
- Zero-knowledge encrypted platform built, tested, and deployed
- QA infrastructure with automated browser testing, auto-generated documentation
- Patch-based development where agents never have repo write access
- Open source, providing building blocks for others to build on

The key message: "I am providing building blocks for you to build your companies on top of."

### Concrete Company Examples

For each domain, a company that an attendee could start tomorrow:

| Domain | Company Idea |
|---|---|
| Threat modelling | Automated threat model generation from architecture diagrams, at scale (1000s not dozens) |
| Code review | AI-powered code review with Explorer/Villager/Town Planner depth levels |
| Architecture review | Automated architecture analysis using semantic graphs and ontologies |
| Agent security | Controlling LLM blast radius, agent identity management, signed agent communication |
| Identity management | Sovereign identity solutions for the mini-app explosion |
| DevSecOps | Security pipelines that run as agentic teams (not just scanners) |
| Supply chain | Open source dependency management with the organic food model |
| Risk management | Continuous risk assessment using vault-based evidence collection |
| User training | AI-generated security training customised to each organisation's actual stack |
| Compliance | Automated GRC with agentic teams (the due diligence team pattern) |
| OWASP Top 10 | A company per vulnerability class, providing detection + remediation + prevention |

Each example should include: the problem, the agentic approach, the open source foundation, and the revenue model.
