## The Opportunity

Conversations with actors, musicians, and recent graduates revealed a real pain point: sharing a professional portfolio is harder than it should be.

The current workflow for most artists: upload videos to YouTube, share the YouTube link. This has significant problems:

- YouTube's UI is cluttered with ads, recommendations, and unrelated content
- The viewing environment is not focused on the artist's work
- Hard to bookmark specific pieces within a collection
- No way to know if the recipient actually watched the material
- No control over the presentation (YouTube decides the layout)
- Not professional-looking for agents, casting directors, or recruiters

When shown the SG/Send folder view and gallery view, the reaction was immediate excitement. A clean, focused, professional presentation of their materials, with a single link.

## The Portfolio View

A portfolio is a vault with a custom UI layer. The artist uploads:

- Videos (showreels, performances, clips)
- Images (headshots, production stills)
- Documents (CV/resume, credits list, biography)
- Audio (voice reels, music demos)
- Text (artist statement, project descriptions)

SG/Send creates a mini website from these materials:

- Gallery view with thumbnails for every piece
- Video playback in the browser (no download required)
- Headshot and photos in a clean grid
- Documents viewable inline (PDF, markdown)
- Metadata per item (title, description, date, category)

The artist shares a single link. The recipient opens it and sees a professional portfolio page. No account needed. No download needed. No YouTube clutter.

## Key Features for This Use Case

### View Tracking

Critically important for artists: has the agent actually watched my showreel? The portfolio should track:

- How many times each item has been viewed
- Which items were opened
- Total time on the portfolio page
- Whether videos were played (and how much was watched)

This information is visible to the portfolio owner. "I sent my portfolio to 5 agencies. 3 opened it. 2 watched the full showreel. 1 only looked at the headshot." This is genuinely valuable data that artists do not have today.

### Lazy Loading

The portfolio should not require downloading all files upfront. The initial load should be fast: structure, thumbnails, metadata. Full files (especially videos) load on demand when the recipient clicks. This means a portfolio with 2GB of video loads instantly and plays specific videos on request.

This is the zip-as-mini-website pattern: the download contains the structure and thumbnails, the full files stream when needed.

### Custom Presentation

The portfolio owner should be able to customise:

- Order of items (featured work first)
- Categories/sections (Acting, Voice, Music, etc.)
- A personal introduction (text or video)
- Colour scheme or theme (from the `<sg-theme>` system)
- Which items are visible to which recipients (different portfolios for different opportunities)

### No Account Required

The recipient does not need an SG/Send account. They click the link and see the portfolio. Zero friction. This is essential for the use case: an agent receiving 50 portfolios a day will not create an account for each one.

## Two Markets, Same Feature

### Artists and Performers

Actors, musicians, dancers, visual artists, photographers. Anyone who needs to share a body of work professionally. The market is large and currently underserved (YouTube is the default, and it is terrible for this purpose).

### Job Seekers and Recruitment

The same portfolio concept works for professionals sharing their work:

- Developers sharing code projects, demos, and technical writing
- Designers sharing a visual portfolio
- Consultants sharing case studies and presentations
- Anyone creating "apps and videos about themselves" as part of modern job applications

The recruitment angle is the same pain point: sharing professional materials in a focused, trackable, professional presentation.

## Connection to Existing Features

This is not a new product. It is a composition of existing features:

| Existing Feature | How It Serves the Portfolio |
|---|---|
| Gallery view with thumbnails | The portfolio grid |
| Video playback in browser | Showreel viewing |
| Folder structure in browser | Organised sections |
| View tracking (download count) | "Has my portfolio been seen?" |
| Vault with custom UI | The portfolio IS a vault |
| `<sg-theme>` theming | Professional customisation |
| Manifest-driven focused UIs | Portfolio as a pre-configured workflow |
| Zero-knowledge encryption | Private portfolios (sensitive audition materials) |

The focused use-case UI brief from earlier this week maps directly: a "Create Portfolio" page that is a pre-configured upload flow with portfolio-specific defaults.
