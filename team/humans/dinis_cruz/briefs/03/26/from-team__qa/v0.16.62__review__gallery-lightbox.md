## Overview

The gallery view displays when a user opens a download link on the `/gallery/` route with a zip containing multiple files. It shows a thumbnail grid with file type badges, and clicking any item opens a lightbox overlay for full viewing.

The review was done using a zip with 2 files: a JPG image ("SG:Send Autonomous AI QA Results.jpg") and a markdown file ("v0.16.53__debrief__browser-automation-qa-execution.md"), accessed via friendly token `creek-reed-0458`.

---

## Issue G-001: Markdown in lightbox has poor typography — REGRESSION

**What the reviewer said:** "I believe in v0.2.x this was different and better"

**What's visible in the screenshots:**

**Gallery view (before click):** Two thumbnails side by side — the JPG shows a preview of an infographic about QA results, the MD shows a raw text preview starting at "1. Environment Setup / What Was Installed". Both have type badges (JPG, MD) in the top-right corner. The gallery header shows "2 files.zip GALLERY 655.5 KB · 2 files ✓ Decrypted" with action buttons.

**Lightbox (after clicking the markdown file):** A full-screen dark overlay with the markdown rendered in a white box. The content shows:
- Title: "Debrief: Browser Automation and QA Test Execution Inside a Claude Session"
- Version/date metadata
- "Executive Summary" heading
- Body text

However, the typography is flat — headings are unstyled (just bold, no sizing differentiation), there are no horizontal rules between sections, no colored code elements, and the white box is narrow and has no box-shadow. The content looks like raw HTML with a white background rather than a properly styled document.

**In contrast — the print preview (same document):** Clicking Print on the same markdown file opens a beautifully formatted preview with:
- SG/Send branded header (logo + filename)
- Teal accent line under the header
- Properly sized headings (large h1, medium h2)
- Horizontal rules between sections
- Well-formatted tables with borders and header backgrounds
- Inline code with pink color and background
- Links in a distinct color
- Clean spacing and typography (DM Sans font)

**The gap is stark:** The print preview looks professional and polished. The lightbox looks like someone pasted HTML into a white div. The reviewer explicitly noted the print preview looks "really good" and the lightbox should aspire to that quality.

**v0.2.x had this right:** The v0.2.7 overlay added rich markdown styling to the lightbox — styled headings with border-bottom on h1, colored inline code (#d63384), pre blocks with background and border, blue links (#0969da), blockquote styling, table styling with header backgrounds, and a box-shadow on the container. This was all lost in the v0.3.0 rewrite.

## Issue G-002: Lightbox is missing Print and Share action buttons — REGRESSION

**What's visible in the screenshots:** The lightbox shows:
- Header: SG/Send logo + brand, filename, close (×) button
- Body: prev (‹) nav, content area, next (›) nav
- Footer: page counter ("2 / 2"), Print button (hidden unless PDF), Save button

There are no Print or Share buttons visible in the lightbox for non-PDF files. The gallery header bar (which does have Copy Link, email, print, Save locally, Folder view, and info buttons) is completely hidden behind the dark lightbox overlay — none of those controls are accessible while viewing content.

**In v0.2.x:** The lightbox header area had three action buttons: "Save locally", a Print icon button, and a Share icon button. Print called `window.print()`. Share copied the download URL to clipboard with "✓ Copied!" visual feedback. For markdown files specifically, later overlays upgraded the Print button to use the SgPrint component (which produces the beautiful A4 print preview the reviewer liked).

## Issue G-003: Lightbox markdown should match print preview quality — FEATURE REQUEST

**What the reviewer said:** "capture the feature request that the markdown in the Gallery view should look as good as the print preview (pic3) which looks really good"

**Details:** The print preview (via SgPrint) sets a much higher quality bar than even v0.2.x's lightbox had. The reviewer explicitly called out the print preview as looking "really good" and wants the lightbox to match it. Specific qualities of the print preview that the lightbox lacks:
- SG/Send branded header with logo and filename
- DM Sans / JetBrains Mono fonts
- Teal accent line
- Properly rendered tables with borders, cell padding, and header background colors
- Inline code with distinct color and background
- Clean heading hierarchy with proper sizing and spacing
- Overall professional document feel

This goes beyond the REG-001 fix (which restores v0.2.7-level styling). This is about elevating to the SgPrint quality bar. The SgPrint component already exports its styles as `SgPrint.PRINT_STYLES` — these could potentially be reused or adapted for the lightbox.
