## The Strategy

The marketing strategy should be case studies and workflows that make people say "hold on, how can you do that?" Every demo should combine encrypted storage (vault/send) with LLM usage (tokens, API calls) because both generate revenue.

The intersection is: workflows that use SG/Send for secure data handling AND tokens for LLM processing. This is where the two revenue streams meet.

## The Investor Review Workflow

The human has published a framework for evaluating startups (a seven-point review criteria). Investors have already asked for reviews based on this framework. Turn this into a self-service tool on the site.

### The Flow

1. Investor visits the review page (e.g. tools.sgraph.ai/investor-review or a dedicated URL)
2. Investor enters their access token (confirms they have credits)
3. Investor enters data about the startup they want to evaluate (structured form: company name, sector, stage, pitch deck upload, financials upload, etc.)
4. LLM processes the input against the seven-point framework (using the provisioned OpenRouter token)
5. The review is generated: structured report, scores per criterion, analysis, recommendations
6. A vault is created in the browser containing the report, uploaded documents, and generated analysis
7. The vault currently exists only in the browser (local encrypted state)
8. The investor can: download the report, push the vault to SG/Send (gets an access key), share the vault with others

### What Makes This Powerful

- The review happens entirely in the browser. Documents are encrypted before they touch any server.
- The LLM call goes directly from the browser to OpenRouter (via the provisioned token). SG/Send never sees the startup data.
- The vault is created locally first. The investor decides whether to push it to SG/Send for sharing or keep it local.
- If pushed, the recipient gets the vault key and can access the full review pack: report, source documents, analysis.

This is a complete end-to-end demonstration of the technology: encrypted upload, LLM processing, vault creation, secure sharing. And every step generates revenue (upload credits + LLM token usage).

## Write-Only Vault Links (Upload to Vault)

A feature that enables this workflow and many others: a link that allows someone to upload a file to a vault without being able to read the vault.

This is a write-only, one-shot capability. The uploader commits a file to a branch. They cannot read other files. They cannot see the tree. They can only append.

Use cases:

- Investor asks a startup founder: "upload your pitch deck to this link"
- A researcher does analysis in ChatGPT/Claude, then uploads the output to a vault via a write-only link
- A customer submits documents to a compliance vault

The CLI already supports commit-only operations. The branch model (see arch spec) naturally supports this: the uploader gets a branch, commits to it, the vault owner merges. This brief is about exposing that capability as a shareable link with a simple browser UI.

## The Marketing Principle

Every case study and demo workflow should:

1. Show a real, useful workflow (not a toy demo)
2. Use both encrypted storage and LLM tokens (dual revenue)
3. Make the viewer ask "how does that work?"
4. Be self-service (the viewer can try it themselves)
5. Be shareable (the output is a vault that can be sent to someone else)

| Competitive analysis | Company name | Web research, comparison | Research pack in vault |

Each one is a case study, a revenue generator, and a technology demonstration.
