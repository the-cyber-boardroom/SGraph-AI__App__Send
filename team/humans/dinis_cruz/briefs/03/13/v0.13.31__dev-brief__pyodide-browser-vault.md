# Pyodide: sg-send-cli Running Entirely in the Browser

**version** v0.13.31
**date** 13 Mar 2026
**from** Human (project lead)
**to** Developer (lead), Architect
**type** Dev brief

---

## The Idea

When telling a user "just run `pip3 install sg-send-cli`," a thought: this should also work entirely in the browser via Pyodide. No Python install. No terminal. No pip. Just a web page where you can clone, browse, and interact with a vault.

Pyodide is CPython compiled to WebAssembly. It runs Python packages in the browser. If `sg-send-cli` and its dependencies are compatible, the entire vault CLI could execute client-side with zero installation.

## Why This Matters

The current onboarding path for vaults requires:

1. Install Python 3
2. Run `pip3 install sg-send-cli`
3. Use the terminal

This is trivial for developers. It is a wall for everyone else. A browser-based vault tool removes steps 1 and 2 entirely. The user visits a page, enters a vault key, and they are in.

Combined with the vault website work (create vault, open vault as primary actions), this means the entire vault experience can happen without leaving the browser. Create a vault, add files, share the key, recipient opens it in their browser. No installs anywhere in the chain.

## MVP Scope

Build a proof of concept:

1. Load Pyodide in a web page
2. Install `sg-send-cli` into the Pyodide environment
3. Clone a vault by key
4. Display the vault tree (file listing)
5. Read and display file contents

That is the MVP. If this works, the vault becomes accessible to anyone with a browser.

## Technical Questions

- Does `sg-send-cli` have any C extensions that won't compile to WASM? (If pure Python, it should work.)
- Does the cryptography dependency (AES-256-GCM, HKDF) work in Pyodide? (Pyodide includes `cryptography` or we could use the Web Crypto API as an alternative.)
- Network access: Pyodide can make HTTP requests via `pyodide.http.pyfetch`. Does this work against the SG/Send API?
- Performance: how fast is vault decryption in WASM compared to native Python?

## Connection to Other Briefs

This intersects with:
