## The Insight

Sharing a collection of photos and videos is one of the most common things people do — and one of the most broken experiences in 2026. Every existing option is either ugly, lossy, privacy-invasive, ecosystem-locked, or requires the recipient to download and unzip files.

SG/Send already has every piece of infrastructure needed to solve this. What's missing is the UX: a beautiful, purpose-built gallery view that makes the photos and videos the entire experience, with encryption invisible underneath.

**This is not a feature. This is a product.**

---

## Why Every Current Option Is Bad

| Method | What Goes Wrong |
|---|---|
| **WhatsApp** | Compresses images (destroys quality), compresses video (destroys quality), sends as individual messages that flood the chat, no gallery view, no organisation, 16MB per file (or larger with quality loss), recipient can't browse — they scroll through chat history |
| **Email** | 25MB attachment limit (Gmail). Can't send more than a handful. No gallery. No preview. Recipient downloads attachments one by one. |
| **iCloud Shared Album** | Requires recipient to have an Apple device. Apple compresses. No encryption. Apple scans content. Locked to one ecosystem. |
| **Google Photos shared album** | Requires Google account. Google scans and analyses every image (facial recognition, object detection, location tracking). No encryption. Privacy nightmare for sensitive content. |
| **Dropbox / Google Drive** | Share a folder link. Recipient sees a file list. No gallery view. Downloads one by one (or downloads a zip and unzips locally). Functional but ugly. |
| **WeTransfer** | Downloads a zip. Recipient unzips. Opens images in whatever viewer they have. No gallery. No preview. Files expire. |
| **AirDrop** | Only works Apple-to-Apple, in physical proximity. Not a sharing solution. |

**Nobody offers:** Encrypted, full-resolution, beautiful gallery view in the browser, accessible via a link, no account required, no app required, no ecosystem lock-in, works on any device.

**SG/Send can.**

## Why This Is More Than a Feature

### The Market

Photo and video sharing is one of the highest-volume use cases on the internet. Everyone does it: families, friends, event photographers, real estate agents, wedding photographers, medical professionals sharing imaging, lawyers sharing evidence photos, insurance adjusters sharing damage photos, construction teams sharing site photos.

Most of these users have never thought about encryption. They will use SG/Send because the gallery experience is better than the alternatives — and the encryption is the bonus.

### The Funnel

```
User has 50 photos to share
  │
  ▼
Discovers SG/Send Gallery (via a link someone sent them,
or via the demo gallery on sgraph.ai)
  │
  ▼
Creates their own gallery (buys £5 credits)
  │
  ▼
Shares the link → recipients see the gallery
  │
  ▼
Recipients think "this is nice, how do I do this?"
  │
  ▼
New user sign-up → viral loop
```

**This is the highest-potential viral loop SG/Send has.** Every gallery shared is a product demo to every recipient. The better the gallery looks, the more likely the recipient becomes a user.

### The DRU Impact

A single gallery upload = 50-100 files = 50-100 credits consumed. Compare with sending a single PDF (1 credit). Gallery sharing drives dramatically higher credit consumption per use, which means:
- Higher revenue per user
- Faster credit depletion → faster re-purchase
- More files in the system → more downloads → more bandwidth (billable)
- More recipients → more potential new users

## The Foundation for Powerful Features

The gallery isn't just a viewer. It's a platform for features that compound:

### Phase 1: Beautiful Gallery (the MVP)

- Grid thumbnail view
- Full-screen lightbox with swipe navigation
- Video plays inline
- Pinch to zoom on mobile
- Download individual images or download all
- Responsive: beautiful on phone, tablet, and desktop
- Friendly encryption key (purple-sunset-442)

### Phase 2: Basic Photo Editing

Once images are in the gallery, basic editing becomes natural:

| Feature | What It Does | Value |
|---|---|---|
| **Crop** | Trim an image | Prepare photos before sharing |
| **Rotate** | Fix orientation | Common phone photo issue |
| **Adjust brightness/contrast** | Basic enhancement | Make photos look better |
| **Filters** | Instagram-style presets | Fun, engagement, makes sharing more appealing |
| **Annotate** | Draw on images, add text | Business use: mark up photos, circle issues |
| **Redact** | Black out sensitive areas | Legal, medical, privacy use cases |

All of this runs client-side (Canvas API, JS image processing). No server involvement. The edited image is saved as a new version in the gallery (encrypted, of course).

### Phase 3: LLM Transformations

This is where it gets genuinely powerful:

| Feature | What It Does | Value |
|---|---|---|
| **Auto-caption** | LLM describes each photo | Accessibility, searchability, documentation |
| **Auto-organise** | LLM groups photos by content (food, people, landscape, documents) | Large galleries become navigable |
| **Generate summary** | "These 42 photos are from a Saturday lunch at a Portuguese restaurant, followed by a walk along the river" | Context for the recipient |
| **Remove background** | AI-powered background removal | Product photography, profile pictures |
| **Enhance** | AI upscaling, noise reduction | Improve phone photos |
| **Create slideshow** | Auto-generate a slideshow with transitions and music | Family events, presentations |
| **Extract text** | OCR on photos of documents, whiteboards, signs | Documentation, note-taking |
| **Translate text in images** | OCR + translate text visible in photos | Multilingual — connects to the translation infrastructure |

All LLM calls happen client-side (user's own API keys, as per the document transformation workspace architecture). The server never sees the images or the LLM interactions. Zero-knowledge all the way through.

### Phase 4: Collaborative Gallery

- Multiple people contribute photos to the same gallery (shared vault key)
- Comments on individual photos
- Favourites / ratings
- Shared editing (one person crops, everyone sees the result)
- Timeline view (photos ordered by EXIF timestamp)
- Map view (photos placed on a map by GPS coordinates)

## Technical Implementation

### What We Already Have

| Component | Status | Gallery Usage |
|---|---|---|
| Folder upload → auto-zip | ✅ Shipped | User drops a folder of photos → zipped and uploaded |
| In-browser zip processing | ✅ Shipped | Extract images from zip client-side |
| Image display | ✅ Shipped | Show individual images |
| Video playback | ✅ Shipped | Play videos inline in the gallery |
| Encryption (AES-256-GCM) | ✅ Shipped | Everything encrypted, server knows nothing |
| Friendly key concept | 🔧 Designed (today's brief) | "blue-mountain-77" instead of hex string |
| Purpose-built views | 🔧 Designed (today's brief) | Gallery is the first purpose-built view |

### What We Need to Build

| Component | Complexity | Notes |
|---|---|---|
| **Thumbnail grid layout** | Medium | CSS grid with lazy loading. Generate thumbnails client-side from the decrypted images. |
| **Lightbox viewer** | Medium | Full-screen overlay with swipe navigation. Use a lightweight JS library or build custom with touch events. |
| **Mobile optimisation** | Medium | Touch gestures (swipe, pinch-zoom), orientation handling, safe-area insets for notch phones. |
| **Gallery metadata** | Small | A gallery.json inside the zip: title, description, photo order, captions (optional). |
| **Progress indicator** | Small | Decrypting and extracting 50 images takes a moment — show a progress bar. |
| **Lazy thumbnail generation** | Medium | Don't generate all 100 thumbnails at once. Generate as the user scrolls. |
| **Download options** | Small | Download single image, download selected, download all (original zip). |

### Performance Considerations

A gallery of 100 photos at 5MB each = 500MB total zip. This is within our confirmed working range (300MB+ validated). The approach:

1. Download the encrypted zip (show progress)
2. Decrypt the zip (fast, AES-256 is hardware-accelerated in browsers)
3. Extract the zip index (file list, sizes) — instant
4. Generate thumbnails on demand as the user scrolls (lazy loading)
5. When user taps a photo, extract and display the full-resolution image
6. Videos: extract and stream via blob URL

**The user never waits for all 100 photos to extract.** They see the grid filling in as thumbnails are generated, and full images load on tap. This is the same pattern as every native photo gallery app.

### The Gallery.json Metadata File

Inside the zip (alongside the images), an optional metadata file:

```json
{
  "title": "Photos from Saturday",
  "description": "Lunch at the Portuguese restaurant + walk along the river",
  "created": "2026-03-03T14:00:00Z",
  "items": [
    {
      "filename": "IMG_4521.jpg",
      "caption": "The restaurant entrance",
      "type": "image",
      "order": 1
    },
    {
      "filename": "VID_4530.mp4",
      "caption": "Live music at the restaurant",
      "type": "video",
      "order": 8
    }
  ],
  "sort": "exif_date"
}
```

If the metadata file doesn't exist, the gallery auto-generates it from the zip contents (sort by filename or EXIF date, no captions).

## Market Positioning

### Consumer: "The Beautiful Way to Share Photos"

**Message:** "Share full-resolution photos and videos with anyone. No compression, no account needed, no app required. Just a link and a code."

**Target:** Anyone who shares photos — families, friend groups, travellers, event attendees. The encryption is the bonus, not the pitch.

### Professional: "Client-Ready Photo Delivery"

**Message:** "Deliver photos to clients in a branded, encrypted gallery. Full resolution, no downloads, instant viewing."

**Target:** Photographers (wedding, event, real estate, product), designers, agencies. They currently use services like Pixieset, ShootProof, or just Dropbox. SG/Send offers encryption + better viewing experience.

### Enterprise: "Secure Image and Evidence Sharing"

**Message:** "Share sensitive visual evidence — medical imaging, legal photos, insurance damage, construction site documentation — encrypted end-to-end with a verifiable audit trail."

**Target:** Legal teams, medical professionals, insurance companies, construction firms. These sectors already need encryption for compliance; the gallery UX is the upgrade.
