## The Epiphany

While emailing materials to investors today, the human realised: **we already built the data room.**

Not intentionally as a "data room feature." It emerged from the combination of four capabilities that were built independently for other reasons:

1. **Folder upload** (built because a customer asked "why can't I drop folders?")
2. **Zip view in browser** (built as the reverse of folder-to-zip — "why download and unzip?")
3. **File preview** (PDF, images, markdown — built for general file viewing)
4. **Large file handling** (300MB+ confirmed working)

These four features, combined, **are** a data room. A folder of files, encrypted, accessible via a link, with in-browser preview of every file. This is a textbook example of Wardley Map evolution — commoditised primitives combining into something of higher value. No additional code changes are needed for this to work — it works today, right now, as deployed.

---

## The Real Use Case That Triggered the Epiphany

### The Old Workflow (Before)

The human reviews startups for investors. The previous workflow was:

1. Receive a deck via email (unencrypted, confidential information sent in the clear)
2. Do research (Claude/ChatGPT deep research on the company)
3. Apply the seven-step evaluation framework (the GenAI cybersecurity framework from the v0.10.1 brief, now generalised)
4. Generate assessment report
5. Create infographic via NotebookLM
6. Create slide deck
7. Email all materials back to the investors (multiple attachments, no organisation, no security)

### What Happened Instead

The human created a folder with numbered files:

```
1. Assessment Criteria (so the reader understands the framework first)
2. Infographic (visual summary — the hook)
3. Slide Deck (presentation-ready analysis)
4. Detailed Report (the full write-up)
5. Deep Research (the raw ChatGPT/Claude research)
6. Assessment Criteria PDF (the framework document)
  source-documents/
    (all markdown source files for transparency)
```

Then: **dropped the folder into SG/Send.** Got a link. Opened it. Saw the UI — every file clickable, every file previewable (PDFs rendered, images displayed, markdown rendered). A subfolder for source documents. Everything encrypted.

**"Holy shit, this is a data room."**

Then emailed the investors: "Here are the materials. Click this link. Here's what you'll find inside." Done.

### Why This Is a Milestone

This is the first time SG/Send was used as a **data room** for a real business purpose — sharing structured, confidential analysis materials with investors in an organised, secure, previewable format. It wasn't a test. It wasn't a demo. It was real work, done better than the previous workflow, using the product as it exists today.

## The Wardley Map Moment

This is exactly the pattern described in the AWS primitives analogy from today's daily brief: the primitives (storage + encryption) are the foundation, but the value is in what's built on top. The data room is the second "killer app" after the vault — and it shipped accidentally.

```
Individual primitives:          Combined capability:

File upload         -+
Encryption          -+
Folder to zip       -+------>  DATA ROOM
Zip preview         -+         (emerged, not designed)
File viewers (PDF,  -+
  image, markdown)  -+
Large file support  -+
```

The same pattern as AWS: storage + compute are the foundation, but the value is Lambda, RDS, and everything built on top. We built the primitives for their own reasons. The data room emerged. Four features that were each useful on their own, but in combination created something none of them were designed to be.

This is marketing content that writes itself because it's a real story.

### 5. The Name Question

What do we call it?

| Name | Pros | Cons |
|---|---|---|
| **Data Room** | Established term, investors/lawyers know it, implies confidentiality | Can sound enterprise-heavy |
| **Vault** | Already using this, implies security | Already used for the persistent storage product |
| **Folder Share** | Simple, descriptive | Too generic |
| **Room** | Short, clean | Too vague |
| **Package** | Describes what it is | Doesn't convey the viewing experience |

**Recommendation:** Use **"Data Room"** in marketing. It's the term the market understands. Distinguish internally between:
- **Instant data room** = folder upload -> shared link (what we have now, one-shot, immutable)
- **Vault** = persistent, editable, synced (what we're building, the evolution)

The instant data room is the gateway drug to the vault.
