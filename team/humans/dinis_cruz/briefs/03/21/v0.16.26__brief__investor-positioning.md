## Current Traction

Customers have completed the full purchasing and usage workflow end-to-end:

- Buy credits (£5)
- Upload files and folders via the UX
- Send a link to a recipient
- Recipient views files successfully

The live product is at sgraph.ai and can be tested by anyone.

### Feedback Received

Positive signals from the first MVP:

- **Ease of use:** Users found the platform straightforward to use
- **Scalability:** Some customers uploaded 1GB of files without issues
- **Folder view:** Almost universally liked. The ability to see files directly in the browser from an uploaded zip or folder is the standout feature

### Honest Assessment of Where We Are

Users have not yet reached the critical adoption thresholds:

| Threshold | Status |
|---|---|
| "I will use this regularly" | Not yet consistently hit |
| "I really need this for my workflow XYZ" | Not yet consistently hit |
| "This opens up new workflows I can't work without" | Not yet hit |
| "Hey, you REALLY need to use this platform" (viral referral) | The target for exponential growth |

The viral property is built into the product: every person who receives a link to a file or folder is already a user, and crucially a potential new customer. The distribution mechanism is the product itself.

### Why the Gap Exists

What is live today is not the most powerful workflow the platform enables. The features about to ship (see roadmap below) are expected to drive adoption from existing customers and the Early Access Programme users.

## Why This Market, Why Now

### The Market Is Stuck

Secure data/file sharing is an industry that has been stuck in time:

- **Barely innovating.** The core experience (upload a file, get a link) has not changed meaningfully in years.
- **Not embracing agentic workflows.** None of the major providers are building for AI agent use cases.
- **Not doing real client-side encryption.** In almost all cases, the provider has access to the raw content and can read it. Zero-knowledge encryption is the exception, not the norm.
- **Poor user experience.** Limited workflows, limited file type support, limited collaboration.
- **Subscription/ARR model friction.** Users and companies are increasingly pushing back against subscription fatigue. "Pay £10/month whether you use it or not" is losing appeal.

### The Initial Hypothesis

The starting hypothesis was: build a more secure and private version of existing platforms. But security alone is not the competitive wedge. The real strategy:

- **Simplicity and user-friendliness** as the primary differentiator
- **Rapid response** to user feedback and feature requests
- **Seamless experience** with no friction points
- **Usage-based billing** (pay for what you use, no subscriptions)
- **Zero tracking** (six-sentence privacy policy, zero cookies)
- **Deploy everywhere** (browser, desktop, mobile, CLI, AI agents)

### The Realisation: We Can Expand the Category

As features improved and the solution started working in practice, the opportunity expanded beyond competing with current players on their existing offerings. The real business opportunity is in expanding the use cases of the technology into areas the current market does not serve.

## Feature Roadmap (Launching in Coming Weeks)

These features have working MVPs and are being prepared for release:

### 1. Secure and Private Data Vaults

Powered by SG/Send and SGit (a PKI-signed and encrypted version of Git, created as part of this project). Vaults are encrypted data stores with version control, branching, and collaboration built in.

### 2. Effective Version Control for Documents

Branches, commits, pushes, pulls for documents and vaults. This is something most users have never experienced. Version control has been a developer tool. We are bringing it to everyone, with a UI that says "timeline" and "sync" instead of "branch" and "merge."

### 3. Powerful Data Viewing UIs

Gallery view with thumbnails for all file types (PDFs, markdown, images, code, audio, video). Folder browsing in the browser. Rich preview with sender-side thumbnail generation. More features and more usable than anything currently available.

### 4. Client-Side LLM Workflows

LLM integration that works with data stored in vaults. Processing happens client-side or via user-provided/purchased API tokens. The server never sees the unencrypted data. Users can analyse, summarise, and transform their vault contents using AI without compromising privacy.

### 5. Agent-to-Human and Agent-to-Agent Communication

Vaults as the communication plane between AI agents and between agents and humans. Signed messages, per-agent PKI, sequential workflow enforcement through cryptography. This is infrastructure for the agentic AI era.

### 6. PKI (Public and Private Keys)

Moving from symmetric encryption (shared decryption keys) to full PKI. Public/private key pairs provide confidentiality, integrity, identity assurance, and a highly effective authorisation and access-control layer. Every user, every agent, every document can have its own cryptographic identity.

### Each Feature Is Use-Case Driven

Every feature is built around specific use cases that add value and are validated by users and customers. The technology serves the workflow, not the other way around.

## The Agentic Development Advantage

The speed at which features ship is itself a competitive advantage. The engineering approach:

- **18+ custom AI agents** organised into Explorer, Villager, and Town Planner teams
- **Effective management via Git:** documentation, communication, briefs, and debriefs all flow through commits
- **Provenance and explainability:** every decision, every code change has a traceable brief
- **Highly focused agentic squads** created per task from the pool of agents
- **Control:** agents work on branches, humans review and merge

This is not a claim about AI. It is a measurable development velocity advantage. Features go from voice memo to shipped in hours, not sprints. The compound effect of this velocity is what allows a small team to compete with established players on breadth of features while maintaining quality.
