# Vault Welcome Experience: README, Video Introduction, and Guided Walkthrough

**version** v0.16.26
**date** 19 Mar 2026
**from** Human (project lead)
**to** Developer (lead), Designer, Sherpa
**type** Dev brief

## The Idea

When someone opens a vault or a shared folder, what is the first thing they see? Right now: a file list. But the sender knows what is in the folder. They know what the receiver should look at first, what matters, what the context is. That knowledge is currently lost. The receiver is left to figure it out by clicking through files.

The idea: let the sender define the opening experience. A welcome layer that appears when the vault is opened, before the receiver starts browsing files.

## What the Welcome Layer Can Be

From simplest to richest:

### README

A markdown file (`_welcome/README.md`) that renders as the first thing the receiver sees. Plain text, formatted, with links to specific files in the vault.

"Welcome to the Q3 investor pack. Start with the executive summary (exec-summary.pdf), then review the financials (financials/). The appendix contains supporting materials."

This is the equivalent of a GitHub README. It provides context, navigation, and instructions. It can be pre-generated by an LLM from the folder contents ("here are the files, write a README that guides the reader"). It can be translated into the receiver's language before sending.

### Video Introduction

A short video (`_welcome/intro.mp4`) that auto-plays (or prompts to play) when the vault opens. The sender records a personal walkthrough: "Hey, here is what I am sending you. Open this first, then look at that, here is why this matters."

This is powerful for complex deliverables. An investor pack with 20 documents benefits enormously from a 60-second video saying "here is what to look at and in what order." A consulting report with multiple sections benefits from the author walking through the structure.

The video is encrypted and stored alongside the vault contents. It is just another file in the `_welcome/` folder. The gallery view knows to present it first.

### Animation or Slideshow

A sequence of images or a lightweight HTML presentation (`_welcome/walkthrough.html`) that guides the receiver through the content visually. Simpler to produce than a video. Could be auto-generated from the folder structure and thumbnails.

### Interactive Guide

A step-by-step walkthrough built from components (similar to the Sherpa demo page concept). "Step 1: Open the executive summary. Step 2: Review the financial projections. Step 3: Check the appendix for due diligence materials." Each step links to or opens the relevant file.

This could be defined in a simple JSON format:

```json
{
  "schema": "welcome_guide_v1",
  "title": "Q3 Investor Pack",
  "steps": [
    { "label": "Start here", "file": "exec-summary.pdf", "note": "2-page overview" },
    { "label": "Financials", "file": "financials/projections.xlsx", "note": "Focus on tab 3" },
    { "label": "Supporting docs", "folder": "appendix/", "note": "Reference only" }
  ]
}
```

## The `_welcome/` Folder

All welcome content lives in a `_welcome/` folder within the vault or shared folder:

```
_welcome/
  README.md              Markdown introduction (simplest)
  intro.mp4              Video introduction (optional)
  walkthrough.html       Interactive guide (optional)
  guide.json             Structured step-by-step (optional)
```

The gallery/folder view checks for `_welcome/` on load. If it exists, the welcome experience renders first. If not, the standard file browser appears directly. The receiver can dismiss the welcome and go straight to browsing at any time.

## Sender-Side Creation

The sender creates the welcome content as part of the upload flow (fits into the three-step UX redesign):

1. Upload files
2. Choose distribution mode
3. **Optional: add a welcome experience** (write a note, record a video, select a template)
4. Upload and get share token

For simple cases: a text box where the sender types a few sentences. That becomes the README.

For richer cases: record a video using the browser's camera, or upload a pre-recorded one. Or select a template ("investor pack walkthrough," "project handoff guide") that auto-generates a structured guide from the folder contents.

## LLM-Generated Welcome

With the LLM pipeline, the welcome content can be auto-generated:

- Scan the folder contents (file names, types, sizes, thumbnails)
- Generate a README: "This folder contains 12 files across 3 categories..."
- Generate a suggested reading order based on file types and names
- Translate the welcome into the receiver's language

The sender reviews and edits before sending. The LLM does the first draft. This is another one-shot LLM call that adds value and consumes credits.

## Connection to Other Briefs

This extends several existing concepts:

- **Rich preview gallery:** The welcome layer sits on top of the thumbnail/preview experience. First you see the welcome, then you browse the rich previews.
- **Pack architecture:** The `_welcome/` folder is analogous to `_pack.json` from the vault UI components brief. Both are metadata that defines how the content is presented.
- **Sherpa demos:** The interactive guide format is the same two-column walkthrough pattern. The Sherpa teaches users how to use SG/Send. The welcome guide teaches receivers how to consume specific content.
- **Musicians vertical:** An album vault could open with a video from the artist, or liner notes as a README, or a track-by-track listening guide.
