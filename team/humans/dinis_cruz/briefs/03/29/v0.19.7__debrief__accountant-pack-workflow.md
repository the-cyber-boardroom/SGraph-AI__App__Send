## What Happened

The accountant use case from the earlier brief was executed for real. Not a demo. Not a test. An actual tax document pack prepared and delivered to an actual accountant. The workflow worked spectacularly.

## The Workflow, Step by Step

### 1. Mobile: Download and Upload

On the phone, downloaded credit card statements from multiple providers. Then used SG/Send (mobile browser) to upload:

- Selected multiple files on the phone
- The upload created a zip file automatically
- Typed the simple token to authenticate
- Files uploaded successfully

Pain point noted: downloading the statements from the credit card providers was painful (their apps and sites are not optimised for batch download). But the SG/Send upload from mobile worked well. The simple token is becoming the natural way to authenticate: just type three words.

### 2. Desktop: Clone to Local via CLI

On the laptop, used the SGit CLI to clone the uploaded folder:

```bash
sg-send-cli clone <simple-token>
```

The CLI now supports cloning from simple tokens directly. This created a local folder with all the credit card statements.

### 3. Create a Vault, Share with Claude

Created a vault from the local folder. Got the vault key. Opened a Claude session and said: "Clone this vault."

Claude cloned the vault. Claude could see all the files. Two-way communication was established: the human made changes locally and pushed, Claude could see them. Claude made changes and pushed, the human could pull them.

### 4. Claude Packages the Materials

Working interactively with Claude in the vault:

- Claude reorganised files into a cleaner folder structure
- Claude created markdown files (summaries, cover letters, briefing documents)
- Claude created a mini HTML app for navigation
- Claude pushed everything to the vault
- The human pulled and reviewed

### 5. Browse View as the Review Tool

Instead of reviewing the files locally in a file manager, used the SG/Send share function to view the folder in the Browse view on the website. This worked brilliantly because the Browse view:

- Has a tree navigation on the left
- Renders markdown files with formatting
- Shows file contents inline

### 6. Surgical UI Improvements During the Workflow

While using the Browse view to review the accountant pack, several small improvements were made to the website (via the web dev team, surgical IFD changes):

| Feature Added | Impact |
|---|---|
| Markdown hyperlinks detected and clickable | Can navigate between markdown files |
| Markdown images rendered inline | Infographics and charts visible in the documents |
| HTML files loaded in iframe | Claude's mini navigation app renders inside Browse |
| CSV files rendered as tables | Financial data displayed as proper tables |
| SG Layout bug fix | Improved the overall browse experience |

Each of these is a small change. Together, they transformed the Browse view from a file viewer into a document browsing experience.

### 7. Claude Creates the Navigation

Asked Claude to:
- Create a README for every folder
- Link all READMEs together
- Create a hardcoded navigation (a "start here" page that links to everything)

Claude did this brilliantly. The result: the folder now has a "start here" page that links to the briefing, the evidence, the financial summaries, and the individual statement folders. Clicking through the markdown links navigates the entire pack like a mini website.

### 8. Delivery

Shared the folder via SG/Send. The accountant receives a link. Opens it. Sees the "start here" page. Navigates through the pack. All the evidence, all the summaries, all the materials, organised and accessible from a single link.

## What This Demonstrates

- **Mobile to vault to Claude to delivery:** The full pipeline works end-to-end
- **Claude never had repo access:** Only vault access. The vault was the communication channel.
- **Two-way vault collaboration:** Human pushes, Claude pulls. Claude pushes, human pulls. Both see the same materials.
- **Browse view as a document platform:** With clickable links, inline images, CSV tables, and iframe HTML, the Browse view is becoming a genuine document browsing experience
- **Surgical IFD changes during use:** Found a need (clickable links), made the change, deployed, continued using. The development velocity enabled real-time product improvement during an actual workflow.
- **The simple token is natural:** Typing three words to authenticate is becoming second nature. No passwords, no login, no accounts.

## For the Ambassador

This is a case study. Write it up as "How I prepared my tax documents using AI and encrypted file sharing." The workflow is relatable (everyone does taxes), the technology is invisible (the user just sees files and a website), and the privacy angle is strong (financial documents, zero-knowledge encryption, even SG/Send cannot read the content).
