# Vault Access from Claude: Downloadable Vault Link Pattern

**version** v0.13.30
**date** 12 Mar 2026
**from** Human (project lead)
**to** Developer (lead), Architect
**type** Dev brief

---

## Problem

Claude sessions without `dev.send.sgraph.ai` on the network allowlist cannot use `sg-send-cli` directly. The egress proxy blocks outbound HTTPS. The current workaround (adding the domain to the allowlist in Settings) requires manual configuration and only takes effect in new sessions.

But Claude's `web_fetch` tool can download from any URL that the user pastes into the conversation. This is a separate pathway from the egress proxy.

## Proposed Solution

Create a vault download endpoint that returns the entire vault as a single downloadable bundle. The URL pattern:

```
https://send.sgraph.ai/api/vault/download/{vault_id}
```

Or with a download token:

```
https://send.sgraph.ai/api/vault/download/{vault_id}/{download_token}
```

The workflow in a Claude session:

1. Human pastes the vault download URL into the conversation
2. Claude uses `web_fetch` to download the vault bundle
3. Claude installs `sg-send-cli` from PyPI (already on the egress allowlist)
4. Claude extracts and decrypts the vault using the key (provided separately by the human)
5. Claude can now read, modify, and reason about the vault contents

This works without any network allowlist changes because `web_fetch` operates on user-provided URLs, bypassing the egress proxy entirely.

## Skill Integration

Once this pattern works, it goes into the vault skill. The skill prompt tells Claude: "If the user provides a vault download URL and a key, fetch the bundle, install sg-send-cli, extract, and decrypt."

This makes vault access available to any Claude session, not just those with custom network configuration.

## What This Doesn't Solve

Write-back. Without egress access to send.sgraph.ai, Claude can read from the vault but cannot push changes. For full read/write, the network allowlist is still required. But read-only vault access covers the majority of use cases: consuming briefs, reading documents, reasoning about vault contents.

---

## Acceptance Criteria

| # | Criterion |
|---|---|
| 1 | `/api/vault/download/{vault_id}` endpoint returns full vault as a single bundle |
| 2 | Bundle format is compatible with `sg-send-cli` extraction |
| 3 | Claude session can `web_fetch` the URL, extract, and decrypt without egress allowlist |
| 4 | Vault skill updated with downloadable vault pattern |
| 5 | Download endpoint respects access controls (download token or public vault flag) |
