## Summary

Of 17 Python modules reviewed, **11 are plain classes** that don't inherit from `Type_Safe` at all — receiving zero enforcement, auto-initialisation, or JSON serialisation. Every service class, every provider class, the audit trail, the operation class, and the fleet twin fall into this category. The 6 modules that do inherit from `Type_Safe` mostly use raw `str`, `int`, `float`, or untyped `list`/`dict`. The `@type_safe` decorator is **absent from every method in the entire codebase**.

### RF-1: Plain class — must inherit from `Type_Safe`

Any class that doesn't inherit from `Type_Safe` gets nothing: no auto-initialisation, no `__setattr__` enforcement, no `json()` / `from_json()`, no context manager. Every service, schema, provider, and utility class must inherit from `Type_Safe`.

```python
# ✗ CURRENT
class Service__EC2_Budget:
    def __init__(self, config=None):
        self.config = config or EC2_Budget_Config()

# ✓ TARGET
class Service__EC2_Budget(Type_Safe):
    config : Schema__EC2_Budget_Config   # auto-inits — no __init__ needed
```

---

### RF-2: `__init__` override — never do this

`__init__` overrides that set attributes before or outside `super().__init__(**kwargs)` completely bypass Type_Safe for those attributes. Type_Safe never sees them: no enforcement, no serialisation, no auto-init.

The fix is always to declare the attribute as a class-level annotation. Type_Safe's constructor automatically picks up any keyword argument that matches a declared attribute and assigns it, handling type conversion. For default values it uses the type's own default. **The `__init__` is simply not needed** — Type_Safe handles it.

```python
# ✗ CURRENT — Type_Safe has no knowledge of security_groups or tags
class Schema__Twin__Config__EC2__Instance(Schema__Twin__Config__AWS):
    instance_id : str = ''

    def __init__(self, **kwargs):
        self.security_groups = kwargs.pop('security_groups', [])   # invisible to Type_Safe
        self.tags            = kwargs.pop('tags', {})              # invisible to Type_Safe
        super().__init__(**kwargs)

# ✓ TARGET — declared as class attributes; Type_Safe handles kwargs assignment automatically
class Schema__Twin__Config__EC2__Instance(Schema__Twin__Config__AWS):
    instance_id     : EC2__Instance_Id
    security_groups : List__EC2__Security_Group_Ids
    tags            : Dict__EC2__Instance__Tags
```

### RF-4: Raw primitives — use Safe_* types

Raw `str`, `int`, `float` in a `Type_Safe` class carry no validation. The Safe_* catalogue has 100+ types. Key mappings for this codebase:

| Raw type used for | Correct type |
|---|---|
| EC2 instance ID (`i-...`) | `EC2__Instance_Id(Safe_Str__Id)` — define a named subclass |
| AMI ID (`ami-...`) | `EC2__AMI_Id(Safe_Str__Id)` |
| Security group ID (`sg-...`) | `EC2__Security_Group_Id(Safe_Str__Id)` |
| Key pair name | `EC2__Key_Name(Safe_Str__Id)` |
| Region (`eu-west-2`) | `EC2__Region_Id(Safe_Str__Id)` or `Enum__EC2__Region` |
| VPC / subnet IDs | `EC2__VPC_Id(Safe_Str__Id)`, `EC2__Subnet_Id(Safe_Str__Id)` |
| IPv4/IPv6 address | `Safe_Str__IP_Address` (validates via Python `ipaddress` module) |
| Port number | `Safe_UInt__Port` (0-65535) |
| LLM model name | `Safe_Str__LLM__Model_Id` |
| LLM prompt | `Safe_Str__LLM__Prompt` (32KB limit) |
| Counts (non-negative, bounded) | Named `Safe_UInt` subclass with `max_value` set |
| Duration (ms, seconds) | `Safe_UInt` or a named subclass |
| Financial values | `Safe_Float__Money` (Decimal arithmetic, min 0.0) |
| Admin identifier | `Admin_Id(Safe_Str__Id)` — named subclass |
| Audit action name | `Audit__Action_Id(Safe_Str__Id)` — named subclass |

**Named Id subclasses are preferred over bare `Safe_Str__Id`** whenever the Id has domain meaning and appears in multiple places. One definition, tested once, imported everywhere:

```python
class EC2__Instance_Id      (Safe_Str__Id): pass
class EC2__AMI_Id           (Safe_Str__Id): pass
class EC2__Security_Group_Id(Safe_Str__Id): pass
class EC2__Key_Name         (Safe_Str__Id): pass
class EC2__VPC_Id           (Safe_Str__Id): pass
class EC2__Subnet_Id        (Safe_Str__Id): pass
class Audit__Action_Id      (Safe_Str__Id): pass  # e.g. 'EC2_CREATE', 'EC2_LLM_EXECUTE'
class Admin_Id              (Safe_Str__Id): pass
```

Always verify that `Safe_Str__Id` (alphanumeric + `-` `_`, max 128 chars) is the right base before using it. AWS IDs like `i-0abc123def456` are fine. Anything with `/`, `:`, `.` needs a different base type.

### RF-5: Untyped `list` / `dict` / `set` — always use typed collection subclasses

Bare `list`, `dict`, or `set` annotations carry no type information and receive no enforcement. Every such field needs a `Dict__*` / `List__*` / `Set__*` subclass — defined once, tested independently, imported everywhere.

```python
# Step 1 — define the collection subclass
from osbot_utils.type_safe.type_safe_core.collections.Type_Safe__Dict import Type_Safe__Dict
from osbot_utils.type_safe.type_safe_core.collections.Type_Safe__List import Type_Safe__List

class List__EC2__Security_Group_Ids(Type_Safe__List):
    expected_type = EC2__Security_Group_Id

class Dict__EC2__Instance__Tags(Type_Safe__Dict):
    expected_key_type   = Safe_Str__Key
    expected_value_type = Safe_Str__Text

class Dict__Audit__Entry__Details(Type_Safe__Dict):
    expected_key_type   = Safe_Str__Key
    expected_value_type = Safe_Str__Text

# Step 2 — use as class attribute (no mutable default — Type_Safe auto-inits)
class Schema__Twin__Config__EC2__Instance(Type_Safe):
    security_groups : List__EC2__Security_Group_Ids
    tags            : Dict__EC2__Instance__Tags
```

This gives a single definition of each collection type, makes it independently testable, and makes the schema self-documenting.

### RF-7: `Timestamp_Now` — use for all timestamps

Never call `datetime.now()` or `time.time()` to produce a stored timestamp. `Timestamp_Now` is a `Safe_Int` subclass that auto-captures current UTC epoch milliseconds at init time, serialises cleanly, and is always available wherever osbot-utils is installed.

```python
# ✗ CURRENT
self.timestamp = datetime.now(timezone.utc).isoformat()   # raw string, not Type_Safe

# ✓ TARGET — auto-set to current epoch ms when the object is constructed
class Schema__Audit_Entry(Type_Safe):
    created_at : Timestamp_Now
```

For fields that must store an ISO 8601 string because an external API returns one, use raw `str` and document explicitly as a known gap (pending a `Safe_Str__Iso_Timestamp` type).

---

### RF-8: `json()` method — never define on a Type_Safe class

Defining a method named `json()` **shadows the built-in Type_Safe serialisation method**. The built-in handles nested Type_Safe objects, collection subclasses, Enum values, and Safe_* primitives. A hand-written `json()` almost always does less, and once the class inherits from `Type_Safe` the custom method is actively harmful.

```python
# ✗ CURRENT — shadows Type_Safe and produces a worse result
def json(self):
    return dict(timestamp=self.timestamp, action=self.action, ...)

# ✓ TARGET — delete completely; Type_Safe.json() handles it automatically
```

If a different serialisation shape is required for an external API, name it `to_dict()`, `as_api_dict()`, or similar — never `json`.

---

### RF-9: Hash generation — use `Cache__Hash__Generator`

Manual `hashlib` calls should be replaced with `Cache__Hash__Generator` from osbot-utils. It is battle-hardened, always available (osbot-utils is a universal dependency), returns `Safe_Str__Cache_Hash`, and is itself a `Type_Safe` class with configurable algorithm and hash length via `Schema__Cache__Hash__Config`. It also has `from_type_safe()` which hashes a Type_Safe object directly, with optional field exclusion.

```python
# ✗ CURRENT
data = json.dumps({...}, sort_keys=True)
self.entry_hash = hashlib.sha256(data.encode()).hexdigest()

# ✓ TARGET
from osbot_utils.helpers.cache.Cache__Hash__Generator import Cache__Hash__Generator

hash_generator = Cache__Hash__Generator()
entry.entry_hash = hash_generator.from_type_safe(entry, exclude_fields=['entry_hash'])
```

### RF-10: Enums — prefer over `Literal` for domain values

`Literal` is acceptable for small ad-hoc sets, but `Enum` is preferred when the set of values has domain meaning, may grow, or is referenced from multiple places. Type_Safe has full Enum support — values auto-convert from strings on assignment and serialise cleanly to their string value.

```python
# ✗ AVOID for meaningful domain values — cannot be imported or extended cleanly
max_instance_type : Literal['t3.micro', 't3.small', 't3.medium']

# ✓ PREFER — one definition, importable, testable, extensible
class Enum__EC2__Instance_Type(str, Enum):
    T3_MICRO   = 't3.micro'
    T3_SMALL   = 't3.small'
    T3_MEDIUM  = 't3.medium'
    C7G_XLARGE = 'c7g.xlarge'

    def __str__(self): return self.value

class Schema__EC2_Budget_Config(Type_Safe):
    max_instance_type : Enum__EC2__Instance_Type = Enum__EC2__Instance_Type.T3_MEDIUM
```

---

### RF-11: osbot-utils utilities — use instead of stdlib equivalents

| Stdlib call | Replacement |
|---|---|
| `json.loads(s)` | `json_loads(s)` — returns `{}` on error, no try/catch needed |
| `json.dumps(d)` | `json_dumps(d)` |
| `json.load(f)` | `json_load_file(path)` |
| `os.path.exists(p)` | `file_exists(p)` |
| `os.unlink(p)` | `file_delete(p)` |
| `uuid.uuid4().hex[:8]` | `Obj_Id()` or `Random_Guid()` |
| `hashlib.sha256(...)` | `Cache__Hash__Generator().from_string(...)` |
| `datetime.now(...)` | `Timestamp_Now()` |
