## 1. What Is Type_Safe and What Problem Does It Solve

Python's type hints are **documentation only** — ignored completely at runtime. Type_Safe is a runtime type enforcement framework built on top of standard Python. Every `__setattr__` is intercepted and validated. Every collection operation is checked. Every `@type_safe`-decorated method validates parameters on entry and return values on exit.

**The frame shift:** bugs are caught at the point of assignment, not three stack frames later.

### The three-class architecture

| Class | Role |
|---|---|
| `Type_Safe__Base` | Core type checking logic, `try_convert()`, handles `Union`, `Literal`, `Optional` |
| `Type_Safe__Primitive` | Extends built-in types (`str`, `int`, `float`), preserves type through arithmetic, provides `__to_primitive__()` for serialisation |
| `Type_Safe` | What user classes inherit; auto-initialises attributes, enforces types via `__setattr__`, handles JSON round-trips |

## 2. Core Rules (Non-Negotiable)

### Rule 1: Always inherit from Type_Safe

```python
from osbot_utils.type_safe.Type_Safe import Type_Safe

class Schema__Config(Type_Safe):            # ✓ CORRECT
    name : Safe_Str

class Config:                               # ✗ WRONG — plain class gets nothing
    name: Safe_Str
```

### Rule 2: Type-annotate every attribute

```python
class Schema__Config(Type_Safe):
    host        : Safe_Str__Host            # ✓ Every attribute has a type
    port        : Safe_UInt__Port
    ssl_enabled : bool
    endpoints   : List[Safe_Str__Url]

    # ✗ WRONG:
    # host = "localhost"                    # No annotation — not Type_Safe
    # items: list                           # Untyped collection
```

### Rule 3: No mutable defaults — ever

```python
class Schema__Settings(Type_Safe):
    name        : Safe_Str                  # ✓ Auto-inits to ''
    count       : Safe_UInt                 # ✓ Auto-inits to 0
    items       : List[Safe_Str]            # ✓ Auto-inits to [] (unique per instance)
    timeout     : Safe_UInt = 30            # ✓ Immutable non-default value

    # ✗ NEVER:
    # items: List[Safe_Str] = []            # Mutable default — shared across instances
```

### Rule 4: Ban raw primitives

**Never use raw `str`, `int`, or `float` in a Type_Safe class.** They accept anything without validation.

```python
# ✗ DANGEROUS — raw primitives
class User(Type_Safe):
    name   : str     # SQL injection, XSS, unlimited length
    age    : int     # Can be negative, overflow
    balance: float   # Floating-point rounding errors in finance

# ✓ CORRECT — domain-specific types
class User(Type_Safe):
    name   : Safe_Str__Username     # Sanitised, length-limited
    age    : Safe_UInt__Age         # 0-150 range enforced
    balance: Safe_Float__Money      # Exact decimal arithmetic
```

### Rule 5: Schemas are pure data — no methods

```python
# ✗ WRONG — methods in schema
class Schema__Config(Type_Safe):
    host : Safe_Str__Host
    port : Safe_UInt__Port

    def url(self) -> str:           # NO — move to helper
        return f'{self.host}:{self.port}'

# ✓ CORRECT — logic in separate class
class Schema__Config(Type_Safe):
    host : Safe_Str__Host
    port : Safe_UInt__Port

class Config__Utils(Type_Safe):
    @type_safe
    def url(self, config: Schema__Config) -> Safe_Str__Url:
        return Safe_Str__Url(f'{config.host}:{config.port}')
```

### Rule 6: Collection subclasses are pure type definitions — no methods

```python
# ✓ CORRECT
class Dict__Transfers__By_Id(Type_Safe__Dict):
    expected_key_type   = Transfer_Id
    expected_value_type = Schema__Transfer

# ✗ WRONG — no methods in collection subclasses
class Dict__Transfers__By_Id(Type_Safe__Dict):
    expected_key_type   = Transfer_Id
    expected_value_type = Schema__Transfer

    def find_by_status(self, status):      # NO — move to helper
        pass
```

### Rule 7: Never shadow reserved method names

These names exist as built-in Type_Safe methods. Using them as attribute names silently breaks serialisation:

`json`, `json__compress`, `from_json`, `from_json__compressed`, `obj`, `reset`, `print`, `serialize_to_dict`, `update_from_kwargs`, `__locals__`, `__kwargs__`, `__default_kwargs__`, `__cls_kwargs__`, `__attr_names__`

```python
# ✗ WRONG
class Schema__Bad(Type_Safe):
    json  : Safe_Str    # Shadows json() serialisation method!
    obj   : Safe_Str    # Shadows obj() method!

# ✓ CORRECT
class Schema__Good(Type_Safe):
    json_data     : Safe_Str
    target_object : Safe_Str
```

### Rule 8: Forward references — current class only

```python
class Schema__Tree_Node(Type_Safe):
    value  : Safe_Int
    parent : 'Schema__Tree_Node' = None     # ✓ Same class
    # parent: 'Schema__Other_Node'          # ✗ Cross-class forward refs don't work
```

### Rule 9: Simplified default assignment

```python
# ✗ VERBOSE — unnecessary explicit construction
class Schema__Config(Type_Safe):
    version : Safe_Str__Version = Safe_Str__Version('1.0.0')

# ✓ SIMPLIFIED — auto-conversion handles it
class Schema__Config(Type_Safe):
    version : Safe_Str__Version = '1.0.0'  # auto-converts
    name    : Safe_Str                     # default '' is implicit
```

## 3. Collection Types

### How Python annotations auto-convert to Type_Safe collections

When you annotate with `Dict`, `List`, `Set`, `Tuple` in a `Type_Safe` class, the framework **automatically converts** them at construction:

```python
class Ontology__Registry(Type_Safe):
    cache : Dict[Obj_Id, Schema__Ontology]  # runtime: Type_Safe__Dict
    items : List[str]                       # runtime: Type_Safe__List
    tags  : Set[str]                        # runtime: Type_Safe__Set

registry = Ontology__Registry()
assert type(registry.cache) is Type_Safe__Dict  # ✓ not dict!
```

### Naming convention: always prefix with collection type

```python
# ✓ CORRECT — collection type as prefix
class Dict__Ontologies__By_Id(Type_Safe__Dict):   ...
class List__Node_Type_Ids(Type_Safe__List):        ...
class Set__Permission__Ids(Type_Safe__Set):        ...

# ✗ WRONG — no collection type indicator
class Ontologies_By_Id(Type_Safe__Dict):           ...  # ambiguous
```

### Dict subclass pattern

```python
from osbot_utils.type_safe.type_safe_core.collections.Type_Safe__Dict import Type_Safe__Dict
from osbot_utils.type_safe.primitives.domains.cryptography.safe_str.Safe_Str__Hash import Safe_Str__Hash

class Dict__Content__Hashes(Type_Safe__Dict):      # Maps content hashes to file paths
    expected_key_type   = Safe_Str__Hash
    expected_value_type = str

# Usage
hash_map = Dict__Content__Hashes()
hash_map['abc1234567'] = 'some value'              # Key auto-converts to Safe_Str__Hash
hash_map[True] = 'value'                           # ✗ TypeError!
```

### List subclass pattern

```python
from osbot_utils.type_safe.type_safe_core.collections.Type_Safe__List import Type_Safe__List

class List__User__Ids(Type_Safe__List):
    expected_type = Safe_Id

id_list = List__User__Ids()
id_list.append('user-123')                         # ✓ Auto-converts to Safe_Id
id_list.append(123)                                # ✗ TypeError!
```

### Instantiate collection subclasses directly — never create parent object just to get a collection

```python
# ✗ WRONG — wasteful
def parse(self, data: dict) -> Schema__Ontology:
    node_types = Schema__Ontology().node_types

# ✓ CORRECT
def parse(self, data: dict) -> Schema__Ontology:
    node_types = Dict__Node_Types__By_Id()         # Direct instantiation
```

### Operations preserve subclass type

```python
hash_map = Dict__Content__Hashes({'abc1234567': 'v'})
copied   = hash_map.copy()
assert type(copied) is Dict__Content__Hashes       # ✓ NOT Type_Safe__Dict
merged   = hash_map | {'def1234567': 'other'}
assert type(merged) is Dict__Content__Hashes       # ✓ preserved
```

## 4. The @type_safe Decorator

### What it does — two-phase validation

```
Call-time →  validates all parameters, each list element, Union matching
Return-time → validates return value, auto-converts plain list/dict to Type_Safe collections
```

```python
from osbot_utils.type_safe.type_safe_core.decorators.type_safe import type_safe

class OrderService(Type_Safe):

    @type_safe
    def create_order(self, customer_id : Safe_Id            ,
                           items       : List[OrderItem]    ,
                           total       : Safe_Float__Money  ) -> Order:
        # All params validated before this body runs
        order = Order(customer_id=customer_id, items=items, total=total)
        return self.repository.save(order)
```

### Optional parameters — use `= None`, NOT `Optional[T]`

```python
# ✓ CORRECT — = None makes it optional
@type_safe
def find(self, id: Safe_Id = None) -> Schema__User:
    pass

# ✗ AVOID — Optional[T] is redundant verbosity
@type_safe
def find(self, id: Optional[Safe_Id] = None) -> Optional[Schema__User]:
    pass
```

### Return None — always allowed regardless of return annotation

```python
# ✓ CORRECT — None is implicitly valid return, meaning "not found"
@type_safe
def get(self, id: Ontology_Id) -> Schema__Ontology:
    return self.cache.get(id)              # can return None
```

### Return auto-conversion

```python
# Plain list returned → automatically becomes Type_Safe__List
@type_safe
def get_items(self) -> List[Safe_Id]:
    return ['id-1', 'id-2', 'id-3']       # auto-converts!

result = get_items()
assert type(result) is Type_Safe__List    # ✓
result.append(123)                        # ✗ TypeError! Type safety enforced
```

### List element validation

```python
@type_safe
def process(self, items: List[str]) -> int:
    return len(items)

process(['a', 'b', 123])                  # ✗ Caught at index 2 — not in caller
```

### Performance notes

- ~5x overhead for no-parameter methods (decorator detects, skips validation)
- Full validation overhead for methods with parameters
- Use on **public API methods**; can skip private helpers if perf-critical

## 5. Safe Primitives Reference

### Core string types

| Type | Max length | Character set | Use for |
|---|---|---|---|
| `Safe_Str` | 512 | regex-filtered | generic strings |
| `Safe_Str__Text` | 4KB | more chars allowed | general text |
| `Safe_Str__Id` | 128 | alphanumeric + `_` `-` | identifiers |

### Domain string types (key ones)

| Type | Strips / Validates | Use for |
|---|---|---|
| `Safe_Str__File__Path` | allows `/` | file paths, URL paths |
| `Safe_Str__Url` | URL format | URLs |
| `Safe_Str__Email` | email format | email addresses |
| `Safe_Str__Username` | alphanumeric + `_` | usernames |
| `Safe_Str__Http__Content_Type` | allows `/` | MIME types like `image/jpeg` |
| `Safe_Str__Http__User_Agent` | allows `/` `()` `;` | browser user agents |
| `Safe_Str__Cache_Hash` | 10-96 hex chars | IP hashes, content hashes |
| `Safe_Str__SHA1` | exactly 40 hex | SHA-1 hashes |
| `Safe_Str__Hash` | 10-char hex | short content hashes |
| `Safe_Str__LLM__Prompt` | 32KB | LLM prompts |
| `Safe_Str__Slug` | lowercase + `-` | URL slugs (auto-lowercases) |
| `Safe_Str__Display_Name` | allows spaces, `()` | user-facing names |

### ⚠️ Critical: wrong type = silent data corruption

| Data type | Contains | Wrong type | Right type |
|---|---|---|---|
| MIME types | `/` | `Safe_Str__Id` | `Safe_Str__Http__Content_Type` |
| URL paths | `/` | `Safe_Str__Id` | `Safe_Str__File__Path` |
| ISO timestamps | `:` `+` | any Safe_Str | `str` (no Safe type yet) |
| IP hashes (SHA256) | hex only | `Safe_Str__Id` (wrong semantics) | `Safe_Str__Cache_Hash` |
| User agents | `/` `()` `;` | `Safe_Str__Id` | `Safe_Str__Http__User_Agent` |

**Always test the Safe type with real data before using it.** The primitives reference doc (`v3.28.0__for_llms__osbot-utils-safe-primitives.md`) is the complete catalogue.

### Numeric types

| Type | Range | Use for |
|---|---|---|
| `Safe_Int` | configurable | signed integers |
| `Safe_UInt` | >= 0 | unsigned integers, counts |
| `Safe_UInt__Port` | 0-65535 | network ports |
| `Safe_UInt__FileSize` | >= 0 | file sizes (has `.to_kb()` etc) |
| `Safe_Float` | configurable | floats |
| `Safe_Float__Money` | Decimal arithmetic | currency (no rounding errors) |

### Identity types

| Type | Behaviour |
|---|---|
| `Safe_Id` | **Auto-generates** random value when None/empty |
| `Random_Guid` | Always auto-generates UUID |
| `Timestamp_Now` | Auto-captures current time (milliseconds as int) |
| `Obj_Id` | Unique object identifier |
| `Semantic_Id` | **Never auto-generates**, allows empty — for concept IDs |

### ⚠️ Critical: Safe_Id vs Semantic_Id

`Safe_Id` auto-generates on empty — this **breaks serialisation round-trips** because deserialization generates a new ID instead of restoring the original.

- Use `Safe_Id` / `Obj_Id` for: `Node_Id`, `Edge_Id`, `Graph_Id` — instance identifiers
- Use `Semantic_Id` for: `Ontology_Id`, `Node_Type_Id`, `Category_Id` — concept identifiers

## 6. Enum and Literal Support

### Enum — for fixed sets of values with methods

```python
from enum import Enum

class Enum__Transfer__Status(str, Enum):
    PENDING   = "pending"
    ACTIVE    = "active"
    COMPLETED = "completed"

    def __str__(self): return self.value

class Schema__Transfer(Type_Safe):
    status : Enum__Transfer__Status = Enum__Transfer__Status.PENDING

transfer = Schema__Transfer()
transfer.status = "active"                         # string auto-converts to enum
assert transfer.status == Enum__Transfer__Status.ACTIVE
transfer.status = "invalid"                        # ✗ ValueError — caught immediately

# JSON round-trip works
json_data = transfer.json()
assert json_data['status'] == 'active'
restored  = Schema__Transfer.from_json(json_data)
assert restored.status == Enum__Transfer__Status.ACTIVE
```

### Literal — for small inline enums without methods

```python
from typing import Literal

class Schema__Message(Type_Safe):
    role     : Literal["assistant", "system", "user", "tool"]
    priority : Literal["low", "medium", "high"] = "medium"

msg = Schema__Message()
msg.role = "user"      # ✓ valid
msg.role = "admin"     # ✗ ValueError
```

| Method | Purpose |
|---|---|
| `obj.json()` | Serialize to plain dict |
| `obj.json__compress()` | Compressed JSON with type registry |
| `Schema.from_json(data)` | Deserialize from dict — restores all types |
| `obj.obj()` | Convert to namespace object (for testing) |
| `obj.reset()` | Reset all attributes to defaults |
| `obj.update_from_kwargs(**kw)` | Update multiple attributes at once |
| `obj.print()` | Pretty-print the object |

### JSON utilities (use these, not raw json module)

```python
from osbot_utils.utils.Json import json_load_file, json_loads, json_dumps

# ✗ WRONG — raw json
with open(path) as f:
    data = json.load(f)

# ✓ CORRECT — osbot utilities return safe defaults on error
data = json_load_file(path)    # returns {} on error, not exception
data = json_loads(json_str)    # returns {} on error
```

```python
class Schema__Example(Type_Safe):
```

## 9. No Exceptions for Missing Resources

```python
# ✗ WRONG — exception for a valid state
def load(self, path: Safe_Str__File__Path) -> Schema__Config:
    if not file_exists(path):
        raise FileNotFoundError(f'File not found: {path}')

# ✓ CORRECT — return None, let caller decide
def load(self, path: Safe_Str__File__Path) -> Schema__Config:
    if file_exists(path) is False:
        return None
    data = json_load_file(path)
    if not data:
        return None
    return Schema__Config.from_json(data)
```

## 11. Testing Patterns

### Context manager with `_`

```python
def test__init__(self):
    with Schema__Order() as _:
        assert type(_)         is Schema__Order
        assert type(_.id)      is Safe_Id
        assert type(_.items)   is Type_Safe__Dict   # NOT dict
        assert type(_.tags)    is Type_Safe__List    # NOT list
```

### .obj() for comprehensive state verification

```python
from osbot_utils.testing.__ import __, __SKIP__

def test__init__(self):
    with Schema__Order() as _:
        assert _.obj() == __(id      = _.id          ,  # use __SKIP__ for dynamic values
                              items  = {}             ,
                              total  = 0.00           ,
                              status = "pending"      )
```

### __SKIP__ for auto-generated values

```python
assert order.obj() == __(order_id   = __SKIP__   ,   # auto-generated ID
                          created_at = __SKIP__   ,   # timestamp
                          items      = {}         )
```

### setUpClass for expensive resources (10-100x speedup)

```python
class test_Service__Config(TestCase):

    @classmethod
    def setUpClass(cls):                              # ONE TIME for the whole class
        cls.service = Service__Config()
        cls.service.setup()                           # expensive — done once

    def setUp(self):                                  # per test — lightweight only
        self.test_id = Random_Guid()

    def test_something(self):
        with self.service as _:
            assert _.is_configured() is True
```

### Never use docstrings — inline comments aligned at column 60-80

```python
# ✓ CORRECT — inline aligned comments
def test__init__(self):                               # Test auto-initialization
    with Schema__Persona() as _:
        assert type(_.id)   is Safe_Id               # auto-generated instance ID
        assert type(_.name) is Safe_Str__Text         # sanitised string

# ✗ WRONG — docstrings break visual alignment pattern
def test__init__(self):
    """Test auto-initialization of Schema__Persona"""
```

## 13. Lessons Learned From This Project (SG/Send)

These are real mistakes discovered during the v0.2.41 audit (12 files, 90+ issues) and ongoing code reviews:

### Mistake 1 — Wrong Safe_Str type, silent data corruption

`Safe_Str__Id` strips `/`, `:`, `+`, `(`, `)`, `;`. Using it for MIME types, URL paths, or ISO timestamps silently corrupts data. Test the Safe type with real data before using it.

### Mistake 2 — Plain classes instead of Type_Safe

Services, providers, audit classes all written as plain `class Foo:` with `__init__`. Get zero type enforcement, auto-init, or JSON serialisation. Every one of these should be `class Foo(Type_Safe):`.

### Mistake 3 — `__init__` overrides bypass Type_Safe

```python
# ✗ WRONG — attributes set in __init__ are unknown to Type_Safe
class Schema__Config(Type_Safe):
    def __init__(self, **kwargs):
        self.security_groups = kwargs.pop('security_groups', [])  # raw list!
        super().__init__(**kwargs)

# ✓ CORRECT — declare as class attributes
class Schema__Config(Type_Safe):
    security_groups : List[Safe_Str__Id]   # Type_Safe knows about this
```

### Mistake 4 — Untyped `list` and `dict` fields

```python
# ✗ WRONG
class Schema__Request(Type_Safe):
    checks : list   # What's in it? No enforcement.
    meta   : dict   # Same problem.

# ✓ CORRECT
class Schema__Request(Type_Safe):
    checks : List[Schema__QA__Check]
    meta   : Dict[Safe_Str__Key, Safe_Str__Text]
```

### Mistake 5 — Default values visible in Swagger

When `usage_limit: int = 0`, Swagger shows `0` as the default. Users created tokens with unlimited uses. **Type_Safe defaults are part of your API contract.** Always choose meaningful defaults.

### Mistake 6 — Business logic in Schema classes

Methods added to Schema classes make them hard to serialise correctly and violate single responsibility. Always separate data (Schema__*) from logic (Service__* or helper classes).

### Mistake 7 — Using `datetime.now()` instead of `Timestamp_Now`

Raw `datetime.now()` calls scattered through the code. `Timestamp_Now` is the Type_Safe-native way to capture the current time consistently, and it serialises correctly.

### Mistake 8 — Missing `@type_safe` on all public methods

Service methods that accept `str`, `int`, `list` without `@type_safe` pass through unchecked. A single missed decorator means an entire method's parameter contract is unenforced.

### Mistake 9 — No Safe type for ISO timestamps (known gap)

No existing Safe_Str type preserves `:` and `+` (needed for `2026-02-13T10:30:00+00:00`). `Timestamp_Now` stores milliseconds as `int`. For ISO string timestamps, either use `Timestamp_Now` (int) or document the field as raw `str` pending a future `Safe_Str__Iso_Timestamp`.

### Mistake 10 — `str()` casts on Safe types

Explicit `str(some_id)` casts break the type chain and indicate misunderstanding. Type_Safe handles conversions automatically.

## 14. Complete Checklist

When writing any Type_Safe code:

**Schema classes:**
- [ ] Inherits from `Type_Safe`
- [ ] Every attribute has a type annotation
- [ ] No mutable defaults (`= []`, `= {}`)
- [ ] No methods or business logic
- [ ] No `__init__` override
- [ ] No reserved names (`json`, `obj`, `reset`, etc.)
- [ ] Uses Safe_* types — no raw `str`, `int`, `float`
- [ ] Uses appropriate domain type (check character set before choosing)
- [ ] `list`/`dict` fields use typed versions (`List[T]`, `Dict[K,V]`)

**Collection subclasses:**
- [ ] Named with `Dict__`, `List__`, `Set__` prefix
- [ ] Only `expected_key_type` / `expected_value_type` / `expected_type` — no methods

**Service / helper classes:**
- [ ] Inherits from `Type_Safe`
- [ ] `@type_safe` on all public methods
- [ ] No `Optional[T]` — use `param: T = None`
- [ ] Returns `None` for not-found, no exceptions
- [ ] No `str()` casts
- [ ] Uses `json_load_file` / `json_loads` not raw `json` module
- [ ] Uses `file_exists` not raw `os.path.exists`
