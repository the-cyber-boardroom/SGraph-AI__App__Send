# Two-Tier Team Model: Meta-Team and Lightweight Project Teams

**version** v0.16.26
**date** 20 Mar 2026
**from** Human (project lead)
**to** Architect (lead), Developer, Ambassador
**type** Brief

---

## The Two Teams

There are two distinct types of agentic teams in play, and they serve completely different purposes.

### Team 1: The Meta-Team (We Build This)

This is the full SG/Send agentic team. It creates the architecture, the workflows, the documentation, the templates, and the infrastructure that makes everything else possible. This is the team with 20+ agents: Conductor, Architect, Developer, Designer, QA, AppSec, SRE, Sherpa, Ambassador, Librarian, Alchemist, and the rest.

This team builds the dev packs, the role definitions, the CI pipelines, the vault templates, the component library. It is large, sophisticated, and understands the full depth of the platform.

The meta-team's output is not research or deliverables. Its output is the machinery that other teams use to produce research and deliverables.

### Team 2: The Project Team (Partners Spin These Up)

This is a small, focused team that a partner (or any user) spins up for a specific project. It should be three roles, maybe four. Lightweight. Easy to understand for people who are new to agentic workflows.

Suggested roles for a research project team:

| Role | What They Do |
|---|---|
| Researcher | Finds information, analyses sources, produces findings. The core intelligence of the team. |
| Journalist / Librarian | Connects the dots, structures the narrative, catalogues and cross-references findings. Turns raw research into coherent output. |
| Developer | Builds mini tools as needed: scrapers, data processors, visualisations, custom analysis scripts. |

Three roles. That is it. Anyone can understand "a researcher, someone who writes it up, and someone who builds tools." No jargon. No 18-agent complexity. A team small enough to explain in one sentence.

## The Relationship

```
Meta-Team (SG/Send)
  │
  │  creates
  ▼
Dev Pack / Skill / Template
  │
  │  used to spin up
  ▼
Project Team (partner/user)
  │
  │  does the actual work
  ▼
Research Output (in a vault)
```

The meta-team creates the dev pack once. The dev pack is used to spin up project teams many times. Each project team is independent, focused on one task, and communicates via a vault.

## What "Spin Up" Means in Practice

A partner or user should be able to create a project team from:

- **Claude Code:** Start a session, point it at the dev pack repo, the three agents are ready
- **Claude Web:** Upload the skill/briefing pack, the session adopts the researcher role
- **ChatGPT:** Paste the role definition as a system prompt, same workflow
- **A website:** Go to the SG/Send tools page, select "Research Team," enter your topic, the agents start working

The barrier must be near zero. The person does not need to understand agentic architecture. They need to say "research this topic" and get a structured output in a vault.

## The Skill as the Launcher

The project team should be packaged as a skill (in the Claude sense) or a briefing pack:

- A single file (or small folder) that contains the three role definitions
- A setup prompt that initialises the vault, creates the branches, and assigns roles
- A set of pre-built prompts for common research tasks ("research company X," "analyse document Y," "compare these three options")
- A workflow definition: what happens in what order, what gets committed where

The skill fires up the three agents, sets up the environment (vault, branches), maybe creates a mini website for the output, and the team starts working. The vault is the communication channel between agents and between the team and the human.

## Why This Matters for Partners

The partner from the earlier meeting does not want to learn 20 agent roles. They want to say "set up a research team for this topic" and get going. The two-tier model gives them that:

- We (the meta-team) handle the complexity of creating the infrastructure
- They (the project team) get a simple, focused, three-agent setup
- The vault handles all the collaboration, storage, and sharing
- The output is a vault they can share with their clients

The meta-team is our competitive advantage. The project teams are the product we are enabling others to create.

## Making It Super Easy

The project team creation needs to work from everywhere:

| Platform | How It Works |
|---|---|
| Claude Code | Clone the dev pack repo, agents are defined in CLAUDE.md files |
| Claude Web | Upload the skill zip, session adopts the role |
| ChatGPT | Paste the role definition as system prompt |
| Website | Select template, enter topic, agents run server-side or via browser LLM |
| CLI | `sgit-ai init --template research-documents`, vault and roles created |

Each platform gets its own launcher format, but the role definitions are the same core content adapted for the platform's input method.

## Acceptance Criteria

| # | Criterion |
|---|---|
| 1 | Three-role project team defined: Researcher, Journalist/Librarian, Developer |
| 2 | Skill/briefing pack that launches the three-agent team with one action |
| 3 | Vault created automatically on team launch (branches per agent) |
| 4 | Works from Claude Code (dev pack repo) |
| 5 | Works from Claude Web (uploaded skill) |
| 6 | Works from a website (template selection, topic input) |
| 7 | Partner can spin up a project team without understanding the meta-team architecture |
| 8 | Pre-built prompts for common research tasks included in the skill |
