## What Happened

The sg-send-cli team has been converted into the SGit-AI team. SGit now has:

- Its own GitHub organisation
- PyPI package reserved as `sgit-ai` (because `sgit` was already taken)
- Domain mapped to Route 53 (static S3 bucket + CloudFront distribution, same model as other sites)

This is SGit becoming its own product, not just a feature of SG/Send.

## Positioning

**A lightweight, PKI-signed and encrypted Git.**

Unpack that:

- **Lightweight:** It is a convention and a set of light implementations, not a full Git reimplementation. No SSH required. No heavy dependencies. Pure Python and browser-based JavaScript libraries. This is a genuine advantage: Git in Lambda functions has always been painful because Lambda does not have SSH. SGit does not need SSH. It runs anywhere Python or a browser runs.

- **PKI-signed:** Every commit is signed by the branch key. Provenance is mandatory, not optional. This is stronger than Git's optional GPG signing.

- **Encrypted:** All content is encrypted. The server stores opaque blobs. File names, folder structure, and content are all encrypted. The server cannot read the data even if compromised.

- **Git:** Content-addressed storage, tree structures, commits, branches, merges, remotes. The mental model is Git. The implementation is vault-native.

The one-line pitch: "Git, but encrypted and signed by default. Runs anywhere. No SSH."

## Website and Documentation

The website needs to be professional and developer-focused. This is the first touch point for developers who might adopt SGit.

Requirements:

- **README:** Clear, well-structured. Installation (`pip install sgit-ai`), quickstart (init, add, commit, push, pull, clone), and a "why SGit?" section.
- **Documentation site:** Getting started, CLI reference, architecture overview, agentic usage guide.
- **Homepage:** Clean, professional. The positioning statement. A terminal demo (animated or live) showing the workflow. Links to GitHub, PyPI, docs.

### Agentic Usage as the Lead

The primary developer audience right now is people building agentic workflows. Lead with this:

- "Give your AI agent its own identity, not your credentials"
- "Agents commit to their own branches, signed with their own keys"
- "No SSH, no Git credentials, no overpowered tokens"
- Show the workflow: agent installs from PyPI, clones a vault, develops, commits, pushes. Human pulls and reviews.

This is the hook that will get GitHub stars and developer attention. The agentic AI community is actively looking for better ways to give agents controlled access to codebases. SGit solves this.

## Traction Strategy

- Get the README and website polished first (developers judge projects by their README)
- Post on Hacker News, Reddit (r/programming, r/MachineLearning), and Twitter/X with the agentic angle
- Write a blog post: "Why we built an encrypted Git for AI agents"
- Target the Claude Code and agentic development communities specifically
- Aim for GitHub stars as the initial metric of developer interest
