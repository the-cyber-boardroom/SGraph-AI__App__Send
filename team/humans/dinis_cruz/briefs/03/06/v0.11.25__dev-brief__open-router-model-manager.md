## The Problem

Choosing the right LLM model is currently guesswork. The human uses Sonnet 3.7 for multi-modal and Haiku for text — but found these by accident, not by systematic evaluation. Open Router exposes 339+ models. There is no good way to:

- Filter models by capability (multi-modal, code, text, reasoning)
- Compare prices across models for specific use cases
- Track which models worked and which didn't for specific tasks
- Understand which free models are actually usable
- Know when to use an expensive model vs. when a cheap one will do

The human has handwritten notes in a physical notebook about model experiments. That should be in a tool.

---

## The Tool: open-router.tools.sgraph.ai

A standalone browser-based tool for managing, filtering, comparing, and rating Open Router models. No server. No account required for basic use. Vault integration optional for persistent data.

### Two Modes

**Standalone mode (no vault, no account):**
- Fetches the Open Router models JSON
- Filter, sort, compare, visualise — all in the browser
- User notes and ratings saved in localStorage
- Works as a pure static page — deployable on GitHub Pages, embeddable in other products
- URL: `open-router.tools.sgraph.ai` or `tools.sgraph.ai/open-router/`

**Vault mode (with SG/Send vault):**
- Everything in standalone mode, plus:
- Notes, ratings, and preferences saved to encrypted vault
- Syncs across devices (via vault or Chrome extension)
- History of model usage and performance tracked
- Connects to workspace data (which models were used in which bundles, with what results)

### Why This Becomes a Standalone Tool

This is a tool that EVERYONE who uses Open Router needs, not just SG/Send users. Publishing it as a standalone, open-source tool:

- Drives organic traffic ("open router model comparison tool" is a search query)
- Establishes SGraph as a builder of useful developer tools
- The vault integration is the upsell: "Want to sync your model preferences across devices? Connect your SG/Send vault."
- Can be embedded in other products (iframe or JS import)
- Can be run locally from a downloaded HTML file (zero dependencies)
- First candidate for a properly branded standalone tool: `open-router.tools.sgraph.ai`

| **Filter by context window** | 4K, 8K, 32K, 128K, 200K+ |

| **Search** | Text search across model names and descriptions |
| **Favourites** | Star models for quick access (saved locally or to vault) |

| Feature | Description |
|---|---|
| **Cost calculator** | "If I send 30K tokens and receive 2K tokens, this model costs $X" |
| **Comparison table** | Side-by-side pricing for selected models |
| **Price history** | Track price changes over time (if we cache previous model list fetches) |
| **Free model spotlight** | Dedicated section for free models with capability ratings |

| Feature | Description |
|---|---|

## Technical Architecture

### Data Source

Open Router exposes a models endpoint: `GET https://openrouter.ai/api/v1/models`

This returns JSON with all available models, their pricing, capabilities, and metadata. The tool fetches this on load and caches it in memory (or localStorage for offline access).

No API key required to fetch the model list. An API key is only needed for the A/B test feature (which actually sends prompts).

### Standalone Implementation

```
open-router/
  index.html          <- Single page, vanilla JS
  models-cache.json   <- Optional: cached model list for offline use
```

Pure client-side. Fetches from Open Router API. Renders with vanilla JS. Notes/ratings in localStorage. Under 50KB excluding the model data.

### Vault Integration

When vault mode is active:
- Notes, ratings, tags, and favourites save to `vault://model-preferences/`
- Model usage history (from workspace bundles) is read from `vault://executions/`
- Preferences sync across devices via vault

### Embeddability

The tool should be embeddable as:
- A standalone page (full URL)
- An iframe in other products
- A component imported as `sg-model-manager.js` (following the tools.sgraph.ai module pattern)
- A new tab in the workspace (alongside Settings, Vault, and the transform view)

### GitHub Pages Deployment

In addition to `tools.sgraph.ai/open-router/`, deploy to GitHub Pages from the tool's folder in the repo. This gives a second URL, demonstrates that the tool works independently of SGraph infrastructure, and makes it easy for others to fork and customise.

## Community Feedback (Future)

### Anonymous Model Ratings via PKI

Once the PKI infrastructure matures, users could submit anonymous model ratings:

1. User rates a model (e.g., "4/5 for HTML transformation")
2. Rating is signed with the user's anonymous public key
3. Rating is encrypted and submitted to a shared vault (or public endpoint)
4. Other users see aggregated ratings: "This model has 23 ratings, average 3.8/5 for code generation"

**Privacy:** The rating is signed (proving it came from a real user, not a bot) but the user's identity is not revealed. Same PKI pattern as the whistleblower platform from the v0.11.1 brief.

**Spam protection:** One rating per public key per model. You can update your rating but not submit multiple ratings.

This is P3/future — but it's worth architecting the rating storage so it can evolve into a community system.
