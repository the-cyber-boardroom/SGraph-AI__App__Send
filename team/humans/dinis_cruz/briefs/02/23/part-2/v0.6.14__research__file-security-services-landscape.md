## What to Research

A comprehensive survey of commercial and open source services that can tell us about a file's safety, and services that can deliver files safely to users regardless of whether the original is malicious. These are the services that plug into the PKI-verified scanning chain (v0.6.14 architecture brief).

---

## Two Capabilities to Map

### Capability 1: Is This File Bad?

Services that analyse files and produce a verdict or rating:

| Category | What It Does | Examples to Research |
|---|---|---|
| **Multi-engine scanning** | Runs the file against dozens of antivirus engines simultaneously | VirusTotal, MetaDefender (OPSWAT), Jotti |
| **Sandbox / dynamic analysis** | Executes the file in an isolated environment and observes behaviour | Cuckoo Sandbox (open source), Hybrid Analysis (CrowdStrike), Joe Sandbox, ANY.RUN |
| **Static analysis** | Analyses file structure without executing — macros, scripts, embedded objects | YARA rules (open source), ClamAV (open source), oletools (open source, Office-specific) |
| **Reputation / network signals** | Checks file hash against known databases of good and bad files | VirusTotal hash lookup, NSRL (NIST National Software Reference Library), MalwareBazaar |
| **AI/ML classification** | Machine learning models trained on malware datasets | Cylance, Carbon Black, various academic models |
| **LLM-powered analysis** | Large language models analysing extracted code, macros, scripts | Novel — see scanning pipeline brief. No major commercial offering yet. Opportunity. |

**Important distinction**: there's a difference between "we didn't find anything bad" (absence of evidence) and "we can confirm this file is safe" (evidence of absence). Research which services, if any, can provide positive safety confirmation — not just "no malware detected."

### Capability 2: Can We Deliver the File Safely?

Services that transform or contain files so they can be viewed without risk:

| Category | What It Does | Examples to Research |
|---|---|---|
| **Content Disarm and Reconstruction (CDR)** | Strips potentially dangerous elements (macros, scripts, embedded objects) and reconstructs a clean version of the file | Votiro, OPSWAT Deep CDR, Glasswall, Sasa Software |
| **Secure file viewing / sandboxed rendering** | Renders the file in an isolated environment and delivers only the visual output (images, HTML) — the original file never reaches the user's system | Google Safe Browsing file viewer, various DLP solutions |
| **File format conversion** | Converts the file to a safe format (e.g., DOCX → PDF → flat PDF with no JavaScript) | LibreOffice headless conversion, various online converters |
| **Document isolation / micro-virtualisation** | Opens the file in a micro-VM on the user's device | HP Sure Click, Bromium (now HP), Menlo Security |
| **Secure browser isolation** | Renders web content / documents in a remote browser, streams pixels to the user | Surf Security, Menlo Security, Cloudflare Browser Isolation, Authentic8 Silo |

## Research Questions

| # | Question |
|---|---|
| 1 | Which scanning services offer APIs suitable for client-side integration (called from a browser via the pod)? |
| 2 | What are the pricing models? (Per file? Per GB? Subscription?) What does this mean for our cost model when the sender pays? |
| 3 | Which services can produce PKI-signed results? (Most won't today — can we wrap them?) |
| 4 | Which CDR services can accept a file and return a sanitised version via API? |
| 5 | Can any services provide a "file safety certificate" — positive confirmation that a file is safe, not just "no malware found"? |
| 6 | Which open source tools can run client-side (in-browser via WASM or in a pod)? |
| 7 | What's the latency? A user uploading a file doesn't want to wait 5 minutes for scanning. |
| 8 | Which services handle the file types our users send most? (PDF, DOCX, XLSX, images, archives) |
| 9 | Are there services specifically for polyglot file detection? |
| 10 | Which secure browser isolation services could we integrate with or recommend for high-security receivers? |
