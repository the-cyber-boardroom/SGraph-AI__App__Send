**version** v0.6.14

## What to Build

A simple static website at `sgraph.ai` — the root domain for the SGraph brand. This is the front door. It links to everything else: the products, the open source repos, the tools, the team site, the demos. It starts as a placeholder and grows into the brand's foundation.

---

## Why Now

Two products already exist and need a home:

1. **SG/Send** (`send.sgraph.ai`) — encrypted file transfer
2. **SSH Keygen** (`send.sgraph.ai/tools/ssh-keygen`) — browser-based SSH key generator

Plus the supporting sites:
- **Team site** (`team.send.sgraph.ai`) — agent meeting room
- **Docs** (GitHub Pages) — architecture, briefs, role definitions
- **GitHub repos** — open source code

There's no single place that ties these together under the SGraph brand. `sgraph.ai` is that place.

## Technical Requirements

| Requirement | Spec |
|---|---|
| **Hosting** | Static site. GitHub Pages or equivalent. No server. |
| **Domain** | `sgraph.ai` (root domain) |
| **SSL** | HTTPS required |
| **Build** | Simple HTML/CSS or Jekyll. No complex framework needed. |
| **Branding** | SGraph brand identity (logo, colours, typography). Consistent with SG/Send branding. |
| **Mobile** | Responsive. Works on phone (WhatsApp link → sgraph.ai is a real use case). |
| **Speed** | Instant load. No JavaScript required for initial render. |

---

## Content Strategy

This site is a hub, not a destination. Keep it concise. Every section is 2-3 sentences maximum plus a link. The real content lives at the linked destinations.

The Journalist writes the copy. The Ambassador reviews for positioning. The Designer creates the visual identity. The Alchemist builds the site.

### Future Expansion

As the brand grows, `sgraph.ai` becomes the parent for:

| Subdomain | What |
|---|---|
| `send.sgraph.ai` | SG/Send (existing) |
| `team.send.sgraph.ai` | Agent meeting room (planned) |
| `docs.sgraph.ai` | Documentation hub |
| `status.sgraph.ai` | Platform status page (from QA monitoring) |
| `keys.sgraph.ai` | Public key registry (from key discovery) |
| Future products | Additional tools and services under the SGraph umbrella |

For now, the homepage is a simple, fast, well-branded page that makes it obvious what SGraph does and gives people clear paths to explore.
