**type** Product + marketing brief — public demo infrastructure  

---

## The Strategy

Find users. Let them arrive at the conclusion they should invest. It should be an obvious calculation. To get there, potential users, investors, and partners need to SEE the platform working — not read about it.

Create a series of public, read-only demo environments that show SG/Send in action. Everything is public: decryption keys shared, data rooms open, files accessible. The information is demo data — the point is to demonstrate the platform's capabilities, not protect real secrets.

---

## The Evaluator Journey: Simple to Complex

The demos form a learning path. A WhatsApp link starts the journey. Each level introduces more capability:

| Level | What They See | Complexity | Link Format |
|---|---|---|---|
| **Level 1** | Open a single encrypted file. Type a short code. See the file. That's it. | Minimal — "anyone can do this in 30 seconds" | WhatsApp-friendly link |
| **Level 2** | A shared folder with several files. Browse, download, decrypt. See the file listing. | Folder navigation, multiple files | Link from Level 1 |
| **Level 3** | A branded data room. Logo, identity, multiple folders organised by topic. | Data room with branding | Link from Level 2 |
| **Level 4** | A data room with use-case scenarios: NDA signing, contract exchange, term sheet review. | Business workflows | Link from Level 3 |
| **Level 5** | Live activity — agents performing actions in real-time. Watch an NDA being signed. See files being exchanged. | Live demo with agent automation | Link from Level 4 |

### The Capture-the-Flag Element

Optionally gamified: the link to the next level is INSIDE the current level's data room. If you can decrypt and read the content, you find the next link. This naturally filters: casual browsers stop at Level 1-2, serious evaluators reach Level 4-5. It also demonstrates the product by making the evaluator USE it.

```
WhatsApp message: "Check this out: send.sgraph.ai/demo → enter code: 847293"
    │
    ▼
Level 1: Single file decrypts. Inside: "Want to see more? 
         Open send.sgraph.ai/demo/room1 with code: coral-summit-breeze"
    │
    ▼
Level 2: Shared folder. Inside the README: "The next level is a 
         branded data room: send.sgraph.ai/demo/nda-room"
    │
    ▼
Level 3: Branded NDA data room with sample documents...
    │
    ... and so on
```

## Demo Data Rooms

Each use case gets its own branded data room:

| Data Room | Use Case | Contents |
|---|---|---|
| **Simple Send** | Basic file transfer | A few sample files (text, PDF, image). Shows encrypt → send → receive. |
| **NDA Exchange** | Legal document signing | Sample NDA, signature workflow, two-party exchange. |
| **Contract Negotiation** | Iterative document exchange | Contract v1, redlines, v2, comments, final signed version. |
| **Term Sheet** | Investor materials | Sample term sheet, financial model, cap table. Shows secure investor communication. |
| **Due Diligence Room** | Full data room experience | Multiple folders (Legal, Financial, Technical, HR), dozens of files, branded. |
| **Budget Negotiation** | Multi-party financial discussion | Budget proposals, counter-proposals, approvals. |
| **Password / Credential Sharing** | Secure credential exchange | Demonstrates sending API keys, passwords, secrets securely. |

Each data room has its own brand, logo, and identity. They look like real business environments, not tech demos.

## Public Keys and Access

All demo environments are public. The access approach:

| Element | How It's Shared |
|---|---|
| **Short codes** | Published on the SG/Send website, included in WhatsApp messages |
| **Decryption keys** | Shared openly (these are demo environments — the data is public) |
| **PKI keys for demo users** | Published in a GitHub repo or on the website. "Here are the keys you need." |
| **Rotating keys** | Keys can rotate daily or per website build — keeps the demo fresh, shows the key management working |

The main SG/Send website (`send.sgraph.ai`) should have a visible "Try It" section with links to live demo encryption keys that work right now.

## Live Agent Demos (Level 5)

The most advanced demo: real agents performing real actions in real-time that evaluators can watch.

### What the Agents Do

```
Agent A (Sender):
  - Uploads a file to a data room
  - Sends a message to Agent B
  - Waits for confirmation

Agent B (Receiver):
  - Receives notification
  - Opens the data room
  - Downloads and decrypts the file
  - Sends confirmation back to Agent A

Agent C (Observer — the evaluator):
  - Watches the entire exchange in read-only mode
  - Sees messages appear, files being uploaded, confirmations arriving
  - Can inspect the encrypted data, the decryption process, the audit trail
```

### Connection to QA Infrastructure

These demo agents are the SAME agents the QA team builds for smoke testing:

| QA Purpose | Demo Purpose |
|---|---|
| Smoke test: does the platform work? | Live demo: watch the platform working |
| Performance baseline: are response times normal? | Live monitoring: see real-time performance |
| Regression detection: did something break? | Confidence signal: the platform is tested continuously |
| End-to-end flow validation | End-to-end flow demonstration |

The QA agents run continuously. The demo site shows their activity. Two purposes from one investment.

## Pace Matching: Fast Track for Sophisticated Evaluators

The journey accommodates two speeds:

**Casual explorer**: arrives from a WhatsApp link, opens one file, browses a bit, maybe reaches Level 2-3. Gets a sense of what SG/Send does. Takes 10-30 minutes.

**Sophisticated evaluator** (the target): arrives, immediately understands the encryption model, jumps through levels quickly, inspects the PKI infrastructure, watches the live agents, checks the GitHub repos, reads the architecture docs. Reaches the conclusion: "this is real, this works, I should get involved." Takes 1-2 hours.

Both paths use the same infrastructure. The fast path is just someone who moves through the levels quickly because they already understand the concepts.
