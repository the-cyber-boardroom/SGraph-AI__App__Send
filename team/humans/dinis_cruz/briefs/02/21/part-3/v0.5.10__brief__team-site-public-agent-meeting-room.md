## What to Build

A public static website at `team.send.sgraph.ai` where anyone can meet, read about, and talk to the project's 19 agents. Fully client-side. No server. No authentication required. All data loaded dynamically from GitHub Pages.

This is three things at once:

1. **The human's daily tool** â€” a place to talk to agents, get updates, check status
2. **A marketing asset** â€” carries the SG/Send brand, showcases the agentic workflow, drives traffic
3. **An open-source community hub** â€” contributors can talk to the dev, understand the architecture, find where to help

## LLM Provider Strategy for Public Access

Visitors don't have API keys. The site needs to work without requiring one.

| Provider | Visitor Experience | Cost to Us |
|---|---|---|
| **Chrome AI (Gemini Nano)** | Works immediately in Chrome Canary. No API key. Runs locally in browser. | Zero |
| **Ollama** | Works if visitor has Ollama running locally. | Zero |
| **Visitor's own API key** | Visitor enters their Anthropic/OpenAI/OpenRouter key. Stored in browser pod (never sent to our servers). | Zero |
| **Our API key (budgeted)** | We provide a limited free tier via our pod â€” N free messages per visitor per day. | Our cost, capped by budget |

### Default flow:

```
1. Visitor arrives â†’ check for Chrome AI availability
   â”œâ”€â”€ Chrome AI available â†’ use it (free, instant, no setup)
   â””â”€â”€ Chrome AI not available â†’
       2. Show options:
          â”œâ”€â”€ "Use your own API key" â†’ pod stores it, full access
          â”œâ”€â”€ "Try with Ollama" â†’ if localhost:11434 responds, use it
          â””â”€â”€ "Free preview (limited)" â†’ our budgeted key, N messages/day
```

The free preview tier is optional and budget-controlled. The human can decide whether to offer it based on cost. Even without it, Chrome AI provides a zero-cost option for Chrome users, and "bring your own key" works for everyone else.

## Guest Access (from v0.4.11)

The "guest" concept mapped in earlier sessions applies here. A visitor to `team.send.sgraph.ai` is a guest:

- No authentication
- No stored state (unless they choose to install the pod for their own API key)
- Read access to all public agent context
- Chat access via Chrome AI / own key / free tier
- Cannot access admin functions, data rooms, or encrypted content

The guest experience is the entry point to the SG/Send ecosystem. A guest who finds the agents useful might:
- Star the GitHub repo
- Try SG/Send for encrypted file transfer
- Become a contributor
- Become a customer (for data rooms)
