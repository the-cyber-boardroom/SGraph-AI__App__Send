## Origin Story

This tool was born from a real workflow. The human submitted a GDPR Subject Access Request (SAR) to Otter.ai. Otter.ai responded:

> *"Attach a copy of your government-issued ID showing your full name. For privacy reasons, you may redact any other sensitive information such as ID number, photo, address, date of birth, or expiration date."*

The human had no image editing tool at hand. Rather than open Photoshop, GIMP, or a random online tool of unknown provenance, the human asked Claude:

> *"Hi, how hard would it be for you to write a mini app that allowed me to: add/upload a copy of my gov ID (only available in the browser), allow me to redact the data that I want (via mouse), create a screenshot of it, and allowed me to download that as an image?"*

Claude built the tool in one shot as a single self-contained HTML file. The human used it to redact their Portuguese citizen card (ID number, DOB, address, photo, signature, expiry), keeping only the name visible. The redacted image was then sent to Otter.ai **encrypted via SG/Send** — not as an email attachment.

The result: a government ID was transmitted securely, with only the minimum necessary information disclosed, using zero external services. The tool that made it possible was built on demand, runs entirely in the browser, and never touched a server.

This workflow should be a permanent, polished tool in the SG/Send toolkit.

## The Use Case

GDPR SARs, KYC (Know Your Customer) processes, age verification, and identity checks are increasingly common. In almost every case, the verifying party asks for a government-issued ID and explicitly permits redaction of all fields except the one they need (typically just the name).

The problem: most people have no tool to do this quickly, locally, and safely. Their options are:
- Open Photoshop / GIMP (requires installation, technical knowledge)
- Use an online image editor (requires uploading the ID to a third-party server)
- Take a photo of a printed redacted copy (low quality, laborious)
- Send the full unredacted ID (oversharing, privacy violation)

The result: people routinely send more personal data than required because redacting is too hard. This is a solvable problem.

## The Tool: Image Redactor

```
send.sgraph.ai/tools/id-redactor
```

Same pattern as the SSH keygen, PGP keygen, storage viewer, and password generator — a standalone, transparent utility. Pure client-side. No libraries. No server calls. Nothing leaves the browser.

### The Workflow

```
1. User opens send.sgraph.ai/tools/id-redactor
2. User uploads their ID image (drag-and-drop or file picker)
3. User draws redaction boxes over sensitive fields (drag to select area)
4. User downloads the redacted image (full original resolution, redactions baked in)

Optional step 5:
   User clicks [Send via SG/Send] → encrypted transfer link generated
   User sends the link (not the file) to the requesting party
```

Step 5 is the differentiator. Every other redaction tool stops at "download." SG/Send continues: after redacting, the user can transmit the result via encrypted one-time link — the requesting party sees only what they're meant to see, and the transmission is zero-knowledge.

This replaces the "erase mode" from the MVP — erase mode was confusing because it required spatial memory of where boxes were drawn. ## Full Feature Set

### Image Transform Operations (non-destructive until export)

**Crop**
The user draws a crop rectangle on the canvas. The area outside the crop is dimmed. Pressing Enter (or clicking Confirm) commits the crop. Escape cancels. The crop defines the output area — everything outside is excluded from the downloaded image.

```
CROP mode:
  User drags a rectangle → rest of image dimmed
  Handles on corners and edges for precise adjustment
  [✓ Confirm crop]  [✗ Cancel]
```

**Resize**
Opens a modal with width/height inputs. Aspect ratio locked by default (unlock toggle). Accepts pixel values or percentage. Resizing changes the output canvas dimensions — all overlays scale proportionally.

```
Resize Image
  Width:  [ 1200 ] px    [ 50% ]
  Height: [  760 ] px    (locked ratio ✓)
  [Apply]  [Cancel]
```

**Flip**
Two buttons: Flip Horizontal (mirror left↔right) and Flip Vertical (mirror top↔bottom). Instant, non-destructive. All overlays flip with the image.

### Overlay Objects (the layers panel model)

Every action the user takes creates an **overlay object** — a discrete, named item in the layers panel. Objects are SVG elements rendered on top of the canvas image. They are never "baked in" until export.

This replaces the confusing "erase mode" from the MVP. There is no erase mode. Instead:
- Every object is visible in the layers panel
- Click an object in the list (or on the canvas) to select it
- Drag selected objects to reposition
- Resize via corner handles
- Delete via the 🗑 button in the layers panel or the Delete key
- Reorder by dragging rows in the layers panel (affects render order / z-index)

#### Object Type 1: Redaction Box

The core function. Drag to draw a filled rectangle. Colour from the toolbar swatch. The default is black — correct for most government IDs.

```
Layers panel entry:  ■ Redact #1   [opacity slider]  [colour]  🗑
Canvas:              Filled rectangle, resize handles when selected
Export:              Rendered as filled rectangle at full resolution
```

#### Object Type 2: Freehand Drawing

A draw mode for annotating, highlighting, or adding freehand marks. Useful for circling a specific field, underlining relevant text, or adding an arrow pointing to something.

```
Toolbar: [✏ DRAW]  Brush size: [● ▲▼]  Colour: [■]

Canvas: freehand path follows mouse/touch
Each completed stroke (mouseup/touchend) = one object in the layers panel
Layers panel entry:  ✏ Draw #2   [colour]  [size]  🗑

Selecting a stroke in the layers panel highlights it on canvas
Delete removes the entire stroke
(Strokes are not individually editable point-by-point — that's a future enhancement)
```

#### Object Type 3: Text Annotation

Add a text label to the image. Useful for: adding "NAME ONLY" or "FOR OTTER.AI GDPR REQUEST #664017" or "REDACTED" watermarks. Also useful for marking up documents generally.

```
Toolbar: [T TEXT]

Click on canvas → text input cursor appears at click position
User types → text renders live on canvas
Click elsewhere or press Enter → text committed as an object

Layers panel entry:  T "REDACTED" #3   [font size ▲▼]  [colour]  🗑

Selecting a text object:
  → Shows text content, position handles, font size, colour
  → Double-click to re-edit the text
```

Text objects render at full resolution on export, proportionally scaled from the display canvas.

### Layers Panel: The Control Centre

The layers panel replaces erase mode entirely. It is the canonical list of everything that has been applied to the image.

```
LAYERS  (4 objects)
─────────────────────────────────
  [▲] ■ Redact  #4    ◉ _____ 🗑    ← drag to reorder
  [ ] ✏ Draw    #3    ◉ _____ 🗑
  [ ] T  "ID"   #2    ◉ _____ 🗑
  [▼] ■ Redact  #1    ◉ _____ 🗑
─────────────────────────────────
  (click row to select · drag to reorder)
```

| Interaction | Behaviour |
|---|---|
| **Click row** | Selects the object — shows handles on canvas, shows properties at bottom of panel |
| **Click canvas object** | Same as clicking the row — bidirectional selection |
| **Drag row** | Reorders the layers (z-index changes) |
| **Drag canvas object** | Moves the object's position on the image |
| **Corner handles** | Resize the object |
| **🗑 button** | Deletes the object immediately (with Ctrl+Z undo) |
| **Delete key** | Deletes the currently selected object |
| **Opacity slider** | Adjusts opacity of the selected object (redaction boxes: default 100%, drawing: variable) |

### Technical Implementation

The architecture shifts from the MVP's "canvas-only" model to a **canvas + SVG overlay** model:

```
Rendering stack:
  Layer 1 (bottom): HTML5 Canvas — renders the base image
  Layer 2 (top):    SVG element — renders all overlay objects

Benefits:
  - SVG objects are individually addressable DOM elements
  - Click detection is native (no manual hit-testing)
  - Objects can be selected, moved, resized without re-rendering the canvas
  - The layers panel is just a list of SVG child elements
  - Export: flatten canvas + SVG to a single output canvas at full resolution
```

On export (download or SG/Send), the SVG overlay is rasterised onto the canvas at the original image's full resolution. All redaction boxes, drawing strokes, and text annotations are permanently baked in at native resolution — no scaling artefacts.

**No external libraries.** SVG manipulation is native DOM. Canvas 2D API for the image. No React, no Fabric.js, no Konva. The MVP was zero-dependency and this version stays zero-dependency.

The existing MVP HTML file is the starting point. The canvas-only architecture is replaced with canvas + SVG. The layers panel is new. Crop, resize, and flip are new. Everything else evolves from the existing implementation.

## The SG/Send Integration

The [Send via SG/Send] button is what makes this tool uniquely ours. The workflow after redaction:

```
User clicks [Send via SG/Send]

  → Redacted image is encrypted in the browser (AES-256-GCM)
  → Encrypted blob uploaded to SG/Send
  → One-time download link generated:
    https://send.sgraph.ai/send/v0/v0.1/v0.1.8/download.html#[decryption_key]
  
  User sends the link to Otter.ai (or whoever requested the ID)
  
  Otter.ai opens the link:
  → Sees: "Dinis Cruz - id-redacted.png"
  → Downloads and decrypts in their browser
  → Link self-destructs after first download
  
  SG/Send stored:
  → An encrypted blob. File name: encrypted. File content: encrypted.
  → AES-256-GCM. Decryption key: in the URL fragment (never sent to server).
  → Nothing else.
```

This is the real-world demonstration from 25 Feb 2026 — the human sent their redacted Portuguese citizen card to Otter.ai via exactly this flow. The recipient sees the filename "Dinis Cruz - id-redacted.png" and the full-resolution redacted image. SG/Send stored nothing meaningful.

## Note on the MVP

A fully functional redaction-only version already exists as a single HTML file, built by Claude during the Otter.ai SAR incident on 25 Feb 2026. It uses canvas-only rendering — redaction boxes are baked into the canvas immediately on draw, with an array of rectangle coordinates maintained for undo.

The v0.7.1 brief specifies a more capable architecture: **canvas (base image) + SVG overlay (all objects)**. This is a meaningful architectural change from the MVP, but it is the right foundation for the expanded feature set:
- The layers panel requires addressable, individually selectable objects — SVG delivers this natively
- Drag-to-reposition and resize-via-handles require DOM hit detection — SVG delivers this natively
- Text and freehand drawing integrate cleanly as SVG elements
- The canvas-only approach would require reimplementing all of this manually

The MVP is the reference for the upload, download, and SG/Send integration. The canvas+SVG architecture is the implementation path for everything else. Estimated effort: 2-3 focused sprints to get from MVP to the full feature set in this brief.
