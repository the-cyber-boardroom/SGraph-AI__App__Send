# Task Launcher: The Unified Shell for LLM Task Components

**date** 21 Apr 2026
**from** Human (project lead)

## What This Is

A single tool that wraps all seven task components from the previous brief (`v0.21.9__dev-brief__task-focused-llm-components.md`) and gives the user a clean way to pick a task, execute it, and manage the results. It is also the place where all the shared UI elements live: headers, footers, sg-layout, vault actions, share actions, cost control.

Think of it as the shell. The seven task components are the engines. This tool provides the cockpit.

## Why a Shell

Without the shell, each task component is a standalone page on tools.sgraph.ai. The user has to know which page to go to. They navigate between separate URLs. Each page has its own header, its own layout, its own save flow. There is no shared state, no unified cost tracking, no common vault context.

The shell solves this by providing one place where the user:

1. Drops a document (once)
2. Picks a task (or multiple tasks)
3. Gets results
4. Saves, shares, or continues to the next task

The document stays loaded. The vault context stays active. The cost accumulates across tasks. The user does not re-upload or re-navigate.

## The UI

### Layout (sg-layout)

Use sg-layout (the existing panel management Web Component) for the tool layout:

```
+------------------------------------------+
|  Header: logo, user/login, cost display  |
+--------+---------------------------------+
|        |                                 |
|  Task  |     Active Task Panel           |
|  Picker|                                 |
|        |     (the selected component     |
|  [ ]   |      renders here)              |
|  Review|                                 |
|  [ ]   |                                 |
|  Spell |                                 |
|  [ ]   |                                 |
|  Discuss                                 |
|  [ ]   |                                 |
|  Infog. |                                |
|  [ ]   |                                 |
|  Image |                                 |
|  [ ]   |                                 |
|  Research                                |
|  [ ]   |                                 |
|  Evidence                                |
|        |                                 |
+--------+---------------------------------+
|  Footer: vault actions, share, save      |
```

The left panel is the task picker. The right panel renders whichever task component is selected. sg-layout handles resizing, collapsing on mobile, and fractal nesting if we need sub-panels later.

### Header (Shared)

The header is consistent across the tool and reusable across all tools on the platform:

- SG/Send logo and navigation
- User/login status (uses `<sg-login>` from the auth MVP when ready, gracefully hidden when auth is not active)
- **Session cost display:** running total of all LLM calls in this session. Updated after every task completion. "Session cost: 12.4 cents (7 tasks)"
- Model indicator: which model is active for the current task (informational, not a selector here)

### Footer (Shared)

The footer provides actions that apply to any task's output:

- **Save to vault:** save the current output to the active vault (or create a new vault). Uses vault components from the existing vault UI.
- **Share:** create an SG/Send share token for the output. Uses `<sg-send-drop>` from the send/receive reference implementation.
- **Download:** download the output as a file (markdown, PDF, SVG, image, depending on the task).
- **Copy:** copy the output to clipboard.
- **Cost breakdown:** expand to see per-task cost breakdown for the session.

### Task Picker

The left panel lists all seven tasks with clear icons and one-line descriptions:

| Icon | Task | One-liner |
|------|------|-----------|
| (doc icon) | Review Document | Get a structured review with suggestions |
| (spell icon) | Spell Check | Fix spelling, grammar, and style |
| (chat icon) | Discuss Ideas | Explore arguments, alternatives, blind spots |
| (chart icon) | Create Infographic | Turn a document into a visual summary |
| (image icon) | Create Image | Generate an image from a description |
| (search icon) | Research Topic | Get a sourced research summary |
| (link icon) | Find Evidence | Verify claims with source links |

Clicking a task loads the corresponding component into the main panel. The input (if a document was already dropped) carries over. The user does not re-upload.

## Shared State

### Document Context

When the user drops a document into any task, that document becomes the session context. Switching to another task pre-loads the same document. "I dropped my report into Review. Now I want to Spell Check it too." One drop, multiple tasks.

The document context is held in memory (not re-uploaded). Each task component receives the document via its API, not via a new file upload.

### Vault Context

If the user has an active vault (opened or created during the session), all save actions default to that vault. The footer's "Save to vault" button saves the current task output as a new file in the active vault, committed with a descriptive message ("Review of report.pdf by Claude, 21 Apr 2026").

If no vault is active, the save action offers: create a new vault, save as SG/Send file (get a token), or download locally.

### Cost Accumulation

The header tracks total session cost across all tasks. The footer's cost breakdown shows per-task detail:

```
Session cost: 18.7 cents
  Review Document (Claude):     4.2 cents
  Spell Check (Claude):         2.1 cents
  Create Infographic (Claude):  3.8 cents
  Research Topic (Perplexity):  5.1 cents
  Find Evidence (Perplexity):   3.5 cents
```

This is the cost transparency principle applied at the session level, not just the task level.

## Multi-Task Workflow

The shell enables a natural multi-task workflow on a single document:

```
Drop report.pdf
    |
    +--> Spell Check (fix language)
    |       +--> Accept corrections -> updated document
    |
    +--> Review (get structural feedback)
    |       +--> Read suggestions
    |
    +--> Find Evidence (verify all claims)
    |       +--> See which claims have sources, which do not
    |
    +--> Create Infographic (visual summary)
    |       +--> Save SVG to vault
    |
    +--> Share vault with colleague
```

Five tasks, one document, one session, one vault, one share action. Total cost: visible in the header.

## Shared Components Library

The shell is also the forcing function for creating a proper shared components library. These components are used in the shell and reusable across all tools and pages:

| Component | Used In Shell | Reusable Across |
|-----------|--------------|-----------------|
| `<sg-header>` | Top of shell | All tools, main website, vault pages |
| `<sg-footer-actions>` | Bottom of shell | Any tool that produces saveable output |
| `<sg-cost-display>` | Header + footer breakdown | Any tool with LLM costs |
| `<sg-task-input>` | Shared document input | All seven task components |
| `<sg-vault-actions>` | Footer save-to-vault | Any tool interacting with vaults |
| `<sg-share-actions>` | Footer share/download/copy | Any tool producing shareable output |
| `<sg-login>` | Header user status | All tools (when auth MVP ships) |
| `<sg-layout>` | Panel management | Already exists, used everywhere |

Building the shell properly means building these shared components properly. Every future tool benefits.

## Deployment

The shell deploys as a single page on tools.sgraph.ai (e.g. tools.sgraph.ai/tasks or tools.sgraph.ai/workspace). It should also be linkable directly to a specific task (e.g. tools.sgraph.ai/tasks/research) for cases where someone wants to jump straight to one task.

The seven task components remain independently deployable as standalone pages too. The shell is the unified experience. The standalone pages are the direct-link experience.
