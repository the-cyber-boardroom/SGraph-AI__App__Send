# SG Tools Python Package: CLI and API Wrapper for Browser Tool Automation

**version** v0.20.38
**date** 09 Apr 2026
**from** Human (project lead)
**to** Developer (lead), Architect
**type** Dev brief

## The Problem

Every time Claude (or any agent) needs to create infographics, it writes a full Playwright script from scratch: proxy setup, browser launch, page navigation, API calls. This is 30-50 lines of boilerplate that is identical every session. The JS API refactoring solved the tool interaction problem, but the Playwright setup and orchestration still needs to be written each time.

## The Solution

A pip-installable Python package that wraps the entire workflow. Claude clones the repo (or pip installs), imports the module, and calls high-level functions.

### Package Name Candidates

Check PyPI availability for:

- `sg-tools-ai`
- `sg-tools`
- `sgtools-ai`
- `sg-tools-cli`

GitHub repo: `SGit-AI/SG-Tools-AI__CLI` (following existing SGit naming convention).

### Usage (What Claude Should Do)

```python
# Instead of 50 lines of Playwright boilerplate:
from sg_tools import InfographicGenerator

gen = InfographicGenerator(api_key='sk-or-...')
result = gen.create(
    prompt='Architecture diagram of a vault system',
    template='architecture',
    output_path='slides/slide-01.png'
)
print(f"Cost: ${result.cost}, Size: {result.size_bytes} bytes")
```

For batch operations:

```python
from sg_tools import InfographicGenerator, SlideDeck

gen = InfographicGenerator(api_key='sk-or-...')

# Create a full slide deck from a list of prompts
deck = SlideDeck(generator=gen)
for i, prompt in enumerate(slide_prompts):
    deck.add_slide(prompt, template='process-flow')

deck.export('output/slide-deck/')
# Creates: slide-01.png, slide-02.png, ..., manifest.json
```

### What the Package Handles Internally

- Playwright browser launch with proxy detection (reads `HTTPS_PROXY` automatically)
- Chromium setup (installs if needed)
- Navigation to tool URL
- JS API calls via `page.evaluate()`
- Event waiting (generation:complete)
- PNG export and download capture
- Cost data retrieval from OpenRouter
- Browser lifecycle (reuse across multiple generations)

### Architecture

```
sg_tools/
  __init__.py
  browser.py          # Playwright setup, proxy config, browser pool
  infographic.py      # InfographicGenerator class
  slide_deck.py       # SlideDeck class (batch infographic assembly)
  vault.py            # Vault operations (wraps sgit CLI)
  tools_base.py       # Base class for any tool on dev.tools.sgraph.ai
  config.py           # Tool URLs, defaults, timeouts
```

### Extending to Other Tools

The base class pattern means adding a new tool wrapper is straightforward:

```python
class VideoAssembler(ToolBase):
    TOOL_URL = 'https://dev.tools.sgraph.ai/en-gb/video-assembler/'

    def add_frame(self, image_path): ...
    def add_audio(self, audio_path): ...
    def render(self, output_path): ...
```

Each tool wrapper reads the manifest from `window.__tool_manifest` on first connection and validates that the expected methods exist.

### Integration with SGit

The package should also wrap SGit operations so a complete workflow is one script:

```python
from sg_tools import InfographicGenerator, Vault

vault = Vault.clone('both-dart-5487')
gen = InfographicGenerator(api_key='sk-or-...')

# Generate slides
for doc in vault.list_files('sources/'):
    content = vault.read(doc)
    gen.create(prompt=content, output_path=f'slides/{doc.stem}.png')

vault.commit('Generated slide deck from source documents')
vault.push()
token = vault.share()
print(f"Browse at: https://dev.send.sgraph.ai/en-gb/browse/#{token}")
```
