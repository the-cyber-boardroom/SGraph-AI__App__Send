# Debrief: JavaScript API Refactoring Delivers Deterministic Infographic Pipeline

**version** v0.20.38
**date** 09 Apr 2026
**from** Human (project lead)
**to** Developer (lead), Architect, QA
**type** Debrief

## What Happened

The JavaScript API refactoring described in yesterday's arch brief (javascript-api-tool-primitive) was implemented overnight. The infographic generator was refactored so that the JavaScript API is the fundamental layer, with the UI sitting on top of it. This was then tested end-to-end with Claude using Playwright.

## Before and After

| Metric | Before (UI automation) | After (JS API first) |
|---|---|---|
| Execution time | Minutes | ~15 seconds per infographic |
| Reliability | Flaky, frequent timeouts | Deterministic, one-shot execution |
| Script approach | Polling for DOM changes, guessing selectors | Direct API calls via `page.evaluate()` |
| Claude behaviour | Multiple retries, error recovery loops | Writes the Playwright script in one go, runs it in one go |
| Slide deck (6-7 slides) | Not practical (too many failure points) | Works end-to-end in a single session |

The critical improvement: Claude was able to write the entire Playwright script in a single pass and execute it without retries. The API surface is clear enough that Claude does not need to guess at selectors or timing.

## The Full Workflow That Ran

1. Fresh Claude session (Claude.ai with computer use)
2. "Create a vault. Here is a document. Create an infographic and slide deck from it using the tools at dev.tools.sgraph.ai."
3. Claude set up Chromium with Playwright
4. Called the JS API directly: set key, set template, generate, export PNG
5. Repeated for each slide (6-7 infographics, one per slide)
6. Assembled into a slide deck
7. Committed to vault via SGit CLI
8. Shared via `sgit share` for browsing online

## Extended Workflow: Interactive Document Analysis

A second workflow was also tested:

1. Uploaded photo documents for analysis
2. Claude analysed them and created a slide deck
3. Uploaded a second document containing questions
4. Claude asked the questions interactively in the chat
5. Captured answers into a document
6. Created a slide deck from the answers
7. Everything committed to the vault

The vault became the delivery vehicle: all source materials, generated slides, analysis documents, and answers in one shareable package. Accessible via web vault (online editing partially working), SGit CLI, and shared as a folder/zip via SG/Send.

## What Made It Work

1. **JS API as the contract:** Clear methods (setKey, getKey, generate, exportPng, getCost) callable via `page.evaluate()`. No DOM inspection needed.
2. **Async/await model:** Claude wraps generate() in a Promise, awaits completion event. No polling loops.
3. **Deterministic selectors removed:** Claude does not need to find buttons. It calls functions.

## What Is Still Needed

- **Python package wrapper** (see separate brief): Claude should not need to write Playwright boilerplate every session. A pip-installable package with `create_infographic()` would reduce the setup to `pip install sg-tools-ai` and a few function calls.
- **Vault integration in the API:** Commit and share should be callable from the same JS API layer, not via separate CLI commands.
- **Web vault editing:** Partially working. Needs further development for full round-trip editing.
