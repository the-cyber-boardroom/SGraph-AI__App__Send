## Part 1: What Happened (Incident Report)

### The Setup

- Pixel 6 phone running WhatsApp on Android
- WhatsApp backup configured to sync to Google (had previously restored from this backup successfully, with some minor data loss)
- WhatsApp also running on laptop and iPad (linked devices)
- Two-factor authentication enabled
- Google Authenticator for 2FA across services

### The Trigger

The Pixel 6 died. Screen stopped working completely. Not a glitch, not a gradual failure. Total screen death. The phone was likely still powered on but unusable.

### The Migration Attempt

1. Got a new iPhone
2. Moved SIM card to iPhone
3. Transferred Google Authenticator (2FA) successfully
4. Opened WhatsApp on iPhone
5. Entered phone number, completed 2FA verification
6. **WhatsApp asked for first name** (as if creating a new account, not restoring)
7. WhatsApp showed "no backups found" (because Apple cannot read Google backups)
8. No option offered to check for Google backups, no warning about cross-platform incompatibility
9. A small warning said "you might not be able to do this again" but was unclear about the consequences
10. Proceeded with setup

### The Damage

- Logged into WhatsApp on iPhone with zero message history
- **Immediately logged out of all other devices** (laptop, iPad)
- The laptop briefly flashed old messages before they disappeared (WhatsApp revoked the linked device sessions)
- All messages, all media, all conversation history: gone
- Group memberships survived (server-side), but had to re-accept some

### What Should Have Happened But Did Not

| What Went Wrong | What Should Have Happened |
|----------------|--------------------------|
| No cross-platform backup detection | WhatsApp should detect that a Google backup exists and offer migration options |
| Destructive account reset without clear warning | "This will permanently delete all your messages on all devices" should be an explicit, unmissable warning |
| No "are you sure?" gate before destroying linked device sessions | Require explicit confirmation before revoking all linked devices |
| No download-my-data option | GDPR requires data portability. WhatsApp should offer a full export before any destructive action |
| No recovery path once the new device is activated | Should offer a 24-48 hour grace period to recover from a cross-platform mistake |
| Android-to-iOS tool requires the old phone to be working | The only migration tool (Move to iOS) is useless if the old phone is dead |

### Root Cause

WhatsApp's encryption model means backups are client-encrypted and platform-specific. Google backups cannot be read by Apple devices. When you activate a new device on a different platform, WhatsApp treats it as a fresh install and destroys all linked device sessions. The encryption that protects the messages in normal use becomes the weapon that destroys them during migration.

## Part 2: Research Request

Create a research document (using Perplexity, Claude, and OpenAI) investigating:

1. **Is this a known problem?** How common is WhatsApp message loss during Android-to-iOS migration (and vice versa)?
2. **Does it go both ways?** iPhone-to-Android: same problem?
3. **Are there any recovery options?** Can the Google backup be accessed from a computer? Can a broken Android phone's data be extracted?
4. **What does Google show for WhatsApp backups?** Where in Google account settings should the backup appear? Why might it not be visible?
5. **What third-party tools exist?** Are there services or apps that backup WhatsApp messages independently of the phone's built-in backup?
6. **What do competitors do?** How do Telegram, Signal, iMessage handle cross-platform migration?
7. **Legal angle:** Does WhatsApp's destructive migration violate GDPR data portability requirements?

## Part 3: The Business Opportunity

### The Problem (Real, Widespread, Unsolved)

WhatsApp has 2+ billion users. Cross-platform migration destroys message history. Phone death with no working screen makes migration impossible. There is no independent backup service. There is no "download my data" that actually works before a destructive action.

This is not an edge case. Phones die. People switch platforms. Messages matter (personal, business, legal, medical, financial). The loss is permanent and irreversible.

### The Business: WhatsApp Backup as a Service

A service that continuously backs up WhatsApp messages to an SG/Send vault. That is all it does.

**How it works:**

1. User installs the backup app (or connects via WhatsApp Business API or linked device)
2. Messages are encrypted client-side and stored in an SG/Send vault
3. Vault is versioned (every backup is a commit, full history preserved)
4. Vault is encrypted (zero-knowledge, client-side AES-256-GCM)
5. When the user needs to restore (new phone, different platform, phone died), they access the vault

**Why SG/Send vaults are perfect for this:**

- Client-side encryption: the backup service cannot read the messages (privacy preserved)
- Version control: every backup is a snapshot, rollback to any point in time
- Cross-platform: vaults are accessible from any device (browser, CLI, any OS)
- Storage: S3-backed, scales to any size, cost-effective
- Share: if needed, the user can share specific conversations via share tokens (e.g. for legal discovery)

### Revenue Model

- Subscription: small monthly fee for continuous backup (1-3 pounds/month)
- Usage-based: per GB stored (aligns with the SG/Send pricing philosophy)
- Free tier: last 30 days of messages, paid for full history

### What Else Could Be Backed Up

WhatsApp is the most painful example, but the same service applies to:

- Telegram messages (despite having cloud backup, users may want independent copies)
- Signal messages (notoriously difficult to backup)
- SMS/iMessage (for cross-platform portability)
- Any messaging platform where the user wants an independent, encrypted, versioned backup

### Build vs Enable

"I do not want to build this, but somebody should build it."

This is a perfect example of the "primitives for others to build on" philosophy from the OWASP presentation pack. SG/Send provides the vault infrastructure. Someone else builds the WhatsApp integration layer on top. SG/Send earns revenue from vault storage. The integrator earns revenue from the subscription.

Alternatively, this could be a case study and reference implementation: we build the MVP to prove the concept and open-source it, then others build the production service.

### Research: Competitive Landscape

Investigate as part of the research document:

- Do any WhatsApp backup services exist today?
- What do they charge?
- How do they handle encryption?
- Are they cross-platform?
- What are their limitations?
- Have any been shut down by WhatsApp/Meta?
