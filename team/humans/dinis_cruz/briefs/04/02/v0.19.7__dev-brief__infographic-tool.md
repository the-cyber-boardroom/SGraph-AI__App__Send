## The Milestone

Yesterday, infographics were created inside our own UI for the first time. The cost: sub-$1 per infographic. Many were sub-10 cents. Some were sub-1 cent. The infographics looked good (not perfect, but good enough to use).

This is a major milestone because the business model works: if an infographic costs 5 cents to generate and we charge 25% markup, the user pays 6-7 cents for an infographic. That is cheap enough that people will experiment freely. Volume drives revenue.

## IFD Evolution: v0.1.0 to v0.1.1

The Explorer version (v0.1.0) proved the concept. It has too many options, too many moving parts, too much complexity for a casual user. The Villager version (v0.1.1) needs to simplify.

### What v0.1.0 Has

- Infographic generation from text prompts
- Multiple models available
- SG Layout with tabs (multiple infographics side by side)
- Basic settings and configuration
- Client-side rendering

### What v0.1.1 Needs

Two distinct parts, cleanly separated as components:

**Part 1: Settings / Input (left or top panel)**

Multiple modes, from simple to complex:

| Mode | What the User Provides | Complexity |
|---|---|---|
| Simple | A text description ("explain how encryption works") | One text box, one button |
| Document | A source document (upload a PDF or markdown) | Attachment + button |
| Multi-document | Multiple source documents as evidence | Multiple attachments + button |
| Template | A pre-built prompt template + user fills in variables | Template picker + variable inputs |
| Advanced | Custom system prompt, model selection, style options | Full configuration panel |

The default should be Simple mode. One text box. One button. Get an infographic. Everything else is progressive disclosure.

**Part 2: Results / Viewer (right or bottom panel, tabbed)**

Each generated infographic gets its own tab. Each tab is a mini-app containing:

- The generated infographic (viewable, zoomable)
- Generation stats (model used, tokens consumed, cost, duration)
- Controls: stop generation, save, share via SG/Send, download, remix (open a new tab with modifications)
- The bundle (all inputs and outputs saved together for reproducibility)

Each tab should be its own SG Layout instance (fractal nesting). This means within a tab you can have the infographic on one side and the generation stats on the other, or the infographic full-screen.

## Templates and Prompt Library

### Pre-Built Templates

Create a set of infographic prompt templates:

| Template | Style | Best For |
|---|---|---|
| Executive summary | Clean, minimal, data-focused | Business reports, presentations |
| Technical architecture | Boxes and arrows, system diagram | Engineering documentation |
| Timeline | Horizontal or vertical timeline | Project history, roadmaps |
| Comparison | Side-by-side columns | Product comparison, before/after |
| Process flow | Step-by-step with icons | How-to guides, workflows |
| Statistics dashboard | Numbers, charts, key metrics | Data presentation |
| Mind map | Central concept with branches | Brainstorming, topic exploration |

### Templates Per Model

Different image models have different strengths. A template that works well with DALL-E might not work with Stable Diffusion or Midjourney-style models. Maintain per-model template variants and document which combinations produce the best results.

### Shareable Template Library

Publish templates as a shared vault or folder on the website. Users can browse templates, see example outputs, and use them directly. This drives traffic and demonstrates the tool.

## Model Comparison

### Same Prompt, Multiple Models

A powerful feature: send the same infographic prompt to 3-5 different models simultaneously. Display the results side by side in tabs. The user sees which model produced the best output for their use case.

This is:
- Useful for the user (find the best model for their needs)
- Educational (see how different models interpret the same prompt)
- A traffic driver (people come to compare models)
- Revenue-generating (each model call costs tokens, the comparison multiplies usage)

### Model Cost and Quality Page

Publish a page (on the website or tools.sgraph.ai) showing:
- Available models with pricing per generation
- Example outputs per model (same prompt, different models)
- Quality ratings (maintained by the team, updated regularly)
- Cost-per-quality ranking (best value models)

This page becomes a destination for anyone exploring image generation models. Update it as new models release.

## The Bundle: Everything Saved Together

Every infographic generation produces a bundle:

```
infographic-bundle/
  input.json          (prompt, model, settings, template used)
  output.png          (the generated infographic)
  metadata.json       (cost, duration, tokens, model version)
  source-documents/   (if documents were provided as input)
  prompt-history/     (if the prompt was refined across iterations)
```

Bundles are saveable to local storage, downloadable, and shareable via SG/Send. A bundle shared via a vault link lets someone else see exactly how an infographic was created and remix it.

## Live Examples on the Website

Create a gallery of infographics generated by the tool, with:
- The infographic image
- The prompt that created it
- The model used
- The cost
- A "Remix this" button (opens the tool with the same settings)

This gallery is both documentation and marketing. It shows what the tool can do, demonstrates different styles, and invites users to try.
