# Document-Driven Analysis: One Document In, Summary + Infographic + Briefing Out

**version** v0.19.7
**date** 31 Mar 2026
**from** Human (project lead)
**to** Developer (lead), Designer
**type** Dev brief

---

## The Component

A simple, focused component: give it a document, get back a set of outputs. One click, multiple results.

### Input

A single document: PDF, markdown, text file. Dropped, pasted, or referenced by simple token.

### Output (Generated in One Go)

| Output | What It Is |
|---|---|
| Summary | A concise text summary of the document |
| Infographic | A visual summary (using the infographic tool's engine) |
| Infographic from summary | A second infographic generated from the summary (often cleaner than from the raw document) |
| Briefing document | A structured briefing with key findings, recommendations, action items |
| Research document | A deeper analysis if the source is a research paper or technical document |

The user can choose which outputs to generate (or just click "generate all"). Each output appears in its own tab (SG Layout, same pattern as the infographic tool).

### Templates

Pre-built output templates:

- "Executive brief" (summary + infographic + one-page briefing)
- "Research analysis" (summary + detailed analysis + infographic)
- "Quick visual" (just the infographic, fastest and cheapest)
- "Full pack" (everything: summary, infographic, briefing, research doc)

## Inline Integration: The Infographic Button Everywhere

The infographic generation should not live only in a dedicated tool. It should be available as a button wherever documents are viewed:

### In the Browse View

When viewing a markdown or PDF file in the Browse view, add a toolbar button: "Create Infographic." One click generates an infographic from the document contents. The infographic is saved to the user's vault.

### In the Vault View

Same button. When browsing a vault, any document can be turned into an infographic with one click.

### In the Gallery View

When viewing a gallery of documents, each document card could have a "Summarise" or "Infographic" action.

### The Pattern

Every tool should be able to read from SG/Send transfers. This means supporting both:

- Transfer ID + decryption key (the standard access method)
- Simple token (the friendly access method)

Add this as a standard input option across all tools. If a tool can process a document, it should be able to read that document from an existing transfer, not just from a local file upload.

```html
<sg-document-input
  accept="pdf,md,txt"
  sources="local,paste,simple-token,transfer-id"
  on-document="handleDocument"
>
</sg-document-input>
```

This component handles all input modes. Every tool that needs a document uses it. Consistency across the platform.

## Connection to the Content Creation Workflow

This component is the automated version of the voice-memo-to-article workflow:

1. Upload a document (or reference one from a vault)
2. Generate summary + infographic + briefing in one click
3. Results saved to user vault
4. Share via SG/Send link
5. View in Browse with inline images

The human does not need to manage multiple Claude sessions, copy files between tools, or manually assemble outputs. One document in, complete pack out.
