## Context

We already have a video editor component that can clip movies, separate audio and video tracks, and export segments. This brief extends it with three capabilities driven by real use cases: dynamic cropping (to redact sensitive areas of screen recordings), multi-track overlay (titles, text on video), and video stream capture (recording from YouTube or other players).

## Tool 1: Video Cropper with Dynamic Crop Regions

### The Use Case

A screen recording from a camera or demo contains sensitive areas that need to be cropped out. But the sensitive area is not in the same position throughout the video. Every 5-10 seconds, the crop region might need to shift slightly (a name appears in a different corner, a notification pops up, the window moves).

### What It Does

A timeline-based crop editor where the user can define crop regions at specific timestamps, and the crop region interpolates (or steps) between keyframes.

### Workflow

```
Load video
    |
    v
Scrub timeline to a frame
    |
    +--> Draw crop rectangle on the video preview
    +--> Set as keyframe at this timestamp
    |
    v
Scrub to another frame
    |
    +--> Adjust crop rectangle
    +--> Set as another keyframe
    |
    v
Repeat for each region change
    |
    v
Export: crop applied per-frame, output resized to target aspect ratio
```

### Key Requirements

- Timeline scrubber with frame-level (or second-level) control
- Visual crop rectangle overlay on the video preview (draggable, resizable)
- Keyframe system: set crop region at specific timestamps
- Between keyframes: either hard-cut (crop jumps) or smooth interpolation (crop slides)
- Output resizes the cropped region to the target display resolution (so a small crop area gets scaled up to fill the output frame)
- All client-side (Canvas API or OffscreenCanvas for frame processing)

## Tool 2: Multi-Track Overlay (Titles and Text)

### The Use Case

Add titles, captions, or text overlays to a video. For example, adding a title card at the start, chapter headings at specific points, or a persistent lower-third with speaker name.

### What It Does

A simple multi-track timeline (2-3 tracks) where the user can place text or title elements at specific time ranges.

### Tracks

| Track | Purpose |
|-------|---------|
| Video track | The source video (read-only, scrubable) |
| Title track | Full-screen title cards (text on solid/transparent background) |
| Overlay track | Lower-thirds, captions, watermarks positioned on top of video |

### Key Requirements

- Drag text elements onto tracks at specific time positions
- Set duration for each text element
- Basic text styling (font size, colour, position, background)
- Preview: see the composed output with overlays in real time
- Export: burn overlays into the video output
- Start simple: text only, no images or complex graphics on overlay tracks

## Tool 3: Video Stream Capture

### The Use Case

Capture video and audio from a video player (YouTube, Vimeo, or any HTML5 video element) playing in the browser. Record specific clips from presentations, talks, or content that the user has access to.

### Approach (Research Required)

There are several possible approaches, ranked by feasibility:

**Option A: Screen recording with smart crop.** We already have screen recording (from the 17 Apr client-side video recorder brief). The simplest path is: record the screen, then use the crop tool (Tool 1 above) to crop to just the video player area. Not elegant, but works today with existing components.

**Option B: MediaStream capture from video element.** The `captureStream()` API on HTML5 `<video>` elements returns a MediaStream that can be recorded with MediaRecorder. This gives clean capture without screen recording overhead. Limitation: may not work with DRM-protected content or cross-origin video.

**Option C: Python-based download.** Tools like `yt-dlp` can download video from YouTube and other platforms. This runs server-side or via the Playwright container, not client-side. Best quality, but different architecture.

**Recommendation for MVP:** Start with Option A (screen recording + crop). It works now. Investigate Option B for a cleaner future path. Document Option C as an alternative for the Playwright service.

### Clip Recording

The user should be able to record specific segments, not just the entire video. Start recording, pause, resume, stop. Each segment becomes a clip. Clips can be exported individually or stitched together.

## Shared Architecture

All three tools should:

- Build on the existing video editor component
- Use the same timeline/scrubber UI element
- Save output via the send/receive API (`SgSend.sendFile()` from today's other brief)
- Optionally save to vault
- Be deployable on tools.sgraph.ai
- Expose the standard JS API
