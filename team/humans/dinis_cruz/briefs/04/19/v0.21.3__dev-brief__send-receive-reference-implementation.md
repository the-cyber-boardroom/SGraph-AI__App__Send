# Send/Receive Reference Implementation: The JS API for the Core Primitive

**version** v0.21.3
**date** 19 Apr 2026
**from** Human (project lead)
**to** Developer (lead), Designer
**type** Dev brief

---

## The Problem

The core SG/Send workflow (send a file, get a token, paste the token, receive the file) works from the main UI. But when other tools try to use it programmatically, the integration is not smooth. The video creation tool, for example, needs a simple way to upload a recorded video and get a share token. It tried to integrate, but it ended up recreating parts of the workflow instead of calling a clean API.

The root cause: there is no JS API for the send/receive primitive. The UI exists, but the API abstraction that other tools should call does not.

## What We Need

A reference implementation of the send/receive workflow as a proper JS API with reusable Web Components. Any tool on the platform should be able to send or receive files with a single API call.

### The API

```javascript
// Send
const token = await SgSend.sendFile(file)
const token = await SgSend.sendText(text)
const token = await SgSend.sendFolder(files)

// Receive
const file  = await SgSend.receiveFile(token)
const text  = await SgSend.receiveText(token)
const files = await SgSend.receiveFolder(token)
```

That is it. The caller does not need to know about encryption, chunking, upload endpoints, or token formats. The API handles all of it. This is the "API IS the tool" principle applied to the most fundamental primitive in the platform.

The API must expose the standard meta API (`getManifest`, `getMethods`, `getSkills`) per the JS API convention established on 08 Apr.

### The Web Components

Reusable drop-in components for any tool that needs send/receive in its UI:

**`<sg-send-drop>`**: A drag-and-drop zone. Drop a file or folder, get a token. Handles the upload, encryption, and token generation. Emits a `token-created` event with the token value.

**`<sg-send-receive>`**: A token input field. Paste a token, receive the file/folder. Handles the download, decryption, and file delivery. Emits a `file-received` event with the content.

**`<sg-send-panel>`**: A combined component with both send and receive in a single panel (or multi-pane layout). This is the reference implementation UI.

### The Reference Tool

A standalone tool on tools.sgraph.ai that demonstrates the full workflow:

- Left pane: drag and drop a file/folder, get a token displayed
- Right pane: paste a token, receive the file/folder
- Clean, minimal UI that serves as both a usable tool and a reference for how to integrate the components

This is also the tool that appears in demo videos: "Drop a file here. Copy the token. Paste it there. Done."

## Token Storage (Interim)

Until the user account system is fully built (auth MVP from 18 Apr brief), we need a temporary place to store tokens. We already have a local storage viewer component. Extend it or create a simple token history panel:

- Tokens stored in localStorage
- List view showing recent tokens (send and receive)
- Ability to copy, re-use, or delete tokens
- Clear visual distinction between sent tokens and received tokens

This is interim. When the auth MVP ships, token history moves to the user's account (stored as encrypted vault data).

## Why This Matters

Every tool on the platform that produces output (video recorder, infographic generator, PlaybookLM, screen recorder) needs to answer the question: "How does the user save and share this?" The answer should always be: call `SgSend.sendFile()` or drop in `<sg-send-drop>`. One API call. One component. No custom integration code.

This is the missing plumbing that makes the tool ecosystem composable.

## Scope

This brief covers the send/receive primitive only. Not vaults. Not SGit. Not collaboration. Just: send a file (or folder or text), get a token, paste the token, receive the content. The simplest SG/Send workflow, packaged as a clean API and component set.

## Acceptance Criteria

| # | Criterion | Verification |
|---|-----------|-------------|
| 1 | `SgSend.sendFile(file)` returns a valid token | Call from console, token returned, usable for download |
| 2 | `SgSend.receiveFile(token)` returns the original file | Round-trip: send then receive, file matches original |
| 3 | `SgSend.sendFolder(files)` works with multiple files | Send 10 files, receive all 10 via token |
| 4 | `SgSend.sendText(text)` and `receiveText(token)` work | Round-trip with text content |
| 5 | `<sg-send-drop>` component accepts drag-and-drop, emits token | Drop file onto component, `token-created` event fires |
| 6 | `<sg-send-receive>` component accepts token paste, delivers file | Paste token, `file-received` event fires with content |
| 7 | Reference tool deployed on tools.sgraph.ai | Accessible, functional, send and receive panes working |
| 8 | Video recorder tool can use `SgSend.sendFile()` to upload recordings | Record video, save via API, get share token without custom upload code |
| 9 | Token history stored in localStorage with list view | Recent tokens visible, copyable, deletable |
| 10 | Standard JS API meta methods exposed (getManifest, getMethods, getSkills) | Callable from console and Playwright |
