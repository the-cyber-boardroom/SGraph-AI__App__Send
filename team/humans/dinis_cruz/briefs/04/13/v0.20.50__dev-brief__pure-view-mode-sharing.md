# Pure View Mode: Distraction-Free Sharing of Folders and Pages

**version** v0.20.50
**date** 10 Apr 2026
**from** Human (project lead)
**to** Developer (lead), Designer
**type** Dev brief

---

## The Problem

The full infographic workflow works end to end: Claude creates infographics, packages them into a folder, creates a vault, shares via simple token. But when the recipient opens the link (especially on mobile), the experience breaks down. The folder browser shows the left navigation panel alongside the content, which does not render well on small screens. The recipient does not care about the folder structure. They want to see the content.

This is a friction point in the sharing loop. The creation workflow is smooth. The consumption experience is not.

## The Solution: A Pure View Mode

Add a view mode (toggled via URL parameter or folder setting) that hides the folder structure entirely and presents only the rendered content. The `_page.json` layouts already define navigation, hero sections, and content components. In view mode, that is all the recipient sees: a clean mini-website with no file browser, no tree view, no folder panel.

### What View Mode Shows

- The `_page.json` rendered page (hero, navigation bar, sections, images, cards, columns)
- Navigation between pages (if the folder has multiple `_page.json` files or the nav bar links between sections)
- Content: infographics, markdown, images, text

### What View Mode Hides

- Folder tree / file browser (left panel)
- File metadata (sizes, dates, types)
- Upload/edit controls
- Vault management UI

### How to Activate

| Method | Example |
|---|---|
| URL parameter | `send.sgraph.ai/browse/#token?view=pure` |
| Folder setting | A flag in `_page.json`: `"view_mode": "pure"` |
| Share option | When sharing, choose "share as page" vs "share as folder" |

If `_page.json` sets `"view_mode": "pure"`, anyone opening the share link gets the clean view by default. The folder browser is still accessible via a toggle or menu icon for users who want to see the raw files.

### Mobile First

The primary trigger for this is mobile. The view mode must:

- Render correctly on phone screens
- Use responsive layout (single column on narrow viewports)
- Load fast (no unnecessary folder metadata fetched)
- Support touch navigation between sections/pages

## The Sharing Workflow This Enables

```
Claude creates infographics + materials
        |
        v
Package into folder with _page.json
        |
        v
Share link (with view=pure default)
        |
        v
Recipient opens on phone
        |
        v
Sees: clean mini-website with infographics, text, navigation
Does NOT see: folder tree, file browser, vault controls
```

This is the last mile of the PlaybookLM pipeline: the output is not a zip the recipient has to unpack. It is a link that opens as a designed page.

## Connection to Existing Work

The `_page.json` layout system (hero, sections, navigation, cards, columns) already does the heavy lifting. View mode is primarily about hiding the folder chrome around it. The page rendering is already working. The fix is in the container, not the content.
