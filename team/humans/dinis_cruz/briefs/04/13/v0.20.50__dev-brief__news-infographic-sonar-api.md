## The Goal

A new tool on tools.sgraph.ai that generates news reports with infographics, backed by real-time data and source citations. The key differentiator: Perplexity's Sonar API is not a standard LLM. It has live access to current web data and returns citations with every response. This means the reports it generates come with provenance built in.

## Why Perplexity Sonar

Standard LLMs (Claude, GPT, Gemini) have knowledge cutoffs. They can generate text about a topic but cannot tell you what happened today. Perplexity Sonar is different:

| Capability | Standard LLM | Perplexity Sonar |
|---|---|---|
| Current data | No (knowledge cutoff) | Yes (live web access) |
| Citations | No (generates from training) | Yes (returns source URLs per claim) |
| Evidence chain | Not available | References embedded in response tokens |
| News access | No | Yes (real-time) |

This makes Sonar the right model for the first stage of the pipeline: gather current information with sources. Other models can then be used for subsequent stages (analysis, infographic prompts, slide generation).

## The Pipeline

```
TOPIC / QUERY
(e.g. "latest AI regulation news in the EU")
        |
        v
STAGE 1: Sonar API
  - Fetch current news and data
  - Extract citations and references from response
  - Parse provenance metadata (URLs, dates, sources)
        |
        v
STAGE 2: Analysis and structuring
  - Organise findings into report sections
  - Map evidence to claims
  - Identify key data points for visualisation
  - (Can use a different model: Claude, Gemini, etc.)
        |
        v
STAGE 3: Infographic generation
  - Per-section infographic via existing tool
  - Each infographic backed by cited sources
  - Evidence trail preserved in metadata
        |
        v
STAGE 4: Package and share
  - Assemble report + infographics + citations
  - Package as folder/vault with _page.json
  - Share via simple token (with pure view mode)
```

## Citation Extraction: The Key Technical Piece

Perplexity Sonar embeds references in its response tokens. The tool needs to parse these carefully:

- Extract all source URLs returned by the API
- Map each claim in the generated text to its backing source(s)
- Preserve the full citation chain: claim -> source URL -> date accessed
- Store citations as structured data (JSON) alongside the report

This is the provenance layer. Every infographic, every claim in the report, traces back to a specific source. This is what makes these reports credible, not just generated content, but evidence-backed analysis.

## Exploration Phase First

Before building the full pipeline, we need a UI to explore and understand the Sonar API:

- Which Sonar models are available and which work best for news queries?
- What does the citation data look like in the raw response?
- How reliable are the references? Do they resolve to real, current pages?
- What is the optimal sequence of models? (Sonar for data, then Claude/Gemini for analysis, then Gemini for infographics)
- What are the token costs per query?

Build a simple exploration UI first: enter a query, see the raw Sonar response, inspect the citations, test different models. Then build the pipeline on top of what we learn.

## What This Produces

A shareable, evidence-backed news report that looks like a mini-website:

- Hero section with report title and date
- Per-topic sections with infographics
- Each section has a "Sources" footer with clickable citation links
- Full source list at the end
- All backed by data retrieved in real time, not from training data

This is a concrete product that demonstrates the platform: current news, professional infographics, evidence-backed claims, shareable as a link. The kind of thing people would pay for (connecting to the pack delivery service model).
