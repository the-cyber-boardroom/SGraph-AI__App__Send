## The Goal

A new website (examples.sgraph.ai or similar) that showcases what the platform can produce: infographics, document packs, reports, slide decks. Updated daily (or hourly) by an agentic team that runs via Claude Code sessions. All content is stored in vaults and decrypted on the fly in the browser, demonstrating the technology in action.

## Why "Examples" Not "Gallery"

"Gallery" conflicts with the existing gallery view component in the UI. "Examples" is clearer: these are examples of what SG/Send can produce, and examples of the workflows that produced them.

## Architecture

```
ONE ROOT VAULT (the master secret)
        |
        +-- List of simple tokens for content vaults
        +-- OpenRouter API key (for infographic generation)
        +-- SG/Send upload token (for publishing)
        |
        v
CONTENT VAULTS (one per example or category)
        |
        +-- Infographics
        +-- Document packs
        +-- Slide decks
        +-- News reports
        |
        v
STATIC WEBSITE (S3 + CloudFront)
        |
        +-- HTML/JS/CSS (IFD components)
        +-- Reads from vaults at runtime
        +-- Decrypts and renders in browser
```

**One secret unlocks everything.** The agentic team needs a single environment variable: the root vault token. From that vault it reads all other tokens, keys, and configuration. This is the same pattern as the deterministic-safe-paths model: one secret derives access to everything.

### What the Website Shows

- Hero page titled "Examples"
- Daily infographic (auto-generated about today's news or topic)
- Curated examples of document packs, slide decks, reports
- Each example links to a vault or folder that can be browsed (with pure view mode)
- All content decrypted client-side, demonstrating zero-knowledge in action

### The Content is Live Vault Data

The website does not contain static images or pre-built HTML. It fetches vault content at runtime and renders it using existing components. This means:

- The website is a live demonstration of the technology
- Adding a new example means adding content to a vault, not editing HTML
- The agentic team can add examples without touching the website code

## The Agentic Team

A small, focused team that runs via Claude Code (scheduled or on-demand):

| Role | Responsibility |
|---|---|
| **Webmaster** (new role) | Decides where new content goes on the homepage, maintains layout, ensures quality |
| **Ambassador** | Curates what to showcase, writes descriptions, selects highlights |
| **Sherpa** | Ensures examples are accessible and well-explained for new users |

### Daily Workflow

```
Claude Code session starts (scheduled or manual)
        |
        v
Read root vault -> get tokens, keys, config
        |
        v
Generate today's content:
  - Create infographic about today (using Sonar for news + infographic API)
  - Upload to content vault
  - Update vault index
        |
        v
Webmaster agent:
  - Check for new content in vaults
  - Decide placement on homepage
  - Update _page.json layout if needed
        |
        v
Website auto-refreshes from vaults (no deploy needed)
```

### Manual Additions

The human can also add examples at any time:

1. Go to Claude Code session: "Hey, can you add this infographic to the examples site?"
2. Or: update a vault directly, and the webmaster agent picks it up on next run

## First Version: Minimal Viable Showcase

Start simple:

1. Static site hosted on S3 + CloudFront (same pattern as other SG/Send sites)
2. IFD component pipeline for the site structure
3. Hero page with "Examples" title
4. One example: today's auto-generated infographic
5. The infographic is fetched from a vault and rendered live

If this works, the pipeline is proven: agentic team creates content, stores in vault, website renders from vault, all without manual deployment.

## The Setup Pack

A new Claude Code session with a brand new repo needs a brief pack containing:

- Where the main SG/Send repos are (for component reuse)
- Where the tools repos are (for infographic generation)
- CI pipeline configuration (S3 deploy)
- IFD workflow setup
- Agent role definitions (Webmaster, Ambassador, Sherpa)
- Root vault token (the one secret)

This is itself a good example of the agentic team setup pack creator (brief from 8 April).

## Meta: The Website Demonstrates Itself

The examples website is a meta-demonstration: it shows examples of what the platform can do, and it is itself an example of what the platform can do (vault-backed, agentically maintained, zero-knowledge decrypted, component-driven). The first example on the site should be a description of how the site itself works.
