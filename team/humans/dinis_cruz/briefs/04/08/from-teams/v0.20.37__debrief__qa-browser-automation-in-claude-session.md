## Key Learnings

### What Works in Claude.ai Sessions

1. **Playwright headless Chromium** — installs and runs perfectly (`playwright install --with-deps chromium`)
2. **In-memory SG/Send API server** — FastAPI on random port, no S3, no network needed
3. **UI static file server** — `python3 -m http.server` on localhost:10062 (must bind to `localhost` not `127.0.0.1` for Web Crypto secure context)
4. **Screenshot capture** — PNGs at 1280×800 viewport, deterministic variants with QA masks
5. **Full test fixtures** — `conftest.py` session-scoped fixtures all work (send_server, ui_server, browser, screenshots)
6. **Python 3.12** is the container default — matches the SG/Send stack exactly

### What Doesn't Work (or needs workarounds)

1. **Network egress** — only allowlisted domains are accessible. Both curl and Playwright are blocked for non-allowlisted domains. Config changes require a **new session**.
2. **Blocking scripts** — `bash_tool` has a timeout; long-running servers must be backgrounded with `&>/dev/null &` and the PID tracked manually.
3. **Shell differences** — container uses `/bin/sh`, not bash. `source` doesn't work; use `.` or call `/tmp/venv312/bin/python` directly.
4. **File system resets** — `/home/claude` resets between sessions. The venv, cloned repos, and installed packages don't persist. Each session needs the full setup (~2 min).
