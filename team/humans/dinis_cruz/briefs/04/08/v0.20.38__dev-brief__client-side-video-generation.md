## The Goal

Two video workflows, both running entirely in the browser:

1. **Video in:** Take an existing video, extract audio, generate captions, capture screenshots, align visuals with transcript
2. **Document to video:** Take a document, extract topics, generate infographics per topic, generate audio narration, assemble into a 2-5 minute video

All processing is client-side. Server-side is only needed for LLM calls (via OpenRouter) and storage (via SG/Send).

## Why Client-Side

Every workflow described here should run in a browser. This is a deliberate architectural choice:

- Zero server infrastructure for media processing
- Privacy: media never leaves the client unless the user explicitly uploads
- Automation is achieved by automating browsers (Playwright), not by building server pipelines
- Consistent environment: the tool works the same whether a human clicks or an automated browser drives it

## Workflow 1: Video Analysis (Video In)

```
Video file
  |
  +--> Extract audio (client-side, existing code)
  |      |
  |      +--> Transcribe (LLM or speech-to-text API)
  |      |
  |      +--> Generate captions/subtitles
  |
  +--> Capture screenshots at intervals
  |      |
  |      +--> Align with audio timestamps
  |
  +--> Output: folder with video, audio, transcript, screenshots, captions
```

A straightforward first step: add captions to a video. The transcript is timestamped, the screenshots are aligned, and the captions overlay on playback.

## Workflow 2: Document to Video (Document Out)

This is the more ambitious workflow. Take written content and produce a narrated slideshow video.

```
Document (markdown, brief, article)
  |
  +--> Extract 5-7 key topics (LLM call)
  |
  +--> Per topic:
  |      |
  |      +--> Generate infographic (existing infographic tool)
  |      |
  |      +--> Generate audio narration script (LLM call)
  |      |
  |      +--> Generate audio (text-to-speech, OpenRouter models)
  |
  +--> Assemble: sequence infographics as frames, overlay audio per section
  |
  +--> Output: 2-5 minute video file + folder with all intermediate assets
```

### No Animation Required (v1)

The first version is a narrated slideshow: static infographic images with audio narration. Each "slide" displays for the duration of its audio segment. This is achievable today with existing browser APIs (Canvas, MediaRecorder, Web Audio).

### Model Quality Comparison

Use multiple OpenRouter models for the same document and compare output quality. This is both a useful development tool and a potential feature: "generate with Model A vs Model B" side by side.

## Component Architecture

| Component | What It Does | Exists? |
|---|---|---|
| Audio extractor | Extract audio track from video | Yes (existing code) |
| Screenshot capturer | Capture frames at intervals | Yes (existing code) |
| Infographic generator | Create visual from text topic | Yes (existing tool) |
| Text-to-speech | Generate audio from narration script | New (OpenRouter models) |
| Video assembler | Combine images + audio into video | New (Canvas + MediaRecorder) |
| Caption overlay | Add subtitle track to video | New |

## Revenue Connection

Every LLM call in these workflows consumes tokens via the platform. A user generating a video from a document triggers: topic extraction, infographic prompts, narration scripts, text-to-speech. That is 10-20+ LLM calls per video, all billable through OpenRouter at the 25% markup. The workflow drives recurring usage naturally.
