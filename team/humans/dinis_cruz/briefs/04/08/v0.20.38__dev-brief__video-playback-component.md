# Simple Video Playback Component: Play Media Directly from Zip Files and SG/Send

**version** v0.20.38
**date** 08 Apr 2026
**from** Human (project lead)
**to** Developer (lead), Designer
**type** Dev brief

---

## The Goal

A drop-in component for the main website that plays video directly from a zip file hosted on SG/Send. The user shares a simple token, the component loads the manifest, lists available media, and plays it. No download-the-whole-zip-first requirement (eventually), but start with full download for now.

## Why This Matters

We can already create infographics and we can already host files on SG/Send. Combining these with video playback creates a powerful content workflow: upload a video, add screenshots, infographics, and metadata to the same folder, then share a single link that presents all of it.

## Component Behaviour

### Basic Flow

1. Accept a zip file (via simple token, direct URL, or local drop)
2. Download the zip (full download for v1; partial/streaming in future)
3. If the zip contains a manifest file (markdown or JSON), use it for metadata display
4. List available media files
5. Play selected media file in an embedded player

### Manifest Convention

If the zip contains a metadata or manifest file, the component uses it to enrich the playback experience:

| File | Purpose |
|---|---|
| `manifest.json` | Structured metadata: title, description, chapters, timestamps |
| `README.md` | Human-readable description displayed alongside the player |
| `screenshots/` | Thumbnail images for chapter navigation |
| `infographics/` | Generated infographics related to the video content |

### Future: Partial Download

Once server-side zip support allows downloading individual files (see separate brief on server-side zip folder versioning), the component only needs to download the manifest first, then stream individual media files on demand. This is critical for large video collections.

## The Content Workflow

```
Video file
  |
  v
[Drop into folder tool] --> Extract audio, screenshots, topics
  |
  v
[Infographic generator] --> Create infographics per topic
  |
  v
[Package as zip/folder] --> Upload to SG/Send
  |
  v
[Video playback component] --> Share link, play with rich metadata
```

Each step in this pipeline is a reusable tool. The folder is the universal container moving between steps.
