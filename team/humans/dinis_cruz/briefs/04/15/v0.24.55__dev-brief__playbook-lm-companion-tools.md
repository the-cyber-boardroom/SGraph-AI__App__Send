# Three PlaybookLM Companion Tools: Infographic Generator, Infographic Editor, Simple PlaybookLM

**version** v0.24.55
**date** 15 Apr 2026
**from** Human (project lead)
**to** Dev (lead), Designer, Architect
**type** Dev brief

---

## Context

PlaybookLM already has a solid pipeline for source-to-presentation workflows, including a concept of capturing information and bundling artefacts together. These three tools extend that ecosystem by providing focused, simpler entry points that reuse the same underlying components. Each tool serves a distinct user intent but shares the PlaybookLM infrastructure.

---

## Tool 1: Infographic Generator (Simple Mode)

### What It Does

Drop a PDF or text, choose a model, get an infographic. That is it.

This tool already exists in a basic form but should be rebuilt to leverage the PlaybookLM component pipeline and produce the same bundled artefacts (not just a standalone SVG). The key difference from the current infographic tool is integration with the PlaybookLM capture and bundling workflow.

### Workflow

1. User drops a PDF or pastes text
2. User selects a model (Gemini, Claude, Qwen, Llama, DeepSeek)
3. User clicks Generate
4. System produces one or more infographic variants
5. Artefacts are bundled using the PlaybookLM packaging model (not just a raw SVG download)

### Design Notes

- Option to generate multiple versions (different layouts, different emphasis) in a single pass
- Model selector with cost indication per model
- Reuse existing `sg-llm-infographic`, `sg-llm-events`, `sg-llm-request` components from tools.sgraph.ai
- Output should be saveable to vault

## Tool 2: Infographic Editor

### What It Does

Drop an existing infographic (image), get an LLM-powered analysis of what is on it, then make targeted changes and verify those changes were applied correctly.

### Workflow

```
Drop Image
    |
    v
LLM Analysis (review the infographic)
    |
    v
Show analysis to user:
  - "Here are three things we could change"
  - "Here is text that could be updated"
  - "Here are issues spotted"
    |
    v
User confirms/selects changes
    |
    v
LLM Image Generation (apply changes via Gemini or similar)
    |
    v
Verification Pass:
  - Send new image back to LLM
  - Or send both old and new for comparison
  - "Did the change apply correctly?"
  - "Did we lose anything?"
  - "Compare these two images"
    |
    v
Show result + cost breakdown
```

### Key Design Decisions

**Analysis pass vs direct edit:** The tool should always start with an analysis conversation, not jump straight to editing. The LLM reviews the infographic first and proposes what could be changed.

**Change execution has multiple paths:**

- Full LLM image regeneration (round-trip through Gemini or similar image model)
- Browser-based tool execution (for simple operations like redaction, text overlay, obfuscation): this could be a Python tool or a browser-side command run from the prompt. This is potentially the first experiment with running tools in the browser directly from the LLM prompt.
- Hybrid: LLM identifies what to change, browser tool executes the mechanical part, LLM verifies the result

**Verification loop:** After every edit, automatically send the result back for comparison. This can be done by sending just the new image, or both old and new side by side, asking the LLM to confirm the change was applied without unintended side effects.

**Cost transparency UI:** Each step in the loop should display its cost. After the full workflow, show a summary: "Analysis: 10 cents, Image generation: X cents, Verification: Y cents, Total: Z cents." Include a note suggesting cheaper model alternatives if the user is getting good results, or more capable models if they are not.

### Interesting Capability: PDF Image Extraction

Add a sub-feature to extract images from a PDF in the browser, so users can drop a PDF and select which image/slide to edit. This should be browser-side (no server round-trip). The extracted images feed directly into the editor workflow.

## Tool 3: Simple PlaybookLM (One-Click Mode)

### What It Does

The full PlaybookLM pipeline, but with a simple UI: upload sources, provide a brief prompt, click Generate. The tool handles everything behind the scenes and delivers a finished PDF at the end.

### Workflow

1. User uploads one or more source files
2. User writes a short prompt (or uses a default)
3. User clicks Generate
4. UI shows progress status as the pipeline runs (stage indicators, not a blank wait)
5. PDF output appears when complete

### Design Notes

- Behind the scenes, this runs the full PlaybookLM pipeline: source analysis, brief generation, per-slide briefs, infographic generation, assembly
- The UI should show status updates for each stage (e.g. "Analysing sources...", "Generating slide briefs...", "Creating infographics...", "Assembling PDF...")
- Must have an option to "switch to advanced mode" that drops the user into the full PlaybookLM UI with all the intermediate artefacts visible and editable
- Market positioning: "NotebookLM, but you can actually edit the output"

---

## Shared Architecture

All three tools should:

- Reuse PlaybookLM component pipeline (sg-llm-infographic, sg-llm-events, sg-llm-request, page layout renderer)
- Save artefacts to vault using the same bundling conventions
- Display per-operation cost breakdowns
- Support model selection with cost/quality guidance
- Be deployable as standalone pages on tools.sgraph.ai
- Expose the standard JS API (getManifest, getMethods, getSkills) per the "API is the tool" principle
