## Introduction

This document outlines the integration of **SG/Send Vault** into GenAI research platforms that need to securely share AI-generated insights with executives. SG/Send Vault provides zero-knowledge encrypted storage that ensures research outputs remain confidential throughout their lifecycle — from creation by AI agents, through review by analysts, to delivery to senior leadership.

The integration is powered by **SGit-AI** (`pip install sgit-ai` | [sgit.ai](https://sgit.ai)), a Python CLI and library that gives any application programmatic access to encrypted vaults. SGit-AI enables your platform to create, read, write, and sync vault contents without the server ever seeing plaintext data.

**The core guarantee:** SG/Send Vault uses AES-256-GCM encryption performed entirely in the client. The server stores only encrypted ciphertext. Decryption keys never leave the client environment. This is an architectural guarantee — not a policy promise — meaning even a compromised server cannot access your data.

---

## Why This Matters for GenAI Research Delivery

GenAI research platforms face a unique security challenge: AI-generated outputs often contain sensitive strategic insights, proprietary analysis, and confidential data synthesised from multiple sources. When these outputs are shared with executives, they traverse systems that may log, cache, or expose content.

SG/Send Vault eliminates this risk by design:

| Traditional Approach | SG/Send Vault Approach |
|---|---|
| Research stored in plaintext on server | Research encrypted before leaving the client |
| Server administrators can read content | Server mathematically cannot decrypt content |
| Sharing requires trusting intermediaries | Sharing uses cryptographic keys, not trust |
| Audit trails can be tampered with | Audit trails are hash-chained and immutable |
| Access control enforced by application logic | Access control enforced by encryption (no key = no access) |

## Integration Architecture

### How SGit-AI Connects Your Platform to Encrypted Vaults

```
+----------------------------+        +------------------+        +------------------+
|  Your GenAI Research       |        |   SGit-AI        |        |  SG/Send Vault   |
|  Platform                  |        |   (Python lib)   |        |  Server          |
|                            |        |                  |        |                  |
|  1. AI generates research  | -----> |  Encrypt locally | -----> |  Store ciphertext|
|  2. Analyst reviews        | <----- |  Decrypt locally | <----- |  Serve ciphertext|
|  3. Share with exec        | -----> |  Grant vault key | -----> |  Enforce ACL     |
|                            |        |                  |        |                  |
+----------------------------+        +------------------+        +------------------+
                                            |
                                     Keys never leave
                                      this boundary
```

### Typical Workflow

1. **Research Creation** — Your platform's AI agents generate research outputs (reports, analysis, summaries)
2. **Encrypted Storage** — SGit-AI encrypts the output client-side and pushes to a vault: `sgit-ai push --token <access-token>`
3. **Analyst Review** — Analysts open the vault in the browser UI, review and annotate with zero-knowledge preserved
4. **Executive Delivery** — Share the vault key with the executive. They open the vault in their browser — decryption happens on their device, nowhere else
5. **Audit Trail** — Every access, edit, and share event is logged in a hash-chained, tamper-evident audit trail

### SGit-AI Integration Code

```python
from sgit_ai import SGitVault

# Clone a vault to local workspace
vault = SGitVault.clone("vault-key-here")

# Add research output
vault.add_file("/reports/q1-analysis.md", research_content)
vault.commit("Add Q1 strategic analysis")

# Push encrypted changes to server
vault.push(token="access-token")

# Share with executive (they get the vault key via secure channel)
vault_key = vault.get_vault_key()
```
