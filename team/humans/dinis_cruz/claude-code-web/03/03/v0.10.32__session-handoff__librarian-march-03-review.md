## Critical Findings (Next Session Must Know)

### 1. PBKDF2 Iteration Count Bug — MUST FIX BEFORE CLI WORK

The vault CLI brief's Python code sample uses `iterations=100000` but the JavaScript browser implementation uses `iterations=600000`. Three roles (Dev, Architect, AppSec) caught this independently. If anyone implements the CLI using the brief's sample code, files encrypted by Python will be **silently unreadable by JavaScript and vice versa**. The brief's code sample needs to be corrected (it's in `team/humans/dinis_cruz/briefs/03/03/v0.10.30__dev-brief__vault-cli-local-sync.md`). NOTE: briefs folder is read-only for agents — flag to human.

### 2. Vault Auth Trust Model — Needs Documentation

The vault fixes brief proposes making auth tokens only required for upload, not browsing. This means sharing the vault key grants **permanent, irrevocable, undetectable read access**. AppSec rates this HIGH. The trust model must be documented before implementing.

### 3. Gallery XSS — 5 Vectors to Mitigate

AppSec found 5 XSS attack vectors in the gallery sharing design: SVG script execution, polyglot files, subtitle injection, EXIF injection, filename injection. These must be mitigated before exposing galleries to untrusted content.

### 4. Reality Document Is Stale

The reality document (`team/roles/librarian/reality/v0.6.36__what-exists-today.md`) is at v0.6.36, while we're at v0.10.32. There is a newer version at `team/roles/librarian/reality/v0.10.13__what-exists-today.md` but it's still ~40 commits behind. **The Librarian's next priority should be updating the reality document to v0.10.32**, covering vault v0.1.1, zip viewer, folder upload, gallery preview, and all new UI components.
