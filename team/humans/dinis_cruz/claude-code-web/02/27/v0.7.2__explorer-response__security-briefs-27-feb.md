## Brief 2: AppSec Threat Model — SG/Send Skill Workflow

**File:** `team/humans/dinis_cruz/briefs/02/27/v0.7.1__appsec__threat-model-sg-send-skill-workflow.md`

### Summary
Comprehensive threat model for the 7-step encrypted file transfer pipeline. Critical distinction: Symmetric mode = "encrypted storage with key transmitted via email, protected by TLS." PKI mode = true E2E where only the recipient holds the key. Six threat scenarios modelled.

### Reality Check

- One-time use enforcement (auto-delete): **NOT VERIFIED**

### Key Finding
Zero-knowledge architecture means server breach has zero impact — strongest property. Primary risks are in the key transmission chain (Symmetric) and agentic prompt injection.

## Brief 3: GRC Risk Register — SG/Send Skill Workflow

**File:** `team/humans/dinis_cruz/briefs/02/27/v0.7.1__grc__risk-register-sg-send-skill-workflow.md`

### Summary
Formal risk register with accept/mitigate/transfer/avoid decisions. Seven risks registered. Server breach accepted unconditionally (zero-knowledge). Key-in-email accepted with conditions. n8n log retention and prompt injection require active mitigation. Deployment gates defined — **none currently met**.

### Key Decisions
| Risk | Decision | Conditions |
|------|----------|------------|
| Server breach | ACCEPT | Unconditional — zero-knowledge neutralises |
| Key-in-email (Symmetric) | ACCEPT | One-time use + 24h expiry + content classification |
| Claude/Anthropic logs | ACCEPT | With disclosure |
| n8n log retention | MITIGATE | No full body logging, 7-day max |
| Prompt injection | MITIGATE | Security Analyst gate required |
| Gmail compromise | ACCEPT | With one-time use controls |
| Non-repudiation gap | ACCEPT | PKI + signing for cases requiring it |
