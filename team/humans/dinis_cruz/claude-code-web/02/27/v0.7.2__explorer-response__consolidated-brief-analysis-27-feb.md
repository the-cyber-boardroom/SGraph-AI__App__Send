## Brief 1: Sherpa Inbox Workflow and Playbooks

**Source:** `team/humans/dinis_cruz/briefs/02/27/v0.7.1__dev-brief__sherpa-inbox-workflow-and-playbooks.md`

### Summary

| Playbook library | PROPOSED — does not exist yet |
| CRM data room (users.csv, contacts/) | PROPOSED — does not exist yet |
| Morning standup workflow | PROPOSED — does not exist yet |
| Message categorisation engine | PROPOSED — does not exist yet |
| Unsubscribe automation | PROPOSED — does not exist yet |
| Bounce handling | PROPOSED — does not exist yet |

### Key Asks

1. Build a structured morning standup workflow: inbox scan, categorisation, action proposal
2. Create a playbook library (at least ONBOARD-01, FAQ-DATAROOM, FOLLOWUP-01, BOUNCE-01, UNSUBSCRIBE-01) as markdown files in a data room
3. Build CRM state tracking in the outreach data room (users.csv + per-user contact folders)

5. Human-in-the-loop for all actions (automation gated on security review)

### Primary Roles

| User UI localization | EXISTS — 8 languages (EN, ES, FR, DE, PT, JA, ZH, KO) |
