## How This Fits the Workflow

1. **Explorer makes changes** → writes changelog entry documenting what changed + expected test impact
2. **QA agent clones repo** → reads changelog → classifies test failures as good (expected) or bad (regression)
3. **QA agent has questions** → writes in `qa/questions/` → Explorer responds in next session
4. **Villager prepares release** → reads changelog to understand what's new

This replaces the ad-hoc approach of placing QA briefs in `team/roles/dev/reviews/` where they're hard to discover.
