## The Interface Your VaultDataSource Must Implement

Three required methods:

```javascript
class VaultDataSource {

    /**
     * Returns the hierarchical file tree.
     * @returns {{ name: string, children: { [name: string]: TreeNode }, files: Array<{ path: string, name: string, size?: number }> }}
     */
    getTree() { }

    /**
     * Returns file content as ArrayBuffer.
     * @param {string} path — full path within the vault (e.g. 'docs/readme.md')
     * @returns {Promise<ArrayBuffer>}
     */
    async getFileBytes(path) { }

    /**
     * Returns flat list of all files (used for link resolution, auto-open, search).
     * @returns {Array<{ path: string, name: string, dir: boolean, size?: number }>}
     */
    getFileList() { }
}
```

Plus optional properties:

```javascript
    // If true, Browse will show Edit/Save/Delete buttons (future)
    writable = false;

    // Optional: for "Save locally" button (re-generates zip)
    async getZipBlob() { }

    // Optional: archive name for Save dialog
    getOrigName() { }

    // Optional: archive size for header display
    getOrigSize() { }
```

## Tree Node Shape

`getTree()` must return a tree matching this exact shape:

```javascript
{
    name: '',              // root name (empty string for root)
    children: {
        'docs': {
            name: 'docs',
            children: {
                'images': {
                    name: 'images',
                    children: {},
                    files: [
                        { path: 'docs/images/logo.png', name: 'docs/images/logo.png', size: 45000 }
                    ]
                }
            },
            files: [
                { path: 'docs/readme.md', name: 'docs/readme.md', size: 2400 }
            ]
        }
    },
    files: [
        { path: 'INDEX.md', name: 'INDEX.md', size: 1500 }
    ]
}
```

**Key points:**
- `children` is an object keyed by folder name (not an array)
- `files` is an array of file entries at this level
- Each file needs at minimum `path` and `name` (the `name` property is the full zip path in the current implementation — the browse component extracts the basename for display)
- `size` is optional but recommended (shown in file action bar)

## How to Create and Use the Browse Component

```javascript
// In your vault shell, after the vault tree is loaded:

const browse = document.createElement('send-browse');

// Create your adapter
const dataSource = new VaultDataSource(vault, accessKey);

// Pass it to Browse
browse.dataSource = dataSource;

// Optional: set these for the header display
browse.fileName    = vault.name || 'Vault';
browse.transferId  = vault.id;
browse.downloadUrl = window.location.href;

// Add to DOM
document.querySelector('#vault-files-panel').appendChild(browse);
```

## VaultDataSource Reference Implementation

```javascript
class VaultDataSource {

    constructor(vaultInstance, accessKey) {
        this._vault     = vaultInstance;
        this._accessKey = accessKey;
        this.writable   = !!accessKey;
    }

    getTree() {
        // Convert vault's tree format to Browse's expected format
        return this._convertNode(this._vault.getTreeRoot(), '');
    }

    _convertNode(vaultNode, prefix) {
        var result = { name: vaultNode.name || '', children: {}, files: [] };

        for (var [name, child] of Object.entries(vaultNode.children || {})) {
            var childPath = prefix ? prefix + '/' + name : name;
            if (child.type === 'folder') {
                result.children[name] = this._convertNode(child, childPath);
            } else {
                result.files.push({
                    path: childPath,
                    name: childPath,
                    size: child.size || 0
                });
            }
        }
        return result;
    }

    async getFileBytes(path) {
        // Split path into folder + filename for vault API
        var parts      = path.split('/');
        var fileName   = parts.pop();
        var folderPath = parts.join('/') || '/';

        // Fetch + decrypt the file
        var blob = await this._vault.getFile(folderPath, fileName, this._accessKey);
        return blob.arrayBuffer();
    }

    getFileList() {
        // Flatten the vault tree into a flat list
        var list = [];
        this._flatten(this._vault.getTreeRoot(), '', list);
        return list;
    }

    _flatten(node, prefix, list) {
        for (var [name, child] of Object.entries(node.children || {})) {
            var path = prefix ? prefix + '/' + name : name;
            if (child.type === 'folder') {
                list.push({ path: path + '/', name: path + '/', dir: true });
                this._flatten(child, path, list);
            } else {
                list.push({ path: path, name: path, dir: false, size: child.size || 0 });
            }
        }
    }

    // Optional: vault doesn't generate zips, so Save downloads individual files
    // getZipBlob() — not implemented (Save button will be hidden or disabled)

    getOrigName() { return this._vault.name || 'vault'; }
    getOrigSize() { return 0; }  // vault doesn't have a single archive size
}
```

## Testing Your Integration

After creating VaultDataSource, test these scenarios:

1. **Tree renders** — folders and files appear in sidebar
2. **Click file → opens in tab** — content previews correctly (try markdown, PDF, image, CSV)
3. **Markdown links** — click a `[link](other-file.md)` → opens as new tab
4. **Markdown images** — `![alt](image.png)` renders inline
5. **Open 8+ files** — tab bar scrolls, new tabs auto-scroll into view
6. **"Locate" button** — reveals file in tree (expands parents, scrolls)
7. **CSV** — renders as table with source toggle
8. **HTML** — renders in iframe, scripts execute, source toggle works
9. **PDF** — previews + Present button opens fullscreen
