## 1. The Goal

Build the missing middle of the video workflow on `tools.sgraph.ai`:

```
SHIPPED                          NEED TO BUILD                    SHIPPED
┌──────────────┐    ┌─────────────────────────────┐    ┌──────────────────┐
│ Video        │ →  │ Multi-Track Video Editor     │ →  │ YouTube Editor   │
│ Recorder     │    │ (crop, titles, redaction,    │    │ (upload, manage) │
│ (v0.1.48)    │    │  combine, timeline)          │    │ (v0.1.0)         │
├──────────────┤    └─────────────────────────────┘    ├──────────────────┤
│ Video Tools  │                                       │ SG/Send Upload   │
│ (FFmpeg WASM │                                       │ (encrypt + share)│
│  split/trim) │                                       │                  │
└──────────────┘                                       └──────────────────┘
```

**The use case driving this:** Create a demo video showing the browser (SG/Send UI) and SGit CLI console working side by side — with titles, transitions, and redacted sensitive areas. Today that requires screen-recording, downloading, editing in iMovie/DaVinci, exporting, re-uploading. The editor should make this a single-tool workflow.

## 3. Architecture: JSON Transformations (Non-Destructive)

Every edit is stored as a JSON transformation object. The original video is **never modified** until export.

```json
[
  { "type": "trim",    "params": { "start": 2.5, "end": 45.0 } },
  { "type": "crop",    "params": { "x": 10, "y": 20, "w": 640, "h": 480,
                                    "keyframes": [{"time": 0, "x": 10}, {"time": 3, "x": 50}] } },
  { "type": "caption", "params": { "text": "SGit push in action", "start": 5.0, "end": 10.0,
                                    "position": "bottom-center", "style": "white-on-black" } },
  { "type": "mask",    "params": { "mode": "blur", "keyframes": [
                                    {"time": 0, "x": 500, "y": 10, "w": 200, "h": 50},
                                    {"time": 3, "x": 510, "y": 15, "w": 200, "h": 50}] } }
]
```

**Benefits:** Undo/redo = remove from array. Share transformations as JSON. LLM can read/write transformations without seeing the video. Apply same edits to different recordings.

## 5. Core Editing Operations

### 5a. Load Multiple Videos
- Drop multiple video files (or load from SG/Send via token)
- Each becomes an asset in the left panel
- Drag onto the Video track to sequence them
- Trim clip edges by dragging on the timeline

### 5b. Crop with Keyframes
- Draw a crop rectangle on the preview (draggable, resizable)
- Set keyframes at different timestamps (crop region can move/resize over time)
- Interpolation between keyframes (hard-cut or smooth)
- Output resizes cropped region to target aspect ratio

### 5c. Titles and Text Overlays
- Add text blocks to the Title track
- Configure: text, font size, colour, position, background, duration
- Drag to position on timeline
- Preset styles: lower-third, centred title card, subtitle

### 5d. Redaction / Masking
- Draw blur/black regions on the preview
- Keyframe the mask position (for camera movement or scrolling content)
- Modes: blur, black-out, pixelate
- Critical for: patient info in medical videos, credentials in terminal recordings, personal data

### 5e. Basic Transitions
- Cut (default — instant switch between clips)
- Crossfade (optional — Canvas alpha blending over N frames)
- No complex transitions in MVP

## 6. Export Pipeline

```
Timeline JSON + Source Videos
        │
        ▼
Canvas 2D composition (frame by frame)
  - Draw video frame (cropped if applicable)
  - Draw mask regions (blur/black)
  - Draw title overlays
  - Composite at 30 FPS
        │
        ▼
MediaRecorder captures Canvas stream + Audio
  → WebM (VP9 + Opus)
        │
        ▼
(Optional) FFmpeg.wasm converts WebM → MP4
  → H.264 + AAC, -movflags faststart
        │
        ▼
Output options:
  [Download]  [Upload to SG/Send]  [Send to YouTube Editor]
```

**Performance target:** ~1x realtime for composition + recording. A 2-minute video takes ~2-3 minutes to export.

## 7. Phasing

### Phase 1 — Timeline + Multi-Clip Sequencing (1–2 sessions)
- Timeline UI with scrubber and playhead
- Load multiple clips as assets
- Drag clips onto Video track, sequence them
- Trim clip edges on timeline
- Preview plays through sequenced clips
- Export concatenated video (Canvas + MediaRecorder)

**Done when:** User can load 3 screen recordings, arrange them on a timeline, trim dead time, and export a single combined video.

### Phase 2 — Titles + Captions (1 session)
- Title track on timeline
- Add text overlays with position, duration, style
- Preset styles: lower-third, centre title, subtitle
- Preview shows titles composited on video
- Export burns titles into output

**Done when:** User can add "SGit Push Demo" as a title card at the start, and "browser → vault → CLI" as a lower-third during the demo.

### Phase 3 — Crop + Redaction (1 session)
- Crop tool with draggable rectangle on preview
- Keyframe system for crop position over time
- Mask tool: draw blur/black regions
- Keyframe mask positions (for scrolling/moving content)
- Preview shows crop + masks composited

**Done when:** User can blur a terminal password that appears at 0:15 and moves down the screen by 0:20, and crop the recording to just the relevant window.

### Phase 4 — Polish + Integration (1 session)
- Crossfade transitions between clips
- Audio track management (mute, volume, swap audio)
- Save project as JSON (transformations + asset references)
- Load project from JSON (resume editing)
- SG/Send integration: save output + project to vault
- YouTube Editor integration: send exported video directly

**Done when:** Full workflow from screen recording → edit → export → YouTube upload works without leaving `tools.sgraph.ai`.

## 8. Technical Notes

- **All client-side.** No server-side video processing.
- **FFmpeg WASM** for codec operations (already loaded/cached by Video Tools).
- **Canvas 2D + requestAnimationFrame** for frame composition and preview.
- **MediaRecorder API** for capturing the composed output.
- **Web Workers** for export (keep UI responsive during rendering).
- **IndexedDB** for project persistence (video files + project JSON survive refresh).
- **Standard JS API** per the tool primitive pattern (every action callable programmatically).
