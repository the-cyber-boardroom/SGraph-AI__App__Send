# QA Brief: Token Counter Zero-State Behaviour (CR-004)

**Version:** v0.16.54
**Date:** 2026-03-23
**From:** Explorer team (Dev)
**To:** QA (agentic team)
**Priority:** Low — documentation only, no code change

---

## API Contract: Token Check

### Endpoint
```
GET /transfers/check_token/{token_name}
```

### Behaviour
- **Lookup only** — does not consume a use
- Returns token validity and remaining uses

### Response Schema
```json
{
    "valid": true,
    "status": "active",
    "remaining": 47
}
```

### Zero-State Behaviour

When `remaining` reaches 0:
1. **API response:** `{ "valid": false, "status": "exhausted", "remaining": 0 }`
2. **UI behaviour:** Access gate shows error message from `I18n.t('access_gate.exhausted')`
3. **Upload attempts:** Will fail with 401 (token is no longer valid for new transfers)

### Token States

| State | `valid` | `status` | `remaining` | UI Behaviour |
|---|---|---|---|---|
| Active | `true` | `"active"` | `> 0` | Gate opens, shows uses remaining in token bar |
| Exhausted | `false` | `"exhausted"` | `0` | Gate shows exhausted error, token rejected |
| Revoked | `false` | `"revoked"` | any | Gate shows revoked error |
| Not found | `false` | `"not_found"` | — | Gate shows not found error |

### Relevant Code

- **API route:** `sgraph_ai_app_send/lambda__user/fast_api/routes/Routes__Transfers.py` — `check_token__token_name()`
- **UI handler:** `send-access-gate.js` — `submitToken()` method, lines 130-167
- **Token bar display:** `send-access-gate.js` — `updateTokenBar()` (v0.2.6 overlay)
- **Calculation:** `remaining = usage_limit - usage_count` (computed server-side)

### Dev Mode
When no admin service is configured (local dev), `check_token` always returns `{ valid: true, status: "active" }` — no remaining count. Token bar shows "Token active" without a number.

## Recommended Test Assertions

```python
# Valid token with remaining uses
response = httpx.get(f"{api_url}/transfers/check_token/{token}")
assert response.json()["valid"] == True
assert response.json()["remaining"] > 0

# Exhausted token
# (requires admin API to set usage_limit == usage_count)
response = httpx.get(f"{api_url}/transfers/check_token/{exhausted_token}")
assert response.json()["valid"] == False
assert response.json()["status"] == "exhausted"
assert response.json()["remaining"] == 0

# Non-existent token
response = httpx.get(f"{api_url}/transfers/check_token/nonexistent")
assert response.json()["valid"] == False
assert response.json()["reason"] == "not_found"
```

**Note:** In the local in-memory test server (no admin service), all tokens return `valid: true`. To test exhaustion/revocation, the QA team needs the admin service running or should mock the admin service client response.
