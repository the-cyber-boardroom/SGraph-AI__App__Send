## New Bugs Fixed

### BRW-001: Folder Tree Shows Full Zip Path Instead of Filename

**Severity:** P2
**Component:** `send-browse` — folder tree sidebar
**Route:** `/en-gb/browse/#<token>` (any zip with subfolders)

**Before (v0.3.0):**
Files inside subfolders showed the full zip path as their name:
```
▸ 📁 4) SG:Send - Integration
    📄 4) SG:Send - Integration/1) How_SG_Send_Fits_Into_Sportsheali...
    🖼 4) SG:Send - Integration/1) How_SG_Send_Fits_Into_Sportsheali...
    📕 4) SG:Send - Integration/1) How_SG_Send_Fits_Into_Sportsheali...
```

**After (v0.3.1):**
Files show only their basename:
```
▸ 📁 4) SG:Send - Integration
    📄 How_SG_Send_Fits_Into_Sportshealing.md
    🖼 How_SG_Send_Fits_Into_Sportshealing.jpg
    📕 How_SG_Send_Fits_Into_Sportshealing.pdf
```

**Test plan:**
- [ ] Upload a zip containing nested folders (at least 2 levels deep)
- [ ] Open in browse view (`/en-gb/browse/`)
- [ ] Expand a subfolder in the sidebar tree
- [ ] **ASSERT:** Files inside the subfolder show only the filename, not the folder path prefix
- [ ] **ASSERT:** Clicking a file still opens it correctly in the content panel
- [ ] **ASSERT:** The file tab shows the correct filename
- [ ] **ASSERT:** "Save" button downloads the file with the correct filename
- [ ] Test with root-level files (no folder) — should be unchanged

**Screenshot impact:** Folder tree sidebar screenshots will change for any zip with subfolders. Mark as **expected change** (good failure).

### BRW-002: PDF Files Missing Present/Fullscreen Mode in Browse View

**Severity:** P2 (regression vs v0.2.13)
**Component:** `send-browse` — file action bar
**Route:** `/en-gb/browse/#<token>` (any zip or single file containing a PDF)

**Before (v0.3.0):**
PDF files in browse view showed: `filename.pdf | size | Save` — no way to view fullscreen.

**After (v0.3.1):**
PDF files in browse view show: `filename.pdf | size | Save | ⛶ Present` — Present button opens PDF in a new window optimized for presentation.

**Behaviour:**
- Present button appears **only** for PDF files (not images, markdown, text, etc.)
- Click opens: `window.open(blobUrl + '#toolbar=1&navpanes=0&view=Fit', '_blank')`
- If popup is blocked: falls back to `iframe.requestFullscreen()`
- Same behaviour as the gallery lightbox Present button

**Test plan:**
- [ ] Upload a PDF file (or a zip containing PDFs)
- [ ] Open in browse view (`/en-gb/browse/`)
- [ ] Open a PDF file in the content panel
- [ ] **ASSERT:** "⛶ Present" button is visible in the action bar
- [ ] Click "⛶ Present"
- [ ] **ASSERT:** PDF opens in a new browser tab/window
- [ ] **ASSERT:** PDF displays in fit-to-page mode (no sidebar, toolbar visible)
- [ ] Open a non-PDF file (image, markdown, text)
- [ ] **ASSERT:** No "⛶ Present" button — only filename, size, Save

**Screenshot impact:** PDF file action bar screenshots will show an additional button. Mark as **expected change** (good failure).
