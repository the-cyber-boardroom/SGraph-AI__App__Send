## What to Test

v0.3.1 is an IFD overlay on v0.3.0. Only the **browse view** (`/en-gb/browse/`) has patches. All other pages are unchanged v0.3.0.

### Test 1: Version Verification

- [ ] `/en-gb/` — footer shows `v0.3.1`
- [ ] `/en-gb/browse/#<token>` — footer shows `v0.3.1`
- [ ] `window.SGRAPH_BUILD.uiVersion` in console → `'v0.3.1'`
- [ ] Network tab shows `send-browse-v031.js`, `send-browse-v031.css`, `markdown-parser-v031.js` loaded

### Test 2: BRW-001 — Folder Tree Basenames

**Test with:** Zip containing nested folders (2+ levels)

- [ ] Files inside subfolders show only the filename, not the full path
- [ ] Example: `folder/sub/report.pdf` shows as `report.pdf` in the tree (not `folder/sub/report.pdf`)
- [ ] Clicking the file still opens it correctly
- [ ] Root-level files are unaffected

### Test 3: BRW-002 — PDF Present Mode

**Test with:** Zip containing at least one PDF

- [ ] PDF file action bar shows: `filename | size | Save | ⛶ Present`
- [ ] Click "Present" → PDF opens in new window/tab
- [ ] New window shows PDF in fit-to-page mode
- [ ] Non-PDF files do NOT show Present button

### Test 4: BRW-003 — Auto-Open First File (Sorted)

**Test with:** Zip with numbered root files (e.g. `0.a) Start.md`, `0.b) Intro.md`, `0.d) Image.jpg`)

- [ ] First load opens `0.a) Start.md` (alphabetically first), not a random file
- [ ] If no root files, opens the first file alphabetically from the first folder

### Test 5: BRW-004 + BRW-008 — Markdown Internal Links

**Test with:** Zip containing markdown with relative links: `[Link text](other-file.md)`, `[Link](folder/doc.pdf)`

- [ ] Clicking a relative link opens the target file as a new tab
- [ ] Link renders as clickable (blue, cursor: pointer)
- [ ] External links (`https://...`) still open in new browser tab (not intercepted)
- [ ] Anchor links (`#section`) are not intercepted

### Test 6: BRW-005 + BRW-007 — Markdown Images from Zip

**Test with:** Zip containing markdown with image references: `![alt](image.png)`, `![](folder/photo.jpg)`

- [ ] Images render inline in the markdown (not as `[image: alt]` text)
- [ ] Images load from the zip (check Network tab — should be blob: URLs, not HTTP requests)
- [ ] Large images respect `max-width: 100%`

### Test 7: BRW-006 — Markdown Source Toggle

- [ ] Markdown files show `</> Source` button in action bar
- [ ] Click toggles to raw markdown text (monospace, light background)
- [ ] Button label changes to `✎ Rendered`
- [ ] Click again toggles back to rendered markdown

### Test 8: BRW-009 — Save Locally (Valid Zip)

- [ ] Click "Save locally" on browse view header
- [ ] Downloaded .zip opens correctly in macOS Archive Utility / Windows Explorer / 7-Zip
- [ ] All files present and intact inside the zip

### Test 9: BRW-010 — Markdown Link Color

- [ ] Links in rendered markdown are blue (#0969da), readable on white background
- [ ] Links show underline on hover

### Test 10: BRW-011 — URL-Encoded Paths (%20)

**Test with:** Zip containing files with spaces in names (e.g. `ACCOUNTANT DEBRIEF.md`) and markdown linking to them with `%20`

- [ ] Link like `[doc](folder/ACCOUNTANT%20DEBRIEF.md)` opens the correct file
- [ ] File with spaces in name found and opened as tab

### Test 11: BRW-012 — CSV Viewer

**Test with:** Zip containing .csv files

- [ ] CSV renders as a styled table (not plain text)
- [ ] Table has sticky header row (stays visible on scroll)
- [ ] Zebra striping on rows (alternating background)
- [ ] Row hover highlighting
- [ ] `</> Source` button shows raw CSV text
- [ ] `✎ Table` button switches back to table view

### Test 12: BRW-013 — HTML Viewer

**Test with:** Zip containing .html files (ideally with JavaScript like Chart.js)

- [ ] HTML renders in iframe (not as source code)
- [ ] JavaScript inside HTML executes (charts render, interactivity works)
- [ ] `</> Source` button shows raw HTML
- [ ] `🎨 Rendered` button switches back to iframe
- [ ] **SECURITY:** iframe cannot access parent page — run in console inside iframe: `parent.document` should throw a cross-origin error

### Test 13: BRW-014 — Folder Links

**Test with:** Markdown with links to folders: `[MBNA](statements/mbna/)`

- [ ] Clicking expands the folder in the sidebar tree
- [ ] Folder scrolls into view
- [ ] First file in that folder opens as a tab
- [ ] Parent folders also expand (if they were collapsed)

### Test 14: BRW-015 — Tab Bar Scroll

**Test with:** Open 8+ files as tabs

- [ ] Tab bar is horizontally scrollable (thin scrollbar visible)
- [ ] Each new tab scrolls into view automatically
- [ ] Can scroll back to see earlier tabs
- [ ] Tabs remain clickable after scrolling

### Test 15: Gallery Folder Rename

**Test with:** Upload a new folder via the v0.3.1 upload page

- [ ] Inside the zip, the metadata folder is named `__gallery__` + 8-char hash (not `_gallery.` + 16-char hash)
- [ ] Gallery view still works with the new naming
- [ ] Existing uploads with old `_gallery.` naming still work (backward compatible)

### Test 16: No Regressions

- [ ] Upload wizard works end-to-end (6 steps)
- [ ] Encryption/decryption works
- [ ] Friendly tokens resolve
- [ ] Gallery view works (still v0.3.0, no patches)
- [ ] Download/auto-download route works
- [ ] Email link works (uses correct share mode URL)
- [ ] "Save locally" on gallery view works
- [ ] All 17 v0.3.0 QA fixes still in effect
