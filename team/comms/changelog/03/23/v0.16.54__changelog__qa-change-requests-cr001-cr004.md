## CR-001: `data-ready` Signal for `networkidle` Resolution (High)

**Problem:** Playwright's `networkidle` wait strategy never resolves because the UI makes background API calls (e.g. `checkToken`).

**Fix:** Added `document.body.setAttribute('data-ready', 'true')` to the init script of **all 7 pages**:
- `en-gb/index.html` (upload)
- `en-gb/download/index.html`
- `en-gb/v/index.html`
- `en-gb/view/index.html`
- `en-gb/gallery/index.html`
- `en-gb/browse/index.html`
- `en-gb/welcome/index.html`

**QA usage:** Replace `networkidle` with:
```python
page.wait_for_selector('body[data-ready]')
```

### Expected Test Impact
- Tests using `networkidle` wait: **may now complete faster** (good)
- No tests should break from this change

## CR-004: Token Counter Zero-State Behaviour (Low — Documentation Only)

**Documented in:** `team/comms/qa/briefs/03/23/v0.16.54__qa-brief__token-counter-zero-state.md`

**Summary of API contract:**
- `GET /transfers/check_token/{token_name}` — lookup only, no usage consumed
- Returns `{ valid: bool, status: string, remaining: int }`
- `remaining = usage_limit - usage_count`
- When `remaining <= 0`, status changes to `"exhausted"` and `valid = false`
- UI shows error message: "This token has been exhausted"
- No code change needed — this is pure documentation for QA assertions

### Expected Test Impact
- No code changes, no test impact
