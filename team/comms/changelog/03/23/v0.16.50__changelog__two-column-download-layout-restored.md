# Changelog: Two-Column Single-File Download Layout Restored

**Version:** v0.16.50
**Date:** 2026-03-23

**Author:** Explorer team (Dev)

## What Changed

The single-file download view was restored to v0.2.x parity — a two-column layout with file metadata, transparency panel, and content preview side-by-side.

### Files Modified

| File | Change |
|---|---|
| `sgraph_ai_app_send__ui__user/v0/v0.3/v0.3.0/js/components/send-download/send-download.js` | Rewrote `_renderSingleFile()` — now creates a two-column layout (metadata left, preview right) instead of bare `<send-viewer>` |
| `sgraph_ai_app_send__ui__user/v0/v0.3/v0.3.0/js/components/send-download/send-viewer.js` | Minor additions to support the two-column integration |

### Behaviour Changes

| Before (v0.3.0 pre-fix) | After (v0.16.50) |
|---|---|
| Single-file download = bare viewer, thin header | Two-column layout: left panel (file info, Save Locally, transparency, timing) + right panel (content preview) |
| No transparency panel | Transparency panel restored (zero-knowledge encryption explanation) |
| No file metadata display | File name, size, type displayed in left panel |
| No share section | Share section restored |

## How to Verify

1. Start local stack: `SGRAPH_API_ENDPOINT=http://127.0.0.1:{port} ./scripts/user__run-locally.sh`
2. Upload a single file → get download link
3. Open download link → should see two-column layout (metadata left, preview right)
4. Verify transparency panel is present
5. Verify Save Locally button works
6. Compare visually against `main.send.sgraph.ai` for v0.2.x parity
