## What This Enables

The S3 + CloudFront infrastructure is a reusable foundation for:

- **Documentation site** -- developer docs, API reference, getting-started guides
- **Landing pages** -- campaign-specific pages, event pages, investor-specific views
- **Investor materials** -- hosted pitch decks, due diligence documents (encrypted via SG/Send, linked from static pages)
- **Product UI hosting** -- the SG/Send user interface could be served from the same CDN, eliminating Lambda as the static file server and reducing Lambda costs further
- **Additional microsites** -- the deploy script accepts any `--site` parameter; deploying a second site is a configuration change, not new infrastructure

Each additional site adds pennies to the monthly cost. The deployment pipeline, CDN, and cache strategy are already built.

## Risk Reduction

| Risk Eliminated | Impact |
|---|---|
| Third-party hosting dependency | No Vercel/Netlify lock-in. No surprise pricing changes. No vendor outages affecting the marketing site. |
| Manual deployment | Fully automated. No SSH, no FTP, no manual S3 console uploads. Push to `main` and it deploys. |
| Rollback complexity | IFD versioning means every release is preserved. Rolling back is a single S3 copy from `releases/{version}/` to `latest/`. |
| CDN vendor lock-in | CloudFront is the current CDN, but the deploy script targets S3. Switching to Cloudflare R2 + Workers or another CDN requires changing the distribution layer, not the deployment pipeline. |
| Cost surprises | S3 + CloudFront for static sites has a predictable, near-zero cost floor. No autoscaling bills, no compute spikes. |
