## 1. Hero Line — The Central Strategic Asset (Doc 309)

"Version-controlled, client-encrypted vaults. For humans and agents."

This is now the most important strategic asset in the project. Everything the Alchemist produces must
lead with this line or reference it as the anchor.

### Why the Alchemist validates this choice

The previous line ("zero-knowledge encrypted file sharing") was technically accurate but commercially
inert. It failed to:
- Differentiate from 1Password, Bitwarden (password vaults with different UX)
- Signal the Git-semantics advantage (version control as differentiator)
- Name AI agents as first-class users

The new line passes all three tests. It is also verifiably true (the platform ships version control,
client-side encryption, vault model, and agent compatibility — all code-verified). The Alchemist can
commit to this line in investor materials without caveat.

### The subhead question (Decision #18)

The three subhead candidates from the brief are all valid for different audiences:

| Candidate | Audience | Recommendation |
|-----------|----------|----------------|
| "Every commit, every share, every agent: encrypted in your browser. The server is blind." | Technical | Strong technical proof. Good for developer-facing landing pages and GitHub README. |
| "Git semantics, browser-grade encryption, multi-agent ready." | Technical/investor | Compact. Good for one-pagers and deck opening slides. |
| "Encrypt locally. Version globally. Share safely. Across humans, CLIs, and AI agents." | Broad | Most complete statement. Good for the investor deck hero slide. |

**Recommendation for investor materials:** Use candidate 3 as the subhead in pitch decks. The "Across
humans, CLIs, and AI agents" ending reinforces the "For humans and agents" positioning without repeating
it verbatim.

## 5. AWS Marketplace Strategy (Doc 307) — Investor Narrative Value

The three-AMI marketplace strategy is not just a distribution channel — it is a revenue and validation
story for investors.

### What investors hear

"We have three AWS Marketplace AMIs. AMI 1 is already in CI. AMI 2 is the same code that runs our
production service. AMI 3 is our tools platform. Customers can run the entire stack in their own AWS
account with one click. We plan to start free, add paid tiers once we understand demand."

This answers the questions investors always ask:
- **Distribution:** AWS Marketplace, Docker Hub, direct deployment
- **Revenue model:** Usage-based (per API call) or subscription, proven by the OpenRouter credits model
- **Enterprise path:** Customer runs it in their VPC, their security controls, their billing
