### 1.3 The Ecosystem Platforms

#### Platform A: tools.sgraph.ai — Canonical Component Library

Three-tier architecture: core (pure JS logic — crypto, LLM, SSH), components (reusable Web Components), tools (standalone pages/applications). Every product imports FROM tools.sgraph.ai. Updates flow to all platforms.

**Investor appeal:** Velocity (one implementation reaches all platforms), consistency (same crypto everywhere), ecosystem (third-party developers can build), monetisation-ready (CDN-served, tiered access possible).

#### Platform B: library.sgraph.ai — Knowledge & Content Infrastructure

Encrypted knowledge library accessible only with proper keys. Use cases: researchers sharing papers, healthcare sharing guidelines, legal sharing precedents, educational sharing course materials. All zero-knowledge.

**Investor appeal:** Infrastructure for decentralised knowledge economies — what blockchain projects (Arweave, Filecoin) tried but failed because they didn't solve access control. SGraph solves it with PKI.

#### Platform C: babel.sgraph.ai — Translation Infrastructure

17 language variants shipped. Community translation model via GitHub. Hash-based caching (only changed segments re-translated).

**Investor appeal:** Localisation as a moat (competitors starting now can't catch up in non-English markets quickly). Potential standalone translation service for other SaaS companies.

## Part 2: POWERFUL IDEAS (What Could Be Built)

### 2.1 The "Friendly Key" Concept (PROPOSED)

**Problem:** Encryption keys are long, unwieldy, impossible to communicate verbally.

**Solution:** Derive full cryptographic key from memorable passphrase via KDF:
```
Current: a7f2b9c4d8e1f3a5b7c9d2e4f6a8b0c3d5e7f9a1b3c5d7e9f1a3b5c7d9e1f3
Friendly: purple-sunset-442
Voice: "The key is purple-sunset-442"
```

**Investor appeal:** Consumer-grade UX, cryptographically sound, differentiates from all competitors, reduces support friction.

### 2.2 Purpose-Built Views for File Types (PROPOSED)

When file type is known, UI becomes the viewer: video player for videos, PDF reader for PDFs, gallery for images, code viewer for code, spreadsheet grid for data.

**Unit economics:** Each viewer is a new market segment. 17 languages × 12 file viewers = 204 unique market angles.

### 2.3 The Collect-Call Billing Model (PROPOSED)

**The idea:** Invert payment — recipient pays, not sender.

**How it works:** Firm creates branded upload portal → clients upload free → firm pays for receiving → files encrypted throughout.

**Use cases:** Law firms receiving evidence, hospitals receiving patient uploads, companies receiving job applications, freelancers delivering work, regulators receiving submissions.

**Investor appeal:** Flips customer acquisition. Every portal is a user acquisition channel. The firm's clients become free users who experience the product.

### 2.4 The Custody Service as Bank Infrastructure (PROPOSED)

Separate, specialised service storing encryption keys with bank-grade security (HSMs). Key escrow, vault integrity checks, multi-location backup, anonymisation services.

**Markets it opens:** Data sovereignty (EU custody = guaranteed EU data), enterprise/regulated (banks, hospitals, law firms), long-term preservation.

**Investor appeal:** Complementary business with subscription economics. A customer using SG/Send free might also pay £10-50/month for custody. Recurring revenue from security-conscious segments.

### 2.5 The LLM Cost Engineering Story (Thought Leadership)

**Thesis:** LLM usage will have the same cost explosion cloud computing had in 2010-2015. SGraph has already discovered systematic cost reduction practices:

| Practice | Result |
|---|---|
| One-shot requests, no history | 10x reduction vs conversation-based |
| Minimum viable context | Send skeleton (1K tokens) vs full HTML (18K tokens) |
| LLM-as-author, not runtime | LLM writes code once (cost), code runs forever (free) |
| Hash caching per content | Only changed segments, cached identical results |

**Investor appeal:** Thought leadership positions SGraph as "the company that solves LLM economics." Attracts founders and engineers who care about cost efficiency.

### 2.6 One-Shot LLM in Product (PROPOSED)

Agent panel where EVERY interaction is a one-shot request. User sees token count BEFORE sending. No surprise bills. Enterprise-safe with token budgets per workspace.

**Investor appeal:** "The most cost-efficient AI workspace in the world."
