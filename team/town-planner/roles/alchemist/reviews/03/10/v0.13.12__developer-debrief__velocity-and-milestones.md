## Six Major Milestones Shipped

### M1: Vault v0.1.1 — Admin Shell Integration (March 3)

**What shipped:** First time the admin console pattern (3-column Light DOM layout) was extracted and reused in another product. Vault now has hierarchical tree navigation, multi-format file preview (markdown, PDF, images, audio, video, code, text), drag-and-drop upload overlay, and full admin shell debug suite.

**Code-verified components:**
- `vault-shell` (Light DOM, 3-column CSS Grid)
- `vault-tree-view` (collapsible folder hierarchy)
- `vault-file-preview` (8 format renderers with security-first sandbox)
- `vault-upload-dropzone` (path-aware drag-and-drop)
- 3 cross-product modules reused (FileTypeDetect, MarkdownParser, JSZip)

**Investor significance:**
- Demonstrates **pattern reusability** — admin shell designed for a single product has already been reused
- **First offline-first demo** — entire product works on localhost
- **Proves zero-knowledge design scales** — no server involvement in file preview pipeline

**Deliverables:** 42 files, 5,575 lines.

---

### M2: First LLM Call from SG/Workspace (March 4)

**What shipped:** Document loaded from encrypted vault → transformed by OpenRouter LLM → streamed response rendered in browser, real-time. Zero server involvement in the LLM pipeline.

**Verified working:**
- **OpenRouter integration** (cloud LLM provider)
- **Ollama local LLM** (Gemma 3B, CORS-configured)
- **Streaming pipelines** — both SSE (OpenRouter) and NDJSON (Ollama) readers
- **Five-stage transformation pipeline** — vault → source → LLM → transform → vault (all client-side)

**Investor significance:**
- **Proof of concept for agentic capabilities** at the edge (client-side)
- **Cost control narrative** — user can run local Ollama, zero API costs
- **Privacy story:** "Your documents never leave your browser. Neither does the AI processing."

## Development Methodology: What Enables This Velocity

### 1. Iterative Feature Development (IFD)

Every feature ships in thin, testable slices. No waiting for "full" implementations. Vault shipped as v0.1.0 (tree browser only), then v0.1.1 (added preview, admin shell, upload). Each iteration is deployed, tested, measured.

**Principles:**
- Flow state over meetings — minimal planning, maximum building
- UX-first — prototype in the browser before designing the architecture
- Real data — test with actual encrypted files, not mock data
- Zero framework dependencies — Web Components, vanilla JS, CSS Grid

### 2. Brief-Driven Development

Every day starts with a brief from Dinis. The team reads it, understands requirements, and ships code that day. No requirements workshops, no design document cycles, no deployment delays.

### 3. Agentic Workflow (Claude Web + Claude Code)

Claude Web is for thinking (research, debate, architecture). Claude Code is for building (implementation, testing, committing). The brief is the handover artifact between them.

### 4. Role-Based Architecture (Wardley Maps)

18 roles with specific lenses on the code. Architect handles patterns, Dev handles shipping, QA handles coverage, Journalist documents, Librarian maintains knowledge. Without this role specialization, velocity would be 5-10x slower.

### 5. Deployed Reality Document

Every feature that ships is documented in the reality document — a code-verified audit. If it's in the reality document, it's in production. If it's not, it doesn't exist.
