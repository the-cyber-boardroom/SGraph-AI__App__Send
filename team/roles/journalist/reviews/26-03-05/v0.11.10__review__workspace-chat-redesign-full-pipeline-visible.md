# Full Pipeline Visible — SG/Workspace Chat Redesign Milestone

**Version:** v0.11.10
**Date:** 5 March 2026
**Role:** Journalist

---

## What Happened

On 5 March 2026, SG/Workspace achieved full-pipeline visibility: a user can load an HTML document, write a transformation script, execute it, see the result, and have an LLM observe and explain the entire cycle — all in one browser view, with zero server involvement.

This is the second major LLM milestone in two days. On 4 March, the workspace made its first LLM call. On 5 March, the entire interaction model was redesigned so the LLM can see what the user sees.

## Why This Matters

### 1. The LLM Can See the Full Pipeline

Previously, the LLM received a prompt and returned text into the Result panel — overwriting the actual transformation output. Now the LLM has its own dedicated output zone and can observe Source, Script, and Result simultaneously. It becomes a collaborator that can explain, critique, and suggest improvements to transformations it can actually see.

### 2. "No Surprises" — What the LLM Sees Is What You See

The auto-appended context preview (bottom of the System Prompt zone) shows exactly what will be sent to the LLM. If Source is checked, you see a truncated preview of the HTML. If Script is checked, you see the JS. Nothing is hidden. This was a deliberate design response to the previous architecture where context was silently appended with no visibility.

### 3. From Monolith to Event-Driven Components

The old `llm-chat.js` was a single 400+ line component handling prompts, model selection, API calls, streaming, and response display. It has been decomposed into 5 focused components:

1. `llm-system-prompt` — system prompt editor + context preview
2. `llm-prompt-input` — model selector, include toggles, prompt textarea
3. `llm-output` — dedicated response display with copy-to buttons
4. `llm-orchestrator` — invisible coordinator wiring events together
5. `js-highlighter` — regex-based syntax highlighting for the script editor

Each component communicates through DOM events. No component knows about any other component's internals.

### 4. JS Syntax Highlighting

The script editor now shows coloured syntax: keywords in purple, strings in green, comments in grey, builtins in yellow. This is done with a lightweight regex tokenizer (`js-highlighter`) and a transparent textarea over a highlighted `<pre>` overlay — zero dependencies, under 100 lines.
