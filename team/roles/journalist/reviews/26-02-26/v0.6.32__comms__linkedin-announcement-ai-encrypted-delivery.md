## What Happened — The Full Pipeline

This is the first-ever AI-generated, encrypted, email-delivered document through the SGraph Send zero-knowledge pipeline. The complete flow:

1. **Claude.ai Sherpa Agent** generates a PDF document describing its own workflow
2. **SGMETA envelope** wraps the file with metadata
3. **AES-256-GCM encryption** happens client-side via the SGraph Send MCP connector
4. **Upload** to SGraph Send — server stores only ciphertext
5. **n8n workflow** triggered via MCP to compose and send the email
6. **AWS WorkMail** delivers the email with the download link to the recipient
7. **Browser decryption** — recipient clicks the link, browser decrypts locally, PDF renders inline

The decryption key travelled in the URL fragment (after `#`), which browsers never send to servers. The key was generated by the AI agent's MCP connector session and shared only through the download link. The server at send.sgraph.ai never possessed the key, the email infrastructure never possessed the key, and n8n never possessed the key.

## Key Messaging Themes

### 1. AI Agents as Autonomous Secure Communicators

The post demonstrates that an AI agent can independently orchestrate a secure communication pipeline — generating content, encrypting it, uploading it, and delivering it via email — without human intervention in any step. This is not "AI helping a human encrypt a file." This is "AI encrypting and delivering a file on its own."

### 2. Zero-Knowledge Survives Complexity

The pipeline involves five separate systems (Claude.ai, SGraph Send, n8n, AWS WorkMail, the recipient's browser), yet the zero-knowledge guarantee is maintained across all of them. Adding more systems to the chain did not degrade the security property. This is architecturally significant — it means the zero-knowledge guarantee is composable and does not depend on every system in the chain being trusted.

### 3. MCP as the Integration Layer

The post highlights two MCP connectors working in concert: the SGraph Send MCP connector for encryption/upload and the n8n MCP connector for email delivery. This positions MCP (Model Context Protocol) as the universal integration layer for AI agent capabilities. The message to the developer audience: if you expose your service as an MCP tool, AI agents can use it — including for security-critical operations.

### 4. Orchestration, Not Just Tool Use

The 25 Feb milestone was "Claude sends its first encrypted message." The 26 Feb milestone is "Claude orchestrates an entire delivery pipeline." The jump from single-tool use to multi-tool orchestration (encrypt + email) represents a qualitative shift in what AI agents can accomplish with security infrastructure.

### 5. Email as a Delivery Channel for Zero-Knowledge Content

The email itself contains no sensitive data — just a download link. The actual content is encrypted on SGraph Send's servers. This means email, a fundamentally insecure channel, becomes a safe notification mechanism for zero-knowledge content. The security model does not depend on email security.

## AppSec Verification Notes

The following claims in this document require AppSec verification before external publication:

1. **"Decryption key travels only in the URL fragment"** — Verify that the n8n email template does not parse or log the URL fragment.
2. **"n8n connector never sees file content"** — Verify that the n8n MCP connector only receives the download URL, not the file data or decryption key.
3. **"Five systems, zero saw plaintext"** — Verify each system in the chain: Claude.ai session (sees plaintext during generation), SGraph Send MCP connector (encrypts before upload), SGraph Send server (stores ciphertext), n8n (passes URL only), AWS WorkMail (carries URL only). Note: Claude.ai itself sees the plaintext during generation — the zero-knowledge property applies to the transport and storage systems, not the originating AI.
4. **"AWS WorkMail delivers the email"** — Verify that the email body contains only the download URL and no file metadata that could leak information.

**Important clarification for all communications:** The AI agent (Claude.ai) generates the plaintext content and therefore has access to it. The zero-knowledge property applies to all systems downstream of the encryption step — the server, the email infrastructure, and the network. This distinction should be stated clearly in any technical communication to maintain credibility with security-literate audiences.
