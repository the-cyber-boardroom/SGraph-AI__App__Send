## The Story

Today, Claude — Anthropic's AI assistant — generated a PDF document, encrypted it, and sent it to a human through SGraph Send. The PDF was titled **"Hello from Claude.ai Web!"** and contained a self-describing message:

> *"This PDF was generated, encrypted with AES-256-GCM, and uploaded to SG/Send — all from a Claude.ai conversation using the SG/Send MCP connector. The server never saw the plaintext. Only the recipient with the decryption key can read this."*

The server that transmitted it never saw those words. Neither did Claude's MCP transport layer. The only place the plaintext existed was in Claude's session and in the recipient's browser — where it was decrypted at 04:41 UTC and rendered as a clean, readable PDF.

This is the first time an AI agent has participated in a zero-knowledge encrypted communication channel with a human through a production system.

### The Proof

The recipient's browser displayed:
- **Green banner:** "File decrypted and saved successfully."
- **PDF rendered inline** with the full document visible
- **Metadata panel:** 1.9 KB, AES-256-GCM encryption, uploaded 04:34 UTC
- **File name:** "Encrypted (only you can read it)" — the server never knew it was a PDF
- **File content:** "Encrypted (only you can read it)" — the server stored only ciphertext

The fact that Claude generated a *structured PDF document* (not just raw text) makes the demo even more compelling. The AI created a rich format document, encrypted it, and a human's browser reconstructed the original PDF perfectly — all without the server touching any plaintext.

## How It Works (The 30-Second Explanation)

1. Claude calls an MCP tool: "create a transfer"
2. Claude encrypts the content and uploads it
3. SGraph Send stores the ciphertext — it can't read it
4. Claude generates a download link with the decryption key in the URL fragment
5. The human clicks the link — their browser decrypts it locally
6. Nobody in between — not the server, not the network, not the ISP — ever sees the plaintext

The decryption key never leaves the URL fragment (the part after `#`), which browsers don't send to servers. It's the same principle as end-to-end encrypted messaging, but between an AI and a human.

## Why It Matters

### For Users
AI assistants can now send you sensitive information — passwords, keys, confidential documents — through a channel where the AI platform itself can't read what was sent. Today's AI assistants paste sensitive content directly into chat windows that are logged, analysed, and stored. SGraph Send offers an alternative: the AI sends you a link, and only you can open it.

### For Developers
Any AI tool that supports MCP (Model Context Protocol) can now integrate zero-knowledge encrypted file transfers. No cryptography expertise required — the MCP tools handle encryption. The developer just calls `transfer_create`, `transfer_upload`, and gets back a secure download URL.

### For the Industry
This demonstrates that AI agents can be first-class participants in encrypted communication infrastructure — not just tools that humans use, but active agents that initiate and participate in secure exchanges. As AI agents take on more autonomous roles, they need the same security infrastructure humans have. Today, they got it.
