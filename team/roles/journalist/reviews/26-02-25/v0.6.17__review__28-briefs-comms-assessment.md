## 1. Messaging Frameworks Needed

### Zero Lock-In (Brief #27)

Core message: "We make it easy. We never make it the only way." This is the platform's philosophical differentiator and must be embedded in every piece of copy. Three angles:

- **For developers:** "Generate keys here, use them anywhere. GPG, AGE, OpenSSL -- your tools, your choice."
- **For business users:** "Your files are never trapped. Export everything, anytime, in standard formats."
- **For investors/partners:** "Usage is the metric, not revenue. Every encrypted message grows the ecosystem -- even ones that never touch our servers."

The "Documentation Test" from brief #27 is itself a messaging tool: every feature page should show both the SG/Send path and the "without us" path. This builds trust and differentiates from every competitor that hides the exit door.

### "Send to Any GitHub User" (Brief #23)

Headline: **"Encrypt a file for any GitHub user. They already have the keys."**

Supporting copy: GitHub publishes SSH and GPG keys for every user. SG/Send fetches the key, encrypts client-side, and delivers. The recipient decrypts with their existing SSH key. No signup. No new tools. No new accounts.

Campaign hook: tie this to the signed commits narrative. "If your AI agent can commit to your repo, how do you prove which commits are yours? Signed commits. And those same GPG keys let you receive encrypted files."

### Demo Level Copy (Brief #9)

Each demo level needs a short contextual card (2-3 sentences max):

| Level | Card Copy |
|-------|-----------|
| L1 | "Type the code. See the file. That is encrypted file transfer -- 30 seconds, no signup." |
| L2 | "A shared folder, multiple files, all encrypted. Browse and decrypt at your own pace." |
| L3 | "A branded data room. Logo, identity, organised folders. This is what your clients see." |
| L4 | "Watch an NDA exchange: upload, sign, countersign. Real workflow, demo data." |
| L5 | "Live agents exchanging files right now. Watch the audit trail update in real time." |

## 3. Campaign Ideas

### Campaign 1: "Send to Any GitHub User"

- **Format:** Developer blog post + Twitter/X thread + Hacker News submission.
- **Hook:** "There are 100M+ GitHub accounts with published SSH keys. Every one of them can receive encrypted files today."
- **CTA:** Try it at send.sgraph.ai -- enter a GitHub username, drop a file.
- **Timing:** Launch after Phase 3 of brief #23 (GitHub username lookup in UI).

### Campaign 2: "Sign Your Commits" (from Brief #23)

- **Format:** Short explainer post + social thread.
- **Hook:** "In the age of AI agents committing code, how do you prove a commit is yours? GPG-signed commits. Here is how to set it up in 5 minutes."
- **CTA:** Generate your GPG key at send.sgraph.ai/tools, then enable signed commits.
- **Timing:** Can run independently, no product dependency.

### Campaign 3: "The 30-Second Demo" (from Brief #9)

- **Format:** WhatsApp-shareable link with embedded short code.
- **Hook:** "Open this link. Type this code. See the file. That is zero-knowledge encryption."
- **CTA:** The demo itself is the CTA. Level 1 leads to Level 2.
- **Timing:** After demo infrastructure is built.

### Campaign 4: "Zero Lock-In Guarantee" (from Brief #27)

- **Format:** Landing page + comparison table.
- **Hook:** "We document how to leave us. Every feature page shows how to do it without SG/Send."
- **CTA:** Read the docs, verify the claim, then decide if you want to stay.
- **Timing:** Evergreen -- publish once, link from every feature page.

## 5. Key Differentiator Messaging vs. Competitors

| Competitor | Their Positioning | Our Counter |
|------------|------------------|-------------|
| **WeTransfer** | "Simple file sharing" | "Simple file sharing -- but encrypted. The server never sees your data." |
| **Tresorit** | "End-to-end encrypted cloud" | "End-to-end encrypted -- and zero lock-in. Export your keys, use GPG, leave anytime." |
| **ProtonMail/Drive** | "Swiss privacy" | "Client-side encryption like Proton -- plus send to any GitHub user without them signing up." |
| **Firefox Send** (defunct) | "Simple encrypted transfer" | "Firefox Send died. We are open source, self-hostable, and not going anywhere." |
| **OnionShare** | "Tor-based file sharing" | "No Tor required. Works in any browser. Same zero-knowledge principle, mainstream UX." |

**The unique angle no competitor has:** "Send encrypted files to anyone with a GitHub account. They already have the keys. No signup, no new tools, no lock-in." This is the one-liner that separates SG/Send from every alternative.

## Recommendations

1. **Lead with the GitHub story** -- it is the most shareable, most differentiating message.
2. **Make "Zero Lock-In" a brand pillar**, not just a feature. Document the exit path on every page.
3. **Demo Level 1 is marketing content**, not just product. Treat the 30-second demo as the top-of-funnel asset.
4. **Write all copy mobile-first** -- WhatsApp sharing is a primary distribution channel.
