## What Happened

Four commands in a terminal. That's all it took:

```bash
sg-send-cli clone "test-vault-1:7f2f5e54"
cd 7f2f5e54
sg-send-cli status
sg-send-cli pull
```

The CLI downloaded every file from an encrypted vault on send.sgraph.ai, decrypted them locally, and wrote them to disk. Then `sg-send-cli pull` reported "Already up to date." — exactly like `git pull` on a clean repository.

The same vault. The same files. Now in a Finder window and open in Cursor IDE. Including `transform__transformed__2026-03-04T00-52-58.html` — the output from this morning's LLM transformation session in the Workspace.

## Three Ways to Open One Vault

As of today, SG/Send has three completely independent clients accessing the same encrypted vault:

| How | Where | What For |
|-----|-------|----------|
| **Browser** (Vault UI) | Safari at `dev.vault.sgraph.ai` | Browse, edit, manage |
| **Browser** (Workspace) | Chrome at `dev.workspace.sgraph.ai` | Transform with AI |
| **Terminal** (CLI) | Any shell, any machine | Clone, pull, offline work |

All three use the same vault key. All three independently derive the same AES-256-GCM encryption keys. All three independently decrypt the same encrypted blobs. The server doesn't know which is which. It doesn't care.

## Why This Matters

### Git for Encrypted Storage

Developers don't need to learn a new mental model. `clone` downloads everything. `pull` syncs. `status` shows state. The vocabulary is git. The semantics are git. The files are encrypted.

This means encrypted vaults can slot into existing developer workflows:
- Clone a vault into a project directory
- Edit files in VS Code, Cursor, Vim — whatever
- Push changes back to the encrypted vault
- CI/CD pipelines can clone, process, and store results

### Your First Offline Copy

Until today, vault content only existed in browsers or on the server (encrypted). Now it exists on a local hard drive, decrypted, as normal files. Close your laptop lid, get on a plane, and your vault files are right there in your filesystem.

### The Same Fix Unlocked Everything

One technical change — deterministic key derivation using a 2-part vault key — unlocked three milestones on the same day:

1. **M-009:** Cross-browser vault access (Safari + Chrome)
2. **M-010:** CLI vault clone (terminal)
3. The full round-trip: Edit → Transform → Save → Clone → View locally

The 2-part key format (`passphrase:vaultId`) means any client that implements the derivation algorithm can access any vault. No registration. No handshake. No server involvement. Just math.

## The Full Round-Trip

Here's the journey of `test-page.html` today:

Five steps. Three browsers/apps. One vault. Zero plaintext on the server throughout.
