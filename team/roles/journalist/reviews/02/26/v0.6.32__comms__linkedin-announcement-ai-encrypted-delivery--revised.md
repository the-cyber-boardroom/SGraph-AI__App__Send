## What Happened — The Full Pipeline

This is the first-ever AI-generated, encrypted, email-delivered document through the SGraph Send zero-knowledge pipeline. The complete flow:

1. **Claude.ai Sherpa Agent** generates a PDF document describing its own workflow *(Claude has access to the plaintext at this stage)*
2. **SGMETA envelope** wraps the file with metadata (filename preserved client-side only)
3. **AES-256-GCM encryption** happens via the SGraph Send MCP connector before upload *(from this point forward, all systems see only ciphertext)*
4. **Upload** to SGraph Send — server stores only ciphertext
5. **n8n workflow** triggered via MCP to compose and send the email (n8n receives only the download URL)
6. **AWS WorkMail** delivers the email with the download link to the recipient (email contains only the URL)
7. **Browser decryption** — recipient clicks the link, browser decrypts locally, PDF renders inline

The decryption key travelled in the URL fragment (after `#`), which browsers never send to servers. The key was generated during the MCP connector session and shared only through the download link. The server at send.sgraph.ai never possessed the key, the email infrastructure never possessed the key, and n8n never possessed the key.

**Zero-knowledge boundary:** The zero-knowledge property applies to all transport and storage systems (steps 3-7). The originating AI agent (step 1) generates the plaintext and has access to it. This is architecturally equivalent to a human author who writes a document and then encrypts it — the author knows the content, but the delivery infrastructure does not.

## Key Messaging Themes

### 1. AI Agents as Autonomous Secure Communicators

The post demonstrates that an AI agent can independently orchestrate a secure communication pipeline — generating content, encrypting it, uploading it, and delivering it via email — without human intervention in any step. This is not "AI helping a human encrypt a file." This is "AI encrypting and delivering a file on its own."

### 2. Zero-Knowledge Survives Complexity

The pipeline involves five separate systems (Claude.ai, SGraph Send, n8n, AWS WorkMail, the recipient's browser). The zero-knowledge guarantee is maintained across all transport and storage systems downstream of the encryption step. **Clarification:** Claude.ai generates the plaintext content and therefore has access to it prior to encryption — the zero-knowledge property applies to the delivery infrastructure (SGraph Send server, n8n, AWS WorkMail, the network), not to the content originator. Adding more systems to the delivery chain did not degrade the security property. This is architecturally significant — it means the zero-knowledge guarantee is composable and does not depend on every system in the chain being trusted.

### 3. MCP as the Integration Layer

The post highlights two MCP connectors working in concert: the SGraph Send MCP connector for encryption/upload and the n8n MCP connector for email delivery. This positions MCP (Model Context Protocol) as the universal integration layer for AI agent capabilities. The message to the developer audience: if you expose your service as an MCP tool, AI agents can use it — including for security-critical operations.

### 4. Orchestration, Not Just Tool Use

The 25 Feb milestone was "Claude sends its first encrypted message." The 26 Feb milestone is "Claude orchestrates an entire delivery pipeline." The jump from single-tool use to multi-tool orchestration (encrypt + email) represents a qualitative shift in what AI agents can accomplish with security infrastructure.

### 5. Email as a Delivery Channel for Zero-Knowledge Content

The email itself contains no sensitive data — just a download link. The actual content is encrypted on SGraph Send's servers. This means email, a fundamentally insecure channel, becomes a safe notification mechanism for zero-knowledge content. The security model does not depend on email security.
