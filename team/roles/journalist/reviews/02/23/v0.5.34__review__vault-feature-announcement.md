## Headline

**SG/Send Personal Data Vault: Edit, Preview, and Share Encrypted Files — Without Ever Decrypting on the Server**

---

## The Story

The SG/Send admin console now includes a full-featured Personal Data Vault — an encrypted file manager where every operation happens client-side. Files are encrypted before upload, decrypted only in your browser for preview or editing, and re-encrypted before save. The server never sees plaintext. Not during upload, not during edit, not during share.

This is what zero-knowledge actually looks like: not just encrypted storage, but an encrypted workflow.

---

## What's New

### Edit Files In-Place
Open any text or markdown file directly in the vault. The built-in editor decrypts the file in your browser, lets you edit with a proper code editor (monospace, tab support, line count), and re-encrypts when you save. For markdown files, toggle between editor and live preview.

### Preview Without Downloading
Select a file and see its contents in the preview pane. Text files render with syntax highlighting. Markdown renders as formatted HTML. Images display inline. No download required — decryption happens in the browser, preview renders in memory.

### Share via Send Link
Any file in your vault can be shared as a Send link with one click. The vault decrypts the file, creates a new transfer, and generates a shareable URL with the decryption key in the URL fragment. The recipient gets the file; the server never sees the content.

### New File Creation
Create text or markdown files directly in the vault. Start from blank, type your content, and save — the file is encrypted and stored without ever touching the server as plaintext.

### Redesigned Layout
The vault now uses a clean 3-row layout: file table on top, action bar in the middle, preview or editor at the bottom. Resize by dragging. Sort by clicking column headers. Rename by double-clicking. Delete with confirmation.

## Technical Highlights for Developers

- **AES-256-GCM** encryption via Web Crypto API (hardware-accelerated on modern browsers)
- **PKI key selection** — choose which identity key to use for your vault session
- **Event-driven architecture** — components communicate via CustomEvent, no tight coupling
- **IFD-compliant** — all files versioned under v0.1.6, no framework dependencies
- **ResizeObserver** for dynamic layout — editor height recalculates on window resize

---

## What This Means for the Product

The vault transforms SG/Send from a file transfer tool into an encrypted workspace. Users can now:
1. **Store** sensitive files with zero-knowledge encryption
2. **Browse** and preview without downloading
3. **Edit** text and markdown in-place
4. **Share** individual files via Send links

This is the foundation for encrypted data rooms — the next major product milestone.
