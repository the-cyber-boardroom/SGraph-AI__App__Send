## Milestone Summary

SGraph AI now has its own public-facing marketing website at static.sgraph.ai, deployed on the project's own infrastructure (S3 + CloudFront) through a fully automated CI/CD pipeline. The site went from brief to live deployment in a single day across two sessions. It includes 8 pages -- a landing page, product pages, and individual agent profile pages -- built as IFD-versioned releases with zero external dependencies. No third-party fonts, no external CDN, no framework. The site was designed, implemented, and deployed entirely by AI agents working within the project's established team structure. This is the first public-facing website deployed through the project's own CI/CD pipeline, and it demonstrates end-to-end infrastructure ownership: the team that builds zero-knowledge encryption also stands up its own CDN.

## Key Talking Points

**1. Fully automated deployment pipeline.**
The entire deployment is automated through GitHub Actions. Push to the repository and the site deploys: S3 sync with per-file-type cache control, CloudFront invalidation, smoke test verification. No manual steps after the initial infrastructure setup. Two sessions took the site from brief to production. The same CI/CD discipline that ships SG/Send now ships the marketing site.

**2. Zero external dependencies -- self-hosted everything.**
Fonts are vendored as local woff2 files. No Google Fonts CDN, no external JavaScript libraries, no third-party analytics, no tracking scripts. The HTML, CSS, and JS are vanilla -- zero framework dependencies. The site loads from SGraph's own CloudFront distribution. If every third-party service went offline tomorrow, static.sgraph.ai would still work. This is a deliberate security and privacy decision, consistent with the zero-knowledge philosophy of the product.

**3. IFD versioned releases.**
Every deployment is versioned, immutable, and traceable. Each release is stored at a versioned path using IFD nested versioning. The `latest/` path points to the current release. Previous releases are preserved indefinitely. Rolling back is a single S3 copy operation. This is the same versioning methodology used for the SG/Send application UI -- no special tooling, no separate process.

**4. Built entirely by AI agents.**
The website -- HTML, CSS, deployment scripts, CI workflow, CloudFront URL rewrite function, cache-control strategy -- was created by AI agents operating within the SGraph team structure (Explorer, Villager, Town Planner roles). The human stakeholder provided the brief and made deployment decisions. The agents did the implementation. This is a concrete proof point for the project's agentic development model: not a demo, not a prototype, but a production deployment.

**5. 8 pages including agent profile pages.**
The site launched with a homepage, product page, agents directory, architecture overview, contact page, 404 page, and individual agent profiles. The agent profile pages are a deliberate choice -- SGraph's AI team members are part of the project's public identity, not hidden behind a generic "AI-powered" label. Each profile describes the agent's role and contributions.

## What NOT to Say

These are accuracy guardrails. Do not use the following in any external communication.

| Do NOT Say | Why |
|---|---|
| "Self-updating website" | The site is static HTML. Updates require a new deployment through CI/CD. It does not update itself. |
| "Auto-scaling infrastructure" | S3 + CloudFront scales inherently with traffic, but calling it "auto-scaling" implies dynamic compute that does not exist. There is no compute. It is static files on a CDN. |
| "AI-generated content that updates in real-time" | The content was generated by AI agents during development, but it is static once deployed. There is no real-time generation or dynamic content. |
| "The SGraph AI website launched" (without qualification) | This phrasing implies the company launched. Be specific: the marketing/information site at static.sgraph.ai launched. The product (SG/Send) was already live at send.sgraph.ai. |
| "Fully autonomous deployment" | The human approved the deployment and made infrastructure decisions (bucket names, CloudFront distribution IDs, DNS). The pipeline is automated; the strategic decisions were human. |
| "No servers" (as a feature claim) | Technically accurate (it is serverless/static), but misleading to a non-technical audience. Say "static site on CDN" or "runs on S3 and CloudFront" instead. |
