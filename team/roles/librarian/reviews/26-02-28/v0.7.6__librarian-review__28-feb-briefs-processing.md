**CRITICAL -- Google Fonts violation:** Every HTML page in `sgraph_ai__website/` loads fonts from `https://fonts.googleapis.com`. This directly violates the zero-external-dependencies requirement declared as CRITICAL in the v0.7.6 daily brief and arch brief. The fonts used are DM Sans and JetBrains Mono.

**Files affected:**
- `sgraph_ai__website/index.html` (line 8-10)
- `sgraph_ai__website/product/index.html` (line 9-11)
- `sgraph_ai__website/architecture/index.html` (line 8-10)
- `sgraph_ai__website/contact/index.html` (line 8-10)
- `sgraph_ai__website/agents/index.html` (line 9-11)

- `sgraph_ai__website/agents/ambassador/index.html` (line 9-11)
- `sgraph_ai__website/agents/architect/index.html` (line 9-11)

**Absolute path violations:** All navigation links use absolute paths (`href="/"`, `href="/product/"`, etc.) instead of relative paths. The arch brief (v0.7.6) explicitly states: "All links, assets, scripts, and stylesheets use relative paths. No absolute URLs. No hardcoded domains." This will break versioned deployments and offline packaging.

**Existing send.sgraph.ai UI also affected:** The Google Fonts issue also exists in:
- `sgraph_ai_app_send__ui__user/v0/v0.1/v0.1.8/` (index.html, download.html, room.html, join.html)
- `sgraph_ai_app_send__ui__user/v0/v0.1/v0.1.7/vault.html`

- `sgraph_ai_app_send__ui__admin/v0/v0.1/v0.1.4/css/sg-brand.css`

**Deploy workflow notes:** The `deploy-website.yml` is a GitHub Actions YAML workflow that syncs files directly to S3 using `aws s3 sync`. The v0.7.6 arch brief states CI pipeline logic should be in Python scripts invoked from GitHub Actions, not YAML logic. The current workflow uses YAML-based logic. This is functional but deviates from the target architecture.
