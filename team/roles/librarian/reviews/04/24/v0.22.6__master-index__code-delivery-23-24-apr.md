### Stream 1: `sg-site-header` Web Component (v1.0.4 → v1.0.5 → v1.0.6)

| Commit | Date (UTC) | Description |
|--------|-----------|-------------|
| `944c538` | 2026-04-23 21:26 | feat(website): sg-site-header v1.0.4 — replaces inline headers |
| `d80e167` | 2026-04-23 23:04 | feat(website): bake token bar into sg-site-header v1.0.4 Shadow DOM |
| `10d9e2e` | 2026-04-23 23:18 | fix(website): prevent layout shift before sg-site-header upgrades |
| `a0279b5` | 2026-04-23 23:35 | feat(website): sg-site-header v1.0.5 — self-configuring site profiles |
| `4ab1bbb` | 2026-04-24 01:39 | feat(header): v1.0.6 — environment-aware cross-site nav links |

**What shipped:**
- **v1.0.4:** Shadow DOM Web Component replacing all 5 inline `<header>` elements on sgraph.ai. HTML/CSS from Aurora dark theme. Slot-based token bar (`slot='cta'`) and locale picker (`slot='locale'`). Per nav-item `external:true` (target=_blank) and `active:true` (active class). Outside-click handler for mobile nav.
- **v1.0.4 token bar patch:** Token bar baked into Shadow DOM — visitor with a share token can open vault directly from any page.
- **v1.0.4 layout fix:** `:not(:defined)` height reservation on `sg-site-header` in `style.css` prevents layout shift before the component upgrades (FOUC mitigation).
- **v1.0.5:** Self-configuring site profiles. `SITE_CONFIGS` object contains complete nav config for Send and Tools. `HOST_SITE_MAP` maps hostnames to site names. Just `site="Send"` → full nav, token bar, home URL auto-configured. Active nav link auto-detected from `window.location.pathname`. All 5 pages simplified from 8-line header markup to 3 lines. Backward-compatible: explicit attributes still override.
- **v1.0.6:** Environment-aware cross-site navigation. `ENV_PREFIX` derived from `window.location.hostname` at module load: `dev.sgraph.ai` → `dev.`, `main.sgraph.ai` → `main.`, `send.sgraph.ai` → `''`. `xsite()` helper applies prefix to cross-site URLs so dev/main environments link to their equivalents. `HOST_SITE_MAP` extended for `dev.*` and `main.*` variants. Cross-site sgraph.ai navigation no longer opens in a new tab.

**Path:** `sgraph_ai__website/v0/v0.2/v0.2.0/_common/js/components/sg-site-header/v1/v1.0/{v1.0.4,v1.0.5,v1.0.6}/`
**Files per version:** `sg-site-header.js`, `sg-site-header.css`, `sg-site-header.html` (+ `manifest.json` on v1.0.4 and v1.0.5)

**Critical fix (`d1fb32e`):** All 10 components now declare `static jsUrl = import.meta.url`. Without this, `SgComponent` resolves `.html`/`.css` siblings relative to its own install location (tools.sgraph.ai), causing HTTP 403 errors on all templates. Every subclass must declare this field.

`style.css` gains `:not(:defined)` height reservations for all 10 components to prevent layout shift on first paint.
