## Thematic Grouping

### Theme 1: Platform Composability — Send/Receive JS API (Doc 302)

The missing plumbing for the tool ecosystem. Every tool that produces output (video recorder, infographic generator, PlaybookLM) needs a single-call way to save and share that output. The API answers it:

```javascript
const token = await SgSend.sendFile(file)
const file  = await SgSend.receiveFile(token)
```

Web Components (`<sg-send-drop>`, `<sg-send-receive>`, `<sg-send-panel>`) provide drop-in UI for tools. Token history in localStorage bridges the gap until the auth MVP (`<sg-vault-picker>`) ships.

**All items are PROPOSED — no code exists yet.**
**Roles:** Dev (lead), Designer

### Theme 2: Video Editing Suite Extension (Doc 303)

Three tools extending the existing video editor:
1. **Dynamic crop with keyframes** — timeline-based crop regions that shift over time; solves the sensitive-screen-recording redaction use case.
2. **Multi-track overlay** — title cards, lower-thirds, captions burned into the output.
3. **Video stream capture** — Option A (screen recording + crop, works today using existing recorder) recommended for MVP; Option B (`captureStream()`) is the cleaner future path.

All tools save output via `SgSend.sendFile()` and are deployable on tools.sgraph.ai.

**All items are PROPOSED — no code exists yet.**
**Roles:** Dev (lead), Designer

### Theme 3: QA Architecture Overhaul — Three Playwright Modes (Doc 304)

A significant architectural proposal for the QA codebase. All Playwright usage goes through a single `PlaywrightClient(mode=...)` abstraction, routing to:
- **Mode 1 (local Docker):** dev/CI, no MITM proxy
- **Mode 2 (Lambda):** serverless, scalable, no authenticated upstream proxy
- **Mode 3 (EC2+MITM sidecar):** full capability, 100% traffic observability, authenticated proxy support

After the refactoring: hourly traffic generation as a synthetic heartbeat, feeding observability dashboards with files/vaults/response-time data. The AMI packages both the QA infrastructure and the sellable Playwright service.

**All items are PROPOSED — no code exists yet.**
**Roles:** QA (lead), Dev, DevOps
