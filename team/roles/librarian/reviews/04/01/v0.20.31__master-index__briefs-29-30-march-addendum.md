### Theme 1: Browse View as Document Platform (Documents 188, 189, 191, 193, 197)

The most significant theme across this batch. By the end of 28 March the Browse view had crossed a threshold from file viewer to publishing platform. The concept of **fractal vaults** (vaults inside vaults, navigable in the same Browse interface) emerges from this.

### Theme 4: IFD Surgical Overrides + Version-Stamped Filenames (Document 198)

This IFD spec formalises a naming convention that was used informally before:

- **Single dash** (`sg-tree-events.js`): part of component name
- **Double dash** (`sg-tree--v0.1.1.js`): version stamp (surgical override)
- Base class exposed via `window.SgTree = SgTree` after `customElements.define()` — allows surgical overrides to patch methods via prototype
- Loading order guarantee: ES modules are deferred, so all scripts execute before `connectedCallback` fires
- Consolidation at major version: v0.2.0 merges all v0.1.x patches into a clean single file

**Reality status:** This is a **specification** for a pattern that applies to `sg-tree` and future components. The pattern itself is architectural guidance, not code. sg-tree v0.1.0 (which would use this) is **PROPOSED — does not exist yet.**
