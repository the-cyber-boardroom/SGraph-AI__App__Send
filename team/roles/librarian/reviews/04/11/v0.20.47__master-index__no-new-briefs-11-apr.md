### Missed Commit 3 — Vault Fix: No Spurious Write on Read-Only Open

| Field | Value |
|-------|-------|
| **Commit** | `086d00c` |
| **Session** | `01MzzVzXBWitot9wWGhu8nVK` |
| **Type** | Bug fix |

`vault._writeKey` is always derived from the passphrase (not the API access token), so the `if (vault._writeKey)` guard in `open()` was always true — causing a **401 PUT** when opening a vault without an access key. Clone ref is now created lazily on the first commit, which requires write access anyway. Read-only vault opens no longer attempt any writes.
