### Theme 2: Firefox Browser Plugin — First Real Plugin Validation

Doc 335 adds Firefox + MITM proxy as a distinct plugin under `plugins/firefox/`. Key distinction from Playwright:

| Dimension | Playwright | Firefox |
|-----------|-----------|---------|
| Interaction | Programmatic (API) | Interactive (human) |
| MITM role | Capture + audit | Capture + intercept/modify |
| Session type | Stateless | Stateful |
| Authentication | API key | Username/password |

This is framed as the **first real test of the plugin architecture** from Day 48. Success validates the pattern. The brief specifies Firefox-specific UI, vault storage for session data, AMI bake support, health checks, and inline iframe browser view.

**All features are PROPOSED — does not exist yet.**

### Theme 3: SGit Four-Layer Refactoring — PKI Readiness (Villager brief)

Doc 336 is the first **Villager brief** in this batch — explicitly about hardening, not adding features. The four-layer model:

```
Crypto (Layer 1) — pure functions, no deps, heavy review cadence
Core (Layer 2) — all state-changing actions as typed steps
Network (Layer 3) — API client layer, mockable
Plugins (Layer 4) — read-only, feature-flaggable
```

The brief adds: transaction logging (JSON record per Core action), step-based decomposition (each CLI command decomposes into typed steps with per-step timing), and feature flags for plugins.

**This work is scoped to the SGit-AI/SGit-AI__CLI repo**, not this repo. The brief reaches the Villager team via this project's briefs folder because Dinis Cruz coordinates all teams through SGraph Send's team structure.

**All refactoring is PROPOSED — does not exist yet in its four-layer form.**
