## Thematic Grouping

### Theme 1: SGit as Standalone Product (03/19–03/21)

**Documents:** 56, 58, 59, 62, 63, 65, 68, 72

SGit-AI launched as a standalone product with its own GitHub organisation, PyPI package (`sgit-ai`), and domain. Core innovation: **agent identity through cryptography, not credentials**. Agents get their own key pair, commit to their own branch, and never receive the user's Git identity. This was **confirmed working in a real session** where Claude built a 62-test Python CLI tool through vault commits.

Key architectural elements:
- **Git-to-SGit bridge:** Git→export→SGit vault→agent branch→import→Git PR
- **Patch-based workflow:** Agents produce Git patches in vaults with full provenance
- **Read-structure key:** New key type for metadata-only access (diagnostics, auditing)
- **Two-tier team model:** Meta-team (20+ agents) builds dev packs; project teams (3 agents) do work
- **Partner dev pack:** First partner engaged with research documents vault template

**Status:** Core workflow CONFIRMED WORKING. SGit-AI PyPI SHIPPED. Bridge, patches, read-structure key all PROPOSED.
