### Theme 2: Simple Token Vault — SHIPPED in sgit-ai (Document 186)

The CLI team delivered an implementation debrief for the Simple Token Vault feature. This is **code-verified as SHIPPED** in the sgit-ai external repository:

- **Commit:** `fc9ac67`
- **Branch:** `claude/new-explorer-team-cHLXU` (sgit-ai repo)
- **Tests:** 1,424 passing (22 new), 0 regressions

**What was built:**

Three new user-facing commands:
- `sgit init [token]` — creates vault with human-readable token as identity (e.g. `coral-equal-1234`)
- `sgit clone <token>` — clones vault OR imports SG/Send transfer; unified entry point
- `sgit share [--rotate]` — publishes HEAD as AES-GCM encrypted zip to SG/Send; returns browser URL

**Two-token model:**

| Token type | Example | Role |
|-----------|---------|------|
| **Edit token** | `coral-equal-1234` | Vault identity; derives all crypto keys; share only with collaborators |
| **Share token** | `dawn-haven-6034` | Read-only published snapshot; safe to share publicly; cannot push |

**Key derivation chain:**
```
edit_token → PBKDF2-HMAC-SHA256 → aes_key
aes_key → HKDF(info='vault-read-key')  → read_key  (32B)
aes_key → HKDF(info='vault-write-key') → write_key (32B)
aes_key → HKDF(info='vault-ec-seed')   → ec_seed   (32B)
```

**Design principles:**
- Token is the only secret — no separate UUID or key file to lose
- Backward compatible: zero breaking changes; existing EC-key vaults unaffected
- `vault://` prefix is stripped before pattern match (so both `sgit clone coral-equal-1234` and `sgit clone vault://coral-equal-1234` work)

**Clone resolution flow:**
1. Strip `vault://` prefix if present
2. `is_simple_token(input)`? Pattern: `^[a-z]+-[a-z]+-\d{4}$`
3. YES → check SGit-AI server first; if found, clone with derived keys
4. YES + not on SGit-AI → check SG/Send (`/transfers/info/SHA256(input)[:12]`); if found, import as new vault with generated edit token
5. NO → existing vault_key flow (unchanged)

**Files changed (9):** `Simple_Token.py`, `Vault__Crypto.py`, `Safe_Str__Vault_Key.py`, `Vault__Sync.py`, `Vault__Transfer.py`, `CLI__Vault.py`, `CLI__Main.py`, two test files.

**Reality status:** These commands are **SHIPPED** in sgit-ai external repo (branch not yet merged to main). See Section "Reality Document Updates" for changes to apply.

**Note:** The reality document (Section 17) incorrectly listed `sgit share` as PROPOSED. This has been corrected — `sgit share` is now SHIPPED.
