## Thematic Grouping

### Theme 1: Website Redesign — Complete Production-Ready Brief

**Documents:** 149–168 (20 documents)

The website redesign session produced a **complete dev pack** ready for immediate development:

- **North star:** "Encrypted. Browsable. The server never sees your files."
- **Key decisions (non-negotiable):** Remove /agents/+/architecture/+/product/, add /security/+/features/+5×/why/ pages, "Cannot not will not" language throughout, recipient is 70% of traffic
- **Navigation:** Features | Security | Pricing | Tools + persistent "Already have a token?" link
- **Two-PR release strategy:** PR 1 (new content, safe anytime) → 48-72h → PR 2 (redirects after indexing)
- **Blocker:** 8 screenshots + 3 demo vaults needed before development starts
- **120 tasks** across 4 teams (Explorer-Website, Explorer-Platform, Villager-Release, Town Planner-Launch)

### Theme 3: Competitive Intelligence

**Documents:** 146, 156

Critical finding: **Proton Drive also offers client-side E2EE**, so encryption alone is NOT the differentiator. The 6 genuine differentiators are: recipient browsing UX, no-account model, pay-per-use, zero-cookie transparency, friendly tokens, and the 6-sentence privacy policy.

### Theme 4: Token Architecture

**Document:** 147

Simple token system fully documented: `word-word-NNNN` format, SHA-256→transfer_id, PBKDF2→AES key, ~1 billion combinations, client-side only (server never sees the token). Browser ↔ CLI interoperable.
