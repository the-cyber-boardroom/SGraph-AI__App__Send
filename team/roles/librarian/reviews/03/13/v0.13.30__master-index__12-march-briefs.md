## Thematic Grouping

### Vault Architecture (Briefs 4, 5, 6, 12, 13)

The core architecture cluster. The simulation v6 (doc 12) is the most implementation-ready spec — 1014 lines tracing exact state at every step across 4 scenarios. It resolves 13 of 14 architectural gaps. The three arch briefs define the build order:

1. **Bare vault unification** (doc 5) — one format for server + clone
2. **Branch model with PKI** (doc 6) — every clone gets a key pair, signed commits, merge-only main
3. **CLI standalone with remotes** (doc 4) — CLI as vault tool, SG/Send as one backend

The case study (doc 13) proposes simulation-driven architecture as standard methodology.

**Dependency chain:** Unification -> Branch model -> Remotes -> "Lightweight encrypted Git"
