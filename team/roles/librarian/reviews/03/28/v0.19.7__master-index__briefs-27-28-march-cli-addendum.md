## Thematic Grouping

### Theme 1: sgit — Encrypted Git-Compatible CLI (PROPOSED Documentation)

**Documents:** 173–175

The CLI team deposited three documentation files describing `sgit` — an encrypted, git-compatible CLI built on top of the vault/object model. These documents describe a **PROPOSED tool**; the existing `sg-send-cli` v0.5.0 on PyPI provides clone/push/pull/init/status, but the full branch model, diff, log, stash, revert, fsck, and export commands described in these docs do not yet exist.

Key design points documented:
- **Command parity with git** — `init`, `clone`, `commit` (no staging), `push`, `pull`, `diff`, `log`, `branch`, `switch`, `stash`, `revert`, `fsck`, `export`
- **Dual branch model** — Named branches (server, shared) + Clone branches (local, private)
- **Object model** — Content-addressable store (SHA-256), object types: blob, tree, commit, ref
- **Key derivation** — Vault key = passphrase + vault_id; branch key derived from vault key; clone key stays local forever
- **Zero-knowledge** — Server sees only encrypted bytes; plaintext never leaves device
- **Backward compatibility** — Existing `sg-send-cli` vault format is the foundation

**Reality check:** All sgit commands beyond clone/push/pull/status/init are **PROPOSED — do not exist yet.**

### Theme 2: Large Blob Support — Phase 1 Server (EXISTS as of v0.19.5)

**Documents:** 176, 178, 180, 181, 182

The CLI team's spec (doc 176) triggered an immediate Phase 1 server implementation. As of commit `4d5e866` (v0.19.5), four new vault presigned endpoints exist in the User Lambda:

| Endpoint | Status |
|----------|--------|
| `POST /api/vault/presigned/initiate/{vault_id}` | **EXISTS** — v0.19.5 |
| `POST /api/vault/presigned/complete/{vault_id}` | **EXISTS** — v0.19.5 |
| `POST /api/vault/presigned/cancel/{vault_id}` | **EXISTS** — v0.19.5 |
| `GET /api/vault/presigned/read-url/{vault_id}/{file_id}` | **EXISTS** — v0.19.5 |

**In S3 mode:** produces real presigned URLs for direct-to-S3 upload/download, bypassing Lambda payload limits (4.7 MB upload / 5.7 MB download).
**In memory mode:** returns `{ "error": "presigned_not_available" }` — clients fall back to normal vault read/write.

**Phases 2–4 (client-side) remain PROPOSED** — `Vault__API`, `Vault__Batch`, `Vault__Fetch` changes not yet implemented.

Also in this session: `abort` → `cancel` rename across the entire presigned API surface (doc 181). Breaking change to `POST /api/presigned/abort/` → `POST /api/presigned/cancel/`.
