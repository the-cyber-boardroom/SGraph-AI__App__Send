### 4. tools.sgraph.ai as the Canonical Component Library

| Field | Value |
|-------|-------|
| **File** | `v0.11.08__arch-brief__tools-canonical-component-library.md` |
| **Category** | Architecture |
| **Version** | v0.11.08 |
| **Lead roles** | Architect (lead), Developer, DevOps |

**Summary:** Defines the dependency inversion architecture: tools.sgraph.ai becomes the single source of truth for all shared JavaScript across the SGraph ecosystem. Three tiers: `core/` (pure JS logic, no UI), `components/` (reusable UI elements), `tools/` (standalone pages composing the other two). Each module independently versioned via folder-based versioning on S3/CloudFront. Production consumers pin versions; tools use `latest`. Defines a five-phase migration path from current duplicated code to single-source imports. Documents CDN caching strategy (immutable for pinned, 5-min for latest).

**Key decisions recorded:**
- Dependency direction inverted: send, vault, workspace, and Chrome extension import FROM tools.sgraph.ai
- Folder-based versioning (no package manager)
- Per-module IFD CI/CD pipelines
- CDN cache: immutable headers on pinned versions, 300s on `latest`
- crypto.js extraction from send repo as first migration target

### 5. tools.sgraph.ai -- Video Splitter and Tool Platform

| Field | Value |

| **Category** | Dev |
| **Version** | v0.11.08 |
| **Lead roles** | Developer (lead), Ambassador, Designer |

**Summary:** Specifies tools.sgraph.ai as a collection of standalone, browser-based utility tools. Each tool runs client-side only (zero server involvement). Details the Video Splitter MVP: FFmpeg WASM for splitting MP4 files in the browser with fixed-length or custom segment modes, lazy FFmpeg loading, no re-encoding (-c copy), blob URL downloads, and a "Share via SG/Send" bridge. Lists 14 future tools (file hasher, image converter, PDF splitter, etc.) as a traffic acquisition funnel. Notes the SSH Key Generator already exists.

**Key decisions recorded:**
- tools.sgraph.ai deployed as static site (S3 + CloudFront), same pattern as other sites
- Each tool is self-contained, no build step, no cross-tool dependencies
- Video splitter uses FFmpeg WASM loaded lazily from CDN
- File size limit: warn at 200MB, hard limit at 500MB
- Every tool has a natural "share via SG/Send" bridge for conversion
