## Milestone Event: First Browser-to-Browser PKI Encrypted Messaging

On 20 February 2026, the SG/Send project achieved its first end-to-end encrypted message exchange between two separate browser sessions using client-side PKI — a major milestone publicly announced by the project owner on LinkedIn.

- IndexedDB key storage (sg-send-pki)

- SHA-256 fingerprints for key identification

- Encrypt & Sign / Decrypt & Verify message workflow
- Payload v2 format with digital signatures

## Key Architecture Decisions Captured

| Decision | Detail |
|----------|--------|
| Encryption algorithm | RSA-OAEP 4096-bit + AES-256-GCM hybrid |
| Signing algorithm | ECDSA P-256 (added post-MVP, same session) |
| Key storage | IndexedDB, browser-local, non-extractable |
| Server involvement | Zero — all crypto operations client-side |
| Payload format | Base64-encoded JSON (v1 unsigned, v2 signed) |
| Key exchange format | JSON bundle `{v:1, encrypt: PEM, sign: PEM}` |
| Backwards compatibility | v1 payloads and legacy PEM import still work |

## What's Not Yet Implemented (Per LinkedIn Post)

1. **Key verification / fingerprint comparison** — no out-of-band verification step
2. **Key revocation** — no mechanism to revoke compromised keys
3. **Multi-device key sync** — keys are browser-local only
4. **Message history** — no persistent message store (by design for now)
