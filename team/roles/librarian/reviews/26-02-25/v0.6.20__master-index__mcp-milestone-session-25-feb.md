## Session Summary

This session achieved a major project milestone: **Claude.ai web successfully created, encrypted, and transmitted a PDF document to a human through SGraph Send's MCP integration.** The human's browser decrypted it locally. The server never saw the plaintext. This is the first demonstrated AI-to-human zero-knowledge encrypted communication through a production system.

**Evidence:** Working download URL confirmed by human stakeholder. Screenshot shows:
- PDF titled "Hello from Claude.ai Web!"
- PDF body: "This PDF was generated, encrypted with AES-256-GCM, and uploaded to SG/Send — all from a Claude.ai conversation using the SG/Send MCP connector."
- Browser decryption successful (green banner: "File decrypted and saved successfully")
- Transfer metadata showing AES-256-GCM encryption, 1.9 KB file size
- File name and content marked "Encrypted (only you can read it)"

## Key Decisions Made This Session

| Decision | Rationale | Made By |
|----------|-----------|---------|
| HTTP transport (not SSE) for MCP over Lambda | Lambda's request-response model matches JSON-RPC perfectly | Architect |
| User Lambda only (no Admin MCP exposure) | AppSec principle: minimise attack surface | AppSec |
| Base64 encoding for binary content over MCP | JSON transport can't carry raw binary | Dev |
| Raw bytes extraction in upload handler | MCP wraps content in JSON envelope; must unwrap before storage | Dev (the critical fix) |
