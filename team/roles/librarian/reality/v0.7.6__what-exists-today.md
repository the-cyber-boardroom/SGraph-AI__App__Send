## 4. Infrastructure That Works

### Storage
- **Memory-FS** -- in-memory (dev/test), ~100ms startup
- **S3** -- production storage, auto-bucket creation
- **Auto-detection** via `SEND__STORAGE_MODE` env var or AWS credential presence

### Deployment
- **2 Lambda functions** -- User (public) + Admin (auth-protected)
- **Lambda URL** -- direct HTTPS, no API Gateway
- **3 stages each** -- dev, qa, prod (6 Lambda functions total)
- **CI/CD** -- GitHub Actions: push to `dev` triggers tests + deploy to dev; push to `main` triggers tests + deploy to qa + PyPI publish

### Static Website Deployment (NEW in v0.7.6)
- **deploy-website.yml** -- GitHub Actions workflow: validates HTML files, checks internal links, syncs to S3 with differentiated cache TTLs (HTML 300s, CSS/JS 86400s, images 604800s), invalidates CloudFront, runs smoke test
- **Triggers:** push to `main` on `sgraph_ai__website/**` paths, or manual workflow_dispatch
- **Target:** S3 bucket (via `WEBSITE_S3_BUCKET` secret) + CloudFront distribution (via `WEBSITE_CF_DIST` secret)
- **Region:** eu-west-2

### MCP (Model Context Protocol)
- **Stateless HTTP transport** -- Lambda-compatible (no session persistence)
- **User Lambda MCP tools:** transfers, presigned, vault endpoints
- **Admin Lambda MCP tools:** tokens, keys, vault, users endpoints
- **Verified working:** Claude.ai generated PDF, encrypted it, uploaded via MCP, human decrypted in browser (Milestone M-007)

## 5. Security Properties (Verified in Code)

- Server never sees plaintext (encryption in browser, AES-256-GCM)
- No file names on server (SGMETA envelope stays client-side)
- No decryption keys on server (key in URL hash fragment, never sent)
- IP addresses hashed (SHA-256 with daily salt, stored as `ip_hash`)
- Token-gated uploads (header `x-sgraph-send-access-token` or query param)
- Immutable audit trail (append-only, hash-chained events)
- Vault ACL enforcement (owner/editor/viewer hierarchy)
- Room session tokens (24h expiry, revocable)

## 8. Key Architectural Facts

- **Stack:** Python 3.12 / FastAPI / Mangum / Type_Safe (no Pydantic)
- **Frontend:** Vanilla JS Web Components, zero framework dependencies
- **IFD versioning:** Each UI version patches specific components, inherits others
- **Two Lambda functions:** User (public) + Admin (cookie-authenticated)
- **Inter-Lambda calls:** User Lambda calls Admin Lambda for token validation
- **Cache service:** External MGraph-AI Cache Service (KEY_BASED strategy)
- **10 cache namespaces:** analytics, tokens, costs, transfers, keys, users, rooms, invites, audit, sessions

## 10. Known Violations (Flagged for Remediation)

### CRITICAL: External Dependencies (Google Fonts)

All HTML pages across the project load fonts from `https://fonts.googleapis.com`. This violates the zero-external-dependencies requirement declared as CRITICAL in v0.7.6.

**Affected files:**

| Site | Files |
|------|-------|
| sgraph_ai__website | `index.html`, `product/index.html`, `architecture/index.html`, `contact/index.html`, `agents/index.html`, `agents/sherpa/index.html`, `agents/ambassador/index.html`, `agents/architect/index.html` (8 files) |
| User UI v0.1.8 | `index.html`, `download.html`, `room.html`, `join.html` (4 files) |
| User UI v0.1.7 | `vault.html` (1 file) |
| User UI v0.1.6 | `index.html`, `download.html` (2 files) |
| SSH KeyGen tool | `index.html` (1 file) |
| Admin UI v0.1.4 | `css/sg-brand.css` (1 file, uses CSS @import) |

**Fonts loaded:** DM Sans (400, 500, 600, 700) + JetBrains Mono (400, 500)
**Fix required:** Download woff2 files, vendor them locally, serve from our own infrastructure (or cdn.sgraph.ai when available)

### Absolute Path Navigation Links

The sgraph_ai__website uses absolute paths for navigation (`href="/"`, `href="/product/"`, `href="/agents/"`, `href="/architecture/"`, `href="/contact/"`). The v0.7.6 arch brief requires all links to be relative for versioned deployments and offline packaging.

**Affected:** All 8 HTML files in `sgraph_ai__website/`

### CI Logic in YAML (Not Python)

The `deploy-website.yml` workflow contains deployment logic directly in YAML (aws s3 sync commands, cache control headers, validation scripts). The v0.7.6 arch brief requires CI logic to be in Python scripts invoked from GitHub Actions.

**Status:** Functional but does not match the target architecture.
