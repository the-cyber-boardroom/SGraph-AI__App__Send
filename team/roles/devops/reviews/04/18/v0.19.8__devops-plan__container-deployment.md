# DevOps Plan: SG/Send Container Deployment

**Version:** v0.19.8
**Date:** 2026-04-18
**Role:** DevOps
**Status:** Implementation plan — approved decisions, ready to build

---

## Goal

Package SGraph Send (User Lambda only) as a Docker container, push to Docker Hub + ECR, and create a Docker-on-EC2 AMI via Packer.

---

## Approved Decisions

| Decision | Choice |
|---|---|
| Scope | **User Lambda only** — Admin Lambda excluded |
| Auth | **Existing env-var token** (`SGRAPH_SEND__ACCESS_TOKEN`) — no Admin Lambda needed |
| Storage default | **Memory-FS** (configurable to disk volume or S3 via env vars) |
| AMI strategy | **Docker-on-EC2** via Packer |
| TLS handling | **HTTP only** — external proxy handles HTTPS |

---

## Why This Is Simple

The User Lambda's `Fast_API__SGraph__App__Send__User` already handles the `admin_service_client is None` case gracefully. When the Admin env vars aren't set:

1. `setup_admin_service_client__remote()` raises an exception (line 69-72)
2. `admin_service_client` stays `None`
3. All routes fall back to simple `SGRAPH_SEND__ACCESS_TOKEN` env var comparison
4. If env var not set either, all requests allowed (dev mode)

**No new auth code needed.** The container just runs the existing User FastAPI app with uvicorn instead of Mangum.

## Phase 1: Container Entry Point

**PROPOSED — does not exist yet.**

Create a thin entry point that starts the existing User FastAPI app under uvicorn.

**New file:** `sgraph_ai_app_send/container/app.py`

```python
from sgraph_ai_app_send.lambda__user.fast_api.Fast_API__SGraph__App__Send__User import Fast_API__SGraph__App__Send__User

def create_app():
    with Fast_API__SGraph__App__Send__User() as _:
        _.setup()
        return _.app()
```

That's it. The existing `setup()` method already:
- Auto-creates `Send__Config` (detects storage mode from env vars)
- Auto-creates all services (`Transfer__Service`, `Service__Presigned_Urls`, etc.)
- Gracefully handles missing Admin client (falls back to env-var token)
- Sets up CORS, MCP, routes

### Storage Mode: DISK

Add a `DISK` variant to `Enum__Storage__Mode` and update `Send__Config` to handle it.

**Env var:** `SEND__STORAGE_MODE=disk` + `SEND__DISK_PATH=/data`

Check whether `memory-fs` package has an existing `Storage_FS__File` provider. If yes, use it. If not, implement a simple one that stores files in a local directory.

## Phase 2: Dockerfile

**New file:** `Dockerfile`

```dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY pyproject.toml ./
COPY sgraph_ai_app_send/ ./sgraph_ai_app_send/
COPY sgraph_ai_app_send__ui__user/ ./sgraph_ai_app_send__ui__user/
COPY sgraph_ai_app_send__ui__vault/ ./sgraph_ai_app_send__ui__vault/
COPY sgraph_ai_app_send__ui__workspace/ ./sgraph_ai_app_send__ui__workspace/

RUN pip install --no-cache-dir -e .

RUN mkdir -p /data

EXPOSE 8080
ENV SEND__STORAGE_MODE=memory
ENV SEND__DISK_PATH=/data

CMD ["uvicorn", "sgraph_ai_app_send.container.app:create_app", \
     "--factory", "--host", "0.0.0.0", "--port", "8080"]
```

**New file:** `.dockerignore`
```
.git

.claude
.issues
tests/
team/
library/
*.md
__pycache__
.venv
.pytest_cache
sgraph_ai_app_send__ui__admin/
```

Note: `sgraph_ai_app_send__ui__admin/` excluded from Docker context since Admin is not included.

### pyproject.toml Consideration

The `pyproject.toml` lists `sgraph_ai_app_send__ui__admin` as a package. The Dockerfile only COPYs the user/vault/workspace UI packages, so the admin UI won't be in the image. This is fine — Python won't fail on a missing package that's listed but not imported. The container only imports User-side code.

### Multi-arch Build

```bash
docker buildx build --platform linux/amd64,linux/arm64 \
  -t sgraphai/sg-send:v0.19.8 \
  -t sgraphai/sg-send:latest \
  --push .
```

## Phase 3: CI/CD — GitHub Actions Workflow

**New file:** `.github/workflows/docker-build.yml`

Triggered on:
- Git tags matching `v*`
- Manual workflow_dispatch

Steps:
1. Checkout code
2. Set up Docker Buildx
3. Login to Docker Hub (`DOCKERHUB_USERNAME`, `DOCKERHUB_TOKEN` secrets)
4. Login to ECR (`aws-actions/amazon-ecr-login`)
5. Build multi-arch image
6. Push to Docker Hub: `sgraphai/sg-send:{tag}` + `sgraphai/sg-send:latest`
7. Push to ECR: `{account}.dkr.ecr.{region}.amazonaws.com/sg-send:{tag}`
8. Run smoke test: start container, hit `/info/health`, verify 200

## Phase 5: AMI via Packer (Docker-on-EC2)

**New file:** `packer/sg-send.pkr.hcl`

### AMI Spec
- **Base AMI:** Amazon Linux 2023 (arm64)
- **Software:** Docker CE
- **Container config:** Systemd unit pulls and runs `sgraphai/sg-send:latest`
- **Ports:** 8080
- **Data:** `/data` directory mapped to EBS volume

### Systemd Unit

```ini
[Unit]
Description=SGraph Send
After=docker.service
Requires=docker.service

[Service]
Restart=always
ExecStartPre=-/usr/bin/docker pull sgraphai/sg-send:latest
ExecStart=/usr/bin/docker run --rm --name sg-send \
  -p 8080:8080 \
  -v /data:/data \
  -e SEND__STORAGE_MODE=disk \
  -e SEND__DISK_PATH=/data \
  --env-file /etc/sg-send/env \
  sgraphai/sg-send:latest
ExecStop=/usr/bin/docker stop sg-send

[Install]
WantedBy=multi-user.target
```

## Environment Variables Reference

| Variable | Required | Default | Description |
|---|---|---|---|
| `SEND__STORAGE_MODE` | No | `memory` | `memory`, `disk`, or `s3` |
| `SEND__DISK_PATH` | No | `/data` | Local path for disk storage mode |
| `SEND__S3_BUCKET` | If S3 | — | S3 bucket name |
| `AWS_ACCESS_KEY_ID` | If S3 | — | AWS credentials |
| `AWS_SECRET_ACCESS_KEY` | If S3 | — | AWS credentials |
| `AWS_REGION` | If S3 | — | AWS region |
| `SGRAPH_SEND__ACCESS_TOKEN` | No | — | If set, requires this token for uploads |

## Usage (End State)

### Quick start (ephemeral, in-memory, open access)
```bash
docker run -p 8080:8080 sgraphai/sg-send
# → http://localhost:8080/send/
```

### With access token
```bash
docker run -p 8080:8080 \
  -e SGRAPH_SEND__ACCESS_TOKEN=my-secret-token \
  sgraphai/sg-send
```

### Persistent storage (Docker volume)
```bash
docker run -p 8080:8080 \
  -e SEND__STORAGE_MODE=disk \
  -v sg-send-data:/data \
  sgraphai/sg-send
```

### With S3 backend
```bash
docker run -p 8080:8080 \
  -e SEND__STORAGE_MODE=s3 \
  -e SEND__S3_BUCKET=my-bucket \
  -e AWS_ACCESS_KEY_ID=... \
  -e AWS_SECRET_ACCESS_KEY=... \
  -e AWS_REGION=eu-west-1 \
  sgraphai/sg-send
```
