## 3. CloudFront Distribution Design

### 3.1. Distribution Configuration

**Decision: Modify the existing send.sgraph.ai CloudFront distribution** (not create a new one).

```
Distribution: {SGRAPH_SEND__CLOUDFRONT_DISTRIBUTION_ID}
Domain: send.sgraph.ai
SSL: ACM certificate (*.sgraph.ai or send.sgraph.ai) in us-east-1
HTTP Version: HTTP/2 + HTTP/3
Default Root Object: index.html
Price Class: PriceClass_100 (US, Europe)
```

### 3.2. Origins

**Origin 1 -- S3 (Default):**
```
Domain:    745506449035--static-sgraph-ai--eu-west-2.s3.eu-west-2.amazonaws.com
Path:      /websites/send-sgraph-ai/latest
ID:        S3-send-static
Access:    Origin Access Control (OAC)
           - Signing behavior: Sign requests (SigV4)
           - Origin type: S3
```

**Origin 2 -- Lambda Function URL (API):**
```
Domain:    {url-id}.lambda-url.eu-west-2.on.aws
Path:      (none)
ID:        Lambda-send-api
Protocol:  HTTPS only
SSL:       TLSv1.2
Timeouts:  Connection: 10s, Read: 30s, Keepalive: 5s
Custom Headers:
  X-CloudFront-Secret: {SEND_CF_VERIFY_SECRET}
```

### 3.3. Behaviors

| Priority | Path Pattern | Origin | Cache Policy | Origin Request Policy |
|----------|-------------|--------|-------------|----------------------|
| 0 | `/transfers/*` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 1 | `/presigned/*` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 2 | `/vault/*` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 3 | `/join/*` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 4 | `/info/*` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 5 | `/mcp` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 6 | `/mcp/*` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 7 | `/set-cookie` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 8 | `/docs*` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| 9 | `/openapi.json` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| Default | `*` | S3-send-static | Custom-Static | S3Origin |

**AllViewerExceptHostHeader** is critical: it forwards all viewer headers (Authorization, Content-Type, x-sgraph-access-token, cookies, query strings) to Lambda, except `Host` which must match the Lambda Function URL domain.

### 3.4. Cache Policies

**API (CachingDisabled -- AWS managed):**
- TTL: 0 (all requests go to origin)
- Forward: all headers, all cookies, all query strings
- Compression: yes

**Static (Custom-Static):**
- Default TTL: 300s (5 min)
- Max TTL: 3600s (1 hour)
- Min TTL: 0s
- Forward: nothing (no query strings, no cookies)
- Compression: gzip + brotli
- Cache-Control headers set per-file-type during S3 upload (same as sgraph.ai)

## 5. Lambda Function URL as CloudFront Origin

### 5.1. Security: Restricting Direct Access

**Phase 1 (Custom header -- recommended for initial deployment):**
1. CloudFront adds `X-CloudFront-Secret: {secret}` to origin requests
2. Lambda middleware validates the header
3. Direct Lambda Function URL access still works (for dev/QA/MCP clients)
4. Production traffic protected by header check

**Phase 2 (OAC for Lambda -- later hardening):**
1. CloudFront OAC signs requests with SigV4
2. Lambda Function URL auth type changed to `AWS_IAM`
3. Lambda resource policy allows only the CloudFront distribution
4. More secure but requires IAM credentials for direct access

### 5.2. Path Mapping

No path transformation needed. CloudFront passes paths as-is:
- `send.sgraph.ai/transfers/create` -> Lambda receives `/transfers/create`
- `send.sgraph.ai/presigned/capabilities` -> Lambda receives `/presigned/capabilities`

### 5.3. Large File Bypass

Presigned URLs already point directly to S3 (not through Lambda or CloudFront). This means large file uploads/downloads bypass CloudFront entirely. Only metadata API calls go through the CloudFront -> Lambda path.

## 6. CloudFront Function

### 6.1. URL Rewrite Logic

```javascript
// url-rewrite-send.js -- CloudFront Function (Viewer Request)

function handler(event) {
    var request = event.request;
    var uri = request.uri;

    // API paths: pass through unchanged (routed to Lambda by behaviors)
    var apiPrefixes = ['/transfers', '/presigned', '/vault', '/join',
                       '/info', '/mcp', '/set-cookie', '/docs', '/openapi.json'];
    for (var i = 0; i < apiPrefixes.length; i++) {
        if (uri === apiPrefixes[i] || uri.startsWith(apiPrefixes[i] + '/')) {
            return request;
        }
    }

    // Legacy /send/* redirects (301 permanent)
    if (uri === '/send' || uri.startsWith('/send/')) {
        var newPath = uri.replace(/^\/send\/?/, '/');
        if (newPath === '') newPath = '/';
        return {
            statusCode: 301,
            statusDescription: 'Moved Permanently',
            headers: {
                'location': { value: 'https://send.sgraph.ai' + newPath },
                'cache-control': { value: 'public, max-age=86400' }
            }
        };
    }

    // Download paths with transfer IDs
    // /download/{id} or /{locale}/download/{id}
    var downloadMatch = uri.match(/^(\/(?:[a-z]{2}-[a-z]{2,3}\/)?)download\/[a-zA-Z0-9_-]+$/);
    if (downloadMatch) {
        request.uri = downloadMatch[1] + 'download/index.html';
        return request;
    }

    // Shared assets -- pass through
    if (uri.startsWith('/_common/')) {
        return request;
    }

    // Files with extension -- pass through
    if (uri.match(/\.[a-zA-Z0-9]+$/)) {
        return request;
    }

    // Directory with trailing slash -- append index.html
    if (uri.endsWith('/')) {
        request.uri = uri + 'index.html';
        return request;
    }

    // Bare path without extension -- redirect to add trailing slash
    return {
        statusCode: 302,
        statusDescription: 'Found',
        headers: { 'location': { value: uri + '/' } }
    };
}
```

### 6.2. Key Routing Examples

| Request URL | Action | Result |
|-------------|--------|--------|
| `/` | Append index.html | S3: `/index.html` |
| `/upload/` | Append index.html | S3: `/upload/index.html` |
| `/upload` | 302 redirect | -> `/upload/` -> `/upload/index.html` |
| `/download/abc123` | Rewrite | S3: `/download/index.html` (JS reads ID) |
| `/pt-pt/` | Append index.html | S3: `/pt-pt/index.html` |
| `/pt-pt/download/abc123` | Rewrite | S3: `/pt-pt/download/index.html` |
| `/_common/css/style.css` | Pass through | S3: `/_common/css/style.css` |
| `/transfers/create` | Pass through | Lambda (via behavior match) |
| `/presigned/capabilities` | Pass through | Lambda (via behavior match) |
| `/send/v0/v0.1/v0.1.8/` | 301 redirect | -> `/v0/v0.1/v0.1.8/` |
| `/docs` | Pass through | Lambda (via behavior match) |

## 7. Deployment Pipeline

### 7.1. New Workflow: `deploy-send-ui.yml`

```yaml
name: Deploy Send UI

on:
  push:
    branches: [dev]
    paths:
      - 'sgraph_ai_app_send__ui__user/**'
      - 'scripts/deploy_static_site.py'
      - 'scripts/generate_i18n_pages.py'
      - '.github/workflows/deploy-send-ui.yml'
  workflow_dispatch:
    inputs:
      environment:
        type: choice
        options: [production, staging]

concurrency:
  group: send-ui-deploy
  cancel-in-progress: false

env:
  AWS_REGION: eu-west-2
  SITE_NAME: send-sgraph-ai

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: production
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - name: Configure AWS
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: eu-west-2
      - name: Read version
        id: version
        run: echo "VERSION=$(cat sgraph_ai_app_send/version)" >> $GITHUB_OUTPUT
      - name: Generate i18n pages
        run: |
          python scripts/generate_i18n_pages.py \
            --site send-sgraph-ai \
            --source sgraph_ai_app_send__ui__user/v2/v2.0/v2.0.0 \
            --i18n sgraph_ai_app_send__ui__user/i18n
      - name: Deploy to S3
        run: |
          python scripts/deploy_static_site.py \
            --site send-sgraph-ai \
            --source sgraph_ai_app_send__ui__user/v2/v2.0/v2.0.0 \
            --version ${{ steps.version.outputs.VERSION }} \
            --bucket ${{ secrets.WEBSITE_S3_BUCKET }} \
            --cf-distribution ${{ secrets.SEND_CF_DIST }}
      - name: Store CI artifacts
        run: |
          python scripts/store_ci_artifacts.py \
            --site send-sgraph-ai \
            --version ${{ steps.version.outputs.VERSION }} \
            --bucket ${{ secrets.WEBSITE_S3_BUCKET }}
```

  s3://{bucket}/websites/send-sgraph-ai/latest/ --delete

# Invalidate CloudFront cache

## 12. Risk Assessment

| Risk | Probability | Impact | Mitigation |
|------|------------|--------|-----------|
| API routes incorrectly routed to S3 | Low | High | Explicit behavior per API prefix + smoke tests |
| Lambda rejects requests (Host header) | Medium | High | Use AllViewerExceptHostHeader policy |
| IFD relative paths break on S3 | Low | Medium | Test all HTML files from S3 paths |
| CORS breaks after CloudFront | Low | Medium | Same-origin eliminates CORS (Lambda has `*` fallback) |
| Locale paths 404 | Low | Low | Run i18n generation before every deploy |
| MCP clients break (direct Lambda access) | Low | Medium | Keep Lambda Function URL accessible during transition |
| Large file uploads fail through CloudFront | N/A | N/A | Presigned URLs already bypass CloudFront |
