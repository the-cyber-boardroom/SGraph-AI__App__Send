## 4. CloudFront Distribution Design

### 4.1. Per-Environment Distribution

Each environment gets its own CloudFront distribution (or can share one with behavior-based routing, but separate distributions are simpler to manage).

```
Distribution (prod): send.sgraph.ai
Distribution (main): main.send.sgraph.ai
Distribution (dev):  dev.send.sgraph.ai

All three have identical configuration (origins, behaviors, CF Function)
but different S3 prefixes and Lambda stages.
```

### 4.2. Origins (per distribution)

**Origin 1 — S3 (Default):**
```
Domain:    745506449035--static-sgraph-ai--eu-west-2.s3.eu-west-2.amazonaws.com
Path:      /websites/send-sgraph-ai{-env}/latest    (prod has no -env suffix)
ID:        S3-send-static
Access:    Origin Access Control (OAC)
           - Signing behavior: Sign requests (SigV4)
           - Origin type: S3
```

**Origin 2 — Lambda Function URL (API):**
```
Domain:    {url-id}.lambda-url.eu-west-2.on.aws     (per stage)
Path:      (none)
ID:        Lambda-send-api
Protocol:  HTTPS only
SSL:       TLSv1.2
Timeouts:  Connection: 10s, Read: 30s, Keepalive: 5s
Custom Headers:
  X-CloudFront-Secret: {per-env secret}
```

### 4.3. Behaviors (Simplified — Same for All Environments)

| Priority | Path Pattern | Origin | Cache Policy | Origin Request Policy |
|----------|-------------|--------|-------------|----------------------|
| 0 | `/api/*` | Lambda-send-api | CachingDisabled | AllViewerExceptHostHeader |
| Default | `*` | S3-send-static | Custom-Static | S3Origin |

**Just two behaviors.** All API routes are under `/api/*`. Everything else is static content from S3.

**AllViewerExceptHostHeader** is critical for the Lambda behavior: it forwards all viewer headers (Authorization, Content-Type, x-sgraph-access-token, cookies, query strings) to Lambda, except `Host` which must match the Lambda Function URL domain.

### 4.4. Cache Policies

**API (CachingDisabled — AWS managed):**
- TTL: 0 (all requests go to origin)
- Forward: all headers, all cookies, all query strings
- Compression: yes

**Static (Custom-Static):**
- Default TTL: 300s (5 min)
- Max TTL: 3600s (1 hour)
- Min TTL: 0s
- Forward: nothing (no query strings, no cookies)
- Compression: gzip + brotli
- Cache-Control headers set per-file-type during S3 upload (same as sgraph.ai)

### 5.2. Version Manifest-Driven Sync

Instead of blindly syncing everything, the CI pipeline reads `manifest.json` to know exactly what to copy to `latest/`:

```python
# deploy_static_site.py (enhanced)
import json

def deploy_to_latest(source_dir, bucket, site_prefix):
    manifest = json.load(open(f"{source_dir}/manifest.json"))

    # 1. Sync common files (shared JS, CSS, fonts, error pages)
    for f in manifest["common_files"]:
        s3_upload(f"{source_dir}/{f}", f"{site_prefix}/releases/{ifd_path}/{f}")

    # 2. Sync locale folders (each contains identical pages with locale-specific text)
    for folder in manifest["locale_folders"]:
        s3_sync(f"{build_dir}/{folder}", f"{site_prefix}/releases/{ifd_path}/{folder}")

    # 3. Copy from releases/{version}/ to latest/ (manifest files + i18n folders only)
    s3_sync(f"{site_prefix}/releases/{ifd_path}/", f"{site_prefix}/latest/", delete=True)
```

This means `latest/` only contains files listed in the manifest — no stale files from previous versions.

### 5.3. S3 Upload with Content-Type and Cache Headers

Same approach as sgraph.ai — each file type synced with specific headers:

```bash
# HTML files (5 min cache)
--content-type "text/html" --cache-control "public, max-age=300"

# CSS files (1 day cache)
--content-type "text/css" --cache-control "public, max-age=86400"

# JS files (1 day cache)
--content-type "application/javascript" --cache-control "public, max-age=86400"

# Fonts (7 day cache)
--cache-control "public, max-age=604800"

# JSON (1 day cache)
--content-type "application/json" --cache-control "public, max-age=86400"
```

## 6. CloudFront Function (Simplified)

### 6.1. URL Rewrite Logic

No legacy redirect handling needed — no live customers on old URLs.

```javascript
// url-rewrite-send.js — CloudFront Function (Viewer Request)

function handler(event) {
    var request = event.request;
    var uri = request.uri;

    // API paths: pass through unchanged (routed to Lambda by /api/* behavior)
    if (uri.startsWith('/api/')) {
        return request;
    }

    // Root path: redirect to default locale
    if (uri === '/' || uri === '') {
        return {
            statusCode: 302,
            statusDescription: 'Found',
            headers: { 'location': { value: '/en-gb/' } }
        };
    }

    // Shared assets — pass through
    if (uri.startsWith('/_common/')) {
        return request;
    }

    // Download paths with transfer IDs (always inside a locale folder)
    // /{locale}/download/{id}
    var downloadMatch = uri.match(/^(\/[a-z]{2}-[a-z]{2,3}\/)download\/[a-zA-Z0-9_-]+$/);
    if (downloadMatch) {
        request.uri = downloadMatch[1] + 'download/index.html';
        return request;
    }

    // Files with extension — pass through
    if (uri.match(/\.[a-zA-Z0-9]+$/)) {
        return request;
    }

    // Directory with trailing slash — append index.html
    if (uri.endsWith('/')) {
        request.uri = uri + 'index.html';
        return request;
    }

    // Bare path without extension — redirect to add trailing slash
    return {
        statusCode: 302,
        statusDescription: 'Found',
        headers: { 'location': { value: uri + '/' } }
    };
}
```

### 6.2. Key Routing Examples

| Request URL | Action | Result |
|-------------|--------|--------|
| `/` | 302 redirect | → `/en-gb/` (default locale) |
| `/en-gb/` | Append index.html | S3: `/en-gb/index.html` |
| `/en-gb/upload/` | Append index.html | S3: `/en-gb/upload/index.html` |
| `/en-gb/upload` | 302 redirect | → `/en-gb/upload/` → `/en-gb/upload/index.html` |
| `/en-gb/download/abc123` | Rewrite | S3: `/en-gb/download/index.html` (JS reads ID from hash) |
| `/pt-pt/` | Append index.html | S3: `/pt-pt/index.html` |
| `/pt-pt/upload/` | Append index.html | S3: `/pt-pt/upload/index.html` |
| `/pt-pt/download/abc123` | Rewrite | S3: `/pt-pt/download/index.html` |
| `/_common/css/style.css` | Pass through | S3: `/_common/css/style.css` |
| `/api/transfers/create` | Pass through | Lambda (via `/api/*` behavior) |
| `/api/info/health` | Pass through | Lambda (via `/api/*` behavior) |
| `/api/docs` | Pass through | Lambda (via `/api/*` behavior) |

## 7. Deployment Pipelines

### 7.1. Workflow: `deploy-send-ui.yml`

```yaml
name: Deploy Send UI

on:
  push:
    branches: [dev, main]
    paths:
      - 'sgraph_ai_app_send__ui__user/**'
      - 'scripts/deploy_static_site.py'
      - 'scripts/generate_i18n_pages.py'
      - '.github/workflows/deploy-send-ui.yml'
  workflow_dispatch:
    inputs:
      environment:
        description: 'Target environment'
        type: choice
        options: [production]
        default: production

concurrency:
  group: send-ui-deploy-${{ github.ref_name }}
  cancel-in-progress: false

env:
  AWS_REGION: eu-west-2

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: ${{ github.event_name == 'workflow_dispatch' && 'production' || (github.ref_name == 'main' && 'main' || 'dev') }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - name: Configure AWS
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: eu-west-2

      - name: Set environment variables
        id: env
        run: |
          if [ "${{ github.event_name }}" = "workflow_dispatch" ]; then
            echo "SITE_PREFIX=send-sgraph-ai" >> $GITHUB_OUTPUT
            echo "CF_DIST=${{ secrets.SEND_CF_DIST_PROD }}" >> $GITHUB_OUTPUT

            echo "SITE_PREFIX=send-sgraph-ai-main" >> $GITHUB_OUTPUT
            echo "CF_DIST=${{ secrets.SEND_CF_DIST_MAIN }}" >> $GITHUB_OUTPUT
          else
            echo "SITE_PREFIX=send-sgraph-ai-dev" >> $GITHUB_OUTPUT
            echo "CF_DIST=${{ secrets.SEND_CF_DIST_DEV }}" >> $GITHUB_OUTPUT
          fi

      - name: Read version
        id: version
        run: echo "VERSION=$(cat sgraph_ai_app_send/version)" >> $GITHUB_OUTPUT

      - name: Generate i18n pages
        run: |
          python scripts/generate_i18n_pages.py \
            --site ${{ steps.env.outputs.SITE_PREFIX }} \
            --source sgraph_ai_app_send__ui__user/v0/v0.2/v0.2.0 \

            --bucket ${{ secrets.WEBSITE_S3_BUCKET }} \

        run: |
          python scripts/store_ci_artifacts.py \
            --site ${{ steps.env.outputs.SITE_PREFIX }} \
            --version ${{ steps.version.outputs.VERSION }} \
```

## 8. Lambda Function URL as CloudFront Origin

### 8.1. Security: Restricting Direct Access

**Phase 1 (Custom header — initial deployment):**
1. CloudFront adds `X-CloudFront-Secret: {secret}` to origin requests
2. Lambda middleware validates the header
3. Direct Lambda Function URL access still works (for dev/QA/MCP clients)

**Phase 2 (OAC for Lambda — later hardening):**
1. CloudFront OAC signs requests with SigV4
2. Lambda Function URL auth type changed to `AWS_IAM`
3. More secure but requires IAM credentials for direct access

### 8.2. Path Mapping

CloudFront passes `/api/*` paths as-is to Lambda:
- `send.sgraph.ai/api/transfers/create` → Lambda receives `/api/transfers/create`
- `send.sgraph.ai/api/info/health` → Lambda receives `/api/info/health`

### 8.3. Large File Bypass

Presigned URLs already point directly to S3 (not through Lambda or CloudFront). Large file uploads/downloads bypass CloudFront entirely. Only metadata API calls go through CloudFront → Lambda.
