## What's Happening

v0.3.0 is in production and **frozen**. All future UI changes go into **v0.3.1**, which uses the IFD surgical overlay pattern — the same approach used for v0.2.1 through v0.2.17.

A v0.3.1 MVP has been created with a single file (`en-gb/index.html`) that loads all v0.3.0 base code and sets `uiVersion` to `v0.3.1`. This proves the overlay pipeline works before any real patches are added.

## How the IFD Overlay Works

```
v0.3.1/en-gb/index.html
  │
  ├── loads ../../v0.3.0/_common/css/*.css          (all v0.3.0 styles)
  ├── loads ../../v0.3.0/_common/js/*.js             (all v0.3.0 scripts)
  │
  ├── (future) loads ../v0.3.1/_common/js/*-v031.js  (prototype overrides)
  ├── (future) loads ../v0.3.1/_common/css/*-v031.css (CSS overrides)
  │
  └── sets window.SGRAPH_BUILD.uiVersion = 'v0.3.1'
```

The v0.3.1 page is functionally identical to v0.3.0 — it loads the exact same base code. The only difference right now is the version string. When patches are added later, they'll be loaded as additional `<script>` and `<link>` tags after the base.

## What's Different for QA Automation

### URL Changes

If QA tests currently target `/en-gb/` or a specific version path:
- **v0.3.0 tests** should continue to work unchanged at `/v0/v0.3/v0.3.0/en-gb/`
- **v0.3.1 tests** should target `/v0/v0.3/v0.3.1/en-gb/`
- Once v0.3.1 becomes the production default, `/en-gb/` will serve v0.3.1

### Version Detection

The QA harness can detect which version is running:
```javascript
// In Playwright / browser context:
const version = await page.evaluate(() => window.SGRAPH_BUILD?.uiVersion);
// v0.3.0 → 'v0.3.0'
// v0.3.1 → 'v0.3.1'
```

### data-qa-mask Attributes

The `data-qa-mask` attributes added in v0.3.0 are inherited by v0.3.1 (since v0.3.1 loads v0.3.0's components). No changes needed to the screenshot masking pipeline.
