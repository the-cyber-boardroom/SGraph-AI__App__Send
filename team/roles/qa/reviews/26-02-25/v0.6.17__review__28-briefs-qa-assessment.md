## Brief #9: Read-Only Demo Environments (HIGH, Supporting)

### Test Strategy Per Level

| Level | Test Type | What QA Validates |
|---|---|---|
| **Level 1** (single file) | Smoke test | Short code works, file decrypts, loads in under 3 seconds |
| **Level 2** (shared folder) | Integration | Multiple files decrypt, folder navigation works |
| **Level 3** (branded room) | Visual regression | Branding renders, logo loads, responsive layout |
| **Level 4** (use-case) | E2E workflow | NDA signing completes, audit trail populates |
| **Level 5** (live agents) | Continuous smoke | Agents cycle reliably, timeline updates in real-time |

### CTF Validation

Each level's hidden link to the next must resolve correctly. A broken link at any level blocks the evaluator journey -- this is a P0 defect. Links must survive key rotation.

### Automation Cadence

- **Every 5 minutes:** Level 1 smoke test + Level 5 agent cycle
- **Hourly:** Full Level 1-5 journey + CTF link validation
- **Daily:** Visual regression screenshots, performance baseline, key rotation test

QA smoke results should be visible on the demo site as a confidence signal ("Last tested: 2 minutes ago").

## Brief #22: Agent Isolation — Security Testing (CRITICAL, Supporting)

### Security Test Matrix

| Test | What It Proves | Method |
|---|---|---|
| **Memory isolation** | Container A cannot read B's memory | `/proc/<pid>/mem` cross-container access must fail |
| **Key isolation** | Private key only in owner container | Verify key zeroed / container destroyed after task |
| **Network isolation** | No inter-container connectivity | TCP/UDP connection attempts must be refused |
| **Filesystem isolation** | No shared filesystem | Write in A, read from B must fail |
| **Ephemeral destruction** | No state persistence | Verify no key material in swap, /tmp, or core dumps |
| **Workflow integrity** | .sgworkflow is only data channel | Monitor all I/O -- only .sgworkflow transfers |
| **Compromise containment** | One compromise does not spread | Simulate prompt injection in A, verify B and C unaffected |

### AC #8 Test Plan

1. Deploy 3-agent workflow across 3 containers
2. Simulate prompt injection in Container A (memory read, network connect, secret access)
3. Verify all attempts fail with appropriate error codes
4. Verify Containers B and C complete tasks successfully
5. Verify orchestrator detects Container A's anomalous behaviour

### Automation: Pre-merge (memory, network, key isolation). Post-deploy (ephemeral destruction). Quarterly (full compromise simulation).

## Brief #4: Scanning Pipeline — Test Vectors (CRITICAL, Supporting)

### Test Vector Library

| Category | Vectors | Expected Result |
|---|---|---|
| **File type validation** | .exe renamed to .pdf, polyglot files | Flagged: magic bytes mismatch |
| **Known malware hashes** | EICAR test file, known hash samples | Flagged: hash match |
| **Macro detection** | .docx benign macros, .xlsm with Shell() calls | Benign: warning. Malicious: flagged |
| **Script detection** | HTML with JS, SVG event handlers, PDF /JavaScript | Flagged: embedded script |
| **Archive inspection** | Nested ZIPs, ZIP bomb, encrypted ZIP | Recursive scan, bomb blocked, encrypted flagged |
| **LLM analysis** | Obfuscated PowerShell, base64 payload, social engineering doc | LLM identifies risk pattern |
| **Clean files** | Standard .pdf, .docx, .xlsx, .png, .txt | Zero false positives |

### Accuracy Targets

- **False positive rate:** < 2% on clean test set
- **False negative rate:** 0% on known-malicious test set
- **LLM accuracy:** tracked per-model (Claude, GPT-4, Ollama) for comparison

### Automation: Per-commit (full vector library). Weekly (hash DB update, new vectors). Monthly (performance benchmark, browser memory assessment).
