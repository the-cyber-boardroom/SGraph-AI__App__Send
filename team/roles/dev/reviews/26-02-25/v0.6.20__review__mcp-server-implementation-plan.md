## Verdict: Yes, This Is an Easy Win

Adding MCP to the existing FastAPI app is genuinely low-effort. The `fastapi-mcp` package (v0.4.0) auto-discovers all FastAPI endpoints and exposes them as MCP tools. Our two FastAPI apps already have well-defined routes with typed schemas. The core integration is ~10 lines of code per Lambda app.

**One caveat:** Lambda + SSE is problematic. `fastapi-mcp` v0.4.0 supports HTTP transport (JSON-RPC over POST), which works on Lambda. SSE streaming does not work behind Lambda URLs or API Gateway. This is fine for MVP -- Claude Desktop and most MCP clients support HTTP transport.

---

## Implementation Plan

### Step 1: MCP Mount on Admin Lambda (~10 lines)

**File:** `sgraph_ai_app_send/lambda__admin/fast_api/Fast_API__SGraph__App__Send__Admin.py`

Add to `setup_routes()`, after all routes are registered:

```python
from fastapi_mcp import FastApiMCP

# In setup_routes(), at the end:
mcp = FastApiMCP(
    self.app(),
    name="sgraph-send-admin",
    describe_all_responses=True,
    include_tags=["tokens", "keys", "vault", "users"],
)
mcp.mount()
```

This auto-exposes all admin endpoints (tokens, keys, vault, users) as MCP tools at `/mcp`. The `include_tags` parameter filters to only our application routes (excludes info, metrics, static).

### Step 2: MCP Mount on User Lambda (~10 lines)

### Step 3: MCP Config Module (optional, clean separation)

**New file:** `sgraph_ai_app_send/utils/MCP__Setup.py`

```python
from osbot_utils.type_safe.Type_Safe import Type_Safe

class MCP__Setup(Type_Safe):
    name        : str  = ""
    include_tags: list = None
    exclude_tags: list = None

    def mount_mcp(self, fast_api_app):
        from fastapi_mcp import FastApiMCP
        mcp = FastApiMCP(
            fast_api_app,
            name=self.name,
            describe_all_responses=True,
            include_tags=self.include_tags,
            exclude_tags=self.exclude_tags,
        )
        mcp.mount()
        return mcp
```

Follows the project's `Type_Safe` pattern. Keeps MCP config separate from route setup. Both Lambda apps call `MCP__Setup(name=..., include_tags=...).mount_mcp(self.app())`.

## Lambda Deployment Consideration

`fastapi-mcp` uses HTTP transport (JSON-RPC over POST to `/mcp`). This is request/response -- fully compatible with Lambda URLs and Mangum. No SSE, no persistent connections, no streaming required for MVP.

If SSE is later needed for real-time features, that would require a non-Lambda endpoint (Fargate, EC2, or a separate always-on service). But for MCP tool calls, HTTP transport is standard and sufficient.

## Risk Assessment

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| `fastapi-mcp` incompatible with `Serverless__Fast_API` | Low | It operates on the standard FastAPI app object, which `self.app()` returns |
| Schema issues (Type_Safe vs Pydantic) | Low | `fastapi-mcp` reads OpenAPI schema, not Python types. FastAPI generates OpenAPI regardless of schema library |
| Auth token not forwarded to MCP tools | Medium | `fastapi-mcp` v0.4.0 supports passing HTTP headers to tool calls. Existing `check_access_token` on routes still enforces auth |
| Lambda cold start slower with MCP | Low | `fastapi-mcp` adds minimal overhead -- it reads the OpenAPI schema once at startup |
