# Implementation Brief: Vault Key Stability

**Version:** v0.10.36
**Date:** 3 March 2026
**For:** Dev + Architect (Explorer session — separate from this one)
**Status:** Ready for implementation

---

## Objective

Make vault keys permanent. Currently, every file add/delete/rename changes the vault key. After this change, a vault key assigned at creation time will work forever.

---

## The Bug

`sg-vault.js` `_saveTree()` (line 241-256) creates **new transfers** for both tree and settings on every mutation. Since the settings_transfer_id is part of the vault key (`passphrase:vault_id:settings_transfer_id`), the key changes every time.

**Call chain:**
1. User adds/deletes/renames a file
2. `addFile()` / `removeFile()` / `createFolder()` / `removeFolder()` call `_saveTree()`
3. `_saveTree()` encrypts tree → `sgSend.upload()` → **new tree transfer_id**
4. Updates settings with new tree pointer → `sgSend.upload()` → **new settings transfer_id**
5. `vault-app.js` / `vault-shell.js` calls `_updateVaultKey()` which reconstructs key with new settings_transfer_id
6. URL hash updates silently. Old key is now dead.

**Files involved (both v0.1.0 and v0.1.1 versions):**
- `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.{0,1}/_common/js/lib/sg-vault/sg-vault.js` — `_saveTree()` at line 241
- `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.{0,1}/_common/js/components/vault-app/vault-app.js` — `_updateVaultKey()` at line 113
- `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.1/_common/js/components/vault-shell/vault-shell.js` — `_updateVaultKey()` at line 197
- `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.{0,1}/_common/js/components/vault-browser/vault-browser.js` — `_updateVaultKey()` at line 311
- `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.{0,1}/_common/js/lib/sg-send/sg-send.js` — `upload()` at line 48

## Solution: Two Changes

### Change 1: Backend — Add `update_payload` to Transfer__Service + `update__transfer_id` route

Add the ability to replace the encrypted payload of a **completed** transfer while keeping the same transfer_id.

#### 1a. Transfer__Service — New method `update_payload()`

**File:** `sgraph_ai_app_send/lambda__user/service/Transfer__Service.py`
**Insert after:** `complete_transfer()` (line 101)

```python
def update_payload(self, transfer_id, payload_bytes):                   # Replace payload of completed transfer
    if not self.has_transfer(transfer_id):
        return None
    meta = self.load_meta(transfer_id)
    if meta['status'] != 'completed':
        return None
    self.storage_fs.file__save(self.payload_path(transfer_id),
                               payload_bytes                  )
    meta['file_size_bytes'] = len(payload_bytes)
    meta['events'].append(dict(action    = 'update'                          ,
                               timestamp = datetime.now(timezone.utc).isoformat()))
    self.save_meta(transfer_id, meta)
    return dict(status      = 'updated'    ,
                transfer_id = transfer_id  ,
                size        = len(payload_bytes))
```

**Design notes:**
- Only works on `completed` transfers (not `pending` — those use `upload_payload`)
- Updates `file_size_bytes` in metadata (new payload may differ in size)
- Appends an `update` event to the audit trail
- Returns `None` if transfer not found or not completed (same pattern as other methods)

#### 1b. Routes__Transfers — New route `update__transfer_id`

**File:** `sgraph_ai_app_send/lambda__user/fast_api/routes/Routes__Transfers.py`
**Insert after:** `complete__transfer_id()` (line 136)

```python
async def update__transfer_id(self, transfer_id  : Safe_Str__Id,          # POST /transfers/update/{transfer_id}
                                    request       : Request     ,
                                    access_token  : str = ''   ,          # MCP tool parameter
                                    data          : str = ''              # MCP tool parameter — base64
                               ) -> dict:
    self.check_access_token(request, access_token)
    if data:
        body = base64.b64decode(data)
    else:
        body = await request.body()
        body = self.unwrap_mcp_payload(body)
    result = self.transfer_service.update_payload(transfer_id   = transfer_id,
                                                   payload_bytes = body       )
    if result is None:
        raise HTTPException(status_code = 404,
                            detail      = 'Transfer not found or not in completed state')
    return result
```

#### 1c. Tests

def test__update_payload__happy_path(self):
    # Create + upload + complete a transfer
    create  = self.service.create_transfer(file_size_bytes=8, content_type_hint='', sender_ip='')
    tid     = create['transfer_id']
    payload = b'\x00' * 8
    self.service.upload_payload(transfer_id=tid, payload_bytes=payload)
    self.service.complete_transfer(tid)

    # Update with new payload
    new_payload = b'\xff' * 16
    result = self.service.update_payload(transfer_id=tid, payload_bytes=new_payload)
    assert result is not None
    assert result['status']      == 'updated'
    assert result['transfer_id'] == tid
    assert result['size']        == 16

    # Download should return new payload
    downloaded = self.service.get_download_payload(transfer_id=tid, downloader_ip='', user_agent='')
    assert downloaded == new_payload

    # Metadata should be updated
    info = self.service.get_transfer_info(tid)
    assert info['file_size_bytes'] == 16

def test__update_payload__not_found(self):
    result = self.service.update_payload(transfer_id='nonexistent', payload_bytes=b'\x00')
    assert result is None

def test__update_payload__pending_transfer_rejected(self):

    tid    = create['transfer_id']
    # Don't upload or complete — still pending

### Change 2: Browser — Use `update()` instead of `upload()` for tree and settings

#### 2a. Add `update()` method to SGSend

**File:** `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.{0,1}/_common/js/lib/sg-send/sg-send.js`
**Insert after:** `upload()` method (line 73)

```javascript
async update(transferId, data) {
    const response = await this._fetch('POST', `/api/transfers/update/${transferId}`, {
        body: data,
        rawBody: true
    })
    return response.json()
}
```

#### 2b. Modify `_saveTree()` in SGVault

**File:** `sgraph_ai_app_send__ui__vault/v0/v0.1/v0.1.{0,1}/_common/js/lib/sg-vault/sg-vault.js`
**Replace:** `_saveTree()` at line 241-256

**Before:**
```javascript
async _saveTree() {
    this._tree.version = (this._tree.version || 0) + 1
    this._tree.updated = new Date().toISOString()

    const treeData      = new TextEncoder().encode(JSON.stringify(this._tree))
    const encrypted     = await this._sgSend.encrypt(treeData, this._derivedKey)
    const result        = await this._sgSend.upload(new Uint8Array(encrypted))
    this._treeTransferId = result.transferId

    // Update settings to point to new tree, re-upload
    this._settings._tree_transfer_id = this._treeTransferId
    const settingsData  = new TextEncoder().encode(JSON.stringify(this._settings))
    const encSettings   = await this._sgSend.encrypt(settingsData, this._derivedKey)
    const settingsResult = await this._sgSend.upload(new Uint8Array(encSettings))
    this._settingsTransferId = settingsResult.transferId
}
```

**After:**
```javascript
async _saveTree() {
    this._tree.version = (this._tree.version || 0) + 1
    this._tree.updated = new Date().toISOString()

    const treeData  = new TextEncoder().encode(JSON.stringify(this._tree))
    const encrypted = await this._sgSend.encrypt(treeData, this._derivedKey)

    if (this._treeTransferId) {
        // Update existing tree transfer in place (key stays stable)
        await this._sgSend.update(this._treeTransferId, new Uint8Array(encrypted))
    } else {
        // First save — create new transfer
        const result = await this._sgSend.upload(new Uint8Array(encrypted))
        this._treeTransferId = result.transferId
    }

    // Update settings to point to (possibly same) tree transfer
    this._settings._tree_transfer_id = this._treeTransferId
    const settingsData = new TextEncoder().encode(JSON.stringify(this._settings))
    const encSettings  = await this._sgSend.encrypt(settingsData, this._derivedKey)

    if (this._settingsTransferId) {
        // Update existing settings transfer in place (key stays stable)
        await this._sgSend.update(this._settingsTransferId, new Uint8Array(encSettings))
    } else {
        // First save — create new transfer
        const settingsResult = await this._sgSend.upload(new Uint8Array(encSettings))
        this._settingsTransferId = settingsResult.transferId
    }
}
```

**Key change:** If `_treeTransferId` / `_settingsTransferId` already exist (i.e. vault was opened, not just created), use `update()` instead of `upload()`. The transfer IDs never change after creation.

#### 2c. Remove `_updateVaultKey()` from UI components

Since the vault key no longer changes, `_updateVaultKey()` becomes a no-op. These calls can be removed:

- **vault-app.js** (v0.1.0 + v0.1.1): Remove `_updateVaultKey()` method and calls at lines 104, 110, 113-121
- **vault-shell.js** (v0.1.1): Remove `_updateVaultKey()` method and calls at lines 173, 197-205, 256
- **vault-browser.js** (v0.1.0 + v0.1.1): Remove `_updateVaultKey()` method and calls at lines 266, 311-315

**Note:** The `vault-key-changed` event and its listeners can also be removed since the key is now immutable.

---

## Breaking Changes

**This intentionally breaks existing vaults.** Per the human's direction: "it is ok to push breaking changes at this stage (and break content inside existing vaults), since nobody is really using these vaults."

Existing vaults will continue to work for **read** operations (download/decrypt). But the next time a file is added/removed, the tree and settings will be updated in-place rather than creating new transfers.
