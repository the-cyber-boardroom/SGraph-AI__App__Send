### Development Guide for Container Runtime Abstraction

The correct pattern (from doc 343):
```python
# DO: use the abstraction
runtime = get_container_runtime()
container = await runtime.start(image="...", ports={...})

# DON'T: import Docker or Podman directly in plugins
import docker  # NEVER in plugin code
```

The `get_container_runtime()` function returns whichever adapter is configured. The adapter implements the same interface regardless of backend. Tests use a mock adapter (the only acceptable use of a mock in this codebase — the mock replaces the external Docker/Podman socket, not the application logic).

#### Day 2 (continued): Pre-warming Hook

**Where:** Add to the existing push-completion endpoint handler.

```python
# In the existing push finalisation endpoint:
async def finalise_push(vault_id, commit_id, ...):
    # ... existing push logic ...
    response = PushResponse(...)

    # Fire-and-forget pre-warming
    async def _prewarm():
        try:
            cache = PackCache(vault_storage=storage_for_vault(vault_id))
            for flavour in ['full', 'head']:
                builder = PackBuilder(vault_id=vault_id, commit_id=commit_id,
                                      flavour=flavour, storage=storage_for_vault(vault_id))
                cache.put(commit_id, flavour, builder.build())
        except Exception as e:
            log.warning(f"Pre-warming failed for {vault_id}/{commit_id}: {e}")

    asyncio.create_task(_prewarm())
    return response
```

**Note:** In Lambda, `asyncio.create_task` may not complete before the Lambda invocation ends. Accept this — the pack will build on-demand for the first clone. This is better than delaying the push response.
