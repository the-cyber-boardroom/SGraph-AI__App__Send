## New File

### `sgraph_ai_app_send/lambda__user/storage/Storage__Paths.py`

Single source of truth for all storage key construction.

```python
import os

STORAGE__BASE       = 'sg-send__data'
STORAGE__VERSION    = 'sg-send-api__v1.0'
STORAGE__DEPLOYMENT = os.environ.get('SEND__DEPLOYMENT_ID', 'shared')

_ROOT = f'{STORAGE__BASE}/{STORAGE__VERSION}/{STORAGE__DEPLOYMENT}'

def path__transfer_meta(transfer_id: str) -> str:
    return f'{_ROOT}/transfers/{transfer_id[:2]}/{transfer_id}/meta.json'

def path__transfer_payload(transfer_id: str) -> str:
    return f'{_ROOT}/transfers/{transfer_id[:2]}/{transfer_id}/payload'

def path__transfer_prefix(transfer_id: str) -> str:
    return f'{_ROOT}/transfers/{transfer_id[:2]}/{transfer_id}/'

def path__vault_manifest(vault_id: str) -> str:
    return f'{_ROOT}/vault/{vault_id[:2]}/{vault_id}/manifest.json'

def path__vault_payload(vault_id: str, file_id: str) -> str:
    return f'{_ROOT}/vault/{vault_id[:2]}/{vault_id}/{file_id}/payload'

def path__vault_prefix(vault_id: str) -> str:
    return f'{_ROOT}/vault/{vault_id[:2]}/{vault_id}/'

def path__vault_zip(vault_id: str, content_hash: str) -> str:
    return f'{_ROOT}/cache/vault-zips/{vault_id}/{content_hash}.zip'

def path__vault_zip_prefix(vault_id: str) -> str:
    return f'{_ROOT}/cache/vault-zips/{vault_id}/'
```

Note: `_ROOT` is evaluated at import time. In Lambda, this is once per container lifetime — correct behaviour.

## Modified Files

### 1. `Transfer__Service.py`

| Method | Old path | New path |
|---|---|---|
| `meta_path()` | `transfers/{id}/meta.json` | `path__transfer_meta(id)` |
| `payload_path()` | `transfers/{id}/payload` | `path__transfer_payload(id)` |

Remove both methods; inline the `Storage__Paths` calls directly at call sites, or keep thin wrappers that delegate.

### 2. `Service__Vault__Pointer.py`

| Method / location | Old path | New path |
|---|---|---|
| `vault_manifest_path()` | `transfers/vault/{id}/manifest.json` | `path__vault_manifest(id)` |
| `vault_payload_path()` | `transfers/vault/{id}/{fid}/payload` | `path__vault_payload(id, fid)` |
| `list_files()` — `vault_folder` | `transfers/vault/{id}/{prefix}` | `path__vault_prefix(id) + prefix` |
| `list_files()` — `vault_prefix` | `transfers/vault/{id}/` | `path__vault_prefix(id)` |
| `delete_vault()` — `vault_prefix` | `transfers/vault/{id}/` | `path__vault_prefix(id)` |

### 3. `Service__Vault__Zip.py`

| Method | Old path | New path |
|---|---|---|
| `zip_storage_path()` | `vault-zips/{id}/{hash}.zip` | `path__vault_zip(id, hash)` |

In `Routes__Vault__Pointer.destroy__vault_id`, the zip cache cleanup uses `f'vault-zips/{vault_id}/'` — update to `path__vault_zip_prefix(vault_id)`.

### 4. `Service__Presigned_Urls.py`

| Change | Detail |
|---|---|
| Remove `s3_prefix` field | Deployment scope is now in `Storage__Paths` |
| `s3_key()` | Replace with `path__transfer_payload(transfer_id)` |

### 5. `Service__Vault__Presigned.py`

| Change | Detail |
|---|---|
| Remove `s3_prefix` field | Same as above |
| `s3_key()` | Replace with `path__vault_payload(vault_id, file_id)` |

## Test File Updates

## Test Strategy

No mocks required. All tests use `Storage_FS__Memory` as backend. Path change is transparent to test logic — tests that check full paths need values updated; tests that only check API responses (status codes, transfer IDs, etc.) need no changes.

Run full test suite after implementation to confirm no regressions.
