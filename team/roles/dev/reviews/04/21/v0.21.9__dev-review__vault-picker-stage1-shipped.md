## Component Capabilities

### States

| State | Trigger | UI shown |
|-------|---------|----------|
| `idle` | No saved credential in localStorage | Vault name input + Create button + paste-key toggle |
| `open` | Credential found in localStorage on mount, or after create/open | Name, truncated key, Copy/Share/Switch actions |

### Public API

```js
const picker = document.querySelector('sg-vault-picker');

picker.open('apple-river-1234')     // open a vault by key
picker.close()                      // return to idle state
picker.create('My vault')           // generate new key + open
picker.getVaultKey()                // → 'apple-river-1234'
picker.getVaultName()               // → 'My vault'
```

### Events

| Event | `detail` payload | When |
|-------|-----------------|------|
| `vault-created` | `{ vaultKey, vaultName }` | New vault generated |
| `vault-opened` | `{ vaultKey, vaultName }` | Existing vault opened (mount or explicit open) |
| `vault-key-copied` | `{ vaultKey }` | User clicked Copy key |
| `vault-share-requested` | `{ vaultKey }` | User clicked Share |
| `vault-closed` | `{}` | User clicked Switch vault |

### Attributes

| Attribute | Effect |
|-----------|--------|
| `vault-key="apple-river-1234"` | Pre-open vault on mount |
| `compact` | Removes description text — for toolbar embedding |
| `label` | Overrides displayed vault name |

### Auto-open

On `connectedCallback`, the component reads `localStorage` for saved credentials. If any exist, it auto-opens the most recently used vault. This means drop-in adoption in the infographic editor or PlaybookLM tools: add `<sg-vault-picker>` and returning users are automatically connected to their vault.

### Recovery popover

Shown once (suppressed by `sg-vault-picker:recovery-shown` in localStorage) immediately after vault creation. Offers Copy key and Download .txt. Not a modal — does not block interaction.

### Vault key generation

Keys are generated as `{adjective}-{noun}-{4-digit-number}` (e.g. `amber-cove-5821`) using `crypto.getRandomValues`. Word lists are embedded (24 adjectives × 24 nouns × 9000 numbers = ~5M combinations). Format matches what the vault server already accepts for human-readable IDs.

## Design Decisions

| Decision | Choice | Reason |
|----------|--------|--------|
| Storage key | `sg-vault-picker:credentials` | Namespaced to avoid collisions |
| Credential format | `{ vaultKey, vaultName, savedAt }` | savedAt enables MRU ordering |
| Multiple credentials | Stored as array, most-recent-first | Prepares for Stage 2 multi-vault list |
| Recovery flag | `sg-vault-picker:recovery-shown` | One-shot UX — never nag |
| Key truncation | `{word}-{word}-···` | Shows enough to identify which vault |
| No shadow DOM | Plain DOM | Consistent with all other SG/Send components |
| No framework | Vanilla JS IIFE | Consistent with all other SG/Send components |

## Stage 2 Readiness

The Stage 1 implementation is structured so Stage 2 (PasswordCredential API) requires only three changes:

1. Replace `loadCredentials()` / `storeCredential()` / `removeCredential()` with calls to `navigator.credentials.get/store`
2. Add hidden form fallback for Safari/Firefox in `_handleCreate()`
3. `_renderIdle()` gains `autocomplete="new-password"` on the vault-name input

The public API, events, and host-page integration do not change between Stage 1 and Stage 2.

## Integration — First Adoption Site

Per the dev review from 04/21, the best first adoption candidate is the infographic editor or PlaybookLM tools. Adding `<sg-vault-picker>` to either page:

```html
<!-- Load styles -->
<link rel="stylesheet" href="/_common/js/components/sg-vault-picker/sg-vault-picker.css">

<!-- Place picker -->
<sg-vault-picker id="vault-picker" compact></sg-vault-picker>

<!-- Load component -->
<script src="/_common/js/components/sg-vault-picker/sg-vault-picker.js"></script>

<!-- Listen for open event to wire vault into the tool -->
<script>
  document.getElementById('vault-picker').addEventListener('vault-opened', (e) => {
      const { vaultKey, vaultName } = e.detail;
      // Pass vaultKey to sg-vault.js open() here
  });
</script>
```

No other dependencies. Works today in any page that already loads the v0.3.2 assets.
