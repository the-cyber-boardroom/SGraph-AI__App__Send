### Critical Path Rewrite: CI Workflow Trigger

The existing `deploy-website.yml` triggers on:
```yaml
on:
  push:
    branches: [main]
    paths:
      - 'sgraph_ai__website/**'
```

In the new repo, this must become:
```yaml
on:
  push:
    branches: [main]
```

The `paths` filter made sense in the monorepo (to avoid triggering on Send code changes). In the new repo, all content is website content, so `paths` filter is unnecessary and would silently prevent deploys if forgotten.

This rewrite is documented in `05_path-rewrites-and-cross-references.md` but it's high-risk if skipped. The migration agent must verify this is applied before closing Phase 3.
