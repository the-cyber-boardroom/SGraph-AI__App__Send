## Code Fix That Landed (Not in Briefs)

### `fix(browse): intercept img[data-md-src]` — commit `1551a2b`

**What it fixes:** Vault markdown images showing as empty broken boxes.

**Root cause:** `markdown-parser-v031.js` outputs `<img data-md-src="url">` (no `src` attribute) to avoid HTTP 404s while blob URLs load asynchronously. The browse component only queried `img[src]`, missing these entirely.

**Fix:** `send-browse--v0.3.2.js` now queries both `img[src]` and `img[data-md-src]`; reads src from whichever is present; clears `data-md-src` after the blob URL is set.

**Test impact:** Any test that renders vault markdown with images should now show non-empty images. No tests expected to fail from this change.

**Implementation stream. Priority: HIGH. Estimated: 1 session.**

3. "Share as page" option when creating a share link → append `?view=pure` to the share URL
