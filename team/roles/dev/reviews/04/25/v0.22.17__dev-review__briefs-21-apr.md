### Stream 4: Villager Video Hardening (Doc 322) — Villager Team

**Effort estimate:** 1 week (focused refactoring)

This is a Villager deliverable, not Explorer. Notes for the Villager team:

**Phase 1: Decouple recorder from UI** (2–3 days)
- Extract recording logic into standalone `<sg-recorder>` Web Component
- Three sub-components: `<sg-recorder-audio>`, `<sg-recorder-video>`, `<sg-recorder-screen>`
- JS API: `configure()`, `start()`, `pause()`, `resume()`, `stop()` → blob
- Status: `getStatus()`, `getMetrics()`, `getHealthLog()`
- Events: `health-warning`, `recording-error`, `stream-lost`

**Phase 2: Web Worker investigation** (1 day)
- Can `MediaStream` be transferred to a Web Worker via `postMessage`? (browser support varies)
- Alternative: use `visibilitychange` + Wake Lock API if Web Worker transfer is not viable
- Document result in a brief for the Librarian

**Phase 3: Health checks** (1–2 days)
- Every 1000ms: check MediaRecorder state, bytes delta, frame delta, audio level, dropped frames, tab visibility, memory, stream active
- Visual metrics dashboard: duration, data size, data rate chart (KB/s), frame rate, audio VU, health indicator
- Error recovery: warn → partial save → attempt recovery → full log

**Phase 4: Testing** (1 day)
- Simulation tests: tab switch, camera disconnect, memory pressure, long recording (1 hour), rapid start/stop

- Risk acceptance vault: same approach. Create approval-tree.json, decision documents, index.html visualiser. Feature on sgraph.ai.

**These demos can be created TODAY without any new code.** They only depend on an agent session with a vault key and the existing vault browser. Recommend creating both as high-priority tasks in the next Ambassador agent session.

## Implementation Risk Register

| Risk | Severity | Mitigation |
|------|---------|------------|
| Perplexity Sonar citation format changes between API versions | HIGH — breaks `<sg-task-research>` and `<sg-task-evidence>` | Version-pin model; add integration test checking for source links |
| Web Worker MediaStream transfer not supported in current browsers | MEDIUM — video hardening falls back to `visibilitychange` | Research first, document result before committing to Web Worker path |
| Google OAuth single client ID across multiple domains not supported | HIGH — blocks app:data keychain from working on send.sgraph.ai | Resolve Open Decision #22 before any OAuth integration work |
| Vault webhook to GitHub Actions (vault-driven CI) | MEDIUM — SG/Send server must send an HTTP request on push | Confirm server-side hook design with Architect before building CI job |
| Credit auto-refund regulatory complexity | MEDIUM — Stripe refunds have jurisdiction-specific rules | Get legal input before shipping; start with manual refund as MVP |
| VNC performance unacceptable for document viewing | HIGH — invalidates entire VNC use case | Research before any VNC implementation investment |
| Shell document context across Shadow DOM boundaries | MEDIUM — `postMessage` or shared state bus needed | Prototype this specifically before building all 7 task components |
