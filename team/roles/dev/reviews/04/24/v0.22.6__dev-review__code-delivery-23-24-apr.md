### 10 Homepage Section Components

All 10 components follow the same pattern: `.js` + `.css` + `.html` at `v1/v1.0/v1.0.0/`. This is correct IFD practice.

**The `jsUrl` bug and fix:** This was a non-obvious failure mode of the `SgComponent` base class. The bug: `SgComponent` uses `this.constructor.jsUrl` (or falls back to its own `import.meta.url`) to resolve `.html`/`.css` sibling files. If a subclass does not declare `static jsUrl = import.meta.url`, the base class resolves from its own module path (tools.sgraph.ai host) rather than the component's actual location — causing 403 on template fetches. The fix (`d1fb32e`) adds this to all 10 components. Future authors need to know this.

**Recommendation:** Add a comment to `SgComponent` base class noting that `static jsUrl = import.meta.url` is required in every subclass, or add a runtime check with a clear error message.

**Layout shift prevention:** The `:not(:defined)` height reservations in `style.css` are correctly applied for all 10 components. These need to be maintained when component visual sizes change significantly.
