# Design: Chat Zone Redesign + Split Source + JS Syntax Highlighting

**Version:** v0.11.10
**Date:** 2026-03-05
**Status:** Implemented

## Context

The workspace chat zone had several UX problems:
1. System prompt hidden behind a collapse toggle
2. LLM response routed to the Result panel (should only show JS transform output)
3. No visibility into auto-appended context (Source/Script/Result checkboxes silently append)
4. Source panel needed splitting for upcoming HTML + Data workflows

## Design Principles

- **Many small JS components > fewer large components** — each component is focused and isolated
- **Event-driven architecture** — components communicate via the existing EventBus, not direct references
- **No surprises** — auto-appended context is always visible in the system prompt preview

## New Layout

### Bottom Row (chat zone - 3 columns + stats)
```
+------------------+------------------+------------------+-------+
| SYSTEM PROMPT    | PROMPT INPUT     | LLM OUTPUT       | STATS |
| [editable]       | Model: [...]     | [streaming resp] |       |

|  will be sent]   |                  | [Copy->Result]   |       |
+------------------+------------------+------------------+-------+
```

## Event Flow

```
llm-send          -> llm-orchestrator handles API call
llm-cancel        -> llm-orchestrator aborts request
llm-output-copy   -> llm-orchestrator routes to target panel
prompt-context-changed -> llm-system-prompt updates preview
system-prompt-changed  -> stored in localStorage
```

## JS Syntax Highlighting

Lightweight regex tokenizer (`JsHighlighter.highlight(code)`) with 7 token types:
- Keywords (purple), strings (green), numbers (orange), comments (grey italic)
- Builtins (yellow), operators (cyan), punctuation (light grey)

Integration: textarea has transparent text + caret-color; pre overlay behind it shows highlighted tokens. Both scroll together.
