# Dev Review — v0.2.0 UI Fixes: Upload Success Page, i18n, Navigation

**Version:** v0.10.6
**Date:** 2 March 2026
**Role:** Dev

---

## Summary

Fixed four UI regressions/issues in the v0.2.0 User UI:

1. **Upload success page** — ported v0.1.6's polished layout back to v0.2.0
2. **Language selector** — replaced `<select>` dropdown with sgraph.ai-style grid dropdown
3. **Navigation** — aligned max-width and styling with sgraph.ai (1200px)
4. **i18n workflow** — switched from runtime JSON loading to URL-based locale navigation

---

## 1. Upload Success Page (IFD Regression Fix)

**Problem:** v0.2.0 shipped with an older, flat `.result-panel .result-row` layout from v0.1.4 instead of v0.1.6's dark-theme-aware result panel that was live on send.sgraph.ai via v0.1.8.

**Root cause:** IFD blindspot during v0.2.0 consolidation — the renderResult() was based on the v0.1.4 base component, missing the v0.1.6 override that added the improved layout.

**Changes to `send-upload.js`:**

| Element | v0.2.0 (before) | v0.2.0 (after, matching v0.1.6) |
|---|---|---|
| Share link | `.result-row` flex layout | Standalone dark value box, full width, scrollable |
| Copy Link button | `btn-copy btn-sm` | `btn-primary btn-sm` (teal background) |
| Open Tab button | Plain `btn-sm` | `btn-secondary btn-sm` (outlined) |
| Toggle key | Separate div below | Inline in button row, `margin-left: auto` |
| Separate key panel | `.result-panel .result-row` | Contained dark panel with 12px border-radius |
| Timing display | Inline text spans | Visual bar chart with proportional widths |
| Token usage | Missing | Placeholder `#token-usage` div (hidden by default) |
| Download URLs | Hardcoded `/en-gb/` | Locale-aware via `_detectLocalePrefix()` |

## 2. Language Selector

**Before:** Native `<select>` element with custom CSS arrow — works but looks like a form control, not a navigation element.

**After:** Matches sgraph.ai's dropdown pattern:

- **Trigger button**: flag emoji + uppercase locale code + chevron SVG
- **Menu**: Two-column CSS grid, 400px min-width, positioned absolute
- **Active state**: Teal text + medium weight on current locale
- **English link**: Permanent "English" link shown on non-English pages
- **Navigation**: Click navigates to `/{locale}/` URL (no runtime switching)

**Files changed:**
- `send-locale.js` — complete rewrite (81 lines → 81 lines, different approach)
- `send-locale.css` — complete rewrite (native select styles → dropdown grid styles)

## 4. i18n Workflow Fix

**Before:**
- `I18n.setLocale()` called `loadLocale()` which `fetch()`ed `i18n/{code}.json`
- Runtime JSON loading on locale switch
- `localStorage` persistence of locale preference
- Locale list used old codes (`pt`, `de`, `fr` instead of `pt-pt`, `de-de`, `fr-fr`)

**After:**
- `I18n.navigateToLocale()` does `window.location.href = '/' + code + currentPath`
- Zero runtime JSON fetches — all translations pre-rendered at build time
- Locale detected from URL path (not localStorage)
- Locale list matches sgraph.ai website convention (16 locales, language-country codes)
- `setLocale()` is now a legacy wrapper that calls `navigateToLocale()`
