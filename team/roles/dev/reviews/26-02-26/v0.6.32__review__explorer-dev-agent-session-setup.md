### Priority 2: Build — One-Time Secret Link (Brief #1)

The one-time secret link is the highest-value build target from the new batch:

- **Why now:** It's the companion feature for data room key sharing (dogfooding SG/Send)
- **Backend only:** Server-side secret creation, storage, retrieval, and expiry
- **Builds on:** Existing `Transfer__Service` patterns, `Storage_FS` abstraction, `Routes__Transfers` structure
- **Zero new dependencies:** AES-256-GCM encryption already in `SendCrypto`
- **Acceptance criteria:** 15 criteria defined in the brief
- **Estimated effort:** M (medium) — 3-5 days for symmetric mode MVP

**Implementation approach:**
1. New schema: `Schema__Secret` (Type_Safe) — secret_id, encrypted_blob, view_count, max_views, expires_at, created_at
2. New service: `Service__Secrets` — create, retrieve (decrement count), expire, delete
3. New routes: `Routes__Secrets` — POST /secrets/create, GET /secrets/{id}, GET /s/{id}
4. New UI component: `<send-secret-create>` — text input, expiry selector, link display
5. New UI page: secret viewer at `/s/{id}` — displays decrypted secret, shows expiry state

### Priority 3: Build — Browser Storage Viewer (Brief #2)
