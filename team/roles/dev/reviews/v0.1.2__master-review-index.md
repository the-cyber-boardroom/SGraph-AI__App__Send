## Key alignment points captured

1. Use `Memory-FS` as storage abstraction; default single-file mode.
2. Use `OSBot-Fast-API-Serverless` patterns and route classes.
3. Use `Type_Safe` schemas and primitives (no Pydantic in service contracts).
4. Use `osbot-aws` wrappers for AWS interactions (no direct `boto3` usage).
5. Use two lambda functions:
   - public endpoints
   - admin endpoints (protected by built-in Fast_API auth)
6. Use event-per-file in `events/` folder (no single mutable `events.json` write hotspot).
7. Enforce max file size 100MB for MVP.
8. Remove in-app register-interest implementation from MVP and track external provider research instead.
9. Prefix all newly-created markdown files with current repo version.
