## 3. Individual File Storage (P1)

### Upload Pipeline Changes

Current: files -> zip -> encrypt (single blob) -> upload (single PUT)
New: files -> create tree JSON -> encrypt each file -> upload tree + N files

### Storage_FS Changes — Minimal

Use compound keys: `store_content(f"{transfer_id}/tree", tree_data)`. Works with current interface for both memory and S3 backends.

### Transfer Model Changes

Add `storage_mode` field (`single_blob` | `individual_files`) and `tree_blob_id`. All existing transfers default to `single_blob`. Backward compatible.

## 6. P0 Bug Fixes — 1 Hour Total

### Token Input Sanitisation (30 min)

```javascript
function sanitiseToken(input) {
    return input
        .trim()
        .replace(/[\u200B-\u200D\uFEFF]/g, '')  // zero-width chars
        .replace(/[\u2018\u2019]/g, "'")           // smart quotes
        .replace(/[\u201C\u201D]/g, '"')           // smart double quotes
        .replace(/\u00A0/g, ' ')                   // non-breaking space
        .replace(/[^\x20-\x7E]/g, '');             // strip all non-printable ASCII
}
```
