# v0.6.32 -- Dev Review: MCP Upload Fix — `data` Parameter + SGMETA Envelope

**Version:** v0.6.32
**Date:** 26 February 2026
**Role:** Developer (Explorer team)
**Type:** Bug fix + feature alignment

---

## Problem Statement

The MCP upload workflow (Claude.ai web -> SG/Send) had two issues:

1. **Corrupted files** — `transfers_upload` only exposes `transfer_id` and `access_token` as MCP tool parameters. The `Request` object (where `request.body()` lives) isn't an MCP tool argument. When Claude.ai calls the tool via MCP, the encrypted payload cannot reliably reach the server through the request body. Result: `size: 0` in upload response, empty/corrupt files.

2. **Missing filenames** — The browser wraps files in an SGMETA envelope (`[SGMETA\0][metadata_length][{"filename":"..."}][content]`) BEFORE encryption. The MCP client was encrypting raw bytes without SGMETA wrapping, so the download page couldn't extract the filename.

## Root Cause

**Corruption:** The `transfers_upload` FastAPI route reads the payload from `await request.body()`. When fastapi-mcp converts this route to an MCP tool, it only exposes explicit function parameters as tool arguments. The `Request` is a FastAPI internal — MCP clients have no way to pass data through it. The existing `unwrap_mcp_payload()` method assumed the MCP transport would put a `{"data": "<base64>"}` JSON body in the request, but this was unreliable.

**Filename:** The MCP client guide (`v0.6.20__guide__mcp-client-session-prompt.md`) didn't document the SGMETA envelope format. Without the envelope, the download page treats the entire decrypted blob as file content with no filename metadata.

## Fix

### Code Change (Routes__Transfers.py)

Added a `data: str = ''` parameter to `upload__transfer_id`:

```python
async def upload__transfer_id(self, transfer_id: Safe_Str__Id,
                                    request: Request,
                                    access_token: str = '',
                                    data: str = ''  # NEW — base64-encoded encrypted payload
                             ) -> dict:
    if data:                                    # MCP client: payload via tool parameter
        body = base64.b64decode(data)
    else:                                       # Browser/CLI: raw bytes in request body
        body = await request.body()
        body = self.unwrap_mcp_payload(body)    # Fallback for older MCP clients
```

**Impact on existing paths:**
- **Browser upload:** Unaffected. `data` defaults to `''` (falsy) → falls into `else` branch → identical to previous code
- **Old MCP JSON body:** Unaffected. Still handled by `unwrap_mcp_payload()` in the `else` branch
- **New MCP `data` param:** MCP clients can now reliably pass the encrypted payload as a base64-encoded tool argument
