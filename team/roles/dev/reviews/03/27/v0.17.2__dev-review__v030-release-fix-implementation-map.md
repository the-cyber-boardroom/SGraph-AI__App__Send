## Batch 1: Security + Default Change (Critical Path)

### U-030: Email leaks key in "link + key separate" mode — SECURITY FIX

**File:** `v0.3.0/_common/js/components/send-upload/send-upload.js`
**Line:** 582-586 (`_openEmailLink` method)

**Current code (BROKEN):**
```js
_openEmailLink() {
    if (!this.result) return;
    var body = "I've sent you an encrypted file via SG/Send.\n\nOpen this link to download:\n" + (this.result.combinedUrl || '');
    window.open('mailto:?subject=' + encodeURIComponent('Secure file for you') + '&body=' + encodeURIComponent(body));
}
```

**Fixed code:**
```js
_openEmailLink() {
    if (!this.result) return;
    var subject, body;
    if (this._shareMode === 'separate') {
        // SECURITY: only include the link, NOT the key — key must travel separately
        subject = 'Encrypted file via SG/Send';
        body = "I've sent you an encrypted file via SG/Send.\n\n" +
               "Open this link to download:\n" + (this.result.linkOnlyUrl || '') + "\n\n" +
               "You'll need the decryption key to open it — I'll send that separately.";
    } else if (this._shareMode === 'token' && this.result.friendlyKey) {
        // U-020: use friendly token URL when share mode is 'token'
        subject = 'Encrypted file via SG/Send';
        var tokenLink = this.result.tokenLink || this.result.combinedUrl || '';
        body = "I've sent you an encrypted file via SG/Send.\n\n" +
               "Open this link to view it:\n" + tokenLink;
    } else {
        subject = 'Encrypted file via SG/Send';
        body = "I've sent you an encrypted file via SG/Send.\n\n" +
               "Open this link to view it:\n" + (this.result.combinedUrl || '');
    }
    window.open('mailto:?subject=' + encodeURIComponent(subject) + '&body=' + encodeURIComponent(body), '_blank');
}
```

**Also fixes:** U-019 (email text), U-020 (use selected share mode), D-006b (open in new tab via `'_blank'`).

**Test:** Upload a file with each of the 3 share modes. Click "Email Link" on each Done page. Verify:
- **Token mode:** email contains friendly token URL, NOT raw hash URL
- **Combined mode:** email contains combined URL (has key)
- **Separate mode:** email contains link-only URL, does NOT contain the key

### U-006: Prevent browser default drop outside drop zone

**File:** `v0.3.0/_common/js/components/send-upload/upload-step-select/upload-step-select.js` (or the page-level HTML/JS)

**Add at document level (outside Shadow DOM):**
```js
// Prevent browser from opening dropped files
document.addEventListener('dragover', function(e) { e.preventDefault(); });
document.addEventListener('drop', function(e) { e.preventDefault(); });
```

**Note:** This must be added in the page-level JS, not inside a Shadow DOM component. The component's internal drop handler already calls `preventDefault()` — this catches drops that miss the zone.
