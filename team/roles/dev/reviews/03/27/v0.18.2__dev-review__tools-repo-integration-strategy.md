# Tools Repo Integration Strategy — sgraph.ai Website Redesign
**v0.18.2 | 27 March 2026**
**Role:** Developer
**Scope:** How sgraph.ai uses, leverages, maintains, and owns code from `SGraph-AI__Tools`

---

## The Principle: One Place for Shared Code

> "We should only have ONE place where shared code exists."
> — Dinis Cruz, 27 March 2026

The `SGraph-AI__Tools` repo (`https://github.com/the-cyber-boardroom/SGraph-AI__Tools`) is the canonical source for:
- Top and bottom banners (shared across all SGraph properties)
- Web Components used across multiple sites
- Client-side tools (SSH keygen, file hasher, file encryptor, key generator, vault browser, etc.)
- CSS design tokens and shared stylesheets

The sgraph.ai redesign is the **right moment** to make structural decisions about what lives where and how all sites consume shared code. Changes made now will propagate to all current and future SGraph properties.

## Consumption Model: latest vs pinned

The `SGraph-AI__Tools` repo publishes assets via:

| Channel | URL pattern | Use case |
|---------|------------|---------|
| **latest** | `https://tools.sgraph.ai/components/latest/...` | Development and preview environments |
| **pinned** | `https://static.sgraph.ai/v{version}/...` | Production — locked to a specific release |

**Rule for sgraph.ai (marketing site):** Production always pins to a specific version via `static.sgraph.ai`. Development and preview branches may use `latest`. This prevents a __Tools deploy from breaking the marketing site unexpectedly.

## Ownership: What sgraph.ai Owns as Master

The main sgraph.ai site is the **master definition** for certain shared assets. Changes here should propagate back to __Tools, not the other way round.

### sgraph.ai is master for:
| Asset | Location in this repo | What it defines |
|-------|----------------------|----------------|
| Site header / top banner | `sgraph_ai__website/_common/components/sg-site-header.js` | Logo, nav links, locale picker, "Already have a token?" link |
| Site footer / bottom banner | `sgraph_ai__website/_common/components/sg-site-footer.js` | Footer columns, copyright, links to all properties |
| Design tokens | `sgraph_ai__website/_common/css/design-tokens.css` | Colours, typography, spacing, radius — the Aurora theme |
| Locale picker | `sgraph_ai__website/_common/components/sg-locale-picker.js` | Language selector UI |

These components must be:
1. Built here first (this repo owns the master version)
2. Published to `SGraph-AI__Tools` as part of the redesign release
3. Consumed by all other SGraph properties via `static.sgraph.ai`

### __Tools repo is master for:
| Asset | What it defines |
|-------|----------------|
| Client-side tools (File Encryptor, Key Generator, etc.) | Tool logic, UI, and integration |
| `<sg-public-viewer>` Web Component | Core vault viewing component |
| Pyodide / WASM integrations | Python REPL, CLI, vault Pyodide |
| Shared utility functions | Crypto helpers, formatting, etc. |

## Web Components Architecture

**All UI on the new sgraph.ai should be Web Components with IFD versioning.**

### Component inventory and home

| Component | Home repo | IFD path | Status |
|-----------|-----------|---------|--------|
| `<sg-site-header>` | **this repo** → publish to __Tools | `v0/v0.1/v0.1.0/sg-site-header.js` | Build as part of redesign |
| `<sg-site-footer>` | **this repo** → publish to __Tools | `v0/v0.1/v0.1.0/sg-site-footer.js` | Build as part of redesign |
| `<sg-locale-picker>` | **this repo** → publish to __Tools | `v0/v0.1/v0.1.0/sg-locale-picker.js` | Extract from existing JS |
| `<sg-public-viewer>` | __Tools → consume here | Consume via static.sgraph.ai | Build in __Tools, consume here |
| `<sg-feature-card>` | **this repo** | Local only — not shared | Build as part of redesign |
| `<sg-share-mode-card>` | **this repo** | Local only — not shared | Build as part of redesign |
| `<sg-privacy-policy>` | **this repo** | Local only — not shared | Build as part of redesign |
| `<sg-how-it-works>` | **this repo** | Local only — not shared | Build as part of redesign |
| File Encryptor tool | __Tools | Consume via iframe or embed | Already live |
| Key Generator tool | __Tools | Consume via iframe or embed | Already live |
| Vault Browser tool | __Tools | Consume via iframe or embed | Already live |

### IFD versioning rules for components
All components follow: `v{major}/v{major}.{minor}/v{major}.{minor}.{patch}/component.js`

Example:
```
_common/components/
  v0/
    v0.1/
      v0.1.0/
        sg-site-header.js   ← pinned production version
    v0.2/
      v0.2.0/
        sg-site-header.js   ← next version
  latest/
    sg-site-header.js       ← symlink or copy to latest minor
```

Pages reference pinned versions:
```html
<script type="module"
  src="/_common/components/v0/v0.1/v0.1.0/sg-site-header.js">
</script>
```

## How sgraph.ai Consumes from __Tools

### Production (pinned)
```html
<!-- In sgraph_ai__website/_common/head.html or each page -->
<script type="module"
  src="https://static.sgraph.ai/v0.1.0/sg-site-header.js">
</script>
<script type="module"
  src="https://static.sgraph.ai/v0.1.0/sg-public-viewer.js">
</script>
```

### Development (latest — for feature branches only)
```html
<script type="module"
  src="https://tools.sgraph.ai/components/latest/sg-site-header.js">
</script>
```

### Version pinning in CI
The CI deploy workflow should:
1. Read component versions from `_common/component-versions.json`
2. Verify those versions exist on `static.sgraph.ai` before deploy
3. Fail the build if a pinned version is not available

`_common/component-versions.json`:
```json
{
  "sg-site-header": "0.1.0",
  "sg-site-footer": "0.1.0",
  "sg-locale-picker": "0.1.0",
  "sg-public-viewer": "0.1.0"
}
```

## "Tools" as a Top-Level Nav Item

`tools.sgraph.ai` must appear in the main sgraph.ai navigation as a first-class destination.

**Why:**
- The tools (File Encryptor, Key Generator, Vault Browser) directly prove SG/Send's zero-knowledge architecture
- They are free, client-side, and require no account — perfectly aligned with SG/Send's brand values
- Tools like the File Encryptor serve as security education: "use the same encryption SG/Send uses, right now, in your browser"

**Nav treatment:**
```html
<a href="https://tools.sgraph.ai" data-i18n="nav.tools"
   target="_blank" rel="noopener">Tools</a>
```

**Footer treatment:** `tools.sgraph.ai` listed alongside `send.sgraph.ai` and `qa.send.sgraph.ai` in the footer's property list.

**Homepage callout (optional):** A "Free tools, no account" section near the bottom of the homepage listing 3 key tools with one-sentence descriptions and links.
