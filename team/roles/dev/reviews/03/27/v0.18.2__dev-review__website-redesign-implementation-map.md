```html
<!--    A: Attackers get encrypted ciphertext. Useless without the key. -->

<!-- DevTools instruction -->

<!-- Section 8: Open source — audit it yourself -->
<!-- GitHub link -->
```

**Schema to add:**
```json
{
  "@type": "WebPage",
  "name": "Security — SG/Send",
  "about": {"@type": "Thing", "name": "Zero-Knowledge Encryption"}
}
```

**FAQ schema** — add 5 questions:
1. "Does the server ever see my files?" — No. Encryption happens in your browser.
2. "What happens if SG/Send's servers are breached?" — Attackers get ciphertext only.
3. "Can SG/Send be compelled to hand over files?" — Only ciphertext. Unreadable without the key.
4. "How is the encryption key shared?" — Via URL fragment, never sent to the server.
5. "Does SG/Send use cookies?" — No. Zero cookies, zero tracking, verifiable in DevTools.

### Implementation sketch
```javascript
class SgPublicViewer extends HTMLElement {
  connectedCallback() {
    const vaultId = this.getAttribute('vault-id');
    const fallbackSrc = this.getAttribute('fallback-src');

    // Attempt to load from send.sgraph.ai
    this._loadVault(vaultId)
      .then(content => this._render(content))
      .catch(() => this._renderFallback(fallbackSrc));
  }

  async _loadVault(vaultId) {
    // Fetch encrypted content from send.sgraph.ai/t/{token}/{id}
    // Demo key embedded in component config (intentionally public)
    // Decrypt using Web Crypto API
    // Return decrypted content list
  }

  _render(content) {
    // Render gallery/folder view matching send.sgraph.ai styles
  }

  _renderFallback(src) {
    this.innerHTML = `<img src="${src}" alt="${this.getAttribute('fallback-alt')}"
      class="sg-public-viewer__fallback">
      <p class="sg-public-viewer__fallback-note">
        <a href="https://send.sgraph.ai">Try the live product →</a>
      </p>`;
  }
}
customElements.define('sg-public-viewer', SgPublicViewer);
```
