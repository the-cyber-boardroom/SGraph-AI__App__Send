## 3. The LLM Provider Layer — Technical Detail

This is the most architecturally significant new code. Here's how I'd build it:

### 3.1 Provider Interface

```javascript
// llm-provider.js — base interface
class LLMProvider {
  constructor(name, config) {
    this.name = name;          // 'openrouter', 'ollama', 'anthropic', etc.
    this.config = config;      // { apiKey, baseUrl, model }
  }

  async testConnection() { /* returns { ok: boolean, model: string, error?: string } */ }

  async *streamCompletion(messages, options) {
    // Yields { type: 'text_delta', text: '...' } chunks
    // Yields { type: 'done', usage: { input_tokens, output_tokens } } at end
    // Throws on error
  }

  estimateTokens(text) { /* rough char/4 estimate */ }
}
```

### 3.2 OpenRouter Implementation (Phase 1 Primary)

```javascript
class OpenRouterProvider extends LLMProvider {
  constructor(apiKey, model = 'anthropic/claude-sonnet-4') {
    super('openrouter', { apiKey, baseUrl: 'https://openrouter.ai/api/v1', model });
  }

  async *streamCompletion(messages, options) {
    const response = await fetch(`${this.config.baseUrl}/chat/completions`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${this.config.apiKey}`,
        'Content-Type': 'application/json',
        'HTTP-Referer': 'https://workspace.sgraph.ai',
        'X-Title': 'SGraph Workspace'
      },
      body: JSON.stringify({
        model: this.config.model,
        messages,
        stream: true
      })
    });

    // Read SSE stream (OpenAI-compatible format)
    const reader = response.body.getReader();
    const decoder = new TextDecoder();
    // ... parse SSE chunks, yield deltas
  }
}
```

**Why OpenRouter first:**
- Explicitly supports browser CORS (designed for web apps)
- One API key → access to Claude, GPT-4, Gemini, Llama, Mistral, and 200+ models
- OpenAI-compatible API (well-documented, battle-tested)
- Avoids the CORS uncertainty of calling Anthropic/OpenAI directly from a browser

### 3.3 Ollama Implementation (Phase 1 Local)

```javascript
class OllamaProvider extends LLMProvider {
  constructor(model = 'llama3.2') {
    super('ollama', { baseUrl: 'http://localhost:11434', model });
  }

  async *streamCompletion(messages, options) {
    const response = await fetch(`${this.config.baseUrl}/api/chat`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ model: this.config.model, messages, stream: true })
    });
    // ... parse NDJSON stream
  }
}
```

**Why Ollama:**
- Runs locally — zero network, zero cost, maximum privacy
- No CORS issues (localhost)
- Good for testing and development
- Appeals to the privacy-conscious audience (SGraph's core market)

### 3.4 Phase 2 Direct Providers

After validating the workspace UX with OpenRouter + Ollama, add direct connections to individual providers. OpenRouter already covers all of them, so direct connections are an optimization (lower latency, no middleman), not a requirement.

## 4. Document Viewer — Technical Detail

### 4.1 Component Structure

```javascript
// document-viewer.js
class DocumentViewer extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
    this._content = null;       // raw content (decrypted)
    this._filename = null;      // for type detection
    this._renderType = null;    // 'markdown' | 'text' | 'html' (Phase 1)
    this._mode = 'rendered';    // 'rendered' | 'source'
    this._editable = true;      // both source and transform are editable
    this._version = 0;          // current version number
  }

  // Public API
  load(content, filename) { /* detect type (text/html/markdown), render with appropriate preview */ }
  clear() { /* empty state */ }
  getContent() { /* return current content (may have been edited) */ }
  getVersion() { /* return current version number */ }
  setEditable(bool) { /* toggle edit mode */ }
}
```

**Preview modes by type:**

| Type | Rendered View | Source View | Editable |
|------|-------------|------------|----------|
| **Markdown** | HTML-rendered markdown (default) | Raw markdown text in `<textarea>` | Yes — edit raw MD |
| **HTML** | Sandboxed `<iframe srcdoc="...">` (default) | Raw HTML in `<textarea>` | Yes — edit raw HTML |
| **Plain text** | Monospace `<pre>` | Same as rendered | Yes — edit in `<textarea>` |

### 4.2 Render Pipeline

```
1. User clicks file in vault-panel
   ↓
2. vault-panel emits 'file-selected' event via EventBus
   ↓
3. workspace-shell catches event, calls:
     vaultAPI.getFile(vaultKey, fileGuid)
   ↓
4. Encrypted blob received
   ↓
5. vaultCrypto.decrypt(privateKey, blob) → plaintext
   ↓
6. FileTypeDetect.detect(filename, mimeType) → renderType
   ↓
7. source-viewer.load(plaintext, filename)
     → renders in left panel based on renderType
```

### 4.3 Transformation Flow

```
1. User types prompt in llm-chat panel
   ↓
2. llm-chat reads source content from source-viewer.getContent()
   ↓
3. Constructs LLM messages:
     [{ role: 'system', content: 'You are a document transformation assistant...' },
      { role: 'user', content: `${prompt}\n\n---\n\n${documentContent}` }]
   ↓
4. Calls provider.streamCompletion(messages)
   ↓
5. Streams chunks into transform-viewer (right panel)
   ↓
6. On completion, enables "Save to Vault" button
   ↓
7. User clicks save → encrypt → store as new vault file → update index
```

## 12. Technical Concerns

### C-1: OpenRouter Streaming from Browser (MEDIUM)

**Risk:** While OpenRouter is designed for browser use, we should verify streaming works from a real domain early.
**Action:** Session 1 includes a quick streaming test from workspace.sgraph.ai origin.
**Mitigation:** If streaming has issues, fall back to non-streaming (wait for full response) temporarily.

### C-2: Large Document Handling (LOW — Phase 1 scope)

**Risk:** LLM context windows limit how large a document can be transformed.
**Observation:** Claude Sonnet 4 via OpenRouter supports 200K tokens (~150K words). Most documents fit easily.
**Action:** Show estimated token count before sending. Warn if document exceeds model's context window.

### C-3: Vault API Base URL Configuration (LOW)

**Risk:** Hard-coding `admin.send.sgraph.ai` makes the workspace inflexible.
**Solution:** Read API base URL from a config constant. Default to `admin.send.sgraph.ai` but allow override via URL parameter or localStorage config.

### C-4: Manifest Complexity (MEDIUM)

**Risk:** The source/transform folder pair model with manifest tracking adds UX complexity.
**Mitigation:** Good defaults — auto-create folder structure on first save, auto-increment version numbers, auto-link pairs in manifest. The user shouldn't have to think about manifests.
