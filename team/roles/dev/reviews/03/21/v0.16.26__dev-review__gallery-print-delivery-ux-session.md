## 1. Gallery Thumbnail System — Content-Hashed Folders

### What was built

The `_gallery.{hash}/` folder is now content-addressed: the folder name includes a hash computed from all file hashes in the upload. This was already computed in v0.2.13 (via `computeFolderHash(fileHashes)`) but was never used in the folder name — it was hardcoded as `_preview` then `_gallery`.

### Key design decision: Gallery folder only for gallery delivery

The `_gallery.{hash}/` folder is now **only generated when the user selects "Gallery" delivery**. When "Folder" or "Download" delivery is selected, the zip contains the raw files with zero modifications (apart from the existing dot-file and empty-directory filtering from the base zip compression).

This is architecturally correct: the folder view is a faithful representation of what's in the zip and what the user will download. Adding invisible processing folders would break that contract.

### Implementation

**v0.2.13** (`send-upload-v0213.js`):
- `previewDir` now set to `'_gallery.' + folderHash` (content-hashed)
- `_v0213_addPreviewToZip()` only called when `delivery === 'gallery'`

**v0.2.12** (`send-upload-v0212.js`):
- Same conditional — `addPreviewToZip()` gated on gallery delivery

**v0.2.9** (`send-download-v029.js`):
- Removed ALL folder tree filtering — `filterGalleryFolders()` function deleted
- Removed file list filtering override
- Folder view now shows everything exactly as it exists in the zip

**v0.2.8** (`send-download-v028.js`):
- `findPreviewFolder()` regex already handles `_gallery[^/]*` pattern — no changes needed

## 2. Print-Friendly Markdown — SgPrint Module

### What was built

A complete print pipeline for markdown files viewed in gallery or lightbox:

1. **Print button** in gallery lightbox and markdown viewer
2. **SgPrint module** (`sg-print.js`) — opens a professional A4 print preview window
3. **Print preview page** with toolbar (Print, Close buttons) and SG/Send branding
4. **A4 CSS** — proper `@page` margins, heading hierarchy, code blocks, tables, page breaks

### Print preview features

- **Screen view:** Dark toolbar with SG/Send branding + print/close buttons, white A4-styled document below
- **Print view:** Clean document with header (title + timestamp) and footer (SGraph AI branding), proper page margins
- **Cross-browser:** Removed `@page { size: A4 }` which was limiting Chrome's paper size options

### Architecture

```
User clicks Print → SgPrint.printMarkdown(html, filename)
  → Opens new window with PRINT_STYLES + toolbar
  → User clicks Print button → window.print()
  → Browser's native print/save-as-PDF dialog
  → Clean A4 output with headers/footers
```

The `SgPrint` module was extracted as a standalone utility in `v0.2.0/_common/js/sg-print.js`, symlinked into v0.2.3 and v0.2.13. This follows IFD principles — the module is independently deployable and reusable across all UI versions.

## 3. Unified Delivery UX — Three Always-Visible Options

### What was built

The upload Step 2 "Choose delivery" was redesigned to always show three options:

| Card | Route | Description |
|------|-------|-------------|
| **Browse** | `/browse/` | Folder tree view — shows everything in the zip |
| **Gallery** | `/gallery/` | Visual gallery with thumbnails, lightbox, metadata |
| **Download** | `/download/` | Direct download (original behaviour) |

### Key changes

1. **Always visible:** All three cards shown regardless of file type (single file or folder)
2. **Smart defaults:** Gallery pre-selected for folders, Download for single files
3. **Consistent order:** Always Browse → Gallery → Download (left to right)
4. **Mode naming:** Standardised on "browse"/"gallery"/"download" internally (was inconsistent)
5. **Card hover:** Default card dims slightly on hover to indicate other options available

### Stepper and navigation fixes

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| Step clicks broken on Done page | `setupDynamicListeners()` not called when step circles change | Added hook in `setupDynamicListeners` for dynamic step element binding |
| Delivery selection lost on Back | `_v023_selectedDelivery` cleared when re-rendering Step 2 | Preserved selection state across renders |
| Friendly key collision on Back | Same `transfer_id` reused after navigating back | Regenerate friendly key on back navigation |
| Carousel message panel height | Stats column taller than message panel | Flex stretch on message panel container |

## Technical Decisions

| Decision | Rationale |
|----------|-----------|
| Content-hashed `_gallery.{hash}/` folder | Deduplication potential, cache-friendly, hash was already computed |
| Gallery folder only on gallery delivery | Folder view must faithfully represent what's downloaded |
| No folder view filtering | Users should see exactly what's in the zip — `_gallery` is real content |
| `SgPrint` as shared module in `v0.2.0/_common/` | Reusable across all UI versions via symlink |
| `@page` without `size: A4` | Chrome restricts paper options when size is locked |
| Regenerate friendly key on Back | Prevents transfer ID collision with server |
