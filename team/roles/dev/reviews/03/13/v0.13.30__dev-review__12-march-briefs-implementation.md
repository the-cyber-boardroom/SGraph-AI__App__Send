## P0 Bug: 5MB Upload Failure (IMMEDIATE)

- The encrypted payload (base64 + AES-256-GCM overhead) exceeds the 6MB Lambda payload limit
- Fix: client-side size check before upload. If payload > ~5.5MB after encryption, route through chunked upload path
- Location: browser upload JS (in `sgraph_ai_app_send__ui__user/`)
- This is a ~20 line fix but critical -- silent failure makes users think the service is broken
- Should be fixed today, before any architecture work begins

## Implementation Streams

### Stream 1: Vault Architecture (8-12 weeks)

Phase order (strict dependencies):

1. **Bare vault unification (2-3 weeks)** -- one format for server + clone. Security audit in parallel.
2. **Branch model (3-4 weeks)** -- simulation v6 is the spec. Build init, commit, push first. Then clone, merge, conflict.
3. **CLI remotes (2-3 weeks)** -- pluggable backend abstraction. Local folder + SG/Send API first. S3/URL/zip later.
4. **Signed commits + diff + log (1-2 weeks)** -- completes "lightweight encrypted Git" claim.

The simulation v6 document is directly translatable to test cases. The QA predictions table (file counts at each step) becomes the acceptance test suite.
