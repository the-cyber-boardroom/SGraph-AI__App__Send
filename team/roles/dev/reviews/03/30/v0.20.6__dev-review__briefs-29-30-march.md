### What the team needs to know

**New commands available in sgit-ai (from this commit):**

```bash
sgit init                           # auto-generates token (e.g. coral-equal-1234)
sgit init coral-equal-1234          # init with specific edit token
sgit clone coral-equal-1234         # clone from SGit-AI server
sgit clone dawn-haven-6034          # import from SG/Send transfer (generates new edit token)
sgit clone vault://coral-equal-1234 # vault:// prefix stripped — same as above
sgit share                          # publish HEAD to SG/Send, returns browser URL
sgit share --rotate                 # generate new share token (old URL becomes stale)
```

**The two-token invariant is critical:**
- `edit_token ≠ share_token` — always different
- Sharing a share token does not give write access to the vault
- Losing the edit token means losing write access (no recovery path)

The `sgit clone <share_token>` → SG/Send transfer lookup uses `SHA256(share_token)[:12]` as the transfer ID. This requires SG/Send's `/transfers/info/{id}` endpoint (which exists). No changes needed to the SG/Send server.

## 5. Code Quality Notes

### Simple Token Key Derivation

The PBKDF2+HKDF derivation in `Vault__Crypto.py` adds a new code path alongside the existing EC key derivation. The backward compatibility note says "existing vaults are unaffected" — this should be explicitly confirmed in the integration test suite (i.e., an existing vault cloned without `mode: simple_token` still works after the upgrade).

### sg-tree: Type_Safe

The sg-tree component is frontend JS, not Python. The `Type_Safe` pattern from osbot-utils does not apply here. Use plain JS objects with JSDoc type annotations for the canonical data format. Do not introduce TypeScript or any type-checking toolchain.

### Video Editor: Non-Destructive Model

The JSON transformation model should be serialised with `Type_Safe` schemas on the Python side if there are any server-side operations (e.g., batch captioning via LLM). For the client-only operations, plain JSON with documented schema is sufficient.
