# Workspace v0.1.1 — Execution Bundles

**Version:** v0.11.10
**Date:** 2026-03-05
**Status:** Implemented (Phase 1 + Phase 2), Phase 3 deferred

## Summary

v0.1.1 adds **Execution Bundles** to the SGraph Workspace — complete state snapshots that enable time travel and forking. Users can save the full workspace state (all panels, config, prompts, response) as a bundle, then load any previous bundle to restore the workspace to that point.

Built using **IFD methodology**: 5 new files, zero v0.1.0 modifications.

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| Surgical IFD overrides only | v0.1.0 is frozen; v0.1.1 patches via `customElements.get()` prototype patching |
| Bundles stored in vault `.bundles/` folder | Encrypted at rest, same lifecycle as vault content |
| `tree.json` index file | Lightweight metadata for listing without decrypting every bundle |
| `parent_id` field on bundles | Enables fork tracking — load old bundle, make changes, save = new branch |
| vault:// references for large content | Source/data files reference vault paths instead of duplicating inline |
| Scripts always inline | Typically <50KB, avoids vault round-trip |
| Screenshots capped at 3, max 2MB each | Prevents bundle bloat from base64 images |

## Bugs Discovered and Fixed

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| `ReferenceError: WorkspaceShell is not defined` | v0.1.0 wraps class in IIFE | Use `customElements.get('workspace-shell')` |
| `llmOutput.getTextContent is not a function` | v0.1.0 exposes `getText()` | Changed to `getText()` |
| `vault.writeFile is not a function` | SGVault API is `addFile()` | Changed in both execution-history and bundle-manager |
| `Already exists: executions` uncaught promise | `createFolder()` called without `await` | Added `await` inside try/catch |
| Save State button not appearing | Shell already rendered before override loads | Added immediate `injectBundleUI()` + duplicate guard |

## IFD Layer Loading Order (index.html)

```
Layer 0: CSS (v0.1.0)
Layer 1: Libraries — sg-vault, sg-send (v0.1.0)
Layer 2: Core — workspace-init, messages-service (v0.1.0)
Layer 3: Panels — document-viewer, script-editor, vault-panel (v0.1.0)
Layer 4: LLM — connection, system-prompt, prompt-input, output, orchestrator (v0.1.0)
Layer 5: Shell — workspace-shell, execution-history (v0.1.0)
Layer 6: v0.1.1 Bug Fixes — execution-history override
Layer 7: v0.1.1 Additions — bundle-manager, bundle-list, workspace-shell override
```

## What's Not Included (Phase 3 — Deferred)

- **Tree Visualisation**: Visual graph of bundle relationships showing branches/forks
- **Auto-save on LLM response**: Event listener exists but not wired to auto-trigger
- **Bundle diff view**: Comparing two bundles side-by-side
- **Bundle export/import**: Sharing bundles between vaults
