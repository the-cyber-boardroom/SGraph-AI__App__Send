## Context

The SGraph Workspace currently uses an "LLM-as-runtime" model: HTML is sent to the LLM, and modified HTML comes back. The brief defines a paradigm shift to "LLM-as-author": the LLM writes JavaScript that transforms HTML locally in the browser. This is more token-efficient, deterministic, and scales to pipelines.

Phase 1 delivers: JS editing, JS execution against HTML, LLM-generated JS, prompt composer with artifact toggles, and execution history saving.

**No backwards compatibility required** — this is Explorer-stage code, we can break existing interfaces freely.

## Files to Create

### 1. `components/js-executor/js-executor.js` (NEW)

Utility module for safely executing JS against HTML using sandboxed iframe + postMessage.

- `sandbox="allow-scripts"` (no `allow-same-origin`) — iframe cannot access parent DOM/localStorage
- Injects source HTML + user JS wrapped in try/catch
- User JS can modify DOM in place (returns outerHTML) or return a value (string/JSON)
- Result posted via `parent.postMessage()`, parent listens and resolves promise
- Timeout protection (default 10s)

Public API: `window.sgraphWorkspace.executeJS(sourceHtml, userScript, timeoutMs) -> Promise<{ data, resultType, error }>`

### 2. `components/script-editor/script-editor.js` (NEW)

Web Component for editing and executing JavaScript transformation code.

- Header bar: filename label, "Run" button, "Save" button
- Monospace textarea for code editing
- Error output area (shown on execution failure)
- Run: executes script via js-executor against source HTML, displays result in transform viewer
- Save: writes .js file to current vault folder
- Listens for `llm-response-js` event to auto-load LLM-generated code
- Ctrl+Enter shortcut to run

### 3. `components/execution-history/execution-history.js` (NEW)

Vault-based execution history storage.

- Each execution saved as JSON in vault `/executions/` folder
- Records: prompt, system prompt, model, provider, tokens, latency, response, attachments used
- File naming: `/executions/{timestamp}-{short-id}.json`

## Data Flow

```
User prompt in llm-chat
  -> Prompt composer adds selected artifacts
  -> "Generate JS" mode sets JS system prompt
  -> LLM returns JavaScript code
  -> Execution history saved to vault
  -> Event 'llm-response-js' emitted
  -> script-editor receives code, user reviews/edits
  -> User clicks Run (or Ctrl+Enter)
  -> js-executor creates sandboxed iframe
  -> Source HTML + user JS injected and executed
  -> Result posted back via postMessage
  -> Transform document-viewer displays result
  -> User can save script to vault
```
