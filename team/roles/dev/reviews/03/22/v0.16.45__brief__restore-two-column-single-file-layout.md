# Brief: Restore Two-Column Single-File Download Layout

**Version:** v0.16.45
**Date:** 2026-03-22
**From:** Dev (Explorer team — review session)
**To:** Dev (Claude Code session)
**Priority:** P1 — visible UX regression from v0.2.x

---

## The Problem

v0.3.0's single-file download view is a bare viewer with a thin header bar. The v0.2.x version had a rich two-column layout with file metadata, a transparency panel, share section, and the preview side-by-side. This is a significant UX regression — the transparency panel especially is core to the product's trust story.

**Compare:**
- v0.2.x (`main.send.sgraph.ai`): Two columns — left has file info, Save Locally, transparency panel, timing; right has the content preview. Resizable divider.
- v0.3.0 (`dev.send.sgraph.ai`): Thin header (filename, size, Share, Print, Save locally) and content fills the viewport. No metadata, no transparency, no timing.

---

## Root Cause

v0.3.0's `_renderSingleFile()` (line 586 of `send-download.js`) creates a bare `<send-viewer>` element, which was designed as a lightweight tab content renderer for use inside `send-browse`. It was never meant to be the primary single-file download experience.

```javascript
// Current v0.3.0 — too minimal
_renderSingleFile() {
    const viewer = document.createElement('send-viewer');
    viewer.fileBytes   = this.decryptedBytes;
    viewer.fileName    = this.fileName;
    viewer.fileType    = this._renderType;
    viewer.fileText    = this.decryptedText;
    viewer.downloadUrl = this._downloadUrl;
    viewer.transferId  = this.transferId;
    this.innerHTML = '';
    this.appendChild(viewer);
}
```

## Implementation Plan

### Approach: Build a two-column wrapper in `send-download.js` that embeds `send-viewer` as the content panel

Do NOT modify `send-viewer.js` — it should stay lightweight for browse tabs. Instead, modify `_renderSingleFile()` in `send-download.js` to build the two-column layout itself, using `send-viewer` for the right panel.

### Step 1: Modify `_renderSingleFile()` in `send-download.js`

Replace the current implementation (line 586) with a two-column layout:

```javascript
_renderSingleFile() {
    const filename  = this.fileName || 'download';
    const type      = this._renderType;
    const info      = this.transferInfo;
    const url       = this._downloadUrl || '';
    
    // Type badge label
    const badgeLabel = (type === 'code' && typeof FileTypeDetect !== 'undefined') 
        ? FileTypeDetect.getLanguage(filename) : type;
    const sizeStr    = info ? this._fmtBytes(info.file_size_bytes) : '';
    const uploadDate = info ? this._fmtTime(info.created_at) : '';
    const downloads  = info ? (info.download_count || 0) : 0;
    
    // Build the two-column layout
    this.innerHTML = `
        <div class="sf-layout">
            <div class="sf-details">
                <!-- File info -->
                <div>
                    <h3 class="sf-filename">${this._esc(filename)}</h3>
                    <div class="sf-meta">
                        <span class="sf-badge">${this._esc(badgeLabel)}</span>
                        <span class="sf-size">${this._esc(sizeStr)}</span>
                    </div>
                    <div class="sf-info">
                        <div>Uploaded: ${this._esc(uploadDate)}</div>
                        ${downloads > 0 ? `<div>Downloaded ${downloads} time${downloads !== 1 ? 's' : ''}</div>` : ''}
                    </div>
                </div>
                
                <!-- Actions -->
                <button class="btn btn-primary" id="sf-save" style="width: 100%;">
                    ${SendIcons.DOWNLOAD} Save Locally
                </button>
                <div style="display: flex; gap: 0.5rem;">
                    <button class="btn btn-sm btn-secondary" id="sf-copy-link" style="flex: 1;">
                        ${SendIcons.LINK_SM || '🔗'} Copy Link
                    </button>
                    <button class="btn btn-sm btn-secondary" id="sf-email">
                        ${SendIcons.MAIL || '✉'}
                    </button>
                    <button class="btn btn-sm btn-secondary" id="sf-print">
                        ${SendIcons.PRINT || '🖨️'}
                    </button>
                </div>
                
                <!-- Share section -->
                ${url ? `
                <div class="sf-share">
                    <div class="sf-share-label">Download link</div>
                    <div style="display: flex; gap: 0.5rem;">
                        <input type="text" class="input" value="${this._esc(url)}" readonly 
                               style="flex: 1; font-size: 0.75rem; font-family: monospace;">
                        <button class="btn btn-sm btn-secondary" id="sf-copy-url">Copy</button>
                    </div>
                </div>` : ''}
                
                <!-- Transparency panel -->
                <send-transparency id="transparency-panel"></send-transparency>
                
                <!-- Timing -->
                ${this._renderTimings()}
                
                <!-- Send another -->
                <div style="text-align: center; margin-top: 0.5rem;">
                    <a href="${window.location.origin}/${typeof I18n !== 'undefined' ? I18n.locale : 'en-gb'}/" 
                       style="color: var(--accent); text-decoration: none; font-size: var(--text-sm);">
                        Send another file
                    </a>
                </div>
            </div>
            
            <!-- Resizable divider -->
            <div class="sf-divider" id="sf-divider"></div>
            
            <!-- Content preview -->
            <div class="sf-preview" id="sf-preview"></div>
        </div>
    `;
    
    // Insert send-viewer into the preview panel
    const previewPanel = this.querySelector('#sf-preview');
    const viewer = document.createElement('send-viewer');
    viewer.fileBytes   = this.decryptedBytes;
    viewer.fileName    = this.fileName;
    viewer.fileType    = this._renderType;
    viewer.fileText    = this.decryptedText;
    viewer.downloadUrl = this._downloadUrl;
    viewer.transferId  = this.transferId;
    // Tell viewer to render without its own header (it's wrapped)
    viewer.embedded    = true;
    previewPanel.appendChild(viewer);
    
    this._setupSingleFileListeners();
}
```

### Step 3: Add `_setupSingleFileListeners()` method

```javascript
_setupSingleFileListeners() {
    // Save locally
    const saveBtn = this.querySelector('#sf-save');
    if (saveBtn) saveBtn.addEventListener('click', () => {
        this._saveFile(this.decryptedBytes, this.fileName || 'download');
    });
    
    // Copy link
    const copyLink = this.querySelector('#sf-copy-link');
    if (copyLink) copyLink.addEventListener('click', async () => {
        try {
            await navigator.clipboard.writeText(this._downloadUrl || '');
            copyLink.textContent = 'Copied!';
            setTimeout(() => { copyLink.innerHTML = `${SendIcons.LINK_SM || '🔗'} Copy Link`; }, 2000);
        } catch (_) {}
    });
    
    // Copy URL from input
    const copyUrl = this.querySelector('#sf-copy-url');
    if (copyUrl) copyUrl.addEventListener('click', async () => {
        try {
            await navigator.clipboard.writeText(this._downloadUrl || '');
            copyUrl.textContent = 'Copied!';
            setTimeout(() => { copyUrl.textContent = 'Copy'; }, 2000);
        } catch (_) {}
    });
    
    // Email
    const emailBtn = this.querySelector('#sf-email');
    if (emailBtn) emailBtn.addEventListener('click', () => {
        window.location.href = `mailto:?subject=Shared file via SG/Send&body=${encodeURIComponent(this._downloadUrl || '')}`;
    });
    
    // Print
    const printBtn = this.querySelector('#sf-print');
    if (printBtn) printBtn.addEventListener('click', () => window.print());
    
    // Resizable divider
    this._setupDividerDrag();
}
```

### Step 4: Add `_setupDividerDrag()` method

The v0.2.0 implementation (see line 276 of `send-download-v021.js`) saved the width to localStorage:

```javascript
_setupDividerDrag() {
    const divider = this.querySelector('#sf-divider');
    const layout  = this.querySelector('.sf-layout');
    if (!divider || !layout) return;
    
    let dragging = false;
    
    divider.addEventListener('mousedown', (e) => {
        dragging = true;
        e.preventDefault();
        divider.style.background = 'var(--accent, #4ECDC4)';
    });
    
    document.addEventListener('mousemove', (e) => {
        if (!dragging) return;
        const rect  = layout.getBoundingClientRect();
        const width = Math.max(200, Math.min(e.clientX - rect.left, rect.width - 200));
        layout.style.gridTemplateColumns = `${width}px 4px 1fr`;
        try { localStorage.setItem('sg-send-split-width', width); } catch (_) {}
    });
    
    document.addEventListener('mouseup', () => {
        if (dragging) {
            dragging = false;
            divider.style.background = 'transparent';
        }
    });
}
```

### Step 6: Modify `send-viewer.js` to support `embedded` mode

When `viewer.embedded = true`, the viewer should render without its own header bar (since the two-column layout provides the actions). This is a small change:

In `_build()`:
```javascript
_build() {
    // ... existing CSS injection ...
    
    if (this.embedded) {
        // No header — just content (used when embedded in single-file two-column layout)
        this.innerHTML = `<div class="sv-content" id="sv-content" style="height: 100%; overflow: auto;"></div>`;
    } else {
        // Full header (used standalone or in browse tabs)
        this.innerHTML = `<div class="sv-container">...existing header + content...</div>`;
    }
    
    this._renderContent();
    if (!this.embedded) this._setupListeners();
}
```

### Step 7: Route-aware behaviour (optional but recommended)

Following v0.2.11's pattern, consider making the layout route-aware:

- `/browse/` → two-column layout (the default, described above)
- `/view/` and `/v/` → two-column layout (same as browse)
- `/download/` → auto-download + the existing `_renderDownloadMode()` (already implemented, keep as-is)
- `/gallery/` → two-column layout for single files (can't gallery a single file)

This means `_renderComplete()` just needs to keep routing to `_renderDownloadMode()` for the `/download/` route and to `_renderSingleFile()` (the new two-column version) for everything else. The current routing logic already does this correctly — `_resolveViewMode()` returns `'single'` for non-zip files when route isn't `/download/`.

## Acceptance Criteria

1. Single-file download at `/browse/` or `/view/` shows two-column layout
2. Left panel has: filename, type badge, size, upload timestamp, download count, Save Locally, Copy Link, Email, Print, transparency panel, decrypt timing, "Send another"
3. Right panel shows the content preview (reusing `send-viewer` in embedded mode)
4. Divider is draggable, width persists across loads
5. `/download/` route still uses the existing auto-download confirmation (no change)
6. `send-viewer` still works standalone in browse tabs (embedded=false, default)
7. Responsive: stacks vertically on narrow screens
8. No regression to gallery, browse, or upload flows

---

## Key Principle

**`send-viewer` stays lightweight** — it's a content renderer, not a download experience. The two-column layout with metadata, transparency, and share belongs in `send-download.js` (the orchestrator), not in the viewer. This mirrors how `send-gallery` and `send-browse` are self-contained views but created and configured by the orchestrator.
