## Implementation Streams

### Stream 1: SGit CLI Extensions (P0 — Core Product)

**Documents:** 56, 68, 72

**New CLI commands needed:**
| Command | Purpose | Effort |
|---------|---------|--------|
| `sgit-ai dump --local` | JSON dump of local vault state | Small |
| `sgit-ai dump --remote` | JSON dump of server vault state | Small |
| `sgit-ai diff-state` | Compare local vs server, report divergences | Medium |
| `sgit-ai diff` | Generate Git patch from local changes | Small |
| `sgit-ai apply` | Apply patch from vault to Git repo | Medium |
| `sgit-ai vault-from-zip` | Operate on zip-based vault (no server) | Medium |

- `.sgitignore` support (hard-coded defaults + user-configurable, like `.gitignore`)

**Effort estimate:** 3–4 sessions

### Stream 3: Email Pipeline (P1 — Needed for Announcements)

**Document:** 70

Three-agent pipeline:
1. **Composer:** Reads templates + CRM profiles → generates EML files → `outbox/`
2. **Reviewer:** Renders EML, checks tone/accuracy → `approved/`
3. **Sender:** Reads from `approved/`, sends via SES → `sent/`

Infrastructure: Move from WorkMail to SES. Direct boto3 code (MCP rejected for batch).

Vault-based CRM structure:
```
crm-vault/
  users/           # From Stripe customer list
  early-access/    # From signup list
  templates/       # HTML with vars
  outbox/          # Generated EML
  approved/        # Reviewed & ready
  sent/            # With delivery metadata
```

**Effort:** 2 sessions
