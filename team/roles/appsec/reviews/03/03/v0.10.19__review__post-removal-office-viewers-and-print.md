## 3. Remaining Attack Surface Assessment

After removal, the preview renderers that remain are:

| Renderer | Sandbox | Scripts? | CDN? | XSS Risk | Notes |
|---|---|---|---|---|---|
| Markdown | `allow-same-origin` (blob URL) | Blocked | No | **Low** | Custom parser escapes all text. No HTML passthrough. |
| Image (raster) | N/A (blob URL `<img>`) | N/A | No | **None** | Binary blob, no HTML context. |
| SVG | `""` (empty sandbox, data: URI) | Blocked | No | **Low** | SVG text is `escapeHtml()`-ed before iframe injection. Sandbox blocks all scripts. |
| PDF | N/A (blob URL, native viewer) | N/A | No | **None** | Browser's built-in PDF viewer. Sandboxed by browser. |
| Code | N/A (escaped `<pre>` in DOM) | N/A | No | **Low** | Custom highlighter. Source text escaped via `esc()` before any HTML wrapping. |
| Audio | N/A (blob URL `<audio>`) | N/A | No | **None** | Binary blob. Filename escaped in label. |
| Video | N/A (blob URL `<video>`) | N/A | No | **None** | Binary blob. No user-controlled HTML. |
| Zip browser | Mixed (reuses above renderers) | Blocked | No | **Low** | Zip entry filenames escaped. Inner previews reuse the above safe renderers. |
| Text | N/A (`textContent` assignment) | N/A | No | **None** | Uses `.textContent`, not `.innerHTML`. |

### Key Security Properties Maintained

1. **No third-party CDN scripts** — zero external JavaScript dependencies loaded at runtime
2. **No `allow-scripts` in any iframe sandbox** — no preview iframe permits script execution
3. **All user-controlled strings escaped** — `escapeHtml()` applied consistently before HTML insertion
4. **MarkdownParser escapes all text** — custom parser with `_escape()` on every text node
5. **SVG rendering uses `escapeHtml()`** — SVG source treated as text, not parsed as XML
6. **Blob URLs for same-origin content** — markdown, images, PDFs use blob URLs (revokeObjectURL on cleanup for images/PDFs)

### Residual Items (Low/Info)

- **OV-06 (Low):** Markdown blob URL iframe with `allow-same-origin` — if the MarkdownParser ever adds HTML passthrough, this would be exploitable. Current parser is safe.
- **OV-07 (Info):** Markdown blob URLs are not revoked on re-render (memory leak, no security impact).

## 4. Verdict

**The removal eliminates all material XSS attack vectors identified in the threat model.** The remaining renderers follow a consistent security pattern: escape all user content, sandbox all iframes without `allow-scripts`, use no external CDN dependencies.

**No blocking security issues remain for deployment.**
