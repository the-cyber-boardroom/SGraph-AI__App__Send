# SGraph Send -- MCP Client Guide

You are connected to **SGraph Send**, a zero-knowledge encrypted file sharing service at [send.sgraph.ai](https://send.sgraph.ai). Files are encrypted client-side (AES-256-GCM) before upload. The server only stores encrypted ciphertext. Decryption keys never reach the server.

The MCP server exposes the existing REST API as MCP tools via `fastapi-mcp`. Each FastAPI route becomes an MCP tool automatically. Transport is **HTTP** (JSON-RPC over POST to `/mcp`) -- works on Lambda, no SSE required.

## 1. Architecture

Two Lambda functions, each with its own MCP endpoint:

| Lambda | URL Placeholder | Purpose | MCP Tools |
|--------|----------------|---------|-----------|
| **User** | `{USER_LAMBDA_URL}` | Transfers, presigned URLs, vault (read) | File upload/download workflow |
| **Admin** | `{ADMIN_LAMBDA_URL}` | Tokens, keys, vault (write), users, analytics | Management and key registry |

Auth header: `x-sgraph-access-token: {YOUR_ACCESS_TOKEN}`

## 2. Connection Setup

### Claude Code MCP Configuration

Add to your Claude Code MCP settings (`.claude/mcp.json` or equivalent):

```json
{
  "mcpServers": {
    "sgraph-send-user": {
      "type": "http",
      "url": "{USER_LAMBDA_URL}/mcp",
      "headers": {
        "x-sgraph-access-token": "{YOUR_ACCESS_TOKEN}"
      }
    },
    "sgraph-send-admin": {
      "type": "http",
      "url": "{ADMIN_LAMBDA_URL}/mcp",
      "headers": {
        "x-sgraph-access-token": "{YOUR_ACCESS_TOKEN}"
      }
    }
  }
}
```

Replace `{USER_LAMBDA_URL}`, `{ADMIN_LAMBDA_URL}`, and `{YOUR_ACCESS_TOKEN}` with actual values. If you only need file transfers, the User server alone is sufficient.

## 3. Available MCP Tools

### User Lambda Tools (transfers, presigned, vault)

| Tool | Method | Route | Description |
|------|--------|-------|-------------|
| `transfers_create` | POST | `/transfers/create` | Create a new transfer. Body: `{file_size_bytes, content_type_hint}`. Returns `{transfer_id, upload_url}`. |
| `transfers_upload` | POST | `/transfers/upload/{transfer_id}` | Upload encrypted payload bytes to a pending transfer. |
| `transfers_complete` | POST | `/transfers/complete/{transfer_id}` | Mark transfer as complete. Returns download metadata. |
| `transfers_info` | GET | `/transfers/info/{transfer_id}` | Get transfer status, size, timestamps. |
| `transfers_download` | GET | `/transfers/download/{transfer_id}` | Download encrypted payload (< 5 MB). |
| `presigned_capabilities` | GET | `/presigned/capabilities` | Check available upload modes (direct vs multipart). |
| `presigned_initiate` | POST | `/presigned/initiate` | Start multipart upload; returns presigned URLs for each part. |
| `presigned_complete` | POST | `/presigned/complete` | Finalize multipart upload with ETags. |
| `presigned_upload_url` | GET | `/presigned/upload-url/{transfer_id}` | Get a single presigned PUT URL (files < 5 GB). |
| `presigned_download_url` | GET | `/presigned/download-url/{transfer_id}` | Get a presigned GET URL for large file download. |
| `vault_lookup` | GET | `/vault/lookup/{vault_cache_key}` | Look up vault metadata. |
| `vault_files` | GET | `/vault/files/{vault_cache_key}` | List files in a vault. |
| `vault_list_all` | GET | `/vault/list-all/{vault_cache_key}` | List all vault contents (files, folders, index). |

### Admin Lambda Tools (tokens, keys, users, vault management)

| Tool | Method | Route | Description |
|------|--------|-------|-------------|
| `tokens_create` | POST | `/tokens/create` | Create access token. Body: `{token_name, usage_limit, created_by, metadata}`. |
| `tokens_lookup` | GET | `/tokens/lookup/{token_name}` | Look up token status and usage. |
| `tokens_list` | GET | `/tokens/list` | List all token names. |
| `tokens_list_details` | GET | `/tokens/list-details` | List tokens with full metadata and usage. |
| `tokens_revoke` | POST | `/tokens/revoke/{token_name}` | Revoke an active token. |
| `keys_publish` | POST | `/keys/publish` | Publish a public key. Body: `{public_key_pem, signing_key_pem}`. |
| `keys_lookup` | GET | `/keys/lookup/{code}` | Look up a public key by code/fingerprint. |
| `keys_list` | GET | `/keys/list` | List all published keys. |
| `keys_log` | GET | `/keys/log` | View key registry audit log. |
| `users_create` | POST | `/users/create` | Create user identity. Body: `{display_name, key_fingerprint}`. |
| `users_lookup` | GET | `/users/lookup/{user_id}` | Look up user by ID. |
| `users_list` | GET | `/users/list` | List all users. |
| `vault_create` | POST | `/vault/create` | Create a new vault. Body: `{vault_cache_key, key_fingerprint}`. |
| `vault_file` | POST | `/vault/file` | Store encrypted file in vault. Body: `{vault_cache_key, file_guid, encrypted_data}`. |
| `vault_share` | POST | `/vault/share/{vault_cache_key}` | Grant vault access. Body: `{user_id, permission}`. |
| `vault_permissions` | GET | `/vault/permissions/{vault_cache_key}` | List vault access control entries. |
| `health_pulse` | GET | `/health/pulse` | System health and analytics pulse. |

## 4. Common Workflows

### Upload a File

```
1. transfers_create({file_size_bytes: 1024, content_type_hint: "application/octet-stream"})
   -> {transfer_id: "abc123def456", upload_url: "/transfers/upload/abc123def456"}
2. transfers_upload(transfer_id="abc123def456", body=<encrypted_bytes>)
   -> {status: "uploaded", transfer_id: "abc123def456", size: 1024}
3. transfers_complete(transfer_id="abc123def456")
   -> {transfer_id: "abc123def456", status: "completed", ...}
```

IMPORTANT: You must encrypt the file content client-side BEFORE uploading. The server must never see plaintext.

### Download a File

```
1. transfers_info(transfer_id="abc123def456")
   -> {status: "completed", file_size_bytes: 1024, ...}
2. transfers_download(transfer_id="abc123def456")    # < 5 MB
   -- OR --
   presigned_download_url(transfer_id="abc123def456") # any size
   -> encrypted bytes (decrypt client-side with the key from the sender)
```

### Manage Tokens

```
tokens_create({token_name: "demo-token", usage_limit: 100, created_by: "admin"})
tokens_list()
tokens_lookup(token_name="demo-token")
tokens_revoke(token_name="demo-token")
```

### Publish and Look Up Keys

```
keys_publish({public_key_pem: "-----BEGIN PUBLIC KEY-----\n...", signing_key_pem: ""})
keys_lookup(code="sha256-abc123")
keys_list()
```

### Check System Health

```
health_pulse(window_minutes=5)
```

## 5. Debugging Guide

### Verify MCP Server Is Running

Send the MCP `initialize` handshake:

```bash
curl -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{
    "jsonrpc": "2.0",
    "method": "initialize",
    "id": 1,
    "params": {
      "protocolVersion": "2024-11-05",
      "capabilities": {},
      "clientInfo": {"name": "debug", "version": "0.1"}
    }
  }'
```

Expected: HTTP 200 with `{"jsonrpc":"2.0","id":1,"result":{...}}` containing server capabilities.

### List Available Tools

After initializing, discover tools:

```bash
curl -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":2,"params":{}}'
```

Expected: JSON array of tool definitions with names, descriptions, and input schemas.

### Call a Tool Directly

```bash
curl -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{
    "jsonrpc":"2.0","method":"tools/call","id":3,
    "params":{"name":"transfers_info","arguments":{"transfer_id":"abc123def456"}}
  }'
```

### Test the Underlying REST API Directly

Bypass MCP to isolate issues:

```bash
# Health check (no auth required on admin)
curl {ADMIN_LAMBDA_URL}/health/pulse

# Transfer info (auth required)
curl {USER_LAMBDA_URL}/transfers/info/abc123def456 \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}"

# List tokens
curl {ADMIN_LAMBDA_URL}/tokens/list \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}"
```

### Common Failure Modes

| Symptom | Cause | Fix |
|---------|-------|-----|
| HTTP 401 `Access token required` | Missing or empty `x-sgraph-access-token` header | Set the header in MCP config or curl. |
| HTTP 401 `Invalid access token` | Token not found in admin registry | Create a new token via `tokens_create` or check the token name. |
| HTTP 401 `Access token revoked` | Token was revoked | Create a new token. |
| HTTP 503 `Token validation service unavailable` | User Lambda cannot reach Admin Lambda | Check `SGRAPH_SEND__ADMIN__BASE_URL` env var on the User Lambda. |
| Timeout on first call (10-15s) | Lambda cold start | Retry. Subsequent calls are fast (~200 ms). Warm with a health check first. |
| HTTP 413 `File too large for direct download` | Payload exceeds 5 MB Lambda limit | Use `presigned_download_url` instead of `transfers_download`. |
| MCP `tools/list` returns empty | Routes not tagged or `include_tags` filter mismatch | Check that `fastapi-mcp` is mounted AFTER all routes are registered. |
| `Method not found` on MCP call | Tool name does not match auto-generated name | Run `tools/list` to see exact tool names. |
| Connection refused / DNS error | Wrong Lambda URL | Verify the URL ends in `.lambda-url.{region}.on.aws` or your custom domain. |

### Verify End-to-End Integrity

```bash
# 1. Create transfer
RESP=$(curl -s -X POST {USER_LAMBDA_URL}/transfers/create \
  -H "Content-Type: application/json" \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{"file_size_bytes":11,"content_type_hint":"text/plain"}')
TID=$(echo $RESP | jq -r .transfer_id)

# 2. Upload (encrypted bytes -- this is a demo, real use encrypts first)
curl -X POST {USER_LAMBDA_URL}/transfers/upload/$TID \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -H "Content-Type: application/octet-stream" \
  --data-binary "hello world"

# 3. Complete
curl -X POST {USER_LAMBDA_URL}/transfers/complete/$TID \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}"

# 4. Download and compare
DOWNLOADED=$(curl -s {USER_LAMBDA_URL}/transfers/download/$TID)
echo "Match: $([ "$DOWNLOADED" = "hello world" ] && echo YES || echo NO)"
```

### Reading Logs

MCP server logs go to the Lambda's CloudWatch log group. Look for:
- `/aws/lambda/{LAMBDA_FUNCTION_NAME}` in CloudWatch
- Filter by `MCP` or `fastapi_mcp` for MCP-specific entries
- Cold starts show as `INIT_START` followed by `START RequestId`

## 6. Security Reminders

1. **Zero-knowledge model.** The server stores encrypted blobs only. NEVER send plaintext file content through any MCP tool. Encrypt before `transfers_upload`, decrypt after `transfers_download`.
2. **No private keys on server.** The MCP server handles public keys and encrypted data. Decryption keys stay in your environment.
3. **Token hygiene.** Use scoped tokens with usage limits. Revoke tokens you no longer need. Do not embed tokens in scripts committed to version control.
4. **IP hashing.** Client IPs are SHA-256 hashed with a daily salt before storage. The server never stores raw IP addresses.
5. **Sensitive routes excluded.** `check-token` and `validate-token` are excluded from MCP tool discovery to prevent token probing.
