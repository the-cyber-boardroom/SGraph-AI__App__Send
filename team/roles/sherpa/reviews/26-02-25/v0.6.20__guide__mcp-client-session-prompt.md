# SGraph Send -- MCP Client Guide

You are connected to **SGraph Send**, a zero-knowledge encrypted file sharing service at [send.sgraph.ai](https://send.sgraph.ai). Files are encrypted client-side (AES-256-GCM) before upload. The server only stores encrypted ciphertext. Decryption keys never reach the server.

The MCP server exposes the existing REST API as MCP tools via `fastapi-mcp`. Each FastAPI route becomes an MCP tool automatically. Transport is **HTTP** (JSON-RPC over POST to `/mcp`) -- works on Lambda, no SSE required.

## 1. Architecture

Two Lambda functions, each with its own `/mcp` endpoint:

| Lambda | URL Placeholder | Purpose |
|--------|----------------|---------|
| **User** | `{USER_LAMBDA_URL}` | Transfers, presigned URLs, vault (read) |
| **Admin** | `{ADMIN_LAMBDA_URL}` | Tokens, keys, vault (write), users, analytics |

Auth header on all requests: `x-sgraph-access-token: {YOUR_ACCESS_TOKEN}`

## 2. Connection Setup

Add to your Claude Code MCP settings (`.claude/mcp.json` or equivalent):

```json
{
  "mcpServers": {
    "sgraph-send-user": {
      "type": "http",
      "url": "{USER_LAMBDA_URL}/mcp",
      "headers": { "x-sgraph-access-token": "{YOUR_ACCESS_TOKEN}" }
    },
    "sgraph-send-admin": {
      "type": "http",
      "url": "{ADMIN_LAMBDA_URL}/mcp",
      "headers": { "x-sgraph-access-token": "{YOUR_ACCESS_TOKEN}" }
    }
  }
}
```

Replace the three `{...}` placeholders with actual values. If you only need file transfers, the User server alone is sufficient.

## 3. Available MCP Tools

### User Lambda (transfers, presigned URLs, vault read)

| Tool | Route | Description |
|------|-------|-------------|
| `transfers_create` | POST `/transfers/create` | Create transfer. Body: `{file_size_bytes, content_type_hint}`. Returns `{transfer_id, upload_url}`. |
| `transfers_upload` | POST `/transfers/upload/{id}` | Upload encrypted payload bytes to a pending transfer. |
| `transfers_complete` | POST `/transfers/complete/{id}` | Mark transfer complete. Returns download metadata. |
| `transfers_info` | GET `/transfers/info/{id}` | Get transfer status, size, timestamps. |
| `transfers_download` | GET `/transfers/download/{id}` | Download encrypted payload (< 5 MB direct, else use presigned). |
| `presigned_capabilities` | GET `/presigned/capabilities` | Check available upload modes (direct vs multipart). |
| `presigned_initiate` | POST `/presigned/initiate` | Start multipart upload; get presigned URLs per part. |
| `presigned_complete` | POST `/presigned/complete` | Finalize multipart upload with ETags. |
| `presigned_upload_url` | GET `/presigned/upload-url/{id}` | Single presigned PUT URL (< 5 GB). |
| `presigned_download_url` | GET `/presigned/download-url/{id}` | Presigned GET URL for large file download. |
| `vault_lookup` | GET `/vault/lookup/{key}` | Look up vault metadata. |
| `vault_files` | GET `/vault/files/{key}` | List files in a vault. |
| `vault_list_all` | GET `/vault/list-all/{key}` | List all vault contents. |

### Admin Lambda (tokens, keys, users, vault management)

| Tool | Route | Description |
|------|-------|-------------|
| `tokens_create` | POST `/tokens/create` | Create token. Body: `{token_name, usage_limit, created_by, metadata}`. |
| `tokens_lookup` | GET `/tokens/lookup/{name}` | Token status and usage count. |
| `tokens_list` | GET `/tokens/list` | List all token names. |
| `tokens_list_details` | GET `/tokens/list-details` | Tokens with full metadata. |
| `tokens_revoke` | POST `/tokens/revoke/{name}` | Revoke an active token. |
| `keys_publish` | POST `/keys/publish` | Publish public key. Body: `{public_key_pem, signing_key_pem}`. |
| `keys_lookup` | GET `/keys/lookup/{code}` | Look up key by code/fingerprint. |
| `keys_list` | GET `/keys/list` | List all published keys. |
| `keys_log` | GET `/keys/log` | Key registry audit log. |
| `users_create` | POST `/users/create` | Create user. Body: `{display_name, key_fingerprint}`. |
| `users_lookup` | GET `/users/lookup/{id}` | Look up user by ID. |
| `users_list` | GET `/users/list` | List all users. |
| `vault_create` | POST `/vault/create` | Create vault. Body: `{vault_cache_key, key_fingerprint}`. |
| `vault_file` | POST `/vault/file` | Store encrypted file. Body: `{vault_cache_key, file_guid, encrypted_data}`. |
| `vault_share` | POST `/vault/share/{key}` | Grant vault access. Body: `{user_id, permission}`. |
| `vault_permissions` | GET `/vault/permissions/{key}` | List vault ACL entries. |
| `health_pulse` | GET `/health/pulse` | System health and analytics. |

## 4. Common Workflows

### Upload a File (3-step)
```
1. transfers_create({file_size_bytes: 1024, content_type_hint: "application/octet-stream"})
   -> {transfer_id: "abc123def456", upload_url: "/transfers/upload/abc123def456"}
2. transfers_upload(transfer_id="abc123def456", body=<encrypted_bytes>)
3. transfers_complete(transfer_id="abc123def456") -> {status: "completed", ...}
```
CRITICAL: Encrypt file content client-side BEFORE uploading. The server must never see plaintext.

### Download a File
```
1. transfers_info(transfer_id="abc123def456") -> {status, file_size_bytes, ...}
2a. transfers_download(transfer_id="abc123def456")      # files < 5 MB
2b. presigned_download_url(transfer_id="abc123def456")   # any size
-> Decrypt the returned bytes client-side with the key from the sender.
```

### Manage Tokens
```
tokens_create({token_name: "demo", usage_limit: 100, created_by: "admin"})
tokens_list()  ->  tokens_lookup(token_name="demo")  ->  tokens_revoke(token_name="demo")
```

### Publish / Look Up Keys
```
keys_publish({public_key_pem: "-----BEGIN PUBLIC KEY-----\n...", signing_key_pem: ""})
keys_lookup(code="sha256-abc123")
```

### Health Check
```
health_pulse(window_minutes=5)
```

## 5. Debugging Guide

### Verify MCP Server Is Reachable

```bash
# Initialize handshake
curl -s -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{"jsonrpc":"2.0","method":"initialize","id":1,"params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"debug","version":"0.1"}}}'
# Expected: 200 with {"jsonrpc":"2.0","id":1,"result":{...}}
```

### Discover Tools

```bash
curl -s -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":2,"params":{}}'
# Returns array of {name, description, inputSchema} for every tool
```

### Call a Tool via curl

```bash
curl -s -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{"jsonrpc":"2.0","method":"tools/call","id":3,"params":{"name":"transfers_info","arguments":{"transfer_id":"abc123"}}}'
```

### Test REST API Directly (bypass MCP)

```bash
curl {ADMIN_LAMBDA_URL}/health/pulse                                              # no auth
curl {USER_LAMBDA_URL}/transfers/info/abc123 -H "x-sgraph-access-token: {TOKEN}" # auth required
curl {ADMIN_LAMBDA_URL}/tokens/list -H "x-sgraph-access-token: {TOKEN}"
```

### Common Failure Modes

| Symptom | Cause | Fix |
|---------|-------|-----|
| 401 `Access token required` | Missing `x-sgraph-access-token` header | Set header in MCP config. |
| 401 `Invalid access token` | Token not in registry | Create via `tokens_create` or check name. |
| 401 `Access token revoked` | Token revoked | Create a new token. |
| 503 `Token validation service unavailable` | User Lambda cannot reach Admin Lambda | Check `SGRAPH_SEND__ADMIN__BASE_URL` env var. |
| Timeout 10-15s on first call | Lambda cold start | Retry; warm with `health_pulse` first. |
| 413 `File too large` | Payload > 5 MB Lambda limit | Use `presigned_download_url` instead. |
| `tools/list` returns empty | Routes not tagged or filter mismatch | Ensure `fastapi-mcp` mounts AFTER all routes registered. |
| `Method not found` | Tool name mismatch | Run `tools/list` to see exact auto-generated names. |
| Connection refused | Wrong Lambda URL | URL should end in `.lambda-url.{region}.on.aws`. |

### End-to-End Smoke Test

```bash
# Create -> Upload -> Complete -> Download -> Verify
TID=$(curl -s -X POST {USER_LAMBDA_URL}/transfers/create \
  -H "Content-Type: application/json" -H "x-sgraph-access-token: {TOKEN}" \
  -d '{"file_size_bytes":11,"content_type_hint":"text/plain"}' | jq -r .transfer_id)
curl -s -X POST {USER_LAMBDA_URL}/transfers/upload/$TID \
  -H "x-sgraph-access-token: {TOKEN}" -H "Content-Type: application/octet-stream" \
  --data-binary "hello world"
curl -s -X POST {USER_LAMBDA_URL}/transfers/complete/$TID -H "x-sgraph-access-token: {TOKEN}"
curl -s {USER_LAMBDA_URL}/transfers/download/$TID  # should return "hello world"
```

### Reading Lambda Logs

CloudWatch log group: `/aws/lambda/{FUNCTION_NAME}`. Filter by `MCP` or `fastapi_mcp`. Cold starts appear as `INIT_START` followed by `START RequestId`.

## 6. Security Reminders

1. **Zero-knowledge.** NEVER send plaintext file content through any MCP tool. Encrypt before upload, decrypt after download.
2. **No private keys on server.** Decryption keys stay in your environment. The server handles public keys and encrypted blobs only.
3. **Token hygiene.** Use scoped tokens with usage limits. Revoke when no longer needed. Never commit tokens to version control.
4. **IP hashing.** Client IPs are SHA-256 hashed with a daily salt. The server never stores raw IPs.
5. **Sensitive routes excluded.** `check-token` and `validate-token` are excluded from MCP discovery.
