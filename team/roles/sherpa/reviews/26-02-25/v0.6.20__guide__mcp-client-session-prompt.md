# SGraph Send -- MCP Client Guide

You are connected to **SGraph Send**, a zero-knowledge encrypted file sharing service at [send.sgraph.ai](https://send.sgraph.ai). Files are encrypted client-side (AES-256-GCM) before upload. The server only stores encrypted ciphertext. Decryption keys never reach the server.

The MCP server exposes the existing REST API as MCP tools via `fastapi-mcp`. Each FastAPI route becomes an MCP tool automatically. Transport is **HTTP** (JSON-RPC over POST to `/mcp`) -- works on Lambda, no SSE required.

## 1. Architecture

Two Lambda functions, each with its own `/mcp` endpoint:

| Lambda | URL Placeholder | Purpose |
|--------|----------------|---------|
| **User** | `{USER_LAMBDA_URL}` | Transfers, presigned URLs, vault (read) |
| **Admin** | `{ADMIN_LAMBDA_URL}` | Tokens, keys, vault (write), users, analytics |

### Authentication

**Claude Code CLI** — set the header in MCP config (see Connection Setup below):
```
x-sgraph-access-token: {YOUR_ACCESS_TOKEN}
```

**Claude.ai web** — cannot inject custom headers via MCP transport. Instead, pass `access_token` as a tool parameter on write endpoints (`transfers_create`, `transfers_upload`, `transfers_complete`):
```
transfers_create({file_size_bytes: 1024, access_token: "your-token-name"})
```

Read-only endpoints (`transfers_info`, `transfers_download`, `transfers_download-base64`) do not require a token.

## 2. Connection Setup

Add to your Claude Code MCP settings (`.claude/mcp.json` or equivalent):

```json
{
  "mcpServers": {
    "sgraph-send-user": {
      "type": "http",
      "url": "{USER_LAMBDA_URL}/mcp",
      "headers": { "x-sgraph-access-token": "{YOUR_ACCESS_TOKEN}" }
    },
    "sgraph-send-admin": {
      "type": "http",
      "url": "{ADMIN_LAMBDA_URL}/mcp",
      "headers": { "x-sgraph-access-token": "{YOUR_ACCESS_TOKEN}" }
    }
  }
}
```

Replace the three `{...}` placeholders with actual values. If you only need file transfers, the User server alone is sufficient.

## 4. Common Workflows

### Upload a File (3-step)
```
1. transfers_create({file_size_bytes: <encrypted_size>, content_type_hint: "application/pdf", access_token: "your-token"})
   -> {transfer_id: "abc123def456", upload_url: "/transfers/upload/abc123def456"}
2. transfers_upload(transfer_id="abc123def456", access_token="your-token", data="<base64-encoded-encrypted-payload>")
3. transfers_complete(transfer_id="abc123def456", access_token="your-token") -> {status: "completed", ...}
```
**Note:** `access_token` is required on Claude.ai web (can't inject headers). On Claude Code CLI with header config, it's optional.

CRITICAL: Encrypt file content client-side BEFORE uploading. The server must never see plaintext.

**CRITICAL: Use the `data` parameter** on `transfers_upload` to pass the encrypted payload as a base64-encoded string. The MCP transport cannot reliably pass raw binary bytes via the request body -- the `data` parameter is an explicit tool argument that MCP can use reliably.

### SGMETA Envelope -- Preserving Filenames

The browser wraps file content in an **SGMETA envelope** before encryption. This preserves the original filename inside the encrypted payload. **MCP clients MUST do the same** -- otherwise the recipient sees no filename on download.

**SGMETA binary format:**
```
[7 bytes: "SGMETA\0"]  <- magic bytes (0x53, 0x47, 0x4D, 0x45, 0x54, 0x41, 0x00)
[4 bytes: big-endian]   <- metadata JSON length
[N bytes: UTF-8 JSON]   <- metadata (e.g. {"filename": "report.pdf"})
[remaining bytes]        <- raw file content
```

**Python implementation (matches browser exactly):**
```python
import os, struct, json, base64
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

def upload_file_via_mcp(file_bytes, filename, content_type, mcp_client, access_token):
    """Upload a file through SG/Send MCP, matching the browser workflow exactly."""

    # Step 1: Build SGMETA envelope (preserves filename inside encrypted payload)
    magic    = b'SGMETA\x00'
    metadata = json.dumps({'filename': filename}).encode('utf-8')
    meta_len = struct.pack('>I', len(metadata))
    plaintext = magic + meta_len + metadata + file_bytes

    # Step 2: Encrypt with AES-256-GCM (same as browser's SendCrypto.encryptFile)
    key       = os.urandom(32)               # 256-bit random key
    nonce     = os.urandom(12)               # 96-bit nonce for GCM
    ciphertext = AESGCM(key).encrypt(nonce, plaintext, None)
    encrypted  = nonce + ciphertext          # [12-byte nonce][ciphertext + auth tag]

    # Step 3: Base64 encode for MCP transport
    b64_payload = base64.b64encode(encrypted).decode('ascii')

    # Step 4: Create transfer
    result = mcp_client.transfers_create(
        file_size_bytes   = len(encrypted),
        content_type_hint = content_type,
        access_token      = access_token
    )
    transfer_id = result['transfer_id']

    # Step 5: Upload via data parameter (NOT request body)
    mcp_client.transfers_upload(
        transfer_id  = transfer_id,
        access_token = access_token,
        data         = b64_payload       # base64-encoded encrypted payload
    )

    # Step 6: Complete
    mcp_client.transfers_complete(
        transfer_id  = transfer_id,
        access_token = access_token
    )

    # Step 7: Build download URL (key in fragment -- never sent to server)
    key_b64url = base64.urlsafe_b64encode(key).rstrip(b'=').decode('ascii')
    download_url = f"https://send.sgraph.ai/send/v0/v0.1/v0.1.4/download.html#{transfer_id}/{key_b64url}"

    return download_url
```

**Key points:**
- SGMETA envelope is built BEFORE encryption -- the filename is encrypted with the content
- The download page checks for SGMETA magic bytes after decryption to extract the filename
- Without SGMETA, the file downloads with no name (recipient has to guess the format)
- The key is base64url-encoded (not standard base64) for URL fragment safety
- `file_size_bytes` in `transfers_create` must be the size of the ENCRYPTED payload (not the original file)
- For text-only messages (no file), skip SGMETA -- just encrypt the text directly

### Download a File
```
1. transfers_info(transfer_id="abc123def456") -> {status, file_size_bytes, ...}
2a. transfers_download-base64(transfer_id="abc123def456")  # MCP — returns JSON {data: "<base64>", file_size_bytes: ...}
2b. transfers_download(transfer_id="abc123def456")         # direct HTTP — raw bytes (< 5 MB)
2c. presigned_download_url(transfer_id="abc123def456")     # any size — returns presigned S3 URL
-> Base64-decode the `data` field, then decrypt with the key from the sender.
```
**From MCP, always use `transfers_download-base64`** — raw bytes from `transfers_download` are unusable in JSON-RPC.

## 5. Debugging Guide

### MCP Inspector (recommended for interactive debugging)

The official MCP debugging tool gives you a web UI to browse tools and invoke them:

```bash
npx @modelcontextprotocol/inspector
```

Point it at your MCP endpoint (e.g. `http://localhost:10062/send/mcp` for local dev, or `{USER_LAMBDA_URL}/mcp` for production). The Inspector shows:
- All discovered tools with their input schemas
- Full JSON-RPC request/response for each call
- Connection status and protocol version

For local testing this is the fastest way to verify everything works.

### Verify MCP Server Is Reachable (curl)

```bash
# Initialize handshake
curl -s -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" \
  -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{"jsonrpc":"2.0","method":"initialize","id":1,"params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"debug","version":"0.1"}}}'
# Expected: 200 with {"jsonrpc":"2.0","id":1,"result":{...}}
```

### Discover Tools

```bash
curl -s -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":2,"params":{}}'
# Returns array of {name, description, inputSchema} for every tool
```

### Call a Tool via curl

```bash
curl -s -X POST {USER_LAMBDA_URL}/mcp \
  -H "Content-Type: application/json" -H "x-sgraph-access-token: {YOUR_ACCESS_TOKEN}" \
  -d '{"jsonrpc":"2.0","method":"tools/call","id":3,"params":{"name":"transfers_info","arguments":{"transfer_id":"abc123"}}}'
```

### Common Failure Modes

| Symptom | Cause | Fix |
|---------|-------|-----|
| 401 `Access token required` | Missing `x-sgraph-access-token` header | Set header in MCP config. |
| 401 `Invalid access token` | Token not in registry | Create via `tokens_create` or check name. |
| 401 `Access token revoked` | Token revoked | Create a new token. |
| 503 `Token validation service unavailable` | User Lambda cannot reach Admin Lambda | Check `SGRAPH_SEND__ADMIN__BASE_URL` env var. |
| Timeout 10-15s on first call | Lambda cold start | Retry; warm with `/info/health` (User) or `health_pulse` (Admin). |
| 413 `File too large` (download) | Payload > 5 MB Lambda limit | Use `presigned_download_url` instead. |
| 413 `File too large` (download-base64) | Payload > 3.75 MB (base64 overhead) | Use `presigned_download_url` instead. |
| 401 on MCP tool call | Claude.ai web can't inject headers | Pass `access_token` as tool parameter. |
| `tools/list` returns empty | Routes not tagged or filter mismatch | Ensure `fastapi-mcp` mounts AFTER all routes registered. |
| `Method not found` | Tool name mismatch | Run `tools/list` to see exact auto-generated names. |
| Connection refused | Wrong Lambda URL | URL should end in `.lambda-url.{region}.on.aws`. |
| Upload succeeds but `size: 0` | MCP tool can't pass request body | Use `data` parameter instead of request body. |
| Download fails to decrypt | Missing SGMETA envelope or wrong encryption | Follow the SGMETA + AES-256-GCM workflow above. |
| Download works but no filename | SGMETA envelope not applied before encryption | Wrap content in SGMETA before encrypting (see Section 4). |

## 6. Security Reminders

1. **Zero-knowledge.** NEVER send plaintext file content through any MCP tool. Encrypt before upload, decrypt after download.
2. **No private keys on server.** Decryption keys stay in your environment. The server handles public keys and encrypted blobs only.
3. **Token hygiene.** Use scoped tokens with usage limits. Revoke when no longer needed. Never commit tokens to version control.
4. **IP hashing.** Client IPs are SHA-256 hashed with a daily salt. The server never stores raw IPs.
5. **Sensitive routes excluded.** `check-token` and `validate-token` are excluded from MCP discovery.
