# Encrypted File Send + Email Workflow

You have two MCP connectors available:
1. **SG/Send** — zero-knowledge encrypted file transfer (send.sgraph.ai)
2. **n8n** — workflow automation including email via Sherpa WorkMail

Your task: create an encrypted PDF, upload it to SG/Send, verify the upload works correctly (content + filename), then email the secure download link to a recipient.

Before starting, confirm you have:

- The recipient's email address (ask the user if not provided)

## Step-by-Step Workflow

### Step 1: Create the PDF

Use your computer/sandbox to create a PDF. Install reportlab if needed:

```python
from reportlab.platypus import SimpleDocTemplate, Paragraph, Spacer
from reportlab.lib.styles import getSampleStyleSheet
from reportlab.lib.pagesizes import letter

filename = "message-for-recipient.pdf"  # This will be preserved via SGMETA
doc = SimpleDocTemplate(filename, pagesize=letter)
styles = getSampleStyleSheet()

story = []
story.append(Paragraph("Your Title Here", styles['Title']))
story.append(Spacer(1, 20))
story.append(Paragraph("Your message content here.", styles['Normal']))
# Add more content as needed
doc.build(story)

# Read the PDF bytes
with open(filename, 'rb') as f:
    pdf_bytes = f.read()
print(f"PDF created: {filename} ({len(pdf_bytes)} bytes)")
```

### Step 2: Build SGMETA Envelope + Encrypt

**CRITICAL:** You MUST wrap the file in an SGMETA envelope BEFORE encryption. This preserves the original filename so the recipient sees it when they download. This matches exactly what the browser does.

```python
import os, struct, json, base64
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

# --- SGMETA envelope (preserves filename inside the encrypted payload) ---
magic    = b'SGMETA\x00'                                          # 7-byte magic
metadata = json.dumps({'filename': filename}).encode('utf-8')      # {"filename": "message-for-recipient.pdf"}
meta_len = struct.pack('>I', len(metadata))                        # 4-byte big-endian length
plaintext = magic + meta_len + metadata + pdf_bytes                # Full envelope

# --- AES-256-GCM encryption (same as browser's SendCrypto.encryptFile) ---
key        = os.urandom(32)                                        # 256-bit random key
nonce      = os.urandom(12)                                        # 96-bit nonce for GCM
ciphertext = AESGCM(key).encrypt(nonce, plaintext, None)           # Encrypt with auth tag
encrypted  = nonce + ciphertext                                    # [12-byte nonce][ciphertext + tag]

# --- Encode for MCP transport ---
b64_payload = base64.b64encode(encrypted).decode('ascii')

# --- Encode key for URL fragment (base64url, no padding) ---
key_b64url = base64.urlsafe_b64encode(key).rstrip(b'=').decode('ascii')

print(f"Plaintext size:  {len(plaintext)} bytes (SGMETA + PDF)")
print(f"Encrypted size:  {len(encrypted)} bytes")
print(f"Base64 size:     {len(b64_payload)} chars")
print(f"Key (base64url): {key_b64url}")
```

    transfer_id  = "<transfer_id from 3a>",

### Step 4: Construct the Download URL

```python
download_url = f"https://send.sgraph.ai/send/v0/v0.1/v0.1.8/download.html#{transfer_id}/{key_b64url}"
print(f"Download URL: {download_url}")
```

The URL fragment (`#transfer_id/key`) is never sent to the server. The recipient's browser reads the fragment, fetches the encrypted blob, and decrypts locally.

### Step 5: Verify the Upload (MUST DO)

Download the file back and verify BOTH content and filename are correct:

```
# 5a. Download via MCP (use download-base64 for MCP — raw bytes don't work in JSON-RPC)
transfers_download-base64(transfer_id = "<transfer_id>")
# Returns: {transfer_id, data: "<base64>", file_size_bytes: <n>}
```

Then verify in your sandbox:

```python
import base64, struct, json
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

# Decode the downloaded base64
downloaded_encrypted = base64.b64decode("<data field from download response>")

# Verify size matches
assert len(downloaded_encrypted) == len(encrypted), f"Size mismatch: {len(downloaded_encrypted)} != {len(encrypted)}"

# Decrypt
dl_nonce      = downloaded_encrypted[:12]
dl_ciphertext = downloaded_encrypted[12:]
decrypted     = AESGCM(key).decrypt(dl_nonce, dl_ciphertext, None)

# Extract SGMETA envelope
assert decrypted[:7] == b'SGMETA\x00', "SGMETA magic bytes missing — envelope was not applied"
meta_len_bytes = struct.unpack('>I', decrypted[7:11])[0]
meta_json      = json.loads(decrypted[11:11 + meta_len_bytes])
file_content   = decrypted[11 + meta_len_bytes:]

# Verify filename
assert meta_json['filename'] == filename, f"Filename mismatch: {meta_json['filename']} != {filename}"
print(f"Filename OK: {meta_json['filename']}")

# Verify content matches original PDF
assert file_content == pdf_bytes, "Content mismatch — file is corrupted"
print(f"Content OK: {len(file_content)} bytes match original PDF")
print("VERIFICATION PASSED — file is intact with correct filename")
```

**If verification fails, DO NOT proceed to email. Report the error.**

Then send the email:
```
execute_workflow(
    workflowId = "LNTb1Smmt91x8iGn",
    inputs = {
        type: "webhook",
        webhookData: {
            method: "POST",
            body: {
                operation: "send_email",
                to: "<recipient-email>",
                subject: "Encrypted file from SGraph Send",
                body: "&lt;p&gt;Hi,&lt;/p&gt;&lt;p&gt;I've sent you an encrypted file via SGraph Send.&lt;/p&gt;&lt;p&gt;&lt;strong&gt;Download link:&lt;/strong&gt; &lt;a href='DOWNLOAD_URL_HERE'&gt;Click to download and decrypt&lt;/a&gt;&lt;/p&gt;&lt;p&gt;The file will be decrypted automatically in your browser. The server never sees the contents.&lt;/p&gt;"
            }
        }
    }
)
```

**CRITICAL — HTML in SOAP XML:** The email body goes into an EWS SOAP envelope. Raw HTML breaks the XML parser. You MUST XML-escape the HTML:
- `<` → `&lt;`
- `>` → `&gt;`
- `'` → `&apos;` (in attributes)
- `&` → `&amp;`

The email renders correctly because EWS interprets the escaped HTML.

## Error Handling

| Error | Cause | Action |
|-------|-------|--------|
| `transfers_upload` returns `size: 0` | `data` parameter not used | Ensure you pass `data=<b64_payload>` (not request body) |
| Decryption fails on verify | Key mismatch or corrupted upload | Re-upload — do not email a broken link |
| SGMETA magic missing on verify | Envelope not applied before encryption | Re-do Step 2 — wrap THEN encrypt |
| Filename mismatch on verify | Wrong filename in SGMETA metadata | Fix the filename in Step 2 |
| n8n XML parse error | Raw HTML in email body | XML-escape all HTML entities |
| 401 on SG/Send | Invalid or expired access token | Check token with `transfers_check-token` |

## Security Properties

- **Zero-knowledge:** The server stores only encrypted ciphertext. The decryption key never reaches the server.
- **Filename encrypted:** The filename is inside the SGMETA envelope, which is encrypted. The server never sees it.
- **Key in URL fragment:** URL fragments (`#...`) are never sent to the server. The key travels only in the email link.
- **Fresh key per transfer:** A new random 256-bit key is generated for every file.
- **Verified before sending:** The workflow verifies decryption and filename extraction before emailing the link.
