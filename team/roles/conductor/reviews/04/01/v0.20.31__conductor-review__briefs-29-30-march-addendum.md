## 1. Accountant Pack Workflow — Most Important Signal

**Documents:** 193 (`v0.19.7__debrief__accountant-pack-workflow.md`) + 197 (`debrief__claude_and_sg-vault__workflow-to-create-accountant-pack.md`)

### What happened

A real professional workflow was completed end-to-end:
1. Mobile: Credit card statements downloaded and uploaded → zip created
2. Desktop: SGit CLI clones via Simple Token
3. Vault created → shared with Claude via vault URL
4. Claude parses PDFs, categorises expenses, builds interactive HTML app, pushes to vault
5. Browse view delivery to accountant: navigable document site with README hierarchy, clickable links, rendered markdown, CSV as tables, HTML in iframes

The accountant received a **Simple Token** and navigated a professional deliverable. No installation. No technical knowledge required. Works in any browser.

### Why this matters

This is the **canonical use case** that justifies the entire stack. It demonstrates:
- Zero-knowledge encryption (accountant's financial data never touches the server in plaintext)
- Vault as AI-human collaboration medium (Claude contributes to a shared encrypted workspace)
- Browse view as publishing platform (not a file browser — a mini website)
- Simple Token as professional delivery mechanism (not a technical link — a friendly identifier)

### Actions needed

1. **Ambassador must write the case study.** The raw workflow is documented in docs 193 and 197. The Ambassador should produce a polished, publishable case study: "How I prepared my accountant pack in 2 hours using AI, encryption, and SGraph Send."
2. **This is the primary demo workflow.** Every demo, investor presentation, and onboarding session should reference the accountant pack. It is concrete, relatable, and impressive.
3. **Identify 3 more case studies using the same pattern:** doctor (medical records), lawyer (client documents), designer (portfolio delivery). Dinis mentioned doctor/medical as an upcoming use case — follow up.

## 3. Content Creation Workflow — Documented and Replicable

**Documents:** 191 + 190 (two articles for publication)

### The voice memo → article workflow (Document 191)

This article documents the actual process used to produce all SGraph content:
1. Voice memo (on phone, during commute or walk)
2. Otter.ai transcription (automated)
3. Claude processes transcript → structured article
4. NotebookLM generates infographics from Claude's article
5. Claude receives folder (markdown + infographics) → creates polished article with inline images
6. Push to vault → share link → Browse view renders as webpage
7. Print to PDF or copy to LinkedIn

**Signal:** This is not a proposed workflow — it is the workflow already in use. The fact that it's been documented as an article confirms it's mature enough to publish.
