## Executive Summary

The human's preferred tool — **Vercel Agent Browser** — is real, actively maintained (14,600+ GitHub stars), and does what the brief claims. However, it has **no native Python support** (CLI-only, driven via subprocess) and is optimised for **AI agent steering**, not traditional scripted E2E tests.

**Recommendation:** Use **Playwright for Python** as the primary tool for scripted smoke tests. Use **agent-browser** as a complementary tool for AI-driven exploratory testing and its built-in visual diffing. Both can coexist.

## Vercel Agent Browser (`agent-browser`)

| Field | Value |
|---|---|
| **Repo** | [github.com/vercel-labs/agent-browser](https://github.com/vercel-labs/agent-browser) |
| **Stars** | 14,600+ |
| **Website** | [agent-browser.dev](https://agent-browser.dev/) |
| **Package** | `npm install -g agent-browser` |
| **Architecture** | Rust CLI + Node.js daemon + Playwright/Chromium underneath |
| **Launched** | ~January 2026 |

### What It Does Well

1. **Token efficiency** — Uses a "snapshot + refs" system (`@e1`, `@e2`) that reduces context window usage by ~93% compared to raw DOM dumps. Ideal for AI agents steering a browser.
2. **CLI-first** — 108+ commands covering navigation, interaction, screenshots, recording, tracing, diffing. Designed to be driven from a terminal or subprocess.
3. **Built-in visual diffing** — `diff screenshot` produces a diff image with changed pixels highlighted in red plus a mismatch percentage. Exactly what the QA project needs.
4. **Video recording** — `record start demo.webm` / `record stop`. Could generate YouTube demo videos from test runs (the human mentioned this).
5. **Annotated screenshots** — `screenshot --annotate` overlays numbered labels on interactive elements. Excellent for documentation.
6. **State management** — `state save` / `state load` for session persistence across commands.
7. **Headless by default** — No flag needed. Pass `--headed` to see the browser.

### Limitations for This Project

| Limitation | Impact | Mitigation |
|---|---|---|
| **No Python SDK** | Must drive via `subprocess.run()`. Error handling is clunky. No native async support. | Wrap in a Python helper class. Use `--json` output for structured parsing. |
| **Chromium only** | No Firefox or WebKit testing | Not a blocker — SG/Send users are overwhelmingly Chromium-based |
| **New project (~2 months old)** | May have rough edges, breaking changes | Pin to a specific version. Have Playwright as fallback. |
| **Daemon architecture** | Persistent background process may not suit Lambda's ephemeral model | Use cloud browser providers for Lambda; local Chromium for CI/local |
| **Node.js dependency** | Adds Node.js to a Python project's requirements | Acceptable trade-off if the features justify it |

### Screenshot and Diff Capabilities

```bash
# Basic screenshot
agent-browser screenshot page.png

# Full-page screenshot
agent-browser screenshot --full page.png

# Annotated screenshot (numbered labels on interactive elements)
agent-browser screenshot --annotate page.png

# Visual diff between two screenshots
agent-browser diff screenshot old.png new.png
# Output: diff image + mismatch percentage

# Video recording
agent-browser record start demo.webm
# ... interact with page ...
agent-browser record stop
```

### Cloud Browser Providers

For environments where local Chromium is impractical:

| Provider | Env Vars |
|---|---|
| **Browser Use** | `AGENT_BROWSER_PROVIDER=browseruse` + `BROWSER_USE_API_KEY` |
| **Kernel** | `AGENT_BROWSER_PROVIDER=kernel` + `KERNEL_API_KEY` |

All commands work identically — browser runs remotely. This solves the Lambda problem.

## Playwright for Python

| Field | Value |
|---|---|
| **Repo** | [github.com/microsoft/playwright](https://github.com/microsoft/playwright) |
| **Stars** | 71,000+ |
| **Package** | `pip install playwright` |
| **Maturity** | 5+ years |
| **Python SDK** | Native, first-class |

### What It Does Well

1. **Native Python** — `pip install playwright`. Full async/sync API. Pytest integration.
2. **Multi-browser** — Chromium, Firefox, WebKit.
3. **Built-in visual comparison** — `expect(page).to_have_screenshot()` with anti-aliasing tolerance.
4. **Advanced network** — HAR recording, request interception, response modification.
5. **Tracing** — Timeline, screenshots, DOM snapshots in a trace viewer.
6. **GitHub Actions** — First-class support. `npx playwright install --with-deps`.
7. **Mature ecosystem** — Test reporters, parallel execution, CI integrations, extensive documentation.

### Limitations

| Limitation | Impact |
|---|---|
| **Not AI-optimised** | Context window cost is high if feeding DOM to an LLM |
| **No built-in visual diffing** | Must use third-party (Percy, Applitools) or roll your own |
| **No session recording to video** | Has video recording per context but no CLI replay |
| **No annotated screenshots** | Must implement element labelling manually |

## Recommended Architecture

### Dual-tool approach

```
┌─────────────────────────────────────────────────────┐
│                QA Test Runner                        │
│                                                      │
│  ┌─────────────────┐    ┌──────────────────────┐    │
│  │ Scripted Tests   │    │ AI Exploratory Tests │    │
│  │ (Playwright)     │    │ (agent-browser)      │    │
│  │                  │    │                      │    │
│  │ • Smoke tests    │    │ • Visual regression  │    │
│  │ • Flow tests     │    │ • Exploratory QA     │    │
│  │ • Assertions     │    │ • Annotated docs     │    │
│  │ • pytest native  │    │ • Video demos        │    │
│  └────────┬─────────┘    └──────────┬───────────┘   │
│           │                          │               │
│           └──────────┬───────────────┘               │
│                      │                               │
│           ┌──────────▼──────────┐                    │
│           │ Screenshot Library  │                    │
│           │ Visual Diff Engine  │                    │
│           │ Doc Generator       │                    │
│           └─────────────────────┘                    │
└─────────────────────────────────────────────────────┘
```

### Phase 1 (immediate): Playwright only

- Install: `pip install playwright && npx playwright install chromium`
- Write smoke tests in Python with pytest
- Use Playwright's `page.screenshot()` for captures
- Roll basic visual diff with Pillow (or Playwright's `toHaveScreenshot()`)

### Phase 2 (week 2): Add agent-browser

- Install: `npm install -g agent-browser && agent-browser install`
- Use for: annotated screenshots (numbered element labels)
- Use for: visual diff (`diff screenshot`)
- Use for: video recording of test flows (`record start/stop`)
- Python wrapper class for subprocess integration

### Phase 3 (later): AI-driven testing

- Use agent-browser's snapshot refs for LLM-driven exploratory testing
- "Navigate to the admin panel and check if all navigation links work"
- LLM steers the browser via agent-browser CLI, captures screenshots at each step

## Other Tools Evaluated

| Tool | Assessment |
|---|---|
| **Browser Use** (78K stars) | Python-native, LLM integration built in. Worth evaluating as an alternative to agent-browser for AI-driven tests. But overkill for scripted smoke tests. |
| **Browserbase** | Cloud browser infra. Good for Lambda use case. Supported by agent-browser. Commercial service. |
| **Amazon Nova Act** | Python SDK for browser agents. New. Amazon-specific. Not a good fit for this project's open-source stack. |
| **Playwright MCP** | Playwright exposed as MCP tool for AI agents. Higher token cost than agent-browser. Worth considering if using Claude Code to drive tests. |

## Decision Record

| Decision | Recommendation | Rationale |
|---|---|---|
| Primary tool for scripted tests | **Playwright for Python** | Native Python, mature, pytest integration, guaranteed GitHub Actions support |
| Complementary tool for AI/visual | **agent-browser** | Built-in visual diff, annotated screenshots, video recording, token efficiency for AI agents |
| Lambda execution | **Cloud browser provider** (Browserbase or Browser Use) | Avoid Chromium-in-Lambda complexity. agent-browser supports cloud providers natively. |
| GitHub Actions | **Playwright** (local Chromium) | First-class support, well-documented |
| Visual diff engine | **agent-browser `diff screenshot`** | Built-in, produces diff images with mismatch percentage |
| Video demos | **agent-browser `record start/stop`** | Native `.webm` recording, could generate YouTube demos |
