## Architecture Fit

### Lambda + SSE Constraint

Lambda (via Mangum) is request/response. MCP's SSE transport needs a long-lived connection. Mangum does not support Lambda response streaming.

**Solution:** MCP SDK v1.x supports **Streamable HTTP** transport (POST-based, no SSE). Each MCP call becomes a single HTTP request/response -- works on Lambda as-is. No server-initiated notifications, but we don't need them.

If SSE is later required, deploy the same FastAPI app on always-on compute (Fargate/EC2). One-afternoon task given our existing 7-target deployment matrix.

### Integration Point (3 lines)

```python
# In Fast_API__SGraph__App__Send__User.setup_routes():
from fastapi_mcp import FastApiMCP
mcp = FastApiMCP(self.app(), name="sgraph-send",
                 base_url="https://send.sgraph.ai")
mcp.mount()  # adds /mcp endpoint
```

`self.app()` returns a standard `FastAPI` instance (confirmed in code). `fastapi-mcp` auto-discovers all registered routes and exposes them as MCP tools.

### Which Lambda?

**User Lambda only.** It has transfers, presigned URLs, and vault routes. Admin Lambda endpoints (tokens, users, analytics) must NOT be exposed via MCP in Phase 1.

## API Contract

- **Endpoint:** `/mcp` on User Lambda (POST `/mcp/messages` for Streamable HTTP)
- **Auth:** Same `x-sgraph-access-token` header
- **Tool schemas:** Auto-generated from FastAPI's OpenAPI spec. Each route becomes a tool with name, description, and JSON Schema input/output.
- **Tool filtering:** Use `fastapi-mcp`'s `include_routes`/`exclude_routes` to whitelist safe endpoints and exclude `/transfers/check-token`, `/transfers/validate-token`, `/vault/share`, `/vault/unshare`.

---

## Security

1. **Zero-knowledge preserved.** MCP server handles encrypted blobs only. No plaintext on server.
2. **Auth unchanged.** `check_access_token` runs on every MCP tool call via FastAPI dependency injection.
3. **No private keys on server.** Decryption happens in the calling agent's environment.
4. **Scope restriction.** Admin endpoints excluded. Token/user CRUD not exposed.
5. **Rate limiting.** Token usage tracking exists. Add explicit rate limits in Phase 2.
6. **Tool filtering.** Sensitive routes excluded from auto-discovery (see API Contract above).
