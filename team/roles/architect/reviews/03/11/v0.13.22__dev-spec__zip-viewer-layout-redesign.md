## Context

The zip viewer exists and works. It detects zip files, parses them with JSZip, renders a folder tree + file list, and previews individual files using the same renderers as single-file view (markdown, PDF, image, code, audio, video).

**The problem:** The zip viewer stacks everything vertically (header → browser → preview → transparency → timing). This wastes viewport space and makes PDFs unreadable. The single-file view already uses a superior two-column layout with a resizable divider.

**The solution:** Adopt the same two-column layout for zip view. Put folder tree + file list in the left rail. Put the preview in the right panel. Move transparency below the fold.

## Sprint 1 — Layout Restructure (P1)

### Task 1.1: Replace `_renderZipLayout` with two-column split

**What to change:** The `_renderZipLayout()` method (line 487) currently returns a vertical stack. Replace it with a layout that matches `_renderTwoColumnLayout()` (line 356).

**Current structure (vertical):**
```
status-banner
zip-header (name, badge, size, actions)
zip-browser (folder tree | file list) — max-height: 200px
preview-panel — min-height: 200px
transparency-panel
timing
send-another
```

**New structure (two-column):**
```
status-banner
zip-header (name, badge, size, actions) — sticky, compact, single line
preview-split (grid: left-rail | resize-divider | preview-panel)
  left-rail:
    folder-tree (scrollable)
    file-list (scrollable)
    save-entry-btn (if file selected)
  preview-panel:
    rendered content (fills viewport height)
transparency-panel — below the fold
timing — below the fold
send-another — below the fold
```

**Key changes from current implementation:**
1. Uses `preview-split` grid (same ID as single-file view) so `_setupResize()` works without modification
2. Left rail contains folder tree + file list stacked vertically (replaces the old `zip-browser` horizontal split)
3. Preview panel gets `min-height: calc(100vh - 180px)` — fills the viewport
4. Transparency panel is hidden by default, toggled via the info button
5. Adds folder icon to header for identity signal (Finding #1)
6. Adds progress indicator (Finding #8C)

### Task 3.2: Keyboard navigation

**Add keyboard event handling** for navigating the file list. Add to `setupEventListeners()`:

```javascript
// Keyboard navigation for zip file list
if (this._zipTree) {
    if (!this._keyNavHandler) {
        this._keyNavHandler = (e) => {
            if (!this._zipTree) return;
            const files = this._zipTree.filter(f => !f.dir);
            if (files.length === 0) return;

            const currentIdx = this._selectedZipPath
                ? files.findIndex(f => f.path === this._selectedZipPath)
                : -1;

            if (e.key === 'ArrowDown' || e.key === 'j') {
                e.preventDefault();
                const next = Math.min(currentIdx + 1, files.length - 1);
                this._previewZipEntry(files[next].path);
            } else if (e.key === 'ArrowUp' || e.key === 'k') {
                e.preventDefault();
                const prev = Math.max(currentIdx - 1, 0);
                this._previewZipEntry(files[prev].path);
            } else if (e.key === 's' && !e.ctrlKey && !e.metaKey) {
                // Only if not typing in an input
                if (document.activeElement.tagName !== 'INPUT' && document.activeElement.tagName !== 'TEXTAREA') {
                    e.preventDefault();
                    this._saveCurrentEntry();
                }
            }
        };
        document.addEventListener('keydown', this._keyNavHandler);
    }
}
```

**Clean up** in `cleanup()`:

```javascript
if (this._keyNavHandler) {
    document.removeEventListener('keydown', this._keyNavHandler);
    this._keyNavHandler = null;
}
```

**Note:** The `j`/`k` shortcuts follow Vim convention, which power users expect. Arrow keys work for everyone else.

_renderBreadcrumb(folderPath) {
    if (!folderPath) return '';
    const parts = folderPath.replace(/\/$/, '').split('/');
    let html = '<div class="zip-breadcrumb">';
    html += `<span class="zip-breadcrumb__item" data-folder="">/</span>`;
    let accumulated = '';
    for (const part of parts) {
        accumulated += part + '/';
        html += ` <span class="zip-breadcrumb__sep">&rsaquo;</span> `;
        html += `<span class="zip-breadcrumb__item" data-folder="${this.escapeHtml(accumulated)}">${this.escapeHtml(part)}</span>`;
    }
    html += '</div>';
    return html;
}
