## Strategy: v0.3.2 Complete Rewrite (Not v0.4.0)

The vault-browse unification does NOT require a major version consolidation. Following the IFD v1.3.0 guidance (Scenario 2: Complete Rewrite Mid-Chain), this is a **v0.3.2 file** that replaces `send-browse.js` entirely — with the `dataSource` interface built in and all 16 v0.3.1 BRW fixes merged.

### Why v0.3.2, not v0.4.0

- v0.4.0 would require consolidating ALL components (gallery, download, upload, viewer) — massive scope
- v0.3.2 only replaces ONE component (`send-browse`) — the one that needs the data source interface
- The IFD guide explicitly supports this: "A version-stamped file can be a complete rewrite that replaces the base entirely"
- v0.3.1 QA testing is happening right now — v0.3.2 must produce **zero user-visible changes** since it's a refactor only

### The Guarantee

**v0.3.2 is a refactor, not a feature change.** The QA team is currently testing v0.3.1's 16 BRW fixes. Every fix must work identically in v0.3.2. The only difference is internal architecture — `this.zipTree` becomes `this.dataSource.getFileList()`. The HTML output, CSS, and user interactions are unchanged.

## ZipDataSource Class

```javascript
// zip-data-source.js (~80 lines)

class ZipDataSource {
    constructor(zipInstance, zipTree, zipOrigBytes, zipOrigName) {
        this._zip       = zipInstance;
        this._tree      = zipTree;
        this._origBytes = zipOrigBytes;
        this._origName  = zipOrigName;
        this.writable   = false;
    }

    // Required: returns hierarchical tree
    getTree() {
        var root = { name: '', children: {}, files: [] };
        var files = this._tree.filter(e => !e.dir && !_isGalleryMeta(e.path));
        for (var f of files) {
            var parts = f.path.split('/');
            var node = root;
            for (var i = 0; i < parts.length - 1; i++) {
                if (!node.children[parts[i]])
                    node.children[parts[i]] = { name: parts[i], children: {}, files: [] };
                node = node.children[parts[i]];
            }
            node.files.push(f);
        }
        return root;
    }

    // Required: returns file bytes
    async getFileBytes(path) {
        var entry = this._tree.find(e => e.path === path && !e.dir);
        if (!entry) throw new Error('File not found: ' + path);
        return entry.entry.async('arraybuffer');
    }

    // Required: returns flat file list
    getFileList() {
        return this._tree.filter(e => !e.dir);
    }

    // Zip-specific: re-generate clean zip for Save
    async getZipBlob() {
        return this._zip.generateAsync({
            type: 'blob', mimeType: 'application/zip',
            compression: 'DEFLATE', compressionOptions: { level: 6 }
        });
    }

    getOrigName() { return this._origName; }
}

function _isGalleryMeta(path) {
    return path.startsWith('_gallery.') || path.startsWith('__gallery__');
}
```

## What send-download.js Needs to Change

`send-download.js` creates the browse component and passes data. It needs to create `ZipDataSource` and set it:

```javascript
// In _renderBrowseView() — add dataSource alongside legacy props:
_renderBrowseView() {
    const browse = document.createElement('send-browse');
    const dataSource = new ZipDataSource(
        this._zipInstance, this._zipTree, this._zipOrigBytes, this._zipOrigName
    );
    browse.dataSource   = dataSource;

    // Legacy props (backward compat — browse auto-creates ZipDataSource from these if no dataSource)
    browse.zipTree      = this._zipTree;
    browse.zipInstance   = this._zipInstance;
    browse.zipOrigBytes  = this._zipOrigBytes;
    browse.zipOrigName   = this._zipOrigName;

    browse.fileName      = this._zipOrigName;
    browse.transferId    = this.transferId;
    browse.downloadUrl   = this._downloadUrl;
    this.innerHTML = '';
    this.appendChild(browse);
}
```

This is a v0.3.2 overlay on `send-download.js` — a small prototype override of `_renderBrowseView` only.

## VaultDataSource Interface (For Vault Team)

The vault team builds this independently. Here's the contract they need to implement:

```javascript
class VaultDataSource {
    constructor(vaultInstance, accessKey) {
        this._vault = vaultInstance;
        this._accessKey = accessKey;
        this.writable = !!accessKey;
    }

    // Required: convert vault tree → Browse tree format
    getTree() {
        // vault._tree: { '/': { type: 'folder', children: { ... } } }
        // Browse expects: { name: '', children: { name: TreeNode }, files: [{ path, name, size }] }
        return this._convertTree(this._vault._tree);
    }

    // Required: fetch + decrypt file on demand
    async getFileBytes(path) {
        var parts = path.split('/');
        var fileName = parts.pop();
        var folderPath = parts.join('/') || '/';
        var blob = await this._vault.getFile(folderPath, fileName, this._accessKey);
        return blob.arrayBuffer();
    }

    // Required: flat file list
    getFileList() {
        return this._flattenTree(this._vault._tree);
    }

    // Optional write operations (vault-specific)
    async saveFile(folderPath, fileName, bytes) { ... }
    async renameFile(folderPath, oldName, newName) { ... }
    async deleteFile(folderPath, fileName) { ... }
}
```
