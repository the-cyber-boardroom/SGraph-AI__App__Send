## 4. The Receive Flow Fix (P0)

### Current problem
The "already have a token" link is a small grey text line inside the hero section:
```html
<a href="https://send.sgraph.ai" ...>Already have a token? Go to send.sgraph.ai →</a>
```
It is easy to miss because it is small, secondary, and pushes users off-site without helping them understand what to do next.

### Fix

**Two-part solution:**

**Part A — Persistent nav CTA (highest priority)**
Add a `[Paste a token →]` button to the nav bar at all times. Clicking it expands an inline input:
```
[ Paste your token _________________________________ ] [Open →]
```
This inline input redirects to `https://send.sgraph.ai/en-gb/browse/#{token}` on submit. Zero friction, stays on the current page until submit.

**Part B — Hero dual-CTA layout**
Restructure the hero to have two equal-weight CTAs:
```
[ Send a file →  ]    [ I have a token — open it ]
```
Second CTA is teal outline button that opens the inline token input.

This serves both visitor types from the same hero without making either feel secondary.

## 5. Pricing Page Architecture

### Current pricing page problems
- 4-column grid mixing future aspirational tiers with today's reality
- No clear distinction between "live now" and "coming soon"
- AWS Marketplace page exists but is aspirational — lists not yet submitted
- No mention of Docker container (which NOW EXISTS)

### Proposed pricing page structure

**Narrative approach, not just a table.** Three clear sections:

---

**Section A: Start free**
> "Everything you need to evaluate and self-host. No credit card."

| Offer | Status | Cost |
|-------|--------|------|
| Open source (GitHub) | Live | Free |
| Docker container (ECS, Fargate, EC2) | Live | Free (you pay compute) |
| Early access credits | Live | Free |

**Section B: Hosted service**
> "We run it. You send files."

| Offer | Status | Cost |
|-------|--------|------|
| Starter Pack — 500 transfers | Live | £5 (£0.01/file) |
| Buy credits via Stripe | Live | From £5 |

**Section C: AWS Marketplace (coming)**
> "One-click deploy in your AWS account. We package it; you own the infrastructure."

| AMI | Status | Pricing model |
|-----|--------|---------------|
| Playwright Service (Browser Automation) | Submitting this week | Free tier + usage |
| SG/Send Server (Encrypted VFS + SGit) | In preparation | Free tier + usage |
| Tools Suite (AI Tools Portal) | In preparation | Free tier + usage |

**Label discipline:**
- "Live" = Stripe checkout works today
- "Submitting this week" = AWS listing pending
- "In preparation" = architecture exists, packaging in progress
- Never: "coming soon" without a qualifier

## 6. Technical Constraints

| Constraint | Rule |
|-----------|------|
| Static HTML/CSS/JS | No build step. Files deployed as-is to S3. |
| i18n | All user-facing text via `data-i18n` attributes + JSON string files |
| Web Components | IFD versioning: `v0/v0.1/v0.1.0/component.js` |
| CSS | Extend `_common/css/style.css` — do not fork it |
| Deploy sequence | Build /security/ → deploy → wait for indexing → then redirect /architecture/ |
| AWS stack | The story to tell: Lambda (User + Admin functions), S3 (storage), ECR (Docker), CloudFront (CDN), EC2 (self-hosted AMI) |
| Component reuse | Import from tools.sgraph.ai — do not rebuild shared components |
