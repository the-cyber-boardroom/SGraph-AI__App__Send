## Architecture Decisions Validated

### 1. Zero-Framework Frontend (IFD + Web Components)

The decision to use vanilla JS Web Components with zero framework dependencies is validated by this result. No React/Vue/Svelte bundle to parse — the browser has nothing to do except render the HTML and execute small component scripts.

**Impact:** The largest JS file is the upload component at ~20kB. Compare with a typical React bundle at 150-300kB gzipped.

### 2. CloudFront + S3 Static Origin (First Time in Production)

This is the **first deployment using the new CloudFront distribution** with:

- **Two origins**: S3 (static) and Lambda (API)
- **Two behaviours**: `/api/*` routes to Lambda, `/*` routes to S3
- **URL-rewrite CloudFront Function**: handles `/{locale}/` → `/{locale}/index.html`

The S3 origin delivers content at CDN edge, eliminating Lambda cold-start for UI pages entirely.

### 3. Pre-Rendered i18n (No Runtime JSON Loading)

The i18n architecture was redesigned from runtime JSON loading to build-time pre-rendering:

- **CI generates** locale HTML pages from `en-gb/` templates + `i18n/*.json`
- **16 locale folders** are sibling directories alongside `en-gb/`
- **Shared `_common/` assets** are referenced via relative paths — no duplication
- **No `fetch()` for translations** — zero network overhead for i18n

### 4. Locale-Aware URL Routing

```
send.sgraph.ai/en-gb/       → en-gb/index.html
send.sgraph.ai/pt-pt/       → pt-pt/index.html (pre-rendered)
send.sgraph.ai/en-gb/download/#tid/key → download page
```

Navigation between locales is a page navigation (`window.location.href`), not a runtime re-render. This is correct for pre-rendered content and ensures SEO-friendly locale URLs.

## Topology Change: Two-Origin CloudFront

```
                    ┌──────────────┐
                    │  CloudFront  │
                    │ Distribution │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │                         │
    ┌─────────▼─────────┐   ┌──────────▼──────────┐
    │   S3 Bucket       │   │   Lambda Function   │
    │ (Static Assets)   │   │   (API endpoints)   │
    │                   │   │                     │
    │ /en-gb/           │   │ /api/transfers/*    │
    │ /pt-pt/           │   │ /api/presigned/*    │
    │ /_common/         │   │ /api/vault/*        │
    │ /favicon.svg      │   │ /api/info/*         │
    └───────────────────┘   └─────────────────────┘
```

### Cache Policy

| Content Type | Cache TTL | Rationale |
|---|---|---|
| HTML pages | 300s (5min) | Balance freshness vs. CDN offload |
| JS/CSS | 86400s (1 day) | Versioned paths via IFD — content-addressed |
| Fonts | 604800s (7 days) | Immutable WOFF2 files |
| API responses | 0 (CachingDisabled) | Dynamic data |

## Risk Assessment

| Risk | Severity | Mitigation |
|---|---|---|
| CloudFront cache serves stale HTML after deploy | Medium | Cache invalidation runs post-deploy in CI |
| Locale URLs not generated for new pages | Low | Manifest-driven deployment ensures all folders deployed |
| S3 bucket permissions too broad | Low | OAC (Origin Access Control) restricts S3 to CloudFront only |
