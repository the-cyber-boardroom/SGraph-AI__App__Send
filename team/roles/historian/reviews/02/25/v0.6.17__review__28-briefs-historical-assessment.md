## Brief #20: PKI Historical Analysis — Research Plan

### Question 1: Why PGP Failed — Evidence-Based Post-Mortem

**Research approach:** Investigate each hypothesis from the brief with historical evidence.

| Hypothesis | Historical Evidence to Gather |
|---|---|
| **UX was terrible** | Zimmermann's original CLI (1991). Adoption data for GUI wrappers (PGP Desktop, GPG4Win, Enigmail). Mozilla Thunderbird PGP integration timeline. User studies if available. |
| **Key management was the killer** | SKS keyserver poisoning incident (2019). Key revocation failures documented by the OpenPGP community. "I lost my private key" frequency in support forums. |
| **Web of trust didn't scale** | Key signing party attendance data. The Debian keyring size over time. Academic papers on WoT graph connectivity. |
| **No network effect** | Email encryption adoption rates (EFF surveys, Pew Research). The "both parties need PGP" cold start problem. |
| **Email providers didn't integrate** | Gmail S/MIME support timeline. Outlook S/MIME vs PGP. Google End-to-End project (announced 2014, abandoned). ProtonMail as the exception. |
| **Pain point wasn't big enough** | Snowden revelations (2013) — did they cause a PGP adoption spike? Evidence for duration of spike. |
| **Regulation didn't require it** | GDPR encryption provisions (Art. 32). HIPAA encryption requirements. PCI-DSS. Which actually mandate email encryption? |
| **Alternatives emerged** | Signal Protocol (2013). WhatsApp E2E encryption (2016). iMessage encryption. Timeline of consumer encryption adoption. |

**Key distinction to establish:** Which failures are intrinsic to PKI (the mathematical model) vs failures of specific PKI implementations (PGP/GPG tooling, SKS keyservers, web of trust model)?

### Question 2: Historical Secure Communication Timeline

| Period | Key Events | Lessons for SG/Send |
|---|---|---|
| **Ancient (3000 BCE-1900)** | Caesar cipher, scytale, steganography. Wax seals as tamper evidence. Diplomatic pouches. | Physical tamper evidence (seals) is the direct ancestor of digital signatures. The problem is ancient; only the medium changes. |
| **Telegraph era (1840-1940)** | Commercial code books. One-time pads. Enigma machine (1920s). SIGINT establishment. | Encryption over untrusted channels has been essential since the telegraph. The tension between convenience and security is 180 years old. |
| **Cold War (1945-1991)** | NSA founding (1952). DES standardisation (1977). Public key cryptography (Diffie-Hellman 1976, RSA 1977). | Government monopoly on strong crypto broke with public key discovery. Parallels to today's platform monopoly on identity. |
| **PGP era (1991-2010)** | PGP release (1991). Zimmermann investigation. S/MIME (1995). SKS keyservers. Key signing parties. Peak adoption ~2005. | Technology was right, distribution model was wrong. Manual key management doomed consumer adoption. |
| **TLS success (1995-2015)** | SSL 2.0 (1995). Certificate authorities. Let's Encrypt (2014). Chrome HTTP warning (2018). HTTPS adoption crosses 90%. | TLS won because it was invisible. CA model handled identity. Browser enforcement created adoption pressure. |
| **Messaging encryption (2013-present)** | Signal Protocol. WhatsApp E2E (2016). iMessage. Telegram controversy. | Invisible encryption wins. Key management handled by the platform. Users never see a key. |
| **Blockchain identity (2015-present)** | Self-sovereign identity. DIDs. Verifiable Credentials. ENS. | Over-promised, under-delivered. Naming problems not solved. UX remains poor. But the "user controls their identity" principle is correct. |

### Question 3: The Semantic Web Parallel

**Research focus:** Why the semantic web failed and whether PKI faces the same risks.

| Semantic Web Failure Mode | PKI Risk Assessment |
|---|---|
| Creating RDF mappings was too labour-intensive | Key management is labour-intensive. SG/Send mitigates with auto-generation. Risk: MEDIUM (if auto-generation works). |
| Graph databases weren't mature | Trust web infrastructure doesn't exist. Risk: HIGH (must be built). |
| No economic incentive for data publishers | No economic incentive for key publishers. CC-Signed creates the incentive. Risk: MEDIUM (depends on CC-Signed adoption). |
| Tooling was poor | Web Crypto API is mature. Browser-based crypto works. Risk: LOW. |
| Revived by knowledge graphs + LLMs | Revived by deepfakes + agentic AI + identity crisis. Timing: FAVOURABLE. |

**Historian's preliminary assessment:** The enabling conditions (browser crypto maturity, identity verification demand driven by deepfakes, agentic AI requiring verified identity) suggest NOW may be the right timing — analogous to how LLMs revived knowledge graphs.

### The TLS Success Model — Historical Lessons

The brief asks why TLS succeeded where PGP failed. Five lessons with historical evidence:

1. **Invisibility:** TLS requires zero user action. PGP required key generation, exchange, and management. SG/Send must be closer to TLS than PGP.
2. **Trust anchors:** CAs handled identity for TLS. PGP's web of trust put the burden on users. SG/Send needs a trust anchor (GitHub as PKI registry, or platform-managed identity).
3. **Platform enforcement:** Browsers enforce HTTPS. No platform enforced PGP. SG/Send must make PKI non-optional by design.
4. **Cost elimination:** Let's Encrypt removed the cost barrier for TLS certificates. SG/Send auto-generates keys at zero cost.
5. **Economic incentive:** Google ranked HTTPS sites higher. SG/Send uses trust scores and platform access as the economic lever.

### PGP Historical Timeline (for Brief #20)

| Year | Event | Significance |
|---|---|---|
| 1976 | Diffie-Hellman key exchange published | Public key cryptography invented |
| 1977 | RSA algorithm published | Practical asymmetric encryption |
| 1991 | PGP 1.0 released by Zimmermann | First consumer PKI tool |
| 1993 | US government investigation of Zimmermann | Crypto export controls — chilling effect on adoption |
| 1995 | S/MIME proposed (RFC 1847) | Corporate alternative to PGP — CA-based, not web of trust |
| 1997 | OpenPGP standard (RFC 2440) | Standardisation attempt |
| 1999 | GnuPG 1.0 released | Free/open alternative to commercial PGP |
| 2010 | PGP Corp acquired by Symantec | Commercial PGP effectively dies |
| 2013 | Signal Protocol created | Consumer encryption without key management |
| 2014 | Google End-to-End project announced | Attempted PGP in Gmail — never shipped |
| 2016 | WhatsApp enables E2E encryption (Signal Protocol) | 1 billion users get invisible encryption |
| 2018 | EFF discontinues PGP recommendations | "PGP is on life support" |
| 2019 | SKS keyserver poisoning attack | Web of trust infrastructure compromised |
