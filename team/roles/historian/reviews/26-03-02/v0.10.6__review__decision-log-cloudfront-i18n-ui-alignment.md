# Historian Review — Decision Log: CloudFront i18n, UI Alignment, Performance Milestone

**Version:** v0.10.6
**Date:** 2 March 2026
**Role:** Historian

## Decision Record

### D-2026-03-02-001: First CloudFront Distribution with Two Origins

**Context:** SG/Send previously served all content through Lambda function URLs. The v0.2.0 UI migration moved static content to S3 behind CloudFront.

**Decision:** Deploy a CloudFront distribution with two origins and two behaviours:
- Default behaviour (`/*`) → S3 static origin
- API behaviour (`/api/*`) → Lambda function URL origin

**Rationale:** Eliminates Lambda cold-start for UI pages. CDN edge delivery for static content. Separates concerns (static vs. compute).

**Result:** DOMContentLoaded 112ms, full load 147ms. Previously, Lambda-served pages included ~300-500ms cold-start overhead.

**Status:** Deployed to dev.send.sgraph.ai, validated.

---

### D-2026-03-02-002: i18n Architecture — Pre-Rendered Pages over Runtime JSON

**Context:** The initial v0.2.0 i18n used runtime JSON loading: `I18n.setLocale()` fetched `i18n/{code}.json` via `fetch()`, parsed it, and re-rendered all components.

**Decision:** Switch to build-time pre-rendering of locale pages. CI generates `/{locale}/index.html` for each of 16 locales from `en-gb/` templates + `i18n/*.json`.

**Rationale:**
- Zero runtime network cost for translations
- SEO-friendly URLs (`/pt-pt/`, `/de-de/`)
- Consistent with sgraph.ai website's i18n approach
- Each page is self-contained — no dependency on async JSON loading

**Trade-off:** Language switching requires a page navigation instead of instant client-side re-render. Accepted because the page loads in 150ms anyway.

**Status:** Implemented. CI workflow generates locale pages on push to dev.

---

### D-2026-03-02-003: Locale Code Convention — Language-Country Lowercase

**Context:** v0.2.0 initially used short codes (`pt`, `de`, `fr`) and mixed case (`pt-PT`).

**Decision:** Adopt sgraph.ai website convention: lowercase language-country codes (`pt-pt`, `de-de`, `fr-fr`, `es-es`, etc.).

**Rationale:** Consistency between sgraph.ai and send.sgraph.ai. URL paths are cleaner (`/pt-pt/` not `/pt-PT/`). CI generation simpler with one convention.

**Status:** Applied to i18n.js availableLocales, generate_send_i18n_pages.py LOCALES, and all locale selector UI.

---

### D-2026-03-02-004: Navigation Width — 960px → 1200px

**Context:** SG/Send header used `max-width: 960px`. sgraph.ai uses `max-width: 1200px`.

**Decision:** Align SG/Send to 1200px to match the parent brand.

**Rationale:** Visual consistency across the product family. The upload form itself is narrower, so the wider header gives better breathing room on large screens.

**Status:** Implemented in send-header.css.

---

### D-2026-03-02-005: Favicon — SG/ Brand Mark

**Context:** v0.2.0 had no favicon. The sgraph.ai website uses a text-based `SG/` SVG favicon.

**Decision:** Use the graphical `sg-send-favicon.svg` (S-curve in teal + coral slash + encryption dots on navy background) for SG/Send. Inline the SVG in the header component for zero additional network requests.

**Rationale:** The graphical favicon is distinctive at small sizes (browser tab) while the inline SVG in the header provides instant visual brand recognition without an extra HTTP request.

**Status:** Favicon SVG copied to `_common/favicon.svg`, linked in both upload and download HTML pages, inlined in header component.

---

### D-2026-03-02-006: Upload Success Page — Port v0.1.6 Layout to v0.2.0

**Context:** During v0.2.0 IFD consolidation, the `renderResult()` method was based on the v0.1.4 base component rather than the v0.1.6 dark-theme-aware override that was active on production (via v0.1.8).

**Decision:** Port the v0.1.6 layout to v0.2.0, including standalone value box for share link, teal Copy Link button, visual timing bar chart, and contained separate-key panel.

**Rationale:** The v0.1.6 layout was already validated in production and is visually superior. The v0.2.0 flat layout was a regression.

**Lesson:** IFD overlays need careful tracking during major version consolidation. Each override should be explicitly listed in the consolidation checklist.

**Status:** Implemented. Visual comparison with send.sgraph.ai/send/v0/v0.1/v0.1.8/ recommended for verification.
