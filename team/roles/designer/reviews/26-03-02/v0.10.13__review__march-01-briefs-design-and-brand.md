## Brief 1: Challenge Card Design

**Source:** `team/humans/dinis_cruz/briefs/03/01/v0.10.1__brief__sg-send-challenge-outreach-campaign.md`

### Design Spec for SVG/PNG Card

The challenge card is the campaign's visual anchor. It will be seen cold -- in a WhatsApp message preview, a LinkedIn feed, an email inbox -- before the recipient knows anything about SG/Send. The card must accomplish three things in under two seconds: establish brand recognition, communicate the challenge concept, and feel personal rather than corporate.

**Dimensions and formats:**

| Format | Dimensions | Use Case | Notes |
|--------|-----------|----------|-------|
| SVG | Scalable | Source / archive | Master design file; export to PNG for distribution |
| PNG | 1200 x 630-720px | LinkedIn share preview, email | Landscape aspect for card rendering |
| PNG | 1200 x 1200px | WhatsApp square preview | WhatsApp crops landscape images; square is safer |
| PNG | 1200 x 628px | LinkedIn-specific | Exact LinkedIn card spec |

WhatsApp does NOT support SVG as an inline image. WhatsApp strips SVG attachments and does not render them inline. PNG is mandatory for WhatsApp. SVG is only useful as the editable source.

**Minimum font sizes:** WhatsApp renders shared images at approximately 300px wide. At a 1200px source that is a 4x reduction. Minimum body text must be 64px in the source file (renders at ~16px at thumbnail). Headline text should be at least 80px source (renders at ~20px). Below these sizes, text becomes illegible blurs.

### Colour Usage

The card uses the Aurora palette from `design-tokens.css`:

| Element | Token | Hex | Rationale |
|---------|-------|-----|-----------|
| Background | `--color-bg` | `#1A1A2E` (Dark Navy) | Consistent with app and website dark theme |
| Headline | `--color-primary` | `#4ECDC4` (Teal) | Hero accent -- draws the eye to the challenge title |
| Body text | `--color-text` | `#E0E0E0` (Light Grey) | High readability on dark background |
| Step numbers | `--color-error` | `#E94560` (Coral) | Visual cadence -- numbered steps pop against navy |
| URL / CTA | `--color-primary` | `#4ECDC4` (Teal) | Interactive elements always teal per brand guidelines |
| Reward line / domain | `--color-text-secondary` | `#8892A0` (Slate) | Subdued supporting text |

The card must not use white backgrounds or bright gradients. These would read as generic marketing and break the Aurora identity.

### Typography

| Element | Font | Weight | Size (at 1200px source) |
|---------|------|--------|-------------------------|
| Title ("The SG/Send Challenge") | DM Sans | 700 (Bold) | 80px |
| Body steps | DM Sans | 500 (Medium) | 64px |
| Reward line | DM Sans | 400 (Regular) | 56px |
| URL (sgraph.ai) | DM Sans | 500 (Medium) | 48px |

JetBrains Mono is not appropriate for the challenge card. The card is conversational, not technical. DM Sans throughout maintains a warm, approachable tone.

The slash in "SG/Send" in the title must follow the brand specification: rendered as a custom 75-degree stroke in teal (`#4ECDC4`), 1.2x the weight of surrounding letterforms. In a static image this is baked into the design.

### Layout (1200 x 630 landscape variant)

```
+-------------------------------------------------------------+
|  [48px padding all sides]                                    |
|                                                              |
|     [Concept C icon -- 48px]                                 |
|                                                              |
|     The SG/Send Challenge            <- Teal, Bold 80px     |
|                                                              |
|     Can you send an encrypted file                           |
|     that nobody can read --                                  |
|     not even us?                     <- Light Grey, Med 64px |
|                                                              |
|     1. Go to sgraph.ai              <- Coral numbers         |
|     2. Find the buy button (5 GBP)                           |
|     3. Send an encrypted file                                |
|     4. Get someone to open it                                |
|     5. Tell me -- via SG/Send                                |
|                                                              |
|     Complete it and I'll send you                            |
|     a 5 GBP voucher.                <- Slate, Regular 56px  |
|                                                              |
|     sgraph.ai                        <- Teal, Medium 48px   |
|                                                              |
+-------------------------------------------------------------+
```

The Concept C icon mark should appear in the top-left at 48px (proportional to the card). The full wordmark lockup ("SG/Send") is too wide for a text-dense card -- the icon alone communicates brand without consuming layout space.

### Multilingual Variants

The brief requires at minimum EN, ES, PT, DE, FR. Design considerations:

- German text is typically 30-40% longer than English. The card layout must accommodate "Kannst du eine verschlusselte Datei senden, die niemand lesen kann -- nicht einmal wir?" without breaking the visual composition.
- **Approach: fixed font size, variable card height.** Keep font size constant for legibility; allow the card to grow up to 15% taller for longer languages. Design the English card first as the shortest variant; use the German card as the stress test for maximum expansion.
- The numbered steps are short and translate well. The title and reward line need the most layout flexibility.
- Each variant should include the country flag as a small (24px) indicator in the top-right corner to signal the language at a glance.
- A QR code to sgraph.ai in the bottom-right corner (80px, Slate #8892A0 on Dark Navy, subtle) allows scan-to-visit from forwarded screenshots.

**Deliverables:** 10+ PNG files (5 languages x 2+ aspect ratios), 5 SVG source files.

## Brief 2: Translation Visualisation Tool UI

**Source:** `team/humans/dinis_cruz/briefs/03/01/v0.10.1__brief__translation-tool-and-translator-community.md`

### UI Layout Recommendations

The translation tool lives at `sgraph.ai/tools/translations/` and must use the Aurora design system -- same tokens, same typography, same header/footer chrome as the sgraph.ai website. It is a client-side-only tool with zero tracking. The tool has four distinct UI surfaces that are all data-dense interfaces on the Aurora dark theme.

#### Page Structure

```
+---------------------------------------------------------------+
|  [Header: SG/Send tools nav]                                   |
+---------------------------------------------------------------+
|                                                                |
|  Translation Dashboard                                         |
|  +-----------------------------------------------------------+|
|  |  Coverage heatmap (17 variants, progress bars)             ||
|  |  EN-GB  ==================== 100% (base)                   ||
|  |  DE-DE  ==================== 100% (AI)                     ||
|  |  ES-ES  ==================== 100% (AI)                     ||
|  |  ...                                                       ||
|  +-----------------------------------------------------------+|
|                                                                |
|  [Compare]    [Diff View]    [Glossary]     <- tab navigation  |
|                                                                |
|  [Active panel based on tab selection]                          |
|                                                                |
+---------------------------------------------------------------+
```

### Component Design

**1. Side-by-Side Comparison Table**

Three-column layout: Key | Language A | Language B.

- Key column: fixed width (180px minimum), `JetBrains Mono` (`--font-mono`), `--text-sm` (0.875rem). Truncated with `text-overflow: ellipsis` and tooltip on hover.
- Translation columns: flex-grow, equal width, splitting remaining space. `DM Sans`, `--text-body` (1rem).
- Alternating row backgrounds: `--color-bg` (`#1A1A2E`) and `--bg-surface` (`#1E2A4A`).
- Border between columns: `--color-border` (`rgba(78, 205, 196, 0.15)`).
- An "Edit on GitHub" pencil icon per row: 16px, Slate (#8892A0) default, Teal (#4ECDC4) on hover. Right-aligned in key column. Must not compete with translation text for attention.
- On mobile (<768px): collapse to stacked card layout -- one card per key, Language A above Language B. A three-column table on 375px is unreadable.

**2. Culture Variant Diff View**

This is the most distinctive feature. Show ONLY differing keys between two variants (e.g., PT-PT vs PT-BR).

- **Word-level diff highlighting** is critical. A word-level diff (highlighting "ficheiro" vs "arquivo" within the sentence) is dramatically more useful than a row-level "DIFFERENT" marker.
- Diff highlight colour: Coral (#E94560) at 20% opacity as background (`rgba(233, 69, 96, 0.2)`). Semantically appropriate: Coral marks "what is different" just as it marks "what the server never sees" in the brand.
- Statistics bar at top: "Showing N differences out of M keys (X% identical)" in `--color-text-secondary`.
- Toggle: "Show identical keys" checkbox (off by default).

**3. Coverage Dashboard**

Progress bar colours following the Aurora palette:

| Coverage | Fill Colour | Token |
|----------|------------|-------|
| 100% | Teal #4ECDC4 | `--color-primary` |
| 80-99% | Teal at 70% opacity | `rgba(78, 205, 196, 0.7)` |
| Below 80% | Warning orange #E07C4F | `--color-warning` |

Do not use green/red traffic-light colours. The Aurora palette does not include green, and traffic-light colours on dark navy create a jarring palette break.

**Quality badges** rendered as styled pills next to each language:

| Badge | Background | Border | Text Colour |
|-------|-----------|--------|------------|
| AI | `rgba(136, 146, 160, 0.1)` | `rgba(136, 146, 160, 0.2)` | Slate #8892A0 |
| Reviewed | `rgba(78, 205, 196, 0.08)` | `rgba(78, 205, 196, 0.15)` | Teal #4ECDC4 |
| Edited | `rgba(224, 124, 79, 0.08)` | `rgba(224, 124, 79, 0.2)` | Warning #E07C4F |
| Native | `rgba(78, 205, 196, 0.15)` | `rgba(78, 205, 196, 0.3)` | Teal #4ECDC4, bold |

**4. Glossary Panel**

Searchable table with sticky header. Columns: English | Variant A | Variant B | Notes. Search input styled with `--color-border` focus ring (`rgba(78, 205, 196, 0.25)` expanding to `rgba(78, 205, 196, 0.5)` on focus). Notes column in `--text-sm`, `--color-text-secondary`.

### How It Fits the Design System

Each section should be a self-contained Web Component following IFD methodology:

| Component | Purpose |
|-----------|---------|
| `translation-dashboard` | Coverage progress bars, quality badges, language selector |
| `translation-compare` | Side-by-side key-value comparison table |
| `translation-diff` | Variant diff view with word-level highlighting and statistics |
| `translation-glossary` | Searchable glossary table |

All components consume design tokens from `:root` CSS custom properties (as established in `design-tokens.css` -- tokens cross Shadow DOM boundaries automatically).

**Shared component gap:** The User UI v0.2.0 shared-components CSS has cards, buttons, inputs, status badges, and progress bars -- but no data table component. The Admin UI has tables but these are inside individual Web Component shadow DOMs. Before building the translation tool, extract a shared data table component from Admin UI patterns and add it to the shared-components CSS.

## Brief 3: Website Improvements — UX Recommendations

**Source:** `team/humans/dinis_cruz/briefs/03/01/v0.10.1__brief__website-improvements-and-content-strategy.md`

### Token UX Paths

The brief identifies two user paths that need clearer visual separation:

**Path 1: "I already have a token"**
- Currently a text link in the sub-hero area.
- Keep as a text link -- this is a bypass route for returning users, not a primary conversion path.
- Style: `color: var(--accent)`, DM Sans Regular 400, `var(--text-sm)`. This is already in place and should not be promoted to a button -- it would compete with the primary conversion CTAs.

**Path 2: "I want to get a token"**
- This is the primary funnel. It branches into Free (form) and Paid (Stripe).
- Present as a split card on the pricing page:

```
+-----------------------------------------------------+
|  Get Started                                         |
|                                                      |
|  +-------------------+  +---------------------+     |
|  |  Free              |  |  Instant             |     |
|  |                    |  |                      |     |
|  |  Join Early        |  |  Buy 500 files       |     |
|  |  Access            |  |  for 5 GBP           |     |
|  |                    |  |                      |     |
|  |  Sherpa responds   |  |  Token delivered     |     |
|  |  within 48 hours   |  |  immediately         |     |
|  |                    |  |                      |     |
|  |  [Get Started]     |  |  [Buy Credits]       |     |
|  |  outlined teal     |  |  filled teal         |     |
|  +-------------------+  +---------------------+     |
|                                                      |
+-----------------------------------------------------+
```

- The paid path ("Buy Credits") gets the **primary CTA** (filled teal button, `background: var(--accent)`, `color: var(--color-bg)`) because it has the shorter time-to-value.
- The free path ("Get Started") gets the **secondary CTA** (outlined teal button, `border: 1px solid var(--color-primary)`, `background: transparent`, `color: var(--color-primary)`) because it involves a 48-hour wait.
- Both cards sit on `--bg-surface` (`#1E2A4A`) with `--color-border` divider.

### Pricing Table CSS Fixes for Long Translations

The brief reports that pricing tables break in non-English languages (text overflow in DE-DE, HR-HR, PT-PT). Root cause: fixed column widths cannot accommodate longer translated strings.

**CSS fixes required:**

1. **Replace fixed widths with flex layout:**
   ```css
   .pricing-column {
       min-width: 200px;
       flex: 1 1 auto;
   }
   ```

2. **Handle long compound words (German is the worst case):**
   ```css
   .pricing-feature {
       overflow-wrap: break-word;
       word-break: normal;
       hyphens: auto;
   }
   ```
   The `lang` attribute must be set correctly on each locale's HTML element for `hyphens: auto` to work.

3. **CTA buttons must handle two-line text:**
   German "Jetzt kostenlos starten" (for "Get Started Free") is 60% longer than English. Buttons must use:
   ```css
   .pricing-cta {
       white-space: normal;
       min-height: 48px;
       padding: var(--space-3) var(--space-6);
   }
   ```

4. **Mobile breakpoint -- stack columns vertically:**
   ```css
   @media (max-width: 640px) {
       .pricing-grid {
           flex-direction: column;
       }
   }
   ```

**Stress test languages:** DE-DE (long compound words like "Dateiuebertragung"), HR-HR (diacritics and multi-word translations), PT-PT (longer Romance-language constructions), FR-CA (longer preposition constructions).

### Stripe Integration Design Touchpoints

Two new pages needed:

**Payment Success Page (`/payment/success`):**
- Dark navy background, centred content.
- Teal checkmark icon (`--color-success`).
- Heading: "Payment received" (DM Sans Bold, `--text-h1`).
- Body: "Your access token will arrive within 48 hours." (DM Sans Medium, `--text-body`).
- Secondary link: "Go to SG/Send" (teal, underlined).

**Payment Cancel Page (`/payment/cancel`):**
- Same layout, neutral (Slate) icon.
- Heading: "Payment cancelled".
- Body: "No charges were made."
- Two CTAs: "Try Again" (filled teal) and "Get Free Access" (outlined teal).

| Logo Colour | Hex | Design Token | Token Value | Aligned |

### Gaps Identified

**D-LG3: Missing clear-space rules.** The spec defines minimum widths (32px icon, 280px lockup, 16px monogram) but no minimum clear space around the mark. Recommendation:
- Icon mark: 4px (12.5% of viewBox) on all sides.
- Full lockup: height of the "S" character on all sides.
- No element should touch or overlap the logo at any size.

**D-LG4: Missing usage restrictions.** The spec should state what is not allowed: no rotation, no distortion, no recolouring (except defined monochrome variants), no placement on busy backgrounds, no drop shadows or outlines.

**D-LG5: Missing monochrome variants.** Needed for print, email signatures, and low-colour contexts:
- Monochrome light: all elements in Light Grey (#E0E0E0) on Dark Navy or transparent.
- Monochrome dark: all elements in Dark Navy (#1A1A2E) on white (for print/light-mode).
- Single-colour: all elements in Teal (#4ECDC4) on Dark Navy.

**D-LG6: Slash angle discrepancy.** The spec states "approximately 75 degrees from horizontal." The SVG coordinates are (20,5) to (12,27), giving vector dx=-8, dy=22. The angle from horizontal is `atan(22/8) = 70.0 degrees`, not 75. Either the SVG should be adjusted to 75 degrees (endpoint would be approximately (14.1, 27)) or the spec should state the actual angle of 70 degrees. This is a minor visual difference. The human should decide which is canonical.

**D-LG7: No dedicated coral brand token.** Coral (#E94560) is currently aliased only as `--color-error` and `--danger` in design-tokens.css. In the logo it represents the encryption boundary (positive, identity-defining). A dedicated `--brand-coral: #E94560` token would add semantic clarity without changing any values.

**D-LG8: 16px monogram variant has no SVG source.** The spec lists three variants (full lockup, icon mark, monogram "S/") but only provides SVG for the icon mark. Clarify whether the monogram is a distinct asset or the icon mark at 16px.

**D-LG9: Missing file inventory assets.** The spec lists five files (lockup PNG, lockup SVG, favicon SVG, favicon 512px PNG, favicon 32px PNG). Only the favicon SVG is present in the repo. The lockup SVG/PNG and the PNG favicon exports have not been produced.

### Gaps Summary

| # | Gap | Severity |
|---|-----|----------|
| D-LG3 | Missing clear-space rules | Medium |
| D-LG4 | Missing usage restrictions | Medium |
| D-LG5 | Missing monochrome variants | Medium |
| D-LG6 | Slash angle: spec says 75, SVG is 70 degrees | Low |
| D-LG7 | No brand-coral token in design-tokens.css | Low |
| D-LG8 | 16px monogram unspecified | Low |
| D-LG9 | Missing lockup and PNG exports from file inventory | High |

## Brief 5: OG Image Social Preview

**Source:** `team/humans/dinis_cruz/briefs/03/01/v0.7.19__task__og-image-social-preview.md`

### Dimensions

| Property | Value | Rationale |
|----------|-------|-----------|
| Width | 1200 px | Open Graph standard; Facebook, LinkedIn, Twitter all optimise for this |
| Height | 630 px | 1.91:1 aspect ratio; the standard for og:image |
| Format | PNG | Crisp text and logo rendering; JPEG artefacts on flat dark colours |
| File size target | Under 300 KB | Fast loading for social platform crawlers |

### Content Layout

```
+-------------------------------------------------------------+
|                                                              |
|  Background: solid Dark Navy #1A1A2E                         |
|                                                              |
|           [Concept C icon -- 96px]                           |
|                                                              |
|           SG/Send                                            |
|           (Wordmark lockup, Light Grey #E0E0E0)              |
|                                                              |
|           Send files. Nobody can read them                   |
|           -- not even us.                                    |
|           (DM Sans Medium 500, Teal #4ECDC4, 36px)          |
|                                                              |
|           --------- (Teal rule, 120px, 1px)                 |
|                                                              |
|           sgraph.ai                                          |
|           (JetBrains Mono, Slate #8892A0, 24px)             |
|                                                              |
|  Optional: subtle encryption dots in corners                 |
|  (mirroring the icon mark, at 10% opacity)                  |
|                                                              |
+-------------------------------------------------------------+
```

### Brand Alignment

- Background: `--color-bg` (#1A1A2E) -- consistent with all SG/Send surfaces.
- The icon mark is Concept C rendered at 96px. At this size all four layers are visible: background rect, S-curve, slash, encryption dots.
- Tagline uses the product-specific hero text: "Send files. Nobody can read them -- not even us." The brief's suggested tagline ("AI Agents That Build, Use, and Secure the Same Platform") is the SGraph AI parent brand tagline. For `send.sgraph.ai`, use the product tagline.
- A thin teal horizontal rule (#4ECDC4, 1px height, ~120px wide) separates tagline from URL.
- The domain in JetBrains Mono and Slate grey -- understated but legible.
- No gradients, no abstract shapes, no stock imagery. The Aurora theme is restrained. A dark solid background with typographic elements and the icon mark is more on-brand than aurora-gradient effects.

### Thumbnail Legibility

WhatsApp renders og:image at approximately 300px wide (4x reduction from 1200px source):
- Icon mark at 96px source renders at ~24px -- still recognisable.
- Wordmark at ~280px source renders at ~70px -- legible.
- Tagline at 36px source renders at ~9px -- at the edge of legibility. Keep the tagline short.
- Prioritise icon mark and wordmark in the upper 60% of the image. The tagline and URL are secondary.

### Two Variants Needed

| Variant | Tagline | URL | Use |
|---------|---------|-----|-----|
| sgraph.ai (parent) | "AI Agents That Build, Use, and Secure the Same Platform" | sgraph.ai | Parent site shares |
| send.sgraph.ai (product) | "Send files. Nobody can read them -- not even us." | send.sgraph.ai | Product shares |

### Integration

File location: `sgraph_ai__website/_common/images/og-preview.png` (and `og-preview-send.png` for the variant).

Add to `<head>` of all HTML pages:
```html
<meta property="og:image" content="https://sgraph.ai/_common/images/og-preview.png">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta name="twitter:image" content="https://sgraph.ai/_common/images/og-preview.png">
```

The `twitter:image` tag should be added explicitly even though Twitter/X falls back to `og:image`.

Test with: https://developers.facebook.com/tools/debug/ and https://cards-dev.twitter.com/validator

### Cross-Brief Observations

**The challenge card and OG image share a visual language.** Both are 1200px-wide PNG assets on Dark Navy with DM Sans typography and the Concept C icon. Design the OG image first (simpler -- fewer text elements), then use it as the template foundation for the challenge card.

**Coral carries three semantic roles across the briefs.** In the logo it represents the encryption boundary (positive). In design-tokens.css it represents error/danger (negative). In the translation diff view it represents changed content (neutral). The `--brand-coral` token (deliverable 25) partially addresses this; the design system documentation should explicitly list all three semantic roles.

**Font vendoring is complete in User UI v0.2.0.** DM Sans (400, 500, 600, 700) and JetBrains Mono (400) are vendored as local woff2 files. No external Google Fonts dependency. Note: DM Sans SemiBold (600) is used in shared-components for buttons and labels but is not listed in the brand specification -- the brand spec should acknowledge this weight.
