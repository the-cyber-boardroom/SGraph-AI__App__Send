## 3. Typography Recommendations

| Font stack (`"font"` value) | Best for | Prints cleanly? |
|-----------------------------|----------|-----------------|
| `"sans"` | Investor decks, UI docs, dashboards, general reference | Good — tight, legible on A4 |
| `"serif"` | Articles, case studies, long-form narrative, formal reports | Best — traditional line rhythm, recommended for print-first content |
| `"mono"` | Technical runbooks, API references, code-heavy docs | Fair — monotone tracking can feel dense in print; use `"compact"` density |
| `"system"` | Embedded content, admin consoles, quick notes | Good — renders in the OS native font |

**Recommendation:** Author-facing guidance should default to `"sans"` for `default`, `navy`, `brand`, and `dark-deck`, and `"serif"` for `slate` and `minimal`. Serif particularly benefits long-form text in print, where letter spacing and x-height distinguish characters more reliably than sans-serif at small point sizes.

## 4. Print Recommendations

### Hero pattern

The current default hero background (`#1a1a2e`) prints as a very dark rectangle. Two options:

**Option A — Print-safe hero (recommended for `minimal` and `slate`):** Replace the dark background with a `4px` teal (or accent-coloured) left border on the title, a white background, and dark title text. Set `"hero": { "color": "#ffffff" }` and add a CSS `@media print` rule to override hero background to `#ffffff` and title to `#111111`. This is the cleanest result.

**Option B — Keep dark hero with `print-color-adjust: exact` (recommended for `navy`, `brand`, `dark-deck`):** The existing `@media print` rule already sets `print-color-adjust: exact` on `.plr-hero`. This forces printers to render the dark block. Acceptable for decks where the visual identity of a dark hero is intentional (the receiver expects a coloured printout).

**Implemented recommendation:** Both options can coexist. The CSS adds a `@media print` class `.plr-hero--print-light` that overrides any dark background to white with a left accent bar. Authors who want print-safe heroes set `"class": "print-light"` on the hero component.

### Section `"alt"` background

**Current value:** `rgba(0,0,0,0.03)` — completely invisible in most environments.

**Recommended values:**
- Light mode: `#f4f6f8` (a cool neutral grey, visually distinct but not jarring)
- Dark mode: `rgba(255,255,255,0.06)` (slightly brighter lift against the dark base)

These are close enough to white/dark-base to print cleanly (the `@media print` stylesheet forces `background: #ffffff`, so alt-section tints disappear in print — this is correct behaviour).

The fix is implemented via the `--plr-section-alt-bg` CSS custom property, which the renderer already uses as the inline style fallback.

### Font choice for print

Use `"font": "serif"` when the primary output channel is print or PDF. The Georgia / Times New Roman fallback stack is universally available and renders at small point sizes with better legibility than system sans-serif.

## 5. CSS Implementation Notes

The following changes were made to `send-browse-v031--page-layout.css`:

1. **`--plr-section-alt-bg` defined on `.plr-page--light` and `.plr-page--dark`** — replaces the near-invisible inline fallback with a clearly visible alternating background.

2. **Light mode card colours confirmed consistent** — `.plr-page--light` overrides for `.plr-card__title` and `.plr-card__desc` already existed and match the `default` scheme values.

3. **Light mode nav colours confirmed consistent** — `.plr-page--light .plr-nav` inherits the existing nav rule correctly; no change needed.

4. **Light mode markdown code inline** — added `color: #d63384` override for `.plr-page--light .plr-markdown code` to ensure inline code is readable in light mode (the Browse shell's dark variable would otherwise make it near-invisible).

5. **Light mode blockquote** — added `border-left-color: #4ecdc4` and `color: #2c2c3a` for `.plr-page--light .plr-markdown blockquote`.

No CSS variable system was added — all values are class-scoped hard-coded rules per brief constraints.
