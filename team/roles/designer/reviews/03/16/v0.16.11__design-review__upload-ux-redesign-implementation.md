## 2. Web Component Architecture

**Design principle: each visual piece is a self-contained Web Component.** This keeps files small, HTML/CSS/JS co-located within each component, and enables independent development. The parent `<send-upload>` orchestrates visibility; each child owns its own rendering.

### Component Map

| Web Component | Visual Responsibility | Size Target |
|---------------|----------------------|-------------|
| `<send-step-indicator>` | Step progress bar (1→2→3) | ~80 lines |
| `<send-drop-zone>` | Drop area, browse buttons, file summary, trust badge | ~150 lines |
| `<send-delivery-picker>` | Delivery option cards + content detection | ~120 lines |
| `<send-share-mode>` | Token display, sharing mode selection, actions | ~150 lines |
| `<send-token-editor>` | Word picker for customising simple token | ~100 lines |
| `<send-trust-carousel>` | Rotating privacy/trust messages | ~100 lines |
| `<send-processing>` | Encrypt + upload progress bar + carousel | ~100 lines |

Each component encapsulates its own styles (via `<style>` in the component or a linked CSS file). No global CSS class collisions.

### Card Content Per File Type

| File Type | Icon | Title | Description | Hint |
|-----------|------|-------|-------------|------|
| Download (always) | 📥 | Send as download | Recipient gets a file to save to their device | Best for: large archives, backups |
| View (PDF, md, txt, html, images) | 👁️ | Let them view in the browser | Recipient reads/views directly, no download needed | Best for: documents, reports |
| Browse (folder, zip) | 📂 | Let them browse the folder | Recipient sees files in a browsable view with inline preview | Best for: sharing documents, reports |
| Gallery (images) | 🖼️ | Show as photo gallery | Recipient browses images in a gallery layout | Best for: photo sets, design assets |

### Interaction

- **Click** selects a card (`.delivery-card--selected`)
- **Click again** or **"Continue" button** below cards advances to Step 3
- Single available option: auto-advance after a brief highlight (300ms)
- Two options: show both, require explicit selection
- Three+ options: show all, require selection + Continue button

## 9. Interaction Specification

### Step 1 → Step 2 Transition

1. User drops file/folder
2. `<send-drop-zone>` dispatches `'file-selected'` or `'folder-selected'`
3. Drop zone shows file summary (name, size, type) — "Ready — choose how to share it →"
4. Step indicator advances: Step 1 gets checkmark, Step 2 activates
5. `<send-delivery-picker>` slides in from right (300ms)
6. **No encryption yet** — all decisions first, processing last

### Step 2 → Step 3 Transition

1. User clicks a delivery card
2. Card gets selected state (`.delivery-card--selected`)
3. If only one valid option: auto-advance after 300ms highlight
4. If multiple: user clicks card, then "Continue" button appears below cards
5. Step indicator advances
6. `<send-share-mode>` slides in from right

### Step 3 → Processing → Complete

1. Token is displayed with share link
2. User clicks "Send" (primary action) — this triggers the processing phase
3. `<send-processing>` replaces the share mode area
4. Processing runs: zipping (if folder) → reading → encrypting → creating → uploading → completing
5. `<send-trust-carousel>` shows during processing (the only waiting period)
6. On complete: share credentials return with copy/email actions

### Back Navigation

- "← Back" links between all steps (Step 3 → 2 → 1)
- Always available before processing starts
- Going back is free — no encrypted blob to discard (encryption hasn't happened yet)
- Slide content right-to-left (reverse animation)

## 11. Accessibility Notes

| Element | Requirement |
|---------|-------------|
| Step indicator | `role="navigation"`, `aria-label="Upload progress"`, current step has `aria-current="step"` |
| Delivery cards | `role="radio"` within `role="radiogroup"`, keyboard navigable (arrow keys) |
| Trust carousel | `role="region"`, `aria-live="polite"` for auto-rotating messages, `aria-label="Privacy and security information"` |
| Shuffle button | `aria-label="Generate new share code"` |
| Copy button | `aria-label="Copy share link to clipboard"`, announce "Copied!" via `aria-live` |
| Progress bar | `role="progressbar"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"` |
| Back links | `role="button"` or `<button>`, not just styled text |

## 12. Implementation Priority for Dev

Each web component is independently buildable and testable:

1. **`<send-step-indicator>`** — smallest component, add to page first
2. **`<send-drop-zone>`** — simplified drop area with ghost buttons, trust badge, file summary
3. **`<send-delivery-picker>`** — content detection + delivery cards
4. **`<send-share-mode>`** + **`<send-token-editor>`** — token display, 3 sharing modes, word picker
5. **`<send-trust-carousel>`** — rotating messages (can be developed and tested standalone)
6. **`<send-processing>`** — encrypt + upload progress, embeds trust carousel
7. **Transitions** — CSS animations between steps (added to orchestrator)
8. **Mobile responsive** — test all components at 480px breakpoint

Each component is its own file, independently shippable and reviewable. Components can be developed in parallel since they communicate via events/attributes.
