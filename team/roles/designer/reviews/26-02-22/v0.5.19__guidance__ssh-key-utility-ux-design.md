## Design Goal

Build the **cleanest SSH key generator on the web.** The Ambassador's research shows the competitive bar is low — most tools are visually cluttered or functionally bare. We aim to match Inventive HQ's aesthetic quality while exceeding everyone on trust signals and educational content.

The page must feel like it belongs to SG/Send — same design system, same brand, same quality level — but with a narrower, tool-focused layout.

### Mobile (< 768px)

The layout is already single-column, so mobile adaptation is minimal:

- Algorithm selector: stack vertically (3 rows) instead of horizontal
- Key display areas: full width, horizontal scroll for long lines
- Copy/Download buttons: stack vertically, full width
- How-to cards: full width, no side margins
- FAQ: accordion stays the same

### Content Width

Max-width of `640px` for the main content area, centred. This matches the existing SG/Send upload page and keeps long key strings from stretching the layout uncomfortably wide on desktop.

## Component Design

### Algorithm Selector

Use **segmented control** (button group), not radio buttons. The brief shows radio buttons, but a segmented control is more touch-friendly and visually cleaner in the Aurora dark theme.

```
┌──────────────┬──────────────┬──────────────┐
│   Ed25519    │   RSA 4096   │  ECDSA P-256 │
│ (recommended)│              │              │
└──────────────┴──────────────┴──────────────┘
```

- Active state: `--color-primary` (#4ECDC4) background, dark text
- Inactive state: `--bg-secondary` (#16213E), `--color-text-secondary` text
- Disabled state (Ed25519 unsupported): 50% opacity, cursor: not-allowed, tooltip on hover

### Key Display Areas

Use the existing monospace code block pattern from the upload result panel:

```css
.key-display {
    font-family: var(--font-mono);
    font-size: var(--font-size-sm);
    background: var(--bg-secondary);
    border: 1px solid var(--color-border);
    border-radius: var(--radius-sm);
    padding: var(--space-3) var(--space-4);
    white-space: pre-wrap;
    word-break: break-all;
    max-height: 120px;           /* ~6 lines for private key */
    overflow-y: auto;
    color: var(--color-text);
    user-select: all;            /* click to select entire content */
}
```

**Public key:** single line, horizontal scroll (no wrap) — users expect to copy a single unbroken line.

**Private key:** multi-line, word-break — the PEM format has line breaks at 70 chars, so it wraps naturally.

### Action Buttons

Two button styles:

| Button | Style | Pattern |
|--------|-------|---------|
| **Generate Key Pair** | Primary (`btn-primary`), full width | Same as "Encrypt & Upload" button |
| **Copy** | Secondary (`btn-secondary btn-sm`), inline | Same as copy buttons on upload result |
| **Download** | Secondary (`btn-secondary btn-sm`), inline | New: download icon + filename |
| **Generate New** | Ghost/text button, centred | Same as "Send Another" button |

### Copy Button Feedback

On click: button text changes from "Copy" → "Copied!" with checkmark icon. Resets after 2 seconds. Use `--color-success` (#4ECDC4) for the "Copied!" state.

### Success Banner

After key generation, show a subtle success bar:

```css
.success-banner {
    background: rgba(78, 205, 196, 0.1);     /* accent-subtle */
    border: 1px solid rgba(78, 205, 196, 0.25);
    border-radius: var(--radius-md);
    padding: var(--space-3) var(--space-4);
    color: var(--color-success);
    font-weight: var(--weight-medium);
}
```

Contents: checkmark icon + "Key pair generated" + fingerprint + algorithm name.

### How-To Cards

Use **expandable cards** (accordion style), collapsed by default. This keeps the page scannable. Each card expands to reveal step-by-step instructions.

```
┌─ GitHub ────────────────────────────────────── [▸] ─┐
│  Add your SSH key to GitHub in 4 steps              │
└─────────────────────────────────────────────────────┘

expanded:

┌─ GitHub ────────────────────────────────────── [▾] ─┐
│                                                      │
│  1. Go to github.com → Settings                     │
│  2. Click "SSH and GPG keys" → "New SSH key"        │
│  3. Title: give it a name (e.g. "My Laptop")        │
│  4. Paste your public key and click "Add SSH key"   │
│                                                      │
│  ┌────────────────────────────────────────────┐      │
│  │  [Copy Public Key to Clipboard]             │      │

│                                                      │

```

Cards for: **GitHub**, **GitLab**, **Server (Linux/Mac)**, **AWS EC2**.

### Trust Section

Use icon + text rows. Keep it understated — this isn't a marketing banner, it's a confidence signal. The design should feel factual, not salesy.

```
○  Open source — view the code on GitHub
○  Zero network — keys never leave your browser
○  Web Crypto API — hardware-backed randomness
○  SG/Send — zero-knowledge encryption platform
```

Use small circular icons (16px) in `--color-text-secondary`. Text in `--font-size-sm`.

### FAQ Accordion

Same accordion pattern as the How-To cards. Questions in `--weight-semibold`, answers in regular weight. Collapsed by default.

### CTA Section

Subtle, not aggressive. A thin card with a brief message and a single button:

```css
.cta-card {
    background: var(--accent-subtle);      /* rgba(78, 205, 196, 0.12) */
    border: 1px solid var(--color-border);
    border-radius: var(--radius-md);
    padding: var(--space-6);
    text-align: center;
}
```

## Animation and Transitions

### Key Generation

During generation (especially RSA which takes 0.5-3 seconds):
- Button text changes to "Generating..." with a subtle pulse animation
- A thin progress bar (indeterminate) appears below the button
- Button is disabled (prevents double-click)

### Key Appearance

After generation, the result section fades in with `opacity 0→1` over `200ms`. Use the existing `card-enter` animation from the design system.

### Accordion

Expand/collapse uses `max-height` transition (200ms ease). Content clips cleanly.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | All buttons focusable. Tab order: algorithm → comment → generate → copy/download. Accordion items keyboard-operable. |
| Screen reader | `aria-label` on algorithm selector ("Select SSH key algorithm"). `role="alert"` on success banner. `aria-expanded` on accordions. |
| Colour contrast | All text meets WCAG AA. `--color-text` (#E0E0E0) on `--color-bg` (#1A1A2E) = ratio 11.7:1. `--color-text-secondary` (#8892A0) on `--color-bg` = ratio 5.3:1. Both pass. |
| Touch targets | Minimum 44px height on all buttons. Segmented control segments at least 44px tall. |
| Reduced motion | Respect `prefers-reduced-motion: reduce` — disable pulse animation and transitions. |
