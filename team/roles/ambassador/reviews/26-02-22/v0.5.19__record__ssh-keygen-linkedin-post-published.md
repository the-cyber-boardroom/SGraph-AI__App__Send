## Messaging Assessment

### What Landed Well

1. **"Everything happens in the browser"** — the zero-knowledge promise, clearly stated
2. **"No ads, zero cookies, no tracking"** — direct, trust-building, differentiating
3. **Video format** — shows the tool working, builds credibility more than screenshots
4. **Ed25519 recommended** — signals the tool is opinionated and current (not defaulting to legacy RSA)

### Observations

1. **URL is long.** The current URL (`send.sgraph.ai/send/tools/ssh-keygen/v0/v0.1/v0.1.0/index.html`) is not memorable or shareable. The Journalist's `keys.sgraph.ai/ssh` recommendation should be prioritised for future posts.
2. **Open source claim.** The post mentions open source — this should be backed by a visible link to the GitHub repo from the tool page itself.
3. **No CTA to SG/Send.** The post positions the tool standalone. This is fine for trust-building (not pushy), but future content should explore soft cross-promotion ("by the team behind SG/Send").
