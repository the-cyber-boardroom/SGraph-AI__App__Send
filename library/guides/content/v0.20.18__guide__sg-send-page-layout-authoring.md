# SG/Send Browse ŌĆö Page Layout Authoring Guide (`_page.json`)

**Version:** v0.20.18 | **Audience:** Claude sessions and human authors creating structured page layouts inside SG/Send encrypted zips

---

## What this is

When a zip archive is shared via SG/Send, the Browse view looks for a file named `_page.json` in each folder. If one is present, clicking that folder renders a rich component-based page instead of the default file listing. This guide describes how to write `_page.json` files.

**Use `_page.json` when you want:**
- A hero banner, section headings, and curated layout (investor decks, reports, portfolio pages)
- A navigable hub that links to subfolders with clickable cards
- A slideshow or image gallery with captions
- Control over layout width, theme colour, and font

**Use markdown when you want:**
- A text-first document with prose, inline images, and code blocks
- Standard document hierarchy (README ŌåÆ linked files)
- Something quickly authored or edited

Both can coexist in the same folder. `_page.json` takes priority in the Browse view; the markdown files remain as alternatives.

---

## 1. File structure

A `_page.json` file must be named exactly `_page.json` and placed in the folder it describes.

```json
{
  "title": "Page title",
  "theme": { "mode": "dark", "accent": "#00d4aa" },
  "navigation": [
    { "label": "Overview", "anchor": "overview" },
    { "label": "Articles",  "anchor": "articles" }
  ],
  "components": [
    { "type": "hero",    "props": { "title": "...", "subtitle": "..." } },
    { "type": "section", "props": { "title": "Overview" }, "children": [ ... ] }
  ]
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `title` | yes | Page title (used in the tab label) |
| `theme` | no | Visual theme; string `"dark"/"light"` or v2 object (see ┬¦2) |
| `navigation` | no | Fixed top nav bar; items scroll to matching section anchors |
| `components` | yes | Ordered array of components rendered top to bottom |

---

## 2. Theme

### Quick options

```json
"theme": "dark"      // dark shell palette (matches Browse UI)
"theme": "light"     // white content area (default)
```

### Full control (v2 object)

```json
"theme": {
  "mode":    "dark",
  "accent":  "#00d4aa",
  "font":    "sans",
  "density": "comfortable"
}
```

| Field | Values | Effect |
|-------|--------|--------|
| `mode` | `"dark"`, `"light"`, `"auto"` | Overall palette; `"auto"` ŌåÆ light |
| `accent` | Any CSS colour e.g. `"#00d4aa"`, `"#e63946"` | Nav active indicator colour; image border tint |
| `font` | `"sans"`, `"serif"`, `"mono"`, `"system"` | Font family for the entire page |
| `density` | `"compact"`, `"comfortable"`, `"spacious"` | Section padding multiplier |

---

## 3. Components

All components go inside the `"components"` array (or inside a `"section"`'s `"children"`). Unknown types are silently skipped ŌĆö safe to add future types without breaking existing files.

---

### `hero`

Full-width banner. Always the first component.

```json
{
  "type": "hero",
  "props": {
    "title":    "The Cryptographic Blueprint",
    "subtitle": "A zero-knowledge approach to file sharing",
    "image":    "images/banner.jpg",
    "color":    "#0d1117",
    "height":   "large",
    "align":    "center",
    "overlay":  "gradient"
  }
}
```

| Prop | Default | Values |
|------|---------|--------|
| `title` | ŌĆö | Display text |
| `subtitle` | ŌĆö | Lighter text below title |
| `image` | ŌĆö | Path to background image (relative to folder) |
| `color` | `#0d1117` | CSS background colour (fallback if no image) |
| `height` | *(standard)* | `"small"` (150px), `"medium"` (250px), `"large"` (400px), `"full"` (100vh) |
| `align` | `"left"` | `"left"`, `"center"`, `"right"` ŌĆö text alignment in overlay |
| `overlay` | `"gradient"` with image, `"none"` otherwise | `"gradient"`, `"dark"`, `"light"`, `"solid"`, `"none"` |

---

### `section`

Container with an `<h2>` heading and an anchor id. Wrap all other components in sections to get:
- Anchor navigation (┬¦6)
- Consistent spacing and dividers

```json
{
  "type": "section",
  "props": {
    "title":      "Key Numbers",
    "layout":     "narrow",
    "background": "alt",
    "divider":    "line"
  },
  "children": [
    { "type": "text", "props": { "content": "..." } }
  ]
}
```

| Prop | Default | Values |
|------|---------|--------|
| `title` | ŌĆö | Section heading (also becomes the scroll anchor id) |
| `layout` | *(standard)* | `"full-bleed"` (100% width), `"narrow"` (640px), `"wide"` (1200px) |
| `background` | ŌĆö | Any CSS colour, or `"alt"` for a subtle alternating shade |
| `divider` | `"line"` | `"line"` (horizontal rule), `"space"` (extra padding, no line), `"none"` |

**Sections are the only component that has `children`.** The children are rendered inside the section body. Only leaf components go inside children (not nested sections).

---

### `text`

A styled paragraph block.

```json
{ "type": "text", "props": { "content": "Paragraph text here." } }
```

For multi-paragraph content, use multiple `text` components or use `markdown` (┬¦3.9) with inline `text`.

---

### `bullet-points`

Styled unordered list.

```json
{
  "type": "bullet-points",
  "props": {
    "items": [
      "First key point",
      "Second key point",
      "Third key point"
    ]
  }
}
```

---

### `title`

Standalone heading outside a section (e.g. a subheading between two gallery components).

```json
{ "type": "title", "props": { "text": "Heading text", "level": "3" } }
```

`level` accepts `"1"` through `"6"` (default `"2"`).

---

### `image`

Single image with optional caption and alignment.

```json
{
  "type": "image",
  "props": {
    "file":    "images/diagram.png",
    "caption": "System architecture overview",
    "width":   "80%",
    "align":   "center",
    "shadow":  true,
    "rounded": true,
    "border":  false
  }
}
```

| Prop | Default | Values |
|------|---------|--------|
| `file` | ŌĆö | Path relative to the folder containing `_page.json` |
| `caption` | ŌĆö | Text rendered below the image |
| `width` | `100%` | Any CSS width: `"60%"`, `"400px"`, `"100%"` |
| `align` | `"center"` | `"left"`, `"center"`, `"right"` |
| `border` | `false` | Accent-coloured 1px border |
| `shadow` | `false` | Subtle drop shadow |
| `rounded` | `false` | 8px border radius |

---

### `gallery`

Grid of thumbnails. Clicking any thumbnail opens a full-size lightbox (arrows + ESC to navigate).

```json
{
  "type": "gallery",
  "props": {
    "images": [
      "images/slide-01.jpg",
      "images/slide-02.jpg",
      "images/slide-03.jpg"
    ],
    "captions": [
      "The genesis model",
      "Threat mapping",
      "The four archetypes"
    ],
    "columns":       7,
    "aspect":        "16:9",
    "gap":           "small",
    "show_captions": true
  }
}
```

| Prop | Default | Values |
|------|---------|--------|
| `images` | ŌĆö | Array of file paths |
| `captions` | `[]` | Array of caption strings (positional match with images) |
| `columns` | `3` | Number of grid columns |
| `aspect` | `"16:9"` | `"16:9"`, `"4:3"`, `"1:1"`, `"auto"` |
| `gap` | `"small"` | `"none"`, `"small"`, `"medium"`, `"large"` |
| `show_captions` | `true` when captions present | Explicit boolean to force on/off |

---

### `slides`

Single-image slideshow with previous/next navigation. Keyboard arrow keys work when the component is focused.

```json
{
  "type": "slides",
  "props": {
    "images": [
      "images/archetype-1.jpg",
      "images/archetype-2.jpg",
      "images/archetype-3.jpg",
      "images/archetype-4.jpg"
    ],
    "captions": [
      "The Architect ŌĆö shapes the environment",
      "The Operator ŌĆö runs the machines",
      "The Guardian ŌĆö holds the keys",
      "The Investigator ŌĆö follows the trail"
    ],
    "controls":    "bottom",
    "transition":  "fade",
    "autoplay":    false
  }
}
```

| Prop | Default | Values |
|------|---------|--------|
| `images` | ŌĆö | Array of file paths |
| `captions` | `[]` | Caption for each slide |
| `controls` | `"bottom"` | `"bottom"` (nav below image), `"top"` (nav above image) |
| `transition` | `"fade"` | `"fade"` (smooth opacity change), `"none"` (instant) |
| `autoplay` | `false` | Auto-advance every 3 s; stops on first click |

---

### `pdf`

Embedded PDF viewer (reuses the Browse view PDF iframe).

```json
{
  "type": "pdf",
  "props": {
    "file": "../../strategy/presentations/The_Blueprint.pdf"
  }
}
```

`file` supports `../` traversal within the vault. The PDF must be inside the same vault zip.

---

### `markdown`

Rendered markdown ŌĆö either a file from the vault or inline text.

```json
{ "type": "markdown", "props": { "file": "overview.md" } }

{ "type": "markdown", "props": { "text": "**Bold text** and a [link](other.md)." } }
```

`file` paths are relative to the folder containing `_page.json`. Inline images in the markdown file are resolved from the same folder as the `.md` file. Internal links open as tabs in the Browse view. The `![alt|size](path)` image sizing syntax is supported (see the Markdown Authoring Guide).

---

### `cards`

Navigable card grid. Each card has an image, title, description, and a link that opens a file or folder in the Browse view. Ideal for hub pages that lead to subfolders.

```json
{
  "type": "cards",
  "props": {
    "columns": 2,
    "items": [
      {
        "title":       "Infographic Deep Dive",
        "description": "10 annotated slides explaining the threat model.",
        "image":       "infographic-deep-dive/images/cover.jpg",
        "link":        "infographic-deep-dive/"
      },
      {
        "title":       "Cryptographic Blueprint Walkthrough",
        "description": "4-slide walkthrough with the full PDF deck.",
        "image":       "cryptographic-blueprint-walkthrough/images/cover.jpg",
        "link":        "cryptographic-blueprint-walkthrough/"
      }
    ]
  }
}
```

| Prop | Default | Values |
|------|---------|--------|
| `columns` | `2` | Number of grid columns |
| `items[].title` | ŌĆö | Card heading |
| `items[].description` | ŌĆö | Short body text |
| `items[].image` | ŌĆö | Path to cover image (relative to folder) |
| `items[].link` | ŌĆö | Relative path to file or folder to navigate to on click |

Links behave identically to internal markdown links: files open as tabs, folders navigate the tree.

---

### `columns`

Side-by-side layout. Each direct child goes in its own column. Stacks to a single column below 768px.

```json
{
  "type": "columns",
  "props": {
    "ratio":          "1:2",
    "gap":            "medium",
    "vertical_align": "top"
  },
  "children": [
    {
      "type": "image",
      "props": { "file": "images/portrait.jpg", "rounded": true }
    },
    {
      "type": "markdown",
      "props": { "text": "**Name**\n\nRole and description paragraph." }
    }
  ]
}
```

| Prop | Default | Values |
|------|---------|--------|
| `ratio` | `"1:1"` | `"1:1"`, `"1:2"`, `"2:1"`, `"1:3"`, `"3:1"` |
| `gap` | `"medium"` | `"none"`, `"small"`, `"medium"`, `"large"` |
| `vertical_align` | `"top"` | `"top"`, `"center"`, `"bottom"` |

`columns` only has direct children (no nesting of `columns` inside `columns`).

## 4. Navigation bar

Add a `"navigation"` array to the top-level object to render a sticky nav bar with anchor links.

```json
"navigation": [
  { "label": "Overview",   "anchor": "overview" },
  { "label": "Gallery",    "anchor": "gallery" },
  { "label": "Archetypes", "anchor": "the-four-archetypes" },
  { "label": "Full Deck",  "anchor": "full-deck" }
]
```

`anchor` must match the kebab-cased version of the section title. The renderer auto-derives anchors from section titles:

| Section title | Anchor id |
|--------------|-----------|
| `"Overview"` | `overview` |
| `"The Four Archetypes"` | `the-four-archetypes` |
| `"Key Numbers & Metrics"` | `key-numbers--metrics` |

The active section is highlighted as the user scrolls (IntersectionObserver).

**Rule:** Only add `navigation` when the page has 3+ sections and the content is long enough to scroll. Short pages with 1ŌĆō2 sections don't need it.

## 5. File path rules

All `file` props are resolved relative to the folder containing the `_page.json`.

| `_page.json` location | Prop value | Resolves to |
|----------------------|------------|-------------|
| `/articles/infographic/` | `"images/slide-01.jpg"` | `/articles/infographic/images/slide-01.jpg` |
| `/articles/` | `"infographic/images/cover.jpg"` | `/articles/infographic/images/cover.jpg` |
| `/articles/walkthrough/` | `"../../strategy/deck.pdf"` | `/strategy/deck.pdf` |

**Rules:**
- Use forward slashes only
- `../` traversal is supported and resolved within the vault
- External URLs (`http://`, `https://`) are not supported ŌĆö only vault files
- Folder links in `cards[].link` can have a trailing slash (`"subfolder/"`) or not ŌĆö both work

## 6. Source view

Every page layout tab has a `{ } Source` toggle button in the top-right of the tab panel. Clicking it shows the syntax-highlighted raw JSON. A `ŌÄŗ Copy JSON` button copies the source to the clipboard.

**When someone clicks `_page.json` directly in the file tree**, the tab opens in source view by default (intent: inspect the file). Folder-level auto-detection opens in rendered view.

## 7. Folder structure patterns

### Pattern A ŌĆö Simple folder page

A single article folder with a hero, sections, and images:

```
folder/
  _page.json        ŌåÉ renders this folder as a page
  images/
    banner.jpg
    diagram.png
```

### Pattern B ŌĆö Hub with sub-pages

A top-level folder that introduces the vault and links to subfolders:

```
_page.json                          ŌåÉ root hub (cards linking to articles/)
articles/
  _page.json                        ŌåÉ articles hub
  infographic-deep-dive/
    _page.json                      ŌåÉ article page
    images/
      ...
  cryptographic-blueprint/
    _page.json                      ŌåÉ article page
    images/
      ...
strategy/
  presentations/
    The_Blueprint.pdf               ŌåÉ referenced by articles via ../../ paths
```

### Pattern C ŌĆö Mixed (page + markdown fallback)

Folders that have both `_page.json` and markdown:

```
folder/
  _page.json        ŌåÉ rendered by Browse view (takes priority)
  README.md         ŌåÉ fallback for tools that don't support _page.json
  overview.md       ŌåÉ can be referenced by _page.json markdown component
```

## 9. Authoring checklist

Before finalising a `_page.json`:

- [ ] All `file` paths are relative to the folder containing `_page.json`
- [ ] Every referenced image/PDF exists in the zip at the expected path
- [ ] `../` traversal paths don't escape the zip root (stay within the vault)
- [ ] Section `title` values match the `anchor` strings in `navigation`
- [ ] `navigation` only included if page has 3+ sections and meaningful scroll depth
- [ ] `gallery` and `slides` image arrays have the same length as `captions` (if captions are provided)
- [ ] `cards` items have `link` values that resolve to real files or folders in the zip
- [ ] `"theme"` mode is appropriate for the vault context (`"dark"` for branded decks, `"light"` for document-style pages)
- [ ] The JSON is valid (no trailing commas, all strings quoted, arrays/objects closed)
