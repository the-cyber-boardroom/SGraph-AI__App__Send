## Executive Summary

SG/Send occupies a unique position in the market: **the intersection of end-to-end encryption, zero app requirement for the recipient, open source, self-hostable, and PKI-enabled.** No other product occupies this space. Competitors cover subsets of these properties but sacrifice at least one.

The competitive landscape spans four categories: consumer file sharing, enterprise data rooms, encrypted messaging, and developer/one-time secret tools. SG/Send competes differently in each — and has white space in all of them.

---

## 1. Competitive Landscape Overview

### The Five Properties That Define the Space

| Property | Why It Matters |
|---|---|
| **End-to-end encrypted** | Server cannot read the data — mathematical guarantee, not policy |
| **No app for recipient** | Universal accessibility — works in any browser, no install |
| **Open source** | Verifiable security claims — anyone can audit the cryptography |
| **Self-hostable** | Enterprise control — deploy on your infrastructure, your jurisdiction |
| **PKI-enabled** | Recipient-addressed encryption — no separate key sharing needed |

No competitor has all five.

## 2. Direct Competitors

### 2.1 WeTransfer

**What they are:** Consumer file sharing. Simple drag-and-drop file transfer.

| Dimension | WeTransfer | SG/Send |
|---|---|---|
| **Encryption** | TLS in transit, encrypted at rest (provider holds keys) | **AES-256-GCM, client-side, zero-knowledge** |
| **Server can read files** | Yes | **Never** |
| **Recipient experience** | No account needed | **No account needed** |
| **Open source** | No | **Yes (Apache 2.0)** |
| **Self-hostable** | No | **Yes** |
| **PKI** | No | **Yes** |
| **Pricing** | Free (2GB) / GBP 12-229/month | Free / GBP 49-999/month |

**Assessment:** WeTransfer is the incumbent in frictionless file sharing. SG/Send matches the UX simplicity while adding genuine encryption. WeTransfer adding encryption would validate the market but cannot match zero-knowledge architecture without a complete rebuild.

**Competitive defence:** SG/Send's zero-knowledge architecture is not a feature toggle. It requires the entire system to be designed around client-side encryption. WeTransfer cannot add this incrementally.

### 2.2 Tresorit

**What they are:** Encrypted cloud storage and file sharing for businesses.

| Dimension | Tresorit | SG/Send |
|---|---|---|
| **Encryption** | E2E encrypted (client-side) | **E2E encrypted (client-side)** |
| **Server can read files** | No (with E2E) | **Never** |
| **Recipient experience** | Account required | **No account needed** |
| **Open source** | No (proprietary) | **Yes (Apache 2.0)** |
| **Self-hostable** | Limited (on-premise option) | **Yes (7 deployment targets)** |
| **PKI** | Certificate-based (enterprise) | **Browser-native, zero dependencies** |
| **Pricing** | GBP 12-30/user/month | GBP 49-999/month (not per-user) |

**Assessment:** Tresorit is the closest technical competitor — they take encryption seriously. But they require an account, software, and are closed-source. For a zero-knowledge tool, a security reviewer told us "anyone can verify" is not optional. Tresorit cannot offer this.

**Competitive defence:** Open source + no recipient account + browser-native PKI. Tresorit's per-user pricing also becomes expensive at scale; SG/Send's per-instance pricing is flat.

### 2.3 Bitwarden Send

**What they are:** Encrypted text and file sharing as part of the Bitwarden password manager.

| Dimension | Bitwarden Send | SG/Send |
|---|---|---|
| **Encryption** | E2E encrypted | **E2E encrypted** |
| **Server can read files** | No | **Never** |
| **Recipient experience** | No account needed (for receiving) | **No account needed** |
| **Open source** | Yes | **Yes** |
| **Self-hostable** | Yes (Vaultwarden) | **Yes** |
| **PKI** | No | **Yes** |
| **File focus** | Text-focused, file support secondary | **File and text, equal weight** |
| **Standalone** | Requires Bitwarden account to send | **Standalone product** |
| **Pricing** | Free (text) / USD 10/year (files, with Premium) | Free / GBP 49-999/month |

**Assessment:** Bitwarden Send is the closest in philosophy — open source, encrypted, accessible. But it is a feature of Bitwarden, not a standalone product. File sharing is secondary. No PKI, no data room roadmap, no enterprise deployment flexibility.

**Competitive defence:** SG/Send is purpose-built for encrypted file transfer. PKI, enterprise data rooms, 7 deployment targets, and a clear productisation roadmap separate it from a password manager's sharing feature.

### 2.4 Signal / WhatsApp / Telegram

**What they are:** Encrypted messaging apps with file sharing capabilities.

| Dimension | Signal | SG/Send |
|---|---|---|
| **Encryption** | E2E encrypted (Signal Protocol) | **E2E encrypted (AES-256-GCM + RSA-OAEP)** |
| **Server can read files** | No | **Never** |
| **Recipient experience** | Must install Signal app | **No app needed** |
| **Open source** | Yes | **Yes** |
| **Self-hostable** | No (centralised) | **Yes** |
| **PKI** | Signal Protocol (phone-number based) | **Browser-native (short code, QR)** |
| **Web access** | Linked device only | **Native web — browser is the platform** |

**Assessment:** Signal is the gold standard for encrypted messaging. But the critical limitation is: the other party needs Signal. A security reviewer captured this: "If you use Signal, Proton, or whatever else, the other party needs the same tech, which is not always the case."

SG/Send's positioning: **"Signal for the web"** — the same encryption guarantee, but the recipient needs nothing except a browser.

**Competitive defence:** Architectural. Signal is a messaging app. SG/Send is a transfer tool. Different use cases, different architectures, complementary rather than conflicting.

## 3. Enterprise Data Room Competitors

### 3.1 Intralinks (SS&C)

**What they are:** Enterprise virtual data room for M&A, due diligence, board communications.

| Dimension | Intralinks | SG/Send |
|---|---|---|
| **Encryption** | Encrypted (provider holds keys) | **Zero-knowledge (client-side)** |
| **Server can read files** | Yes (provider-controlled) | **Never** |
| **Multi-user access** | Yes | **Yes (Secure Pod architecture)** |
| **Revocable access** | Yes | **Yes** |
| **Audit trails** | Yes | **Yes (hash-chained, tamper-evident)** |
| **Open source** | No | **Yes** |
| **Pricing** | GBP 3,000-10,000/month | **GBP 199-999/month (planned)** |

**Assessment:** Intralinks is the incumbent in enterprise data rooms. They charge premium prices because the alternative is email. SG/Send's Secure Pod architecture delivers the same multi-user, revocable-access capability with zero-knowledge encryption at a fraction of the price.

**Competitive defence:** Zero-knowledge is SG/Send's structural advantage. Intralinks cannot offer "we cannot read your files" because their architecture requires them to hold the keys. Switching to zero-knowledge would require a complete platform rebuild.

### 3.2 Datasite (formerly Merrill)

**What they are:** Virtual data room and deal management platform.

Similar positioning to Intralinks. Same vulnerability: provider holds encryption keys. Same competitive defence for SG/Send: zero-knowledge architecture is not retrofittable.

### 3.3 Firmex, Ansarada, Brainloop

**What they are:** Smaller enterprise data room providers.

Same structural weakness: provider-controlled encryption. SG/Send's open-source, self-hostable, zero-knowledge model is competitively distinct from all of them.

## 4. Emerging / Adjacent Competitors

### 4.1 1Password (Secrets Sharing)

1Password's sharing feature allows sending items to non-1Password users. E2E encrypted, time-limited. But: requires a 1Password account to send, closed-source, no PKI, no file focus.

### 4.2 Firefox Send (Discontinued)

Mozilla's Firefox Send was the closest historical analogue — browser-based encrypted file sharing. Discontinued in 2020 due to abuse concerns. SG/Send's token-gated access model addresses the abuse vector that killed Firefox Send.

### 4.3 OnionShare

Open-source, Tor-based file sharing. Strong privacy but requires Tor Browser for recipients — the opposite of SG/Send's "any browser" approach. Complementary for the privacy-maximalist segment.

### 4.4 Magic Wormhole

Developer tool for peer-to-peer encrypted file transfer. CLI-based, ephemeral. No web UI, no persistence, no enterprise features. Different audience.

## 6. Competitive Defence Strategy

### Moats (Structural Advantages)

1. **Zero-knowledge architecture** — not a feature, an architectural decision. Cannot be added incrementally to existing products.
2. **Browser-native PKI** — built without external dependencies. Competitors would need certificate authority integration or a complete crypto rewrite.
3. **Encryption trilemma resolution** — the Secure Pod architecture is an original contribution. No competitor has demonstrated all three properties simultaneously.
4. **Open source** — verifiable claims. Enterprise procurement advantage. Community trust. "Anyone can verify" is the strongest security claim possible.
5. **7 deployment targets** — same codebase runs on Lambda, Docker, GCP Cloud Run, Fargate, EC2, PyPI, and CLI. Multi-cloud from day one.

### Speed (Operational Advantages)

6. **24-hour feature delivery** — from investor request to working PKI demo in 24 hours
7. **100-minute incident response** — enterprise-grade operational maturity
8. **Agentic team model** — structural velocity advantage that no competitor can replicate by hiring

### Network Effects (Future)

9. **PKI registry** — as more users publish public keys, the network becomes more valuable
10. **LLM integration** — first mover in "send encrypted from Claude/GPT" creates a new channel
11. **Developer ecosystem** — CLI/SDK/MCP creates integration points that multiply distribution

## 7. Market Positioning

### The Position Statement

**For** professionals and enterprises who share confidential files,
**SG/Send is** a zero-knowledge encrypted transfer service
**that** encrypts in the browser, never sees the plaintext, and requires no app or account for the recipient.
**Unlike** WeTransfer (no encryption), Tresorit (requires account), Signal (requires app), and Intralinks (provider holds keys),
**SG/Send** is the only product that is end-to-end encrypted, recipient-frictionless, open source, self-hostable, and PKI-enabled.
