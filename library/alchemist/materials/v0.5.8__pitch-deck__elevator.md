## Slide 2: The Problem

### Confidential files travel through channels where the server can read everything

Every day, term sheets, NDAs, financial reports, and credentials are shared via:

- **Email** — plaintext, forwarded freely, persists forever
- **Slack** — admins see everything, searchable history
- **WeTransfer** — no encryption, server reads every file
- **Data rooms** — provider holds the keys (GBP 3,000-10,000/month)

**No mainstream tool offers: end-to-end encryption + no app for the recipient + verifiable + self-hostable.**

---

## Slide 3: The Solution

### SG/Send — encrypt in the browser, decrypt in the browser

1. Sender drops a file in the browser
2. File is encrypted (AES-256-GCM) before upload
3. Server stores only encrypted bytes — **never sees the plaintext**
4. Recipient opens a link, decrypts in their browser
5. **No app. No account. No install.**

> "Foolproof — like Google search" — Senior security professional, 25 years experience

---

## Slide 4: The PKI Breakthrough

### Browser-based Public Key Infrastructure — achieved 20 February 2026

- RSA-OAEP 4096-bit encryption + ECDSA P-256 signatures
- Non-extractable private keys — can't be read by JavaScript
- Zero external dependencies — Web Crypto API only
- >97% browser coverage

**Before PKI:** sender shares key via a separate channel
**After PKI:** one link, one click — automatic decryption

> "If you can do that, that's a killer feature" — Investor group, 19 February 2026

**Built in 24 hours** from the investor's request to working demo.
