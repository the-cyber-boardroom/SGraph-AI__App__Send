## Executive Summary

SG/Send's revenue model is built on a fundamental insight: **each encrypted transfer costs fractions of a penny to deliver, and there is no scale at which we lose money on a paying user.** The serverless architecture creates >95% gross margins from the first paying customer.

The model is multi-stream: consumption credits (viral acquisition), managed instances (recurring revenue), professional services (available now), and data rooms (enterprise premium). Each stream corresponds to a different customer segment and buying behaviour.

**Early traction validates the model:** 30 paying customers in 30 days from first commit (8 Feb → 10 Mar 2026), zero churn, built on 2 years of open source infrastructure. We continue to discover the pricing model through hypothesis-driven experimentation. What follows is the framework, unit economics, and pricing structure — informed by real customer behaviour.

## 1. Revenue Streams

### 1.1 Credits (Consumption-Based)

| Parameter | Detail |
|---|---|
| **Model** | Pay-per-use, freemium |
| **Target** | Individual users, freelancers, SMEs |
| **Pricing** | Free tier (5 transfers/month, 100MB, 24h expiry) + paid credits for volume |
| **Purpose** | Viral acquisition funnel — every recipient is a potential sender |
| **Unit economics** | Transfer cost: <£0.01. Credit price: £0.05-0.50 depending on file size and features. |
| **Timeline** | Near-term (available at launch) |

**Hypothesis:** If users experience zero-knowledge encryption once as a recipient, 5-10% will convert to senders. Credits lower the barrier to paid usage below subscription commitment.

### 1.2 Managed Instances (Recurring Subscription)

| Parameter | Detail |
|---|---|
| **Model** | Monthly subscription |
| **Target** | Firms wanting branded, isolated deployments |
| **Pricing** | GBP 49-999/month (see tier table below) |
| **Purpose** | Primary recurring revenue stream |
| **Unit economics** | Provision cost: minutes (templated). Infrastructure: <GBP 15/month. Gross margin: >95% |
| **Timeline** | Near-term (available now as professional service, self-service in 3 months) |

**Hypothesis:** Investment firms, law firms, and security teams will pay GBP 49-199/month for a branded, isolated SG/Send instance rather than use the shared platform — because isolation and branding matter in professional contexts.

### 1.3 Professional Services (Per-Engagement)

| Parameter | Detail |
|---|---|
| **Model** | Per-engagement, time-and-materials or fixed-price |
| **Target** | Firms needing custom configuration, integration, branding |
| **Pricing** | GBP 500-5,000 per engagement |
| **Purpose** | Revenue today + relationship building for managed instances |
| **Unit economics** | 100% margin (founder's time + agentic team) |
| **Timeline** | Available now |

**Hypothesis:** Professional services engagements convert to managed instance subscriptions at >50% rate. The engagement proves value; the subscription retains it.

### 1.4 Data Rooms (Enterprise Premium)

| Parameter | Detail |
|---|---|
| **Model** | Monthly subscription |
| **Target** | M&A, due diligence, board communications, regulated industries |
| **Pricing** | GBP 199-999/month |
| **Purpose** | Enterprise revenue with high retention |
| **Unit economics** | Secure Pod infrastructure adds ~GBP 20-50/month. Gross margin: >90% |
| **Timeline** | Medium-term (prototype in 6 months, GA in 9-12 months) |

**Hypothesis:** If we offer zero-knowledge encrypted data rooms at GBP 199-999/month, we will capture customers from Intralinks (GBP 3,000-10,000/month) because the encryption is provably stronger and the price is 5-10x lower.

### 1.5 API/SDK Credits (Developer Platform)

| Parameter | Detail |
|---|---|
| **Model** | Pay-per-use via API |
| **Target** | Developers integrating SG/Send into their applications |
| **Pricing** | Per-API-call or per-transfer credits |
| **Purpose** | Distribution multiplier — other products embed SG/Send |
| **Unit economics** | Near-zero marginal cost per API call |
| **Timeline** | Future (CLI/SDK launches month 2, API monetisation month 6+) |

**Hypothesis:** If developers can add encrypted transfer to their application with a 5-line integration, SG/Send becomes infrastructure rather than a product — and infrastructure has higher retention.

### 1.6 Enterprise SLA (Annual Contract)

| Parameter | Detail |
|---|---|
| **Model** | Annual contract with SLA guarantees |
| **Target** | Large organisations requiring uptime guarantees, compliance certifications, dedicated support |
| **Pricing** | GBP 10,000-50,000/year |
| **Purpose** | Anchor revenue with high-value accounts |
| **Timeline** | Future (requires SOC 2, ISO 27001 — 6-12 month timeline) |

## 3. Unit Economics

### Per-Transfer Cost Breakdown

| Component | Cost per Transfer | Notes |
|---|---|---|
| **Lambda execution** | ~GBP 0.0001 | ~200ms execution, 256MB memory |
| **S3 storage** | ~GBP 0.0002/day | Per MB stored, varies with file size |
| **S3 data transfer** | ~GBP 0.001 | Per MB downloaded |
| **API Gateway / Lambda URL** | ~GBP 0.0001 | Per request |
| **Total per transfer (1MB file, 7-day retention)** | **~GBP 0.003** | Approximately 0.3 pence |

### Managed Instance Economics

| Metric | Value |
|---|---|
| **Infrastructure cost (idle)** | GBP 0.50-2/month (serverless, scales to zero) |
| **Infrastructure cost (light usage)** | GBP 5-15/month |
| **Infrastructure cost (moderate usage)** | GBP 15-50/month |
| **Subscription price** | GBP 49-999/month |
| **Gross margin** | >90-95% |
| **Provision time** | Minutes (templated CloudFormation/Terraform) |
| **Marginal cost of new customer** | Near zero (automated provisioning) |

### Data Room Economics

| Metric | Value |
|---|---|
| **Secure Pod infrastructure** | GBP 20-50/month (Lambda + isolated VPC) |
| **Storage (100 documents, 500MB)** | GBP 0.50/month |
| **Subscription price** | GBP 199-999/month |
| **Gross margin** | >90% |

### Key Insight: Profitable by Design

> "Each encrypted transfer costs fractions of a penny to deliver. There is no scale at which we lose money on a paying user."

The serverless architecture means:
- **No idle costs.** Infrastructure scales to zero when not in use.
- **No minimum spend.** We don't pay for capacity we don't use.
- **Linear scaling.** Costs scale linearly with usage, never ahead of revenue.
- **No kill date.** We don't burn runway waiting for product-market fit. We discover it.

## 5. Financial Strategy

### 5.1 The "No Kill Date" Model

SG/Send's serverless architecture means infrastructure costs are near-zero when idle. Monthly burn (excluding one-off development investment) is:

| Item | Monthly Cost |
|---|---|
| AWS infrastructure (all environments) | GBP 20-50 |
| Domain and DNS | GBP 10 |
| GitHub, tooling | GBP 30 |
| **Total monthly overhead** | **GBP 60-90** |

There is no scenario in which SG/Send "runs out of money" from infrastructure costs. The investment runway is gated by development activity, not idle burn.

### 5.2 Hypothesis-Driven Spending

Every funded experiment has:
- **A testable hypothesis** — "If we build X, Y customers will pay Z"
- **A success metric** — measurable, time-bounded
- **A kill criterion** — if the hypothesis fails, we stop spending and redirect
- **No fixed costs** — contractor engagements, not permanent hires

### 5.3 Path to Profitability

| Milestone | Revenue | Costs | Status |
|---|---|---|---|
| **Break-even (infrastructure)** | GBP 100/month | GBP 60-90/month | Achievable with 2 Professional tier customers |
| **Break-even (with development)** | GBP 5,000/month | GBP 3,000-5,000/month (contractor + infra) | Achievable with 25-30 Professional customers |
| **Self-sustaining** | GBP 20,000/month | GBP 10,000/month (team + infra + compliance) | Achievable with target scenario |

### 5.4 When Is the Next Raise?

The next funding round (if needed) should occur at one of these trigger points:

| Trigger | Timing | Purpose | Expected Valuation |
|---|---|---|---|
| Product-market fit proven | Month 9-12 | Scale customer acquisition | £5-10M |
| Enterprise customer demand | Month 12-18 | Build enterprise sales team | £10-20M |
| Data room traction | Month 12-18 | Scale data room infrastructure | £10-20M |

Alternatively: revenue-funded growth is viable given >95% margins and near-zero infrastructure costs.

## 6. Key Financial Principles

1. **No fabricated projections.** We present scenarios, not forecasts. Revenue will be discovered, not predicted.
2. **Hypothesis-driven spending.** Every pound spent tests a specific hypothesis with measurable outcomes.
3. **No permanent hires.** Agentic team + specialist contractors. Zero payroll obligations.
4. **Profitable by design.** Each user generates positive margin from day one.
5. **No kill date.** Serverless means we don't burn runway waiting for product-market fit.
6. **Each feature = a billable unit.** We don't build features hoping someone will pay — we test the hypothesis first.
