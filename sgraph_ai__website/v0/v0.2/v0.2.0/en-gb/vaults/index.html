Vaults

Version-controlled, client-encrypted file systems.
Access via browser, CLI, or API. The server never sees plaintext.

What is a vault?

A vault is an encrypted, version-controlled collection of files with its own key, commit history, and access controls.

🔐

Client-encrypted

All files are encrypted in your browser using AES-256-GCM before upload. The server stores only ciphertext.

📝

Version-controlled

Every file operation is a commit. The full history is preserved. You can share a point-in-time snapshot with a read-only token.

🔑

Scoped access

Each vault has its own key. You give an agent or colleague that key — not your account credentials. Revoke access by rotating the key.

Three ways to access a vault

The same vault, the same encryption, three interfaces. Use whichever fits your workflow.

🌐

Online — Browser UI

Open any vault at send.sgraph.ai using a vault key or share token. No install required. Folder view, gallery view, markdown rendering.

CLI — SGit

The sgit CLI provides git-like vault operations: init, commit, push, pull, share. Runs on any machine with Python.

pip install sgit-ai sgit init my-vault sgit commit -m "first files" sgit push sgit share # → share token

View modes

The browser UI renders vault contents in the format that suits the files inside.

📁

Folder view

Hierarchical file tree. Navigate directories, open files, browse the vault structure. Default view for mixed-content vaults.

🖼

Gallery view

Image grid layout for photo and media vaults. Thumbnails rendered in-browser from encrypted files. No server-side processing.

📄

Markdown rendering

Markdown files render as formatted articles in-browser. Code blocks, headers, links, images — all rendered client-side from ciphertext.

Two key types

Vault keys and share tokens serve different purposes. Know which to use.

🔑

Vault key

Persistent read-write access. Use this to clone, push, and pull. Share with agents or collaborators who need to write to the vault. Looks like: apple-river-1234

⚠️ Treat like a password. Never commit to git.

🎫

Share token

Read-only snapshot of the vault at a point in time. Created with sgit share. Share with recipients — they can open the vault without an account. Looks like: coral-stamp-5678

✓ Safe to share publicly. Expires. Read-only.

SGit quickstart

From install to first share in under five minutes.

# Install pip install sgit-ai # Create a vault sgit init my-project cd my-project # Add files and commit cp ~/report.md . sgit commit -m "initial report" # Push to SG/Send sgit push # → vault key: apple-river-1234 # Create a read-only share token sgit share # → share token: coral-stamp-5678 # → https://send.sgraph.ai/en-gb/browse/#coral-stamp-5678 # Clone an existing vault (on another machine or agent) sgit clone apple-river-1234 my-project-clone
The recipient opens the share token URL in any browser — no account, no install required. The vault decrypts client-side. The server never sees the content.
