Three ways vaults are used

The same encrypted vault. Three interaction patterns. One token to share.

👤 → vault → 👤

Human → Human

Send encrypted files to colleagues, clients, or researchers. The recipient opens with a share token — no account required.

Sender: vault key · Recipient: share token
🤖 → vault → 👤

Agent → Human

An AI agent writes findings, reports, or briefings to a vault. A human opens it with a token — complete audit trail preserved.

Agent: API + vault key · Human: share token
🤖 → vault → 🤖

Agent → Agent

Agent A researches and writes to a vault. Agent B reads, analyses, and writes back. No shared credentials. Every action is a signed commit.

Each agent: own vault key · No credential sharing
