What people build with SG/Send

From security research to healthcare workflows to multi-agent pipelines — client-encrypted vaults are the common thread.

Security Research

Share vulnerability research without exposure

Security teams create encrypted vaults containing OWASP findings, CVE analysis, and remediation guides. Each vault is a versioned, client-encrypted package. The recipient gets a share token — no login, no account, no plaintext on any server.

  • Vault encrypted before it leaves the researcher's browser
  • Share token gives read-only access to a point-in-time snapshot
  • Full version history — every revision auditable
Multi-Agent Workflows

Agents collaborate via encrypted vaults

Two agents, one vault. Agent A researches and writes findings to the vault. Agent B reads, analyses, and writes back a report — all without sharing credentials or touching a shared file system. The vault is the communication channel.

  • Each agent gets a vault key scoped to one vault only
  • Push/pull model handles concurrent writes cleanly
  • Full commit history — every agent action auditable
Healthcare

Clinical results delivered as a structured patient experience

MRI scans, consultation notes, and treatment plans assembled into a vault with a custom _page.json layout — tabbed sections, plain-language summaries, and a full audit trail. Shared via token. No login. Data never leaves your VPC.

  • _page.json renders richer structured layouts than markdown alone
  • Images, video, and analysis all in one encrypted vault
  • Deploy inside your own AWS account — no third-party server sees plaintext

Browser Automation

Every automation output in one place — view them side by side

A Claude session runs an end-to-end infographic pipeline: reads a brief from the vault, drives the browser, generates PDFs and slide decks, then pushes everything back. Open any two outputs side by side directly in the vault browser — no download needed.

  • Multiple files open simultaneously — compare outputs without leaving the browser
  • All pipeline outputs committed to the vault — fully versioned and auditable
  • Share the vault with a token — downstream agents or humans see the same files
